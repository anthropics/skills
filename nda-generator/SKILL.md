# NDA Generator

## Overview

日本語の機密保持契約書（Non-Disclosure Agreement）を自動生成するスキル。企業間取引における機密情報の保護を目的とした契約書をWord形式（.docx）で作成する。

**重要**: 本スキルは法務リスクを軽減した改善版を生成します。損害賠償の上限設定（500万円）と秘密保持義務の期間制限（終了後5年間）により、双方にとってバランスの取れた内容となっています。

## 使用方法

### 基本ワークフロー

1. **情報収集**: ユーザーに以下の情報を確認
   - 契約当事者（開示者・受領者の企業名）
   - 契約の目的（システム開発、コンサルティング等）
   - 機密情報の流れ（片方向 or 双方向）
   - 特に保護したい情報の種類
   - 契約期間・有効期間

2. **スクリプト実行**: `scripts/create_nda.js`をベースにカスタマイズして実行

3. **出力**: `/mnt/user-data/outputs/`にWord文書を出力

### スクリプトのカスタマイズ

`scripts/create_nda.js`を編集して以下を調整：

- **企業名**: 開示者・受領者の商号を変更
- **目的**: 第1条の目的を具体的な業務内容に修正
- **管轄裁判所**: 第13条の管轄裁判所を適切な地域に変更
- **条項追加**: 必要に応じて追加条項を挿入

### 実行手順

```bash
# スクリプトを編集後、実行
node scripts/create_nda.js
```

## 標準的な契約構成

生成されるNDAには以下の条項が含まれる：

1. **目的**: 契約の目的と背景
2. **機密情報の定義**: 保護対象となる情報の範囲
3. **機密保持義務**: 受領者の義務（目的外使用禁止、第三者開示禁止等）
4. **適用除外**: 機密情報に該当しない情報の定義
5. **外部サービスの利用** ⭐: Google Workspace等企業向けサービスの利用許可、生成AI利用時の配慮
6. **返還・破棄**: 契約終了時の機密情報処理
7. **権利義務の譲渡禁止**: 契約上の地位譲渡の制限
8. **契約期間** ⭐⭐: 有効期間3年・自動更新、機密保持義務は終了後5年間（高リスク解消）
9. **損害賠償** ⭐⭐: 上限500万円、故意・重過失は除外（高リスク解消）
10. **契約の変更**: 変更手続き
11. **分離可能性**: 一部無効時の取扱い
12. **協議事項**: 未定事項の解決方法
13. **管轄裁判所**: 紛争時の裁判管轄

⭐⭐ = 法務リスクを大幅に軽減した改善条項
⭐ = 現代的な業務環境に対応した条項

## カスタマイズオプション

詳細は`references/clauses.md`を参照。

### 双方向NDA（相互開示型）

両当事者が相互に情報を開示する場合：
- 前文で「両当事者は」に変更
- 「開示者」「受領者」を「甲」「乙」に統一
- 各条項を双方向に適用

### 損害賠償の上限額調整

プロジェクト規模に応じて調整可能：
- 小規模プロジェクト: 300万円
- 標準（デフォルト）: 500万円
- 大規模プロジェクト: 1,000万円

スクリプト内の該当箇所を編集：
```javascript
children: [new TextRun("1. 受領者が本契約に違反し、開示者に損害を与えた場合、受領者は開示者に対し、当該損害を賠償する責任を負うものとする。ただし、賠償額の上限は金500万円とする。")]
```

### 秘密保持期間の調整

業種や情報の性質に応じて：
- 短期（技術情報の陳腐化が早い場合）: 3年間
- 標準（デフォルト）: 5年間
- 長期（顧客情報等の重要性が高い場合）: 7年間

### 追加条項

必要に応じて以下を追加：
- **競業避止**: 競合事業への参入制限
- **監査条項**: 開示者による管理状況の監査権

## レイアウトの調整

署名欄がページをまたぐ場合：
- 署名欄の直前に`pageBreakBefore: true`を設定
- これにより署名欄全体が新しいページから始まる

## 技術仕様

- **ライブラリ**: docx.js（Node.js）
- **フォント**: MS Mincho（日本語契約書の標準）
- **出力形式**: .docx（Microsoft Word形式）
- **ページ設定**: A4サイズ、余白各25.4mm（1インチ）
