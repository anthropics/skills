# 機密保持契約書 - 条項リファレンス

## 標準条項（改善版）

本スキルが生成する機密保持契約書は、法務リスクを軽減したバランス型の内容です。

### 第1条 (目的)
契約の目的を明記。システム開発、コンサルティング、新規事業検討など。

### 第2条 (機密情報の定義)
保護対象となる情報を定義。
- 顧客情報（個人情報、取引履歴等）
- 技術情報（システム構成、仕様書、設計書等）
- 営業情報（事業計画、マーケティング情報等）
- その他開示者が機密と明示した情報

### 第3条 (機密保持義務)
受領者の義務：
- 目的外使用の禁止
- 第三者への開示禁止
- 複製時の事前承諾
- 従業員への同等義務の課与

### 第4条 (適用除外)
機密情報に該当しない情報：
- 開示時点で公知の情報
- 開示後に受領者の責によらず公知となった情報
- 開示時点で既に保有していた情報
- 第三者から適法に取得した情報
- 独自に開発した情報

### 第5条 (外部サービスの利用) ⭐改善ポイント
企業向けサービス（Google Workspace、Microsoft 365等）の利用を許可。
生成AI利用時は個人情報の匿名化等の合理的配慮を求める。

**特徴**: 現代的な業務環境に対応し、過度な制約を避けた実用的な規定。

### 第6条 (機密情報の返還・破棄)
契約終了時または要請時の処理方法。

### 第7条 (権利義務の譲渡禁止)
契約上の地位や権利義務を第三者に譲渡する際は事前承諾が必要。

### 第8条 (契約期間) ⭐⭐改善ポイント（高リスク解消）
**改善内容:**
1. 有効期間を3年間と明確化
2. 3ヶ月前通知で解約可能、通知がなければ1年ごと自動更新
3. 機密保持義務は契約終了後5年間（無期限から改善）
4. 個人情報は契約終了後も適切に管理

**改善効果**: 
- 無期限の義務負担を解消
- 契約終了の手続きが明確化
- 業界標準的な5年間に設定

### 第9条 (損害賠償) ⭐⭐改善ポイント（高リスク解消）
**改善内容:**
1. 賠償額の上限を500万円に設定
2. 故意または重大な過失の場合は上限を適用しない

**改善効果**:
- 受領者のリスクを予測可能に
- 通常の過失には上限を設け、故意・重過失には無制限とすることで双方のバランスが取れる
- 無制限リスクから明確な上限設定へ改善

### 第10条 (契約の変更)
書面による合意のみ有効。

### 第11条 (分離可能性)
一部条項が無効でも他の条項は有効とする。

### 第12条 (協議事項)
未定事項や解釈の疑義は協議で解決。

### 第13条 (管轄裁判所)
紛争時の専属管轄裁判所を指定。

## リスク評価について

本スキルが生成する契約書は、contract-analyzerスキルによる分析で以下のリスクレベルとなります：

- **高リスク項目**: 0件
- **中リスク項目**: 0-1件（契約解除条件の詳細化は必要に応じて）
- **低リスク項目**: 多数

双方にとって公平でバランスの取れた内容となっています。

## カスタマイズオプション

### 相互NDA（双方向開示）
両当事者が相互に情報を開示する場合：
- 前文を「両当事者は」に変更
- 「開示者」「受領者」を「甲」「乙」に統一
- 各条項を双方向に適用

### 損害賠償の上限額調整
プロジェクト規模に応じて調整：
- 小規模: 300万円
- 標準: 500万円（デフォルト）
- 大規模: 1,000万円

### 秘密保持期間の調整
業種や情報の性質に応じて：
- 短期: 3年間
- 標準: 5年間（デフォルト）
- 長期: 7年間

### 競業避止条項
競合事業への参入を制限する場合：
```
第●条 (競業避止)
受領者は、契約期間中及び契約終了後○年間、
開示者の事業と競合する事業を行わないものとする。
```

### 監査条項
開示者による監査権を規定する場合：
```
第●条 (監査)
開示者は、受領者に対し、機密情報の管理状況について
合理的な範囲で監査を実施することができる。
```
