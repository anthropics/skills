# Deep Dive 深度研究 Skill

## 核心定位

**普通搜索**找到信息
**Deep Dive**验证信息 + 分析因果 + 评估可信度

当用户需要：深度研究、交叉验证、探求原因、评估可信度、多源对比
**必须**调用此 skill，而非普通搜索。

---

## Phase 1: 广泛采集（必须并行执行）

### 搜索源配置（5+ 源并行）

#### 组 A: 主搜索链（Exa → Tavily → DDG fallback）
```
mcp__exa__web_search_exa
query: <主查询> + <扩展查询1> + <扩展查询2>
numResults: 10
includeRawContent: true

如果 Exa 失败（报错/无结果/额度满）:
→ mcp__tavily-search__tavily_search
query: <主查询>
max_results: 10
search_depth: advanced
include_raw_content: true

如果 Tavily 失败:
→ mcp__ddg-search__search
query: <主查询>
max_results: 10
```

#### 组 B: Jina 多源搜索（学术 + 新闻 + 博客）
```
mcp__jina__search_web
query: <主查询>
num: 10

mcp__jina__search_arxiv
query: <主查询>
num: 10

mcp__jina__search_jina_blog
query: <主查询>
num: 10
```

#### 组 C: Gemini 搜索（覆盖补充）
```
mcp__gemini-cli__ask-gemini
prompt: "使用 Google Search 搜索: <主查询>，返回至少10个最相关的结果，包括标题、URL和摘要"
```

#### 组 D: 扩展查询搜索（语义变体）
使用 mcp__jina__expand_query 生成 3-5 个扩展查询，对每个扩展查询执行：
```
mcp__exa__web_search_exa
query: <扩展查询>
numResults: 5
includeRawContent: true
```

## Phase 2: 内容获取（深度读取）

### 高价值 URL 筛选标准

从 Phase 1 的结果中，筛选以下 URL 进行深度读取：

1. **权威性**: 官方来源、知名媒体、学术机构
2. **相关性**: 与查询主题直接相关
3. **时效性**: 优先较新的内容（除非需要历史背景）
4. **独特性**: 包含其他来源没有的信息

### 并行读取

```
mcp__jina__parallel_read_url
urls: [精选的 URL 列表，最多 5 个]
withAllLinks: false
withAllImages: false
```

**注意**: 不要偷懒，尽可能读取完整内容而非仅依赖摘要。

## Phase 3: 深度分析（核心）

### 3.1 信息抽取

从所有来源中提取：
- **关键事实**: 具体数据、事件、声明
- **观点主张**: 各方立场、预测、评价
- **时间线**: 事件发生的先后顺序
- **参与方**: 涉及的个人、组织、利益相关者

### 3.2 交叉验证矩阵

创建信息映射表：

| 信息点 | 来源A | 来源B | 来源C | 来源D | 确认度 |
|--------|-------|-------|-------|-------|--------|
| 事实1 | ✅ | ✅ | ✅ | - | 高 |
| 事实2 | ✅ | - | ✅ | - | 中 |
| 事实3 | ✅ | - | - | - | 低 |
| 矛盾点1 | 说法X | 说法Y | - | - | 冲突 |

### 3.3 逻辑验证检查清单

对每个关键声明检查：
- [ ] **自洽性**: 数据内部是否一致？
- [ ] **因果合理性**: 原因和结果是否匹配？
- [ ] **时间线合理性**: 时间顺序是否合理？
- [ ] **来源独立性**: 多个来源是否独立？（避免同一原始来源的重复）
- [ ] **证据强度**: 有数据/官方确认 vs 传闻/推测

### 3.4 溯源分析

追溯每个关键信息的**最初来源**：
- **一手来源**: 官方声明、原始数据、直接采访
- **二手来源**: 引用一手的报道
- **无法溯源**: 标记为待验证

### 3.5 因果深挖（向上探求）

对关键事件/现象，逐层分析：

**层级 1 - 直接原因**: 触发事件的具体因素
**层级 2 - 深层动机**: 各方行为背后的利益/目标
**层级 3 - 结构性因素**: 制度、文化、长期趋势
**层级 4 - 反事实思考**: 如果X没发生，会怎样？

### 3.6 可信度评分公式

每个信息点评分（0-100）：
```
基础分 = 来源权威性 (30) + 交叉验证 (30) + 证据强度 (25) + 时效性 (15)

来源权威性:
- 一手官方/顶级学术: 30
- 权威媒体/知名机构: 25
- 普通媒体/自媒体: 15
- 匿名/未知: 5

交叉验证:
- ≥3 独立来源: 30
- 2 独立来源: 20
- 1 来源: 5

证据强度:
- 数据/官方文件/直接证据: 25
- 专家分析/引用来源: 15
- 推测/观点: 5

时效性:
- 最新/持续更新: 15
- 较新（<1年）: 10
- 较旧（>1年）: 5

总分分级:
- 85-100: 高置信度
- 60-84: 中置信度
- 40-59: 低置信度
- <40: 待验证
```

## 强制约束

### 不偷懒原则

1. **搜索**: 必须触发所有可用的搜索源，不能省略
2. **读取**: 高价值 URL 必须读取完整内容
3. **分析**: 每个分析维度必须执行，不能跳过
4. **输出**: 输出必须完整，不能因为长度而省略章节
5. **引用**: 每个关键结论必须有来源支撑

### 质量检查清单

输出前自检：
- [ ] 是否覆盖 ≥5 个独立搜索源？
- [ ] 是否对高价值 URL 执行了深度读取？
- [ ] 是否包含交叉验证分析？
- [ ] 是否包含因果推断？
- [ ] 是否包含可信度评分？
- [ ] 是否列出信息矛盾点？
- [ ] 是否识别偏见和局限性？
- [ ] 输出是否详尽？是否有省略关键章节？

## 错误处理

### 搜索源失败

如果某个搜索源失败：
1. 记录失败信息
2. 继续执行其他搜索源
3. 在报告中注明哪些源未成功
4. 评估对整体研究的影响

### 无结果处理

如果所有搜索源都无结果：
1. 尝试简化/泛化查询词
2. 尝试英文查询（如果原查询是中文）
3. 尝试相关关键词
4. 如果仍然无果，报告中如实说明搜索范围和建议

### 内容读取失败

如果 URL 读取失败：
1. 记录失败 URL
2. 尝试通过其他来源获取相同信息
3. 在来源列表中标注"内容未获取"
