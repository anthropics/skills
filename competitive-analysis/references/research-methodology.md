# Stage 2 — 信息搜集方法论

## 搜索执行流程

```
分析规划（来自 Stage 1）
  │
  ├─ 1. 生成搜索关键词矩阵
  ├─ 2. 按优先级逐源搜索
  ├─ 3. 初步结果评估与信息缺口识别
  ├─ 4. 追加定向搜索（填补缺口）
  ├─ 5. 高价值资料精读
  ├─ 6. 截图与素材归档
  └─ 7. 汇总搜集清单，标注覆盖率与待核查项
```

## 1. 搜索关键词矩阵

针对每个竞品 × 每个分析维度，生成组合关键词：

| 维度 | 关键词模板 | 示例 |
|------|-----------|------|
| 产品概览 | `{品牌} {产品} 官网 / about / overview` | "Figma overview" |
| 功能详情 | `{产品} {功能} tutorial / how to / documentation` | "Figma auto layout tutorial" |
| 更新动态 | `{产品} {年份} update / changelog / what's new` | "Figma 2026 updates" |
| 定价 | `{产品} pricing / plans / comparison` | "Figma pricing 2026" |
| 用户评价 | `{产品} review / vs / Reddit / complaints` | "Figma vs Sketch Reddit 2026" |
| 技术架构 | `{产品} architecture / API / SDK / tech stack` | "Figma plugin API docs" |

**搜索语言**：中英文各搜一轮，英文资料通常更全面。

## 2. 多源搜索策略

### 一手来源（优先级最高）

| 来源 | 操作 | 留存 |
|------|------|------|
| 官网 | 浏览产品页、功能页，截图关键界面 | 截图 + URL |
| 帮助文档 | 搜索目标功能的使用说明 | 摘要 + 链接 |
| API/SDK 文档 | 评估技术开放度与集成能力 | 要点摘录 |
| 更新日志 | 近 6 个月的功能迭代方向 | 时间线摘要 |
| 应用商店 | 评分、评论、更新频率 | 数据记录 |
| 官方博客/公众号 | 产品理念、设计决策说明 | 引用 + 链接 |

### 二手来源

| 来源 | 操作 | 注意 |
|------|------|------|
| 专业媒体测评 | 搜索近 1 年的权威评测文章 | 注意发布日期 |
| 行业报告 | Gartner / Forrester / IDC 等 | 标注报告日期和版本 |
| YouTube 测评 | 视频解析（见下方工作流） | 注意视频时效性 |
| Reddit / 论坛 | 真实用户痛点和吐槽 | 区分个例与共性 |
| 竞品对比文章 | `A vs B` 类第三方对比 | 警惕软文/affiliates |

### 来源可信度分级

| 等级 | 来源类型 | 处理方式 |
|------|----------|----------|
| A 高可信 | 官方文档、学术论文、权威媒体 | 直接引用 |
| B 中可信 | 专业博客、KOL 评测、行业报告 | 交叉验证后引用 |
| C 低可信 | 论坛评论、社交媒体、匿名帖 | 仅作参考线索，标注来源等级 |
| D 存疑 | AI 生成内容、过期信息、利益相关方 | 必须独立验证，否则不引用 |

## 3. 信息缺口识别（主动探寻）

初步搜索后，对照分析规划中的维度做覆盖率检查：

```
对每个竞品 × 每个维度：
  ✅ 已覆盖（有可信来源）
  ⚠️ 部分覆盖（信息不完整或来源单一）
  ❌ 未覆盖（缺失）

对 ⚠️ 和 ❌ 项：
  → 调整关键词，换搜索引擎/渠道重新搜索
  → 尝试间接途径（如从竞品的竞品资料中找线索）
  → 实在找不到 → 在报告中明确标注"该信息未能获取"
```

**向用户汇报**：完成缺口分析后告知用户覆盖情况，对关键缺口征求意见（是否需要通过其他方式补充，如用户自行体验产品）。

## 4. 特性完整性清单

防止遗漏产品重要特性（根据分析维度选取相关项）：

- [ ] 核心功能列表与版本差异
- [ ] 定价方案与免费/付费边界
- [ ] 平台支持（Web/Desktop/Mobile/API）
- [ ] 协作能力（多人/权限/版本管理）
- [ ] 集成生态（插件/第三方/导入导出）
- [ ] 性能表现（加载速度/大文件处理/离线支持）
- [ ] 无障碍与国际化
- [ ] 安全与合规
- [ ] 独特卖点 / 差异化功能
- [ ] 近期重大更新与路线图

## 5. YouTube 视频分析工作流

```
1. 告知用户 → "正在解析视频：{标题}，预计 3~5 分钟"
2. 获取字幕 → 优先英文/中文字幕
3. 分段摘要 → 按主题切分，每段：
   [MM:SS - MM:SS] 主题标题
   - 关键观点 1
   - 关键观点 2
4. 关键帧 → 尝试截取关键画面（产品界面、演示瞬间）
5. 输出格式：
   ## 视频摘要：{标题}
   - 来源：{URL}
   - 发布日期：{date}
   - 时长：{duration}
   ### 关键观点
   - [02:15] xxx 功能的交互设计采用了...
   - [08:30] 与竞品 Y 的主要差异在于...
   ### 截图
   - ![关键帧描述](materials/{project}/{filename})
```

## 6. 浏览器截图工作流

```
1. 使用 browser_navigate 访问目标 URL
2. 等待页面完全加载（browser_wait + browser_snapshot 确认）
3. browser_screenshot 截取关键界面
4. 保存到 materials/{project-name}/{competitor}-{feature}-{序号}.png
5. 记录元信息到素材清单：
   - 来源 URL
   - 截取时间
   - 页面/状态描述
   - 服务于哪个分析维度
```

截图优先级：核心功能界面 > 关键交互状态 > 定价页 > 错误/空状态。

## 7. 素材归档

所有搜集的素材统一存放在 `materials/{project-name}/` 下：

```
materials/{project-name}/
├── screenshots/          # 界面截图
├── docs/                 # 下载的文档/白皮书摘录
├── videos/               # 视频摘要笔记
└── sources.md            # 素材索引（来源、日期、用途）
```

`sources.md` 格式：
```markdown
| 序号 | 类型 | 文件名 | 来源 URL | 获取日期 | 备注 |
|------|------|--------|----------|----------|------|
| 1 | 截图 | figma-autolayout.png | https://... | 2026-03-03 | 核心功能界面 |
```
