# 事实核查与减少幻觉协议

## 核心原则

1. **不确定就标注** — 宁可标「⚠️ 待核查」也不编造
2. **硬数据必溯源** — 金额、日期、版本号、技术参数必须附来源链接
3. **交叉验证** — 关键事实至少两个独立来源确认
4. **时效标注** — 所有信息标注获取日期，超过 6 个月的标注「信息可能已过时」

## 高风险数据类型（必须二次核查）

| 类型 | 示例 | 核查方式 |
|------|------|----------|
| 金额/定价 | "月费 $20" | 访问官网定价页验证 |
| 时间/日期 | "2026年1月发布" | 查官方博客/更新日志 |
| 技术参数 | "支持 10万 并发" | 查官方文档/白皮书 |
| 版本/型号 | "v3.5 新增xx" | 查更新日志/发行说明 |
| 用户数据 | "1亿用户" | 查官方披露/权威报道 |
| 对比结论 | "A比B快3倍" | 查原始测试报告/方法论 |

## 常见幻觉模式与防范

| 幻觉类型 | 表现 | 防范 |
|----------|------|------|
| **功能虚构** | 声称竞品有某功能，实际没有 | 必须有截图或文档链接证实 |
| **数据捏造** | 编造具体数字（价格、性能） | 硬数据只从官方来源引用 |
| **过度推断** | 从少量信息得出强结论 | 标注推断的前提和置信度 |
| **时序混淆** | 混淆不同版本的信息 | 明确标注版本和日期 |
| **因果虚构** | 编造设计决策的原因 | 用"推测"而非"因为"表述 |

## 搜索结果评估清单

对每条搜索结果，快速评估：

- [ ] **时效**：发布日期在 1 年以内？
- [ ] **来源**：是否为利益相关方（警惕软文/PR稿）？
- [ ] **具体性**：有具体数据还是泛泛而谈？
- [ ] **可验证**：核心主张能否通过其他来源验证？
- [ ] **原创性**：是一手信息还是转述/聚合？

## 输出前自检

生成报告或回答前，逐项检查：

1. 所有硬数据是否附了来源链接？
2. 是否有未标注的推测性表述？
3. 引用的截图/链接是否真实可访问？
4. 是否混淆了不同竞品或不同版本的信息？
5. 对比结论是否基于同一维度和条件？
