# Skill Batch Creator

職業・ペインポイントを記載したスプレッドシート（CSV）から、複数人分のAgent Skillsを一括生成するワークフローです。

## ワークフロー概要

```
1. CSVファイルを読み込む
2. 各行の職業・ペインポイントを分析
3. 職業分析パターンに基づいてスキルを提案
4. ユーザー確認後、各スキルのSKILL.mdを生成
5. 生成結果のサマリーを出力
```

## 使用方法

### 入力CSVの形式

以下の列を含むCSVファイルを用意（テンプレート: `skill_request_template.csv`）：

| 列名 | 必須 | 説明 |
|------|------|------|
| 記入者名 | ○ | リクエスト者の名前 |
| 部門/チーム | ○ | 所属部門 |
| 職業/役職 | ○ | 職種 |
| ペインポイント | ○ | 解決したい業務課題 |
| 解決したいタスク | ○ | 具体的な自動化タスク |
| 希望するスキル名 | △ | 空欄の場合は自動提案 |
| 出力形式の希望 | △ | Excel, PDF, Markdown等 |
| 優先度 | △ | 高/中/低 |
| 備考 | - | 補足情報 |

## Phase 1: 職業分析とスキル提案

CSVを読み込んだら、まず**各行に対してスキルを提案**する。すぐに作成せず、ユーザーに確認を取る。

### 分析フロー

```
職業 → 職業カテゴリ特定 → ペインポイント分類 → スキルパターン選択 → 提案生成
```

### 職業分析の実行

職業分析の詳細パターンは **[references/job-analysis-patterns.md](references/job-analysis-patterns.md)** を参照。

このファイルには以下が含まれる：
- 職業カテゴリ別の業務特性と推奨スキルパターン
- ペインポイント分類と対応するスキルタイプ
- スキル名の命名パターン
- 出力形式の選定ガイド

### 提案フォーマット

各リクエストに対して以下の形式で提案を出力：

```markdown
## 提案 #1: {記入者名}さん

**職業分析結果**
- カテゴリ: {営業系/開発系/人事系/etc.}
- ペインポイント分類: {時間系/情報整理系/ドキュメント系/etc.}

**提案スキル**
| 項目 | 内容 |
|------|------|
| スキル名 | `{提案するスキル名}` |
| 目的 | {1文で説明} |
| 入力 | {想定される入力} |
| 出力 | {出力形式} |
| 効果 | {時間削減・品質向上の見込み} |

---
```

### 提案後の確認

全リクエストの提案を表示したら、ユーザーに確認：

```
以上の提案でスキルを作成してよろしいですか？
- 全て作成: 「はい」
- 一部修正: 修正したい番号と内容を指定
- 特定のみ作成: 作成したい番号を指定
```

## Phase 2: スキル生成

ユーザー承認後、各スキルのSKILL.mdを生成。

### 生成するファイル構造

```
generated-skills/
├── {スキル名}/
│   └── SKILL.md
├── {スキル名}/
│   └── SKILL.md
└── ...
```

### SKILL.mdテンプレート

```yaml
---
name: {スキル名}
description: {ペインポイントとタスクから生成した説明文}。{職業}向け。「{トリガーキーワード}」で使用。
---

# {スキル名}

## 概要
{ペインポイントを解決するためのスキル説明}

## 対象ユーザー
- {職業/役職}
- {部門/チーム}

## 入力
{想定される入力データ}

## 出力
{出力形式の希望に基づく成果物}

## ワークフロー
1. {ステップ1}
2. {ステップ2}
3. {ステップ3}

## 使用例

**トリガー例:**
- 「{具体的な依頼文1}」
- 「{具体的な依頼文2}」

**入力例:**
{具体的な入力例}

**出力例:**
{具体的な出力例}
```

## Phase 3: 生成結果サマリー

生成完了後、以下のサマリーを出力：

```markdown
## 生成完了サマリー

| # | 記入者 | スキル名 | ステータス |
|---|--------|----------|-----------|
| 1 | 田中太郎 | sales-report-generator | ✅ 生成完了 |
| 2 | 鈴木花子 | interview-evaluator | ✅ 生成完了 |
| 3 | 山田次郎 | api-doc-generator | ✅ 生成完了 |

### 次のステップ

生成されたスキルを確認:
\`\`\`bash
ls generated-skills/
cat generated-skills/{スキル名}/SKILL.md
\`\`\`

スキルをテスト後、必要に応じて調整してください。
```

## スキル名の命名規則

希望するスキル名が空欄の場合、職業分析に基づいて自動命名：

| ペインポイント分類 | 命名パターン | 例 |
|------------------|------------|-----|
| 時間系（作成に時間がかかる） | `{対象}-generator` | `weekly-report-generator` |
| 情報整理系（集計・比較） | `{対象}-analyzer` | `interview-score-analyzer` |
| ドキュメント系（更新・作成） | `{対象}-doc-creator` | `api-doc-creator` |
| コミュニケーション系（メール・報告） | `{対象}-writer` | `followup-email-writer` |
| チェック系（確認・レビュー） | `{対象}-checker` | `contract-risk-checker` |

---

## 注意事項

- 生成されたスキルは `generated-skills/` ディレクトリに出力
- 既存の同名スキルがある場合は上書き確認
- 1つのCSVで最大50件程度のリクエストを推奨
- 複雑なスキルは後から手動で調整が必要な場合あり
- 職業分析の詳細は `references/job-analysis-patterns.md` を参照
