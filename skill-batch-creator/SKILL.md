# Skill Batch Creator

職業・ペインポイントを記載したスプレッドシート（CSV）から、複数人分のAgent Skillsを一括生成するワークフローです。

## ワークフロー概要

```
1. CSVファイルを読み込む
2. 各行のリクエストを解析
3. 職業・ペインポイントに基づいてスキルを設計
4. 各スキル用のディレクトリとSKILL.mdを生成
5. 生成結果のサマリーを出力
```

## 使用方法

### 入力CSVの形式

以下の列を含むCSVファイルを用意してください：

| 列名 | 必須 | 説明 |
|------|------|------|
| 記入者名 | ○ | リクエスト者の名前 |
| 部門/チーム | ○ | 所属部門 |
| 職業/役職 | ○ | 職種 |
| ペインポイント | ○ | 解決したい業務課題 |
| 解決したいタスク | ○ | 具体的な自動化タスク |
| 希望するスキル名 | △ | 空欄の場合は自動命名 |
| 出力形式の希望 | △ | Excel, PDF, Markdown等 |
| 優先度 | △ | 高/中/低 |
| 備考 | - | 補足情報 |

### 実行コマンド

```
CSVファイル「skill_requests.csv」を読み込んで、Agent Skillsを一括作成してください
```

## スキル生成ルール

### 1. スキル名の命名規則

希望するスキル名が空欄の場合、以下のルールで自動命名：
- 英語・小文字・ハイフン区切り
- `{タスクのキーワード}-{出力形式/generator/analyzer}`
- 例: `sales-report-generator`, `contract-analyzer`, `meeting-summary`

### 2. 生成するファイル構造

```
generated-skills/
├── {記入者名}-{スキル名}/
│   └── SKILL.md
├── {記入者名}-{スキル名}/
│   └── SKILL.md
└── ...
```

### 3. SKILL.mdのテンプレート

```yaml
---
name: {スキル名}
description: {ペインポイントとタスクから生成した説明文}。{職業}向け。「{トリガーキーワード}」で使用。
---

# {スキル名}

## 概要
{ペインポイントを解決するためのスキル説明}

## 対象ユーザー
- {職業/役職}
- {部門/チーム}

## 入力
{想定される入力データ}

## 出力
{出力形式の希望に基づく成果物}

## 使用例
{具体的な使用シナリオ}

## 処理フロー
1. {ステップ1}
2. {ステップ2}
3. {ステップ3}
```

## 生成時の考慮事項

### ペインポイント分析

- 時間的コスト（〜に時間がかかる）→ 自動化スキル
- 繰り返し作業（毎週、毎月）→ テンプレート + 自動生成
- 情報整理（集計、比較、抽出）→ データ処理スキル
- ドキュメント作成 → 生成 + フォーマットスキル

### 職業別の特性

| 職業カテゴリ | 主なニーズ | 推奨スキルタイプ |
|-------------|-----------|-----------------|
| 営業 | レポート、提案書、顧客管理 | generator, tracker |
| 人事 | 評価、採用、研修 | evaluator, planner |
| 開発 | ドキュメント、コードレビュー | generator, analyzer |
| マーケティング | コンテンツ、分析、レポート | creator, analyzer |
| 経理・財務 | 帳票、集計、チェック | processor, validator |
| 法務 | 契約書、リスク分析 | analyzer, checker |
| 管理職 | 報告、承認、計画 | summarizer, planner |

## 実行手順

1. **CSVファイルの確認**
   - 必須列が揃っているか確認
   - 空行・不正データをスキップ

2. **各行を順次処理**
   - 職業・ペインポイントを分析
   - 最適なスキル設計を決定
   - SKILL.mdを生成

3. **生成結果の出力**
   ```
   ## 生成完了サマリー

   | # | 記入者 | スキル名 | ステータス |
   |---|--------|----------|-----------|
   | 1 | 田中太郎 | sales-report-generator | ✅ 生成完了 |
   | 2 | 鈴木花子 | interview-evaluator | ✅ 生成完了 |
   ...
   ```

4. **生成されたスキルの確認方法**
   ```bash
   ls generated-skills/
   cat generated-skills/田中太郎-sales-report-generator/SKILL.md
   ```

---

## 注意事項

- 生成されたスキルは `generated-skills/` ディレクトリに出力
- 既存の同名スキルがある場合は上書き確認
- 1つのCSVで最大50件程度のリクエストを推奨
- 複雑なスキルは後から手動で調整が必要な場合あり
