# 行业动态报告 · 视觉证据采集 Checklist

本清单用于执行行业动态趋势分析时，按需配备**图文/视频证据**。细则与引用格式见 `references/multimedia-curation.md`；具体是否必须、数量多少由行业实例约定（如 AI for Design 为可选、有则更好、不必多张）。**多媒体采集与分析**：可根据已找到的链接调用浏览器爬取/下载图片；图片与视频的内容分析建议 handoff 给具备多模态能力的模型执行。

## 目标

- 每份报告是否配图/视频、至少几条，见当前加载的行业实例（如 `references/industries/ai-for-design.md`）；部分行业为可选、有则更好、不必多张。
- 若配备截图/视频，须**服务于论证**：能体现新功能界面、工作流、数据可视化或演讲/发布会关键结论。

## 有效证据标准（满足其一即可）

- 新功能/新产品的界面或工作流（官方演示图、产品截图）。
- 数据可视化或官方指标图（基准测试、版本说明中的图表）。
- 演讲/发布会/评测视频中的关键片段（须标注时间点 mm:ss 与来源 URL）。

尽量高清、可读；截图须可追溯至具体来源页，视频须标注时间点与访问日期。

## 推荐来源优先级（证据强度从高到低）

1. 官方文档 / 官方博客 / 帮助中心 / Changelog / 发布会
2. 产品官网功能页、演示页
3. 权威媒体评测或设计社区报道（含发布日期）
4. 视频评测或大会演讲（可引用片段，标注视频来源与时间点）
5. 社区或用户生成内容（仅作补充，需标注来源强度）

## 采集执行（浏览器与多模态）

- **根据链接采集**：在检索得到信源链接后，**根据链接调用浏览器**（如 MCP cursor-ide-browser 或其它 web browser 能力）打开目标页面，进行**整页或区域截图**，或定位并**下载/引用页面内的图片与视频直链**；必要时滚动、展开折叠内容后再截图。截图或下载的素材须可追溯至具体 URL，并在来源索引中注明。
- **图文/视频分析**：对图片、视频进行**内容描述、界面解读、关键帧或演讲结论提取**时，建议 **handoff 给具备多模态能力的模型**执行；报告中引用多模态分析结论时须注明「图片/视频内容由多模态模型分析」，并与来源链接对应。

## 不足时的重试策略

- **换关键词**：加 feature 名、加「UI」「screenshot」「workflow」「documentation」「demo」「case study」。
- **换渠道**：官网 → 帮助中心 → 评测/YouTube → 设计媒体/社区（见 `references/source-discovery.md` 中各维度）。
- **换语言**：中英双语检索。
- **仍不足**：在报告中明确声明「证据不足，以下为推断/待验证」，并降低该条结论力度；来源索引中不虚构链接。

## 引用与索引

- 正文：配图用 `![描述](url)` 并紧跟 `[n]`，视频写「视频 [n]（时间点 mm:ss）」。
- 来源链接索引：截图/视频各占一条，格式见 `references/output-format.md`。
