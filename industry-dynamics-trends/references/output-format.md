# 报告结构与输出格式

本 Skill 的最终交付物为**一份便于快速查看的 Markdown 报告**。须按下列结构组织内容，并遵守引用与索引规范。

## 报告整体结构

1. **标题**：行业名 + 时间范围（如「AI for Design 行业前沿洞察（YYYY年X月）月刊」）。
2. **元信息（可选）**：初稿/校稿人、模型等，可用引用块简短标注。
3. **总览**：一段话（约 100–150 字）概括本周期内行业在「产品应用」与「模型/技术」侧对目标读者的实际影响，点明最值得关注的 1–2 个突破点。
4. **动态 1 ~ N**：每条动态包含固定子项（见下）；行业不同时，子项名称可替换为对应分析维度（如 AI for Design 用「最新动态 / 行业风向 / 设计启发点」）。
5. **总结与展望**：1–2 段，从读者角色（如企业设计管理）总结本周期动态对培训、工作流、工具采购或战略的建议。
6. **来源链接索引**：按正文引用序号 [1]、[2]… 列出完整来源表（见下）。

**配图与多媒体（可选/建议）**：部分动态可含 1 张配图或 1 个视频引用；配图/视频在正文中就近出现在对应动态内，并在「来源链接索引」中占一条（图片/视频 URL，并注明「截图」或「视频，时间点」）。具体最低条数由行业实例约定，详见 `references/multimedia-curation.md`。

## 单条动态结构（示例：AI for Design）

每条动态包含：

- **标题**：一句话总结该条的核心价值（如「Figma AI 自动重命名上线，降低图层规范管理成本」）。
- **【最新动态】**：客观陈述事件/能力及关键参数；句末必须带行内引用 `[n]`。
- **【行业风向】**：该动态对企业级交付标准、产研协同或工具链生态的实质影响（一句话）。
- **【设计启发点】**：针对 Leader 或一线设计师的可落地行动建议（一句话）。

其他行业可替换为「业务影响」「技术启示」「行动建议」等等价维度，但须保持每条结构一致。

## 引用规范

- **行内引用**：在需要标注来源的句末使用方括号数字，如 `[1]`、`[2]`，按出现顺序递增。
- **原声引用**：重要数据、关键结论或争议观点，尽量用引用块或引号给出原文片段，并紧跟 `[n]`。
- **关键数据**：涉及具体数字、版本、日期的，若已二次核查可写「已核查」；若无法核查须标注「待核查」。

## 来源链接索引格式

在报告最底部设「来源链接索引」模块，格式如下：

- **文字来源**：`[n] [文章/新闻标题] - [发布平台] (YYYY年X月X日)：[完整 URL]`
- **截图**：`[n] [描述] - 截图 - [发布平台/页面] (YYYY年X月X日)：[完整 URL]`
- **视频**：`[n] [标题] - 视频（时间点 mm:ss）- [发布平台] (YYYY年X月X日)：[完整 URL]`

示例：

```markdown
### 来源链接索引
[1] [文章/新闻标题] - [发布平台] (YYYY年X月X日)：[完整 URL]
[2] [某功能界面截图] - 截图 - [产品官网] (YYYY年X月X日)：[完整 URL]
[3] [发布会主题演讲] - 视频（时间点 12:34）- [YouTube] (YYYY年X月X日)：[完整 URL]
…
```

正文中出现的每一个引用序号都必须在索引中有且仅有一条对应记录；不得缺号或重复使用同一序号指代不同来源。

## 链接与点击体验（必选）

为便于读者从正文快速跳转到索引、并从索引新开标签页打开来源站，报告须遵守以下约定：

1. **正文中的引用序号**：使用**锚点链接**，格式为 `[n](#ref-n)`（n 为序号）。读者点击正文中的 [1]、[2] 等可**跳转到文末来源链接索引**中对应的那一行。
2. **来源链接索引**：
   - 每条索引项前增加锚点 id，便于正文跳转：`<a id="ref-n"></a>`（n 与引用序号一致）。
   - 索引中的**来源 URL 以可点击链接形式呈现，且在新标签页打开**：使用 HTML `<a href="完整URL" target="_blank" rel="noopener noreferrer">打开</a>`，或保留 URL 为链接文字。确保点击「打开」或 URL 时在**新标签页**打开目标网站。

撰写或生成报告时按上述方式输出，保证「点序号 → 到索引」「点链接 → 新开 tab 到来源」的体验一致。

## 条数建议

- 单次报告**最多筛选 20 条**高价值动态，宁缺毋滥。
- 总览与总结段落保持简洁，便于快速扫读。
