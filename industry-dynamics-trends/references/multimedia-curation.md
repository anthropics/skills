# 多媒体与证据采集（Multimedia Curation）

本规范定义行业动态报告中**图片与视频**的采集标准、采集手段、引用方式及与多模态分析的配合，与 `verification-curation.md` 并行使用：verification 负责可信度与时效，本文件负责证据形态与呈现。检索与撰写阶段须同时遵守两者。

## 适用场景

在以下情况应尽量配备图片或视频引用（具体条数由行业实例约定）：

- 新功能/新产品的**界面或工作流**（截图、官方演示图）。
- **数据可视化**（图表、基准测试图、官方指标截图）。
- **演讲、发布会、评测视频**中的关键结论或演示片段（需标注时间点）。

## 采集原则

- **优先级**：官方/一手（官网、官方博客、帮助中心、发布会、Changelog） > 权威评测/媒体报道 > 社区或用户生成内容；后者仅作补充且需标注来源强度。
- **视频**：须标注**时间点（mm:ss）**与**访问日期**；结论若来自视频内容，在来源索引中注明「视频，时间点 mm:ss」。
- **截图/图片**：须可追溯至具体来源页（URL）；避免无法确认出处的转存图。
- 时间窗口、信源交叉验证、挤水分、硬核过滤等仍按 `verification-curation.md` 执行。

## 采集手段（浏览器与下载）

在已通过检索获得信源链接的前提下，应**根据链接调用浏览器**进行页面级采集：

- **浏览器操作**：使用 MCP 浏览器（如 cursor-ide-browser @browser）或其它 Web 自动化能力，打开目标 URL，对页面进行**截图**或**定位并保存/引用页面内的图片**；必要时可执行滚动、点击以展开折叠内容后再截图。
- **下载与保存**：若页面提供可直链访问的图片/视频 URL，可通过浏览器访问或下载到约定目录，并在报告中引用可追溯的 URL 或本地路径（若报告分发生态支持）。
- 采集过程不改变「仅收录符合时间与信源优先级的内容」这一原则。

## 图文/视频分析（多模态 Handoff）

对已采集的**图片、视频**进行内容层面的分析（如界面元素解读、工作流描述、关键帧总结、演讲结论提取）时，应 **handoff 给具备多模态能力的模型**执行，以保证描述准确、可引用。报告中若引用多模态模型对某图/某视频的分析结论，须在对应位置或来源索引中注明「图片/视频内容由多模态模型分析」或等价说明，并与来源 URL 对应。

## 最低要求（可由行业实例覆盖）

- 通用默认：每份报告**至少若干条动态**配有 1 张图或 1 段视频引用（具体数量在行业实例中规定，如「至少 5 条」）。
- 若证据不足：换关键词（如加 feature 名、UI、screenshot、workflow、documentation）、换渠道（官网/帮助中心/评测/视频）后仍不足时，在报告中注明「证据不足，以下为推断/待验证」，并降低结论力度。

## 引用格式

- **正文**：配图或视频的句末使用与文字来源相同的行内引用 `[n]`；图片可用 Markdown 插图 `![描述](url)` 并紧跟 `[n]`，视频写「视频 [n]（时间点 mm:ss）」。
- **来源链接索引**（与 `output-format.md` 一致）：
  - 文字来源：`[n] [标题] - [发布平台] (日期)：[URL]`
  - 截图：`[n] [描述] - 截图 - [发布平台/页面] (日期)：[URL]`
  - 视频：`[n] [标题] - 视频（时间点 mm:ss）- [发布平台] (日期)：[URL]`

每条来源在索引中占一条，序号与正文引用一一对应，不得缺号或重复。

## 与 verification-curation 的关系

- **verification-curation**：时间戳、信源交叉验证、挤水分、硬核过滤、原声引用、关键数据二次核查、高价值/噪音分级。
- **multimedia-curation**：何时配图/视频、采集优先级、**采集手段（浏览器/下载）**、**图文与视频分析（多模态 handoff）**、最低条数、引用写法。

检索阶段同时遵守两者；撰写时在满足引用规范的前提下，将配图/视频就近放在对应动态中。
