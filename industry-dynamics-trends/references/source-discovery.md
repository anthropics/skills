# 信源发现维度（Source Discovery）

执行行业动态趋势分析时，在「检索」步骤除遵守 `verification-curation.md` 的核查要求外，应按**结构化渠道**展开检索，以减少遗漏、提高可复现性。行业实例可在此通用维度上**增删或重命名**，并指定本行业优先级。

## 通用检索维度（可被行业覆盖）

1. **主流与榜单**
   - 典型关键词/渠道：Best [Domain] tools、Top [Domain] software、年度榜单、行业报告（Gartner、Forrester、研报等）。
   - 用途：锚定主流玩家与权威盘点，便于交叉验证。

2. **新锐与发布**
   - 典型关键词/渠道：Product Hunt、Hacker News、New [Domain] tool、新产品/新版本发布（官方博客、Changelog、Release notes）。
   - 用途：捕捉新上线产品与近期版本更新。

3. **开源与社区**
   - 典型关键词/渠道：GitHub trending、[Domain] alternatives、Open source [Domain]、Reddit、Discord、知乎/即刻等社区讨论。
   - 用途：发现开源替代与社区真实反馈、摩擦点。

4. **视觉与案例**
   - 典型关键词/渠道：Dribbble [Domain]、Behance [Domain] concept、Case study、设计系统/组件库发布、大会演讲。
   - 用途：获取界面证据、案例研究与设计实践一手材料。

5. **生态与版图**
   - 典型关键词/渠道：[Domain] landscape map、[Domain] integrations、大厂官方博客、Design ops / 设计生态动态。
   - 用途：理解集成关系、生态位与平台级动态。

## 使用说明

- 执行分析时，按所选行业实例中定义的「信源发现维度」进行检索；若实例未定义，则采用上述五维作为默认。
- 同一事实尽量多维度交叉（如主流榜单 + 官方博客 + 社区）以符合 `verification-curation.md` 中的信源交叉验证要求。
- 行业实例可注明「典型关键词或站点示例」，便于执行时直接使用。
