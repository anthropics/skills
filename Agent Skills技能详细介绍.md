# 📚 Agent Skills 技能详细介绍

本文档详细介绍 Agent Skills 项目中每个技能的功能、特点和使用场景。

---

## 🎨 创意设计类技能

### 1. algorithmic-art - 算法艺术生成器

**功能描述：** 使用 p5.js 创建基于代码的生成艺术作品，强调种子随机性和交互式参数探索。

**核心特点：**
- 创建算法哲学（计算美学运动宣言）
- 生成交互式 HTML 制品，包含参数控件
- 支持种子随机性（Art Blocks 模式）
- 强调专家级工艺和精心完善

**使用场景：**
- 用户请求创建生成艺术、算法艺术、流场、粒子系统
- 需要带有可调参数的交互式艺术作品
- 创建艺术画廊质量的可视化作品

**避免：** 复制现有艺术家的作品（版权侵权风险）

**技术栈：**
- p5.js（JavaScript 生成艺术库）
- HTML/CSS（交互式控件）
- 种子随机性（可重复输出）

---

### 2. canvas-design - 画布设计

**功能描述：** 创建专业的视觉艺术作品，输出为 PDF 或 PNG 格式。

**核心特点：**
- 两步流程：设计哲学创建 → 视觉表达
- 强调最少文本、视觉优先
- 支持多种美学风格（野兽派、极简主义、有机等）
- 博物馆/杂志质量输出

**使用场景：**
- 用户请求创建海报、艺术品、设计作品
- 需要高质量的静态视觉作品
- 创建品牌宣传材料

**设计风格示例：**
- 混凝土诗歌：宏伟形式、大胆几何
- 色彩语言：色彩作为主要信息系统
- 模拟冥想：纹理、呼吸空间、负空间

**输出格式：** PDF 或 PNG 文件

---

### 3. frontend-design - 前端界面设计

**功能描述：** 创建具有高设计质量的独特、生产级前端界面。

**核心特点：**
- 避免"AI 草稿"美学（紫色渐变、Inter 字体、可预测布局）
- 强调独特字体选择、大胆调色板、运动效果
- 支持多种美学方向（野兽派、极繁主义、复古未来主义等）
- 生产级可用代码（HTML/CSS/JS、React、Vue）

**使用场景：**
- 构建 Web 组件、页面、落地页、仪表板
- 创建 React 组件、HTML/CSS 布局
- 任何 Web UI 的样式/美化

**设计元素：**
- 排版：独特、有趣的字体配对
- 颜色：一致的美学、主导色 + 强调色
- 运动：动画、微交互、滚动触发
- 空间：不对称、重叠、对角线流动
- 背景：渐变网格、噪声纹理、几何图案

### 4. brand-guidelines - 品牌风格指南

**功能描述：** 应用 Anthropic 官方品牌颜色和排版。

**核心特点：**
- 主色调：深色 `#141413`、浅色 `#faf9f5`
- 强调色：橙色 `#d97757`、蓝色 `#6a9bcc`、绿色 `#788c5d`
- 字体：Poppins（标题）、Lora（正文）

**使用场景：**
- 需要应用 Anthropic 品牌风格的制品
- 保持品牌一致性

**颜色规范：**
- 深色：主要文本和深色背景
- 浅色：浅色背景和深色背景上的文本
- 中灰色：次要元素
- 浅灰色：微妙的背景

## 📄 文档处理类技能

### 5. docx - Word 文档处理

**功能描述：** 创建、读取、编辑和操作 Word 文档（.docx）。

**核心功能：**
- **创建新文档**：使用 docx-js（JavaScript）
- **编辑现有文档**：解包 → 编辑 XML → 重新打包
- **支持特性**：修订、评论、目录、表格、图像、页眉页脚

**使用场景：**
- 生成报告、备忘录、信函、模板
- 编辑现有文档（添加修订/评论）
- 从 .docx 提取内容
- 创建专业格式化文档（目录、页码、标题样式）

**技术工具：**
- `docx` (npm) - 新建文档
- `scripts/office/unpack.py` - 解包
- `scripts/office/pack.py` - 重新打包
- `pandoc` - 文本提取

**关键规则：**
- 显式设置页面大小（默认 A4）
- 不使用 unicode 项目符号
- PageBreak 必须在 Paragraph 内
- 始终使用 DXA 设置表格宽度

---

### 6. pdf - PDF 文档处理

**功能描述：** 对 PDF 文件执行各种操作。

**核心功能：**
- **读取/提取**：文本提取（pypdf、pdfplumber）、表格提取
- **操作**：合并、拆分、旋转、添加水印
- **创建**：使用 reportlab 创建新 PDF
- **高级**：OCR（扫描 PDF）、密码保护、图像提取

**使用场景：**
- 合并/拆分 PDF 文件
- 从 PDF 提取文本或表格数据
- 旋转或重新排列页面
- 创建新 PDF 文档
- 填写 PDF 表单
- 对扫描 PDF 进行 OCR

**技术工具：**
- `pypdf` - 基本操作（合并、拆分、旋转）
- `pdfplumber` - 文本/表格提取
- `reportlab` - 创建 PDF
- `pdftotext`、`qpdf` - 命令行工具
- `pytesseract` + `pdf2image` - OCR

**快速参考：**
| 任务 | 最佳工具 |
|------|-----------|
| 合并 PDF | pypdf |
| 拆分 PDF | pypdf |
| 提取文本 | pdfplumber |
| 提取表格 | pdfplumber |
| 创建 PDF | reportlab |

### 7. pptx - PowerPoint 演示文稿

**功能描述：** 创建、编辑、读取 PowerPoint 演示文稿。

**核心功能：**
- **从头创建**：使用 PptxGenJS（JavaScript）
- **基于模板编辑**：解包 → 编辑 XML → 重新打包
- **设计理念**：避免单调布局，强调视觉元素

**使用场景：**
- 创建幻灯片组、推介文稿、演示文稿
- 基于模板创建专业演示
- 从 .pptx 提取内容
- 添加图表、图像、动画

**设计原则：**
- 每张幻灯片都需要视觉元素
- 变化布局（双栏、网格、半出血图像）
- 大胆的调色板（不要默认蓝色）
- 36-44pt 标题，14-16pt 正文

**推荐调色板：**
- 午夜行政：海军蓝 + 冰蓝 + 白色
- 森林与苔藓：森林绿 + 苔藓绿 + 奶油色
- 珊瑚能量：珊瑚色 + 金色 + 海军蓝

### 8. xlsx - Excel 电子表格

**功能描述：** 创建、编辑、分析 Excel 电子表格。

**核心功能：**
- **数据分析**：使用 pandas
- **公式/格式**：使用 openpyxl
- **财务模型**：行业标准颜色编码、数字格式
- **公式验证**：使用 scripts/recalc.py 重新计算

**使用场景：**
- 创建数据报告、财务模型
- 合并/拆分 Excel 文件
- 添加公式、图表、格式化
- 清理杂乱数据
- 从 Excel 提取数据

**颜色标准：**
- 蓝色文本：硬编码输入
- 黑色文本：公式
- 绿色文本：工作簿内链接
- 红色文本：外部链接
- 黄色背景：关键假设

**数字格式规则：**
- 年份：文本字符串（"2024"）
- 货币：$#,##0 格式
- 零值：显示为 "-"
- 百分比：0.0% 格式
- 负数：使用括号 (123)

## 🛠️ 开发工具类技能

### 9. mcp-builder - MCP 服务器构建器

**功能描述：** 创建高质量 MCP（模型上下文协议）服务器。

**核心流程：**
1. **研究和规划**：了解 API、选择工具
2. **实现**：Python (FastMCP) 或 Node/TypeScript (MCP SDK)
3. **审查测试**：代码质量、MCP Inspector 测试
4. **创建评估**：10 个 QA 对问题

**使用场景：**
- 集成外部 API 或服务到 LLM
- 创建自定义工具让 Claude 与第三方服务交互
- 构建标准化的模型上下文协议服务器

**技术栈：**
- TypeScript（推荐）或 Python
- 可流式 HTTP（远程）或 stdio（本地）
- Zod/Pydantic 数据验证

**关键原则：**
- 全面的 API 覆盖 vs 工作流工具
- 清晰、描述性的工具命名
- 简洁的工具描述
- 可操作的错误消息

### 10. skill-creator - 技能创建器

**功能描述：** 创建有效技能的指南。

**核心概念：**
- **简洁性**：上下文窗口是公共资源
- **自由度**：高（文本）、中（伪代码）、低（特定脚本）
- **渐进式披露**：元数据 → SKILL.md 正文 → 打包资源

**技能结构：**
```
skill-name/
├── SKILL.md (必需)
│   ├── YAML frontmatter (name, description)
│   └── Markdown 内容
├── scripts/ (可选) - 可执行代码
├── references/ (可选) - 文档资料
└── assets/ (可选) - 输出资源文件
```

**使用场景：**
- 创建新的自定义技能
- 更新现有技能
- 学习最佳技能设计模式

**关键原则：**
- 只包含直接支持功能的基本文件
- 避免冗余文档（README、INSTALLATION_GUIDE 等）
- 保持 SKILL.md 精简（<500 行）
- 详细参考资料放入 references/

### 11. webapp-testing - Web 应用测试

**功能描述：** 使用 Playwright 测试本地 Web 应用。

**核心功能：**
- 侦察-然后-行动模式
- 支持 `with_server.py` 管理服务器生命周期
- DOM 检查、元素发现、控制台日志捕获

**使用场景：**
- 验证前端功能
- 调试 UI 行为
- 捕获浏览器截图
- 查看浏览器日志

**决策树：**
- 静态 HTML → 直接读取
- 动态 webapp → 侦察 → 行动

**最佳实践：**
- 将捆绑脚本用作黑盒
- 使用 `sync_playwright()` 同步脚本
- 完成后关闭浏览器
- 添加适当的等待

### 16. slack-gif-creator - Slack GIF 创建器

**功能描述：** 创建为 Slack 优化的动画 GIF。

**核心要求：**
- 尺寸：Emoji GIF 128x128，消息 GIF 480x480
- FPS：10-30（越低文件越小）
- 颜色：48-128（越少文件越小）
- 持续时间：<3 秒（表情符号）

**使用场景：**
- 为 Slack 创建自定义表情符号
- 创建消息动画 GIF
- 创建团队动态图

**动画概念：**
- 摇动/振动：使用 sin/cos 振荡
- 脉冲/心跳：缩放对象大小
- 弹跳：使用 bounce_out 缓动
- 旋转/自转：图像旋转
- 淡入/淡出：alpha 通道调整
- 滑动：从屏幕外移动到位置
- 缩放：缩放和定位
- 爆炸/粒子爆发：向外辐射的粒子

**优化策略：**
- 减少帧数（降低 FPS）
- 减少颜色（48 vs 128）
- 减小尺寸（128x128 vs 480x480）
- 移除重复帧
- Emoji 模式自动优化

## 最佳实践

### 1. 技能组合使用
多个技能可以组合使用来完成复杂任务。例如：
- `pptx` + `theme-factory` → 创建主题化演示文稿
- `docx` + `brand-guidelines` → 创建品牌化文档
- `algorithmic-art` + `frontend-design` → 创建交互式艺术网站

### 2. 质量保证
大多数文档类技能都强调质量保证：
