# AI Advisor Workflow トリガーワード ガイド

## 概要
トリガーワードを使用することで、自然な日本語でAI Advisor Workflowを簡単に実行できます。

## 🎯 基本的な使い方

### 1. 分析・調査系
企業のWebサイトを分析したい時に使用します。

```
例:
• "https://example.com を分析して"
• "株式会社サンプルのホームページを調査して"
• "〇〇会社について調べて"
• "このサイトをチェック https://example.com"
```

### 2. 提案書作成系
AI活用提案書を作成したい時に使用します。

```
例:
• "https://example.com のAI活用提案書を作成"
• "〇〇株式会社向けの提案を作って"
• "AI導入プランを提案して"
• "効率化に特化した提案書を準備"
```

### 3. スキル生成系
AgentSkillsを自動生成したい時に使用します。

```
例:
• "在庫管理のスキルを作成して"
• "〇〇会社用のAgentSkillを生成"
• "顧客対応ツールを実装"
• "データ分析スキルを開発"
```

### 4. フルワークフロー系
分析から提案、スキル生成まで全て実行したい時に使用します。

```
例:
• "https://example.com のフルワークフローを実行"
• "分析から提案まですべてお願い"
• "〇〇会社のAI導入をトータルサポート"
• "最初から最後まで全部やって"
```

### 5. クイックチェック系
素早く簡易的な分析をしたい時に使用します。

```
例:
• "ざっくりhttps://example.comをチェック"
• "簡単に〇〇会社を見て"
• "さっと概要を確認"
• "とりあえず軽く分析"
```

## 🔧 オプションの指定方法

### スキル生成の制御
```
• "スキル不要で提案書だけ作成"
• "提案書作成（スキルなし）"
• "no skills でお願い"
```

### ROI計算の制御
```
• "ROI計算不要で分析"
• "ROIなしで提案書作成"
• "no roi でシンプルに"
```

### カテゴリの指定
```
• "効率化に特化した提案"
• "顧客体験向上の観点で分析"
• "データ分析重視でお願い"
• "新規事業の可能性を探る"
```

### 複数URLの指定
```
• "https://example.com と https://example.com/services を分析"
• "以下のURLを調査: URL1, URL2, URL3"
• "メインサイトとサービスページを含めて分析"
```

## 💡 便利な使い方

### 1. 会社名からの自動URL推測
URLが分からない場合、会社名から自動的に推測します。

```
入力: "株式会社テクノロジーを分析"
→ 自動推測: https://technology.co.jp, https://www.technology.com など
```

### 2. 段階的な実行
まずクイックチェックで概要を確認してから、詳細分析に進むことができます。

```
1. "ざっくりhttps://example.comを見て"
2. "詳しく分析して提案書作成"
3. "良さそうならスキルも生成"
```

### 3. カスタマイズした提案
特定の課題や目的に合わせて提案をカスタマイズできます。

```
• "業務効率化と顧客体験向上に絞って提案"
• "中小企業向けの現実的な提案"
• "予算を抑えたAI活用プラン"
```

## 🎨 トリガーワードのカスタマイズ

`trigger_words_config.yaml`を編集することで、独自のトリガーワードを追加できます。

```yaml
trigger_words:
  custom_action:
    keywords:
      - カスタムキーワード
      - 独自の表現
    patterns:
      - "独自のパターン(.+)"
```

## ⚡ パフォーマンスのヒント

### 高速化のコツ
1. **具体的なURLを指定**: 会社名よりもURLの方が高速
2. **必要な機能だけ実行**: フルワークフローよりも個別実行
3. **クイックチェックから開始**: まず概要を把握

### メモリ節約のコツ
1. **URLを絞る**: 必要最小限のページのみ指定
2. **スキル生成を分ける**: 提案確認後に別途生成
3. **カテゴリを限定**: 特定分野に絞って分析

## 🚨 トラブルシューティング

### トリガーが認識されない場合
- より明確なキーワードを使用
- URLを明示的に含める
- ヘルプで使用例を確認

### URLが推測できない場合
- 完全なURLを指定
- 複数の候補URLを列挙
- 会社名を正確に記載

### 処理が遅い場合
- クイックモードを使用
- URLの数を減らす
- 特定カテゴリに限定
