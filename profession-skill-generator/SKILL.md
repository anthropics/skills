# Profession Skill Generator

参加者の職業とペインに最適化された**5機能統合Agent Skill**を**完全動的に生成**するサブエージェント。
事前定義されたテンプレートは使用せず、入力された職業・ペインから全てを動的に分析・設計・生成する。

**配布形式**: 1参加者 = 1ZIP（即利用可能）

## Purpose

1人の参加者情報（職業・ペイン）を受け取り、その人にシンデレラフィットする
5機能を統合した1つのAgent Skillを分析・設計・生成・ZIP化する。

**対応職業**: 任意（エンジニア、看護師、建築家、パティシエ、弁護士、農家、etc.）

## Input Requirements

```
- 名前: {参加者名}
- 職業: {職業名}
- ペイン: {具体的な課題・悩み}
- 出力先: {ディレクトリパス}
```

## Workflow

### Phase 1: 職業特性メタ分析（5軸評価）

対象職業を以下の5軸で動的に評価:

| 軸 | 評価オプション |
|---|---|
| **時間軸** | 過去志向（記録・分析）/ 現在志向（処理・管理）/ 未来志向（計画・創造）|
| **対象軸** | 内部特化（組織内）/ 外部特化（対外）/ 内外統合 |
| **処理軸** | ルーチン重視（定型）/ 判断重視（専門）/ 創造重視 |
| **価値軸** | コンプライアンス / オペレーション / イノベーション |
| **専門軸** | 汎用型 / 専門型 / 統合型 |

**出力**: その職業の特性プロファイル

### Phase 2: 7業務カテゴリー動的生成

職業特性に基づき、MECE原則で7つの業務カテゴリーを**動的に設計**:

- **コアカテゴリー**: 2〜3（職業の核心業務）
- **サポートカテゴリー**: 2〜3（支援業務）
- **成長カテゴリー**: 1〜2（スキルアップ・改善）

命名規則: 「○○・○○業務」形式

**重要**: 事前定義されたカテゴリーは存在しない。職業ごとに完全に新規生成する。

### Phase 3: 35項目AI活用分析（動的）

**CRITICAL**: 参加者のペインを最優先で分析

1. 各カテゴリーに対して5つの業務フローを特定
2. 各業務フローに対してAI活用案を検討
3. 合計35項目（7カテゴリー × 5項目）を分析

**分析スキーマ:**

| 業務フロー | AI活用案 | 活用例詳細 | 難易度 | 評価 |
|---|---|---|---|---|
| {具体的な業務} | {15〜20文字} | {80〜100文字} | ★☆☆〜★★★ | A/B/C |

**評価基準:**
- **A（即採用）**: ペインに直結 or 効果絶大
- **B（検討）**: 有用だが優先度中
- **C（将来）**: 将来的に検討

### Phase 4: Agent Skill提案（動的）

Phase 3の分析結果（特に評価A項目）から、Agent Skill化の候補を**動的に提案**:

各候補について以下を検討:

```
候補スキル: {スキル名案}
- 元となる業務フロー: {Phase 3の項目}
- 解決するペイン: {参加者のペインとの関連}
- 想定利用頻度: 週{N}回
- 繰り返し性: 高/中/低
- 構造化可能性: 高/中/低
- 効果の大きさ: 高/中/低
- 実装難易度: ★☆☆〜★★★
- スキル化適性スコア: {1-10}
```

**提案数**: 8〜12個（選定のための余裕を持たせる）

### Phase 5: 5機能選定（動的）

Phase 4の提案から、以下の優先度マトリクスで**5つを選定**:

| 繰り返し性 | 構造化可能性 | 優先度 |
|---|---|---|
| 高（週5回+） | 高 | 最優先 |
| 高 | 中 | 高 |
| 中 | 高 | 高 |
| 中 | 中 | 中 |
| 低 | 低 | 対象外 |

**シンデレラフィット条件（すべて満たす）:**
1. ペインに直接対応している
2. 週3回以上の利用見込み
3. 入力→処理→出力が明確
4. 30分〜2時間で実装可能
5. 使い始めてすぐに効果を実感できる

**選定時の多様性確保:**
- ペイン直結機能: 2〜3個
- 業務効率化機能: 1〜2個
- 品質向上機能: 1個

### Phase 6: 統合SKILL.md生成（動的）

選定した5機能を**1つの統合スキル**として生成:

**ファイル構造:**
```
{output_path}/
└── {participant-name}-ai-assistant/
    └── SKILL.md
```

**統合SKILL.mdテンプレート:**

```markdown
---

  - 「{機能2のトリガー}」→ 機能2実行
  - 「{機能3のトリガー}」→ 機能3実行
  - 「{機能4のトリガー}」→ 機能4実行
  - 「{機能5のトリガー}」→ 機能5実行
```

```markdown
## 機能1: コードレビュー要約

### Purpose
PRの差分を分析し、変更の全体像を素早く把握できる要約を生成する。

### Workflow
1. PR/diffの内容を受け取る
2. 変更ファイル、変更行数、主な変更点を抽出
3. 構造化された要約を生成

### Input
- GitHubのPR URL、またはdiffテキスト

### Output
- 変更概要（1-2文）
- 変更ファイル一覧（重要度順）
- 主な変更点（箇条書き）
- レビュー時の注目ポイント

### Example
**入力**: PR #123 のdiff（500行の変更）
**出力**:
```
## 変更概要
認証モジュールにOAuth2.0サポートを追加。主にauth/フォルダに変更。

## 変更ファイル（重要度順）
1. auth/oauth.ts (新規) - OAuth2.0クライアント実装
2. auth/index.ts - エクスポート追加
3. config/auth.ts - 設定項目追加

## 注目ポイント
- トークンの保存方法（セキュリティ観点）
- エラーハンドリングの網羅性

## Constraints

- **DO**: 任意の職業に対応する（テンプレートに縛られない）
- **DO**: 5機能を1つの統合スキルとして生成
- **DO**: ZIP形式で配布可能な形で出力
- **DO**: 各機能に明確なトリガーフレーズを設定
- **DO**: インストール手順を含める
- **DO NOT**: 5つの個別スキルを生成しない（統合版のみ）
- **DO NOT**: 汎用的すぎる機能を含めない
- **DO NOT**: 実装困難（★★★）な機能を選ばない
- **DO NOT**: 既存の document-skills (docx/xlsx/pdf/pptx) と重複しない
