# エンジニア・開発者向けスキルテンプレート

## 5軸評価の傾向

| 軸 | 典型的評価 |
|---|---|
| 時間軸 | 現在志向〜未来志向 |
| 対象軸 | 内部特化 |
| 処理軸 | 判断重視〜創造重視 |
| 価値軸 | オペレーション |
| 専門軸 | 専門型 |

## 7業務カテゴリー例

1. **要件定義・設計業務** - 仕様策定、アーキテクチャ設計
2. **開発・実装業務** - コーディング、機能実装
3. **コードレビュー・品質管理業務** - レビュー、静的解析
4. **テスト・検証業務** - ユニットテスト、E2Eテスト
5. **運用・保守業務** - 監視、インシデント対応
6. **技術調査・学習業務** - 技術選定、PoC
7. **チーム連携・ドキュメント業務** - 技術文書、ナレッジ共有

## よくあるペインと対応スキル案

### ペイン: コードレビューに時間がかかる
- `code-review-summarizer` - PR差分を自動要約
- `review-checklist-generator` - レビュー観点チェックリスト生成
- `code-smell-detector` - アンチパターン検出

### ペイン: ドキュメント作成が面倒
- `readme-generator` - README自動生成
- `api-doc-builder` - APIドキュメント生成
- `code-comment-enhancer` - コメント追加・改善

### ペイン: バグ調査に時間がかかる
- `error-log-analyzer` - エラーログ解析
- `stack-trace-explainer` - スタックトレース解説
- `debug-hypothesis-generator` - デバッグ仮説生成

### ペイン: 設計判断に迷う
- `architecture-advisor` - アーキテクチャ相談
- `tech-comparison-matrix` - 技術比較表生成
- `design-pattern-suggester` - デザインパターン提案

### ペイン: テスト作成が大変
- `unit-test-generator` - ユニットテスト雛形生成
- `test-case-designer` - テストケース設計支援
- `mock-data-generator` - モックデータ生成

## スキル命名規則

- 動詞 + 名詞 の kebab-case
- 例: `code-review-summarizer`, `test-generator`, `log-analyzer`
