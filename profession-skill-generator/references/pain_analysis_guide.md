# ペイン分析ガイド

参加者のペイン（課題・悩み）を分析し、最適なスキルを導出するためのガイド。

## ペイン分類フレームワーク

### 1. 時間系ペイン

**キーワード**: 時間がかかる、面倒、手間、毎回、繰り返し

| ペイン例 | スキル化の方向性 | 優先度 |
|---|---|---|
| 「○○に時間がかかる」 | 自動化・効率化 | 高 |
| 「毎週○○するのが面倒」 | 定期実行・テンプレート化 | 高 |
| 「同じ作業の繰り返し」 | バッチ処理・一括実行 | 高 |
| 「手作業が多い」 | 自動生成・変換 | 中 |

**スキル命名パターン**: `{対象}-automator`, `{対象}-generator`, `quick-{対象}`

### 2. 品質系ペイン

**キーワード**: 不安、ミス、品質、正確性、抜け漏れ

| ペイン例 | スキル化の方向性 | 優先度 |
|---|---|---|
| 「ミスが心配」 | チェック・検証 | 高 |
| 「品質が安定しない」 | 標準化・テンプレート | 中 |
| 「抜け漏れがある」 | チェックリスト・網羅確認 | 高 |
| 「正確性が不安」 | バリデーション・レビュー | 中 |

**スキル命名パターン**: `{対象}-checker`, `{対象}-validator`, `{対象}-reviewer`

### 3. 複雑性系ペイン

**キーワード**: 複雑、難しい、わからない、管理が大変

| ペイン例 | スキル化の方向性 | 優先度 |
|---|---|---|
| 「管理が複雑」 | 整理・可視化・簡略化 | 中 |
| 「把握しきれない」 | 要約・ダッシュボード化 | 中 |
| 「どこに何があるかわからない」 | 検索・インデックス化 | 高 |
| 「判断が難しい」 | 分析・提案・意思決定支援 | 中 |

**スキル命名パターン**: `{対象}-organizer`, `{対象}-summarizer`, `{対象}-finder`

### 4. 創造性系ペイン

**キーワード**: アイデア、ネタ、発想、何を書けばいい

| ペイン例 | スキル化の方向性 | 優先度 |
|---|---|---|
| 「アイデアが出ない」 | アイデア生成・ブレスト支援 | 中 |
| 「何を書けばいいかわからない」 | ドラフト生成・構成提案 | 高 |
| 「毎回ゼロから考える」 | テンプレート・雛形提供 | 高 |
| 「マンネリ化している」 | バリエーション提案 | 低 |

**スキル命名パターン**: `{対象}-idea-generator`, `{対象}-drafter`, `{対象}-suggester`

### 5. コミュニケーション系ペイン

**キーワード**: 伝え方、説明、報告、調整

| ペイン例 | スキル化の方向性 | 優先度 |
|---|---|---|
| 「うまく伝えられない」 | 文章作成・翻訳・要約 | 中 |
| 「報告資料作成が大変」 | レポート自動生成 | 高 |
| 「会議の準備が面倒」 | アジェンダ・資料作成 | 中 |
| 「議事録が追いつかない」 | 議事録・要約生成 | 高 |

**スキル命名パターン**: `{対象}-report-builder`, `{対象}-summarizer`, `{対象}-translator`

## ペイン→スキル変換マトリクス

```
ペインの深刻度 × 発生頻度 = スキル優先度

高頻度（毎日〜週数回）+ 高深刻度 → 最優先でスキル化
高頻度 + 低深刻度 → 効率化系スキル
低頻度 + 高深刻度 → 品質保証系スキル
低頻度 + 低深刻度 → スキル化不要
```

## シンデレラフィット判定基準

スキルが参加者にシンデレラフィットするための条件:

### 必須条件（すべて満たす）
1. **ペイン直結**: 記載されたペインを直接解決する
2. **即効性**: 使ったその日から効果を実感できる
3. **低学習コスト**: 5分以内で使い方がわかる
4. **高頻度利用**: 週3回以上使う見込み

### 加点条件
- 職業の核心業務に関わる
- 定量的な効果が見込める（時間削減、品質向上）
- 既存ツールでは解決できない

### 減点条件
- 汎用的すぎる（誰でも使える = フィットしていない）
- 実装が複雑（★★★）
- 稀にしか使わない

## ペイン深掘り質問（必要に応じて）

ペインが抽象的な場合、以下を推測または想定:

1. **具体的な業務は？** → どの業務で発生するか
2. **頻度は？** → 毎日/週/月/四半期
3. **影響は？** → 時間損失/品質低下/ストレス
4. **現状の対処法は？** → 手作業/ツール/諦め
5. **理想の状態は？** → 自動化/半自動/サポート

## 出力例

**入力:**
```
職業: プロダクトマネージャー
ペイン: ステークホルダーへの説明に時間がかかる
```

**分析:**
- ペイン分類: 時間系 + コミュニケーション系
- 深刻度: 中〜高（意思決定に影響）
- 頻度: 高（週複数回）
- 優先度: **最優先**

**スキル候補:**
1. `stakeholder-deck-builder` - 説明資料自動生成
2. `feature-explainer` - 機能説明文作成
3. `trade-off-visualizer` - トレードオフ可視化
4. `meeting-prep-pack` - 会議準備パック
5. `question-anticipator` - 想定質問と回答準備
