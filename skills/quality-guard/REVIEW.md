## 按主题归类

### 主题一：流程完整性（状态机缺陷）

> 核心问题：流程是严格的单向管道，缺乏回退、中断恢复和状态失效处理。

- **H1** 歧义检测状态（behavior-map 含 `[❓]`）
- **H2** 无回退机制（Phase 2 → Phase 1）
- **H3** Staleness 检查时机未定义
- **M20** Phase 3 部分完成后重入不明确
- **M21** init 在 project.md fresh 时行为含糊
- **M23** scope.md 状态转换规则不完整
- **M24** 灾难性失败无恢复策略
- **M25** 用户无法跳过/选择 Phase
- **L2** Phase 间衔接方式不明确

### 主题二：遗留系统适配

> 核心问题：Skill 假设代码有类型、有文档、可本地运行、有 DI——遗留系统通常全部不满足。

- **H4** Mock 约束在无 DI 场景下不可行（E2E 也不可行时怎么办？）
- **H6** 无法本地运行代码的处理策略缺失
- **H12** 无类型/动态语言行为映射缺乏指导
- **H13** Tier 1 ↔ Tier 3 之间的覆盖空白
- **M7** "公共接口"概念模糊
- **M8** 单体应用 scope 划分策略不足
- **M9** 死代码处理缺失
- **M10** 混合测试框架
- **M11** 全局状态/单例测试隔离
- **M12** Monorepo
- **M14** 测试数据库替代方案

### 主题三：AI 代码特有问题

> 核心问题：AI 代码的缺陷模式与人写代码不同，需要专项检测机制和适配工作流。

- **H7** 幻觉 API 检测
- **H8** 跨函数/跨文件契约一致性
- **H9** 对抗性思维缺乏可操作清单
- **H10** "刚生成整个模块"入口模式缺失
- **H11** 全新代码意图锚点全部缺失的处理
- **M16** AI 代码 behavior-map 可靠性存疑
- **M17** 复制粘贴模式检测
- **M18** Tier 1 未包含 AI prompt 文件
- **M19** 一次性验证轻量模式

### 主题四：测试方法论完善

> 核心问题：测试层级推荐、优先级判断、mock 边界判定等核心逻辑需要细化。

- **H5** 测试环境预检缺失
- **H14** 测试代码 bug vs 行为差异未区分
- **M1** 层级推荐表覆盖不足
- **M2** 优先级判断矩阵缺失
- **M3** 隐式行为未被捕获
- **M4** 测试间依赖无表达方式
- **M6** 无意图锚点的优先级传递规则缺失
- **M13** mtime 刷新机制不可靠（建议改用 git hash）
- **M15** behavior-map 自给自足规则在巨型类中不实际

### 主题五：报告与产物

> 核心问题：Discovery Report 缺少可操作性和持久化。

- **M5** 报告无建议行动、无严重程度分级
- **M26** 报告未持久化为 artifact
- **M27** AI 代码依赖预检
