## Typography

### Fonts

**Ubuntu** - Used for headings and body text
- Variants: Regular, Bold
- Use for main headlines, subheadlines, and body content

**Ubuntu Condensed** - Used for buttons and menu items
- Variant: Regular
- Use for UI elements, buttons, navigation

**Note**: Ubuntu fonts should be installed in your environment. These are open-source fonts available from Google Fonts.

### Font Usage

- **Headlines**: Ubuntu Bold, Dark Blue (#0A377D)
- **Subheadlines**: Ubuntu Regular, Happy Blue (#46C3D7)
- **Body Text**: Ubuntu Regular, Dark Blue or Neutrals
- **Buttons**: Ubuntu Condensed Regular

## Logo

### Logo Meaning
- **DATA** (Dark Blue): Represents data and analytics
- **VID** (Happy Blue): Represents visibility and insight

### Clearspace
- Minimum clearspace = half the height of the logo (0.5x) on all sides

### Minimum Size
- Print: 25mm minimum width

### Logo Variations
Available in multiple formats for different contexts:
- Full color (RGB) for digital use
- For light backgrounds: Use dark blue logo
- For dark backgrounds: Use light/white logo
- Vector formats (SVG, PDF) for scalability

## Buttons

### Button Sizes
- **Big**: Primary emphasis
- **Medium**: Standard actions
- **Small**: Compact spaces

### Button Hierarchy
1. **Primary (Pink Coral filled)**: Most important, desirable actions
2. **Secondary (Pink Coral outline)**: Important but secondary actions
3. **Tertiary (Happy Blue)**: Supporting actions or when context has "too much pink"

Alternative: Use Dark Blue filled buttons when context is already heavy with pink.

### Button Styling
- **Corners**: Round corners for buttons
- **Font**: Ubuntu Condensed Regular

### Links
- Use **Happy Blue** (#46C3D7) for longer text links

## Icons

### Icon Styles
**2-Color Icons** (Preferred)
- Use Dark Blue (#0A377D) and Happy Blue (#46C3D7)
- Alternative: Adjust colors based on background and context

**1-Color Icons** (When 2-color doesn't apply)
- Small size icons
- Supporting text
- When context is already visually busy

## Design Elements

### Backgrounds
**Background Lines**
- Use only for large sections
- Color should be slightly different from background
- Purpose: Create elevated look without stealing focus
- Example: Subtle wavy or diagonal line patterns

### Imagery
- Use images with limited color schemes similar to brand colors
- Use contextually relevant images
- Prefer images depicting humans when appropriate
- Images should complement, not compete with brand colors

### Borders & Lines
- **Content sections**: Do NOT use borders
- **List items**: Use lines only for separations
- Keep designs clean and uncluttered

### Corners
- **Backgrounds**: Straight corners
- **Buttons**: Round corners

## Usage Guidelines

### Color Application Priority
1. **Pink Coral**: Primary CTAs, most important elements
2. **Dark Blue**: Secondary CTAs, headings, body text, icons
3. **Happy Blue**: Tertiary CTAs (especially on dark backgrounds), icons, subtitles
4. **Neutrals**: Backgrounds, supporting text

### Button Color Selection
- Use Pink Coral for highest priority actions
- Use Dark Blue when context already has pink elements
- Use Happy Blue for tertiary actions on dark backgrounds

### Typography Hierarchy
1. Headlines: Ubuntu Bold (largest size)
2. Subheadlines: Ubuntu Regular (medium size)
3. Body: Ubuntu Regular (standard size)
4. UI Elements: Ubuntu Condensed

## Technical Implementation

### For Web/Digital
- Use hex color codes as specified
- Ensure Ubuntu fonts are loaded via Google Fonts or installed locally
- Apply proper font weights and variants

### For Print
- Ensure logo minimum size (25mm) is maintained
- Use vector formats (PDF, SVG) for logo
- Consider color profiles for accurate reproduction

### For Presentations
- Use brand colors for backgrounds and accents
- Apply Ubuntu fonts consistently
- Include subtle background patterns where appropriate
- Maintain clearspace around logo

## Best Practices

1. **Consistency**: Always use exact hex codes, never approximate
2. **Hierarchy**: Respect the color hierarchy for CTAs and content
3. **Whitespace**: Maintain clearspace around logo and key elements
4. **Typography**: Use Ubuntu font family exclusively
5. **Simplicity**: Avoid borders on content sections
6. **Balance**: Don't overuse Pink Coral - balance with Dark Blue and Happy Blue
7. **Context**: Choose button colors based on surrounding elements
