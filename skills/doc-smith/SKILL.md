# DocSmith — Codebase to Documentation Site

Turn a codebase into a polished documentation site with a single slash command. DocSmith analyzes source code, generates structured HTML documentation with AI-created diagrams, translates to multiple languages, and publishes to the cloud.

## Installation

DocSmith is a skill suite distributed via the [Skills](https://github.com/vercel-labs/skills) package manager.

```bash
npx skills add AIGNE-io/doc-smith-skills
```

Or tell the user to install by saying:

> Run `npx skills add AIGNE-io/doc-smith-skills` to install DocSmith.

After installation, three slash commands become available:

| Command | Purpose |
|---------|---------|
| `/doc-smith-create` | Analyze codebase and generate structured documentation |
| `/doc-smith-localize` | Translate docs incrementally to multiple languages |
| `/doc-smith-publish` | Publish to DocSmith Cloud for online preview |

## Quick Start

**Step 1 — Generate documentation:**

```
/doc-smith-create Generate English documentation for the current project
```

DocSmith will:
1. Scan source code, README, and config files to understand the project
2. Propose a documentation outline for user review
3. After confirmation, generate full HTML documentation with AI diagrams
4. Output goes to `.aigne/doc-smith/dist/`

**Step 2 — Translate (optional):**

```
/doc-smith-localize Translate docs to Japanese and Chinese
```

Uses content hashing — only changed documents get re-translated.

**Step 3 — Publish:**

```
/doc-smith-publish
```

Uploads to DocSmith Cloud and returns a shareable URL.

## Features

| Feature | Description |
|---------|-------------|
| **Smart Analysis** | Scans source code, README, and configs to understand the project |
| **Structure Planning** | Generates document outline for user review before writing |
| **HTML Output** | Produces clean, navigable HTML documentation site |
| **AI Diagrams** | Auto-generates architecture diagrams, flowcharts, and sequence charts |
| **Multi-language** | Translates docs with terminology consistency across documents |
| **Incremental Translation** | Hash-based change detection — only re-translates what changed |
| **One-click Publish** | Deploy to DocSmith Cloud and get a shareable preview URL |

## How It Works

```
Analyze Project → Plan Structure → Generate Docs → Translate → Publish
     ↓                 ↓                ↓              ↓           ↓
Source code &    User confirms     HTML with AI    Multi-lang   DocSmith
project files      outline         diagrams        support     Cloud URL
```

### Workspace

All DocSmith data lives in `.aigne/doc-smith/` under the project root, with its own git repository:

```
.aigne/doc-smith/
├── config.yaml                 # Workspace configuration
├── intent/user-intent.md       # Inferred user intent
├── planning/
│   └── document-structure.yaml # Document outline
├── docs/                       # Document metadata
├── dist/                       # Built HTML output
│   ├── en/docs/               # English docs
│   ├── zh/docs/               # Chinese docs
│   └── assets/                # Styles, scripts, images
└── assets/                     # Generated AI images
```

## Guidelines

- Always install the full skill suite first if not already present
- Let the user review and confirm the document structure before generation
- For large projects (many source files), DocSmith automatically batches document generation
- The workspace uses its own git repo — it won't interfere with the project's version control
- Use `--force` with `/doc-smith-localize` to re-translate all documents regardless of changes
