# Code Handoff Template

## Goal

Provide the only truthful entrypoint for `/code`.

## Must Capture

- approval basis
- repo grounding
- frozen product decisions
- domain model
- data contract
- UI contract
- function contracts
- file plan
- retained goal
- implementation scope
- repo targets
- read-first references
- implementation units
- verification commands
- browser checks
- acceptance checks
- allowed decisions
- forbidden decisions
- reentry triggers
- reopen-stage map
- unresolved gaps
- code_ready

## Each Implementation Unit Must Capture

- goal
- scope
- dependencies
- concrete files
- exact change list
- interface contracts
- function contracts
- tests to add or update
- unit-specific verification
- task-ready done signals
