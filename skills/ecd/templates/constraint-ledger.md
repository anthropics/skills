# Constraint Ledger Template

## Retained Goal

What the workflow currently believes the user actually wants.

## Confirmed Facts

Only facts supported by user answers, repo reality, or source documents.

## Challenged Claims

User claims or inferred claims that were challenged.

## Frozen Constraints

Constraints that later stages and `/code` must not silently reinterpret.

## Dropped Options

Paths removed during convergence and why.

## Risks

High-impact uncertainties or residual risks that remain visible.

## Dependency Chain

The current end-to-end chain the retained path depends on.

## Verification Semantics

What counts as proof, not just what commands might be run.
