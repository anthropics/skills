此技能指导创建独特、生产级的前端界面，避免通用的"AI 草稿"美学。实现真正可用的代码，对美学细节和创意选择给予极大的关注。

用户提供前端需求：要构建的组件、页面、应用程序或界面。他们可能包括有关目的、受众或技术约束的上下文。

## 设计思维

在编码之前，了解上下文并致力于大胆的美学方向：
- **目的**：此界面解决了什么问题？谁使用它？
- **基调**：选择一个极端：野兽派极简主义、极繁主义混乱、复古未来主义、有机/自然、奢华/精致、好玩/玩具般、编辑/杂志、野兽派/原始、装饰艺术/几何、柔和/粉彩、工业/实用主义等。有太多风味可供选择。使用这些作为灵感，但设计一个真正符合美学方向的。
- **约束**：技术要求（框架、性能、可访问性）。
- **差异化**：是什么让这令人难忘？人们会记住的一件事是什么？

**关键**：选择清晰的概念方向并精确执行。大胆的极繁主义和精致的极简主义都有效——关键是意图性，而不是强度。

然后实现真正的可用代码（HTML/CSS/JS、React、Vue 等），这些代码是：
- 生产级和功能性的
- 视觉上令人印象深刻和难忘的
- 与清晰的美学观点一致的
- 在每个细节上都经过精心完善

## 前端美学指南

专注于：
- **排版**：选择美丽、独特和有趣的字体。避免通用字体，如 Arial 和 Inter；而是选择提升前端美学的独特选择；意想不到、富有特色的字体选择。将独特的显示字体与精致的正文字体配对。
- **颜色和主题**：致力于一致的美学。使用 CSS 变量以保持一致性。具有鲜明强调色的主导颜色优于胆小、均匀分布的调色板。
- **运动**：使用动画进行效果和微交互。优先为 HTML 使用仅 CSS 的解决方案。在可用时为 React 使用 Motion 库。专注于高影响力的时刻：一个精心编排的页面加载，带有错开的显示（animation-delay）比分散的微交互产生更多的快乐。使用滚动触发和令人惊讶的悬停状态。
- **空间构图**：意想不到的布局。不对称。重叠。对角线流动。网格破坏元素。慷慨的负空间或受控的密度。
- **背景和视觉细节**：创建氛围和深度，而不是默认为纯色。添加与整体美学相匹配的上下文效果和纹理。应用创意形式，如渐变网格、噪声纹理、几何图案、分层透明度、戏剧性阴影、装饰边框、自定义光标和颗粒叠加。

绝不使用通用的 AI 生成美学，如过度使用的字体系列（Inter、Roboto、Arial、系统字体）、陈词滥调的配色方案（尤其是白色背景上的紫色渐变）、可预测的布局和组件模式，以及缺乏特定于上下文特色的千篇一律的设计。

创造性地解释并做出真正为上下文设计的意想不到的选择。没有设计应该是相同的。在明暗主题、不同字体、不同美学之间变化。永远不要在几代中收敛于常见的选择（例如，Space Grotesk）。

**重要**：将实现复杂度与美学愿景相匹配。极繁主义设计需要具有大量动画和效果的精心制作的代码。极简主义或精致的设计需要克制、精确以及对间距、排版和微妙细节的仔细关注。优雅来自于很好地实现愿景。
