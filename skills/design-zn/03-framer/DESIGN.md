# Framer 设计系统灵感参考

## 1. 视觉主题与氛围

Framer 的网站是一块充满电影感、对工具痴迷的深色画布，散发着由爱好匠艺的设计师打造的设计工具的自信。整个体验沉浸于纯黑之中——不是暖炭灰或温馨深灰，而是绝对虚空（`#000000`），令每一个元素、每一张截图、每一个排版细节都仿佛漂浮在深邃宇宙之中。这是一个将自身产品 UI 作为英雄艺术品的网站，将完整保真度的截图和交互演示直接嵌入叙事流程。

排版是其标志性举措：GT Walsheim 配合激进的紧缩字间距（110px 展示文字极端达 -5.5px），创造出压缩的、充满动感的、几乎如弹簧蓄力的标题——仿佛处于张力之下随时可能扩展的词语。向 Inter 正文文字的过渡无缝流畅，大量使用 OpenType 特性（`cv01`、`cv05`、`cv09`、`cv11`、`ss03`、`ss07`），即使在小字号下也赋予文字精致的定制感。Framer 蓝（`#0099ff`）被克制但决断地运用——作为链接颜色、边框强调和微妙的光环阴影——在温情全无的黑色中创造出冷峻的电光贯穿线。

整体效果是一个面向网页设计师的夜总会：黑暗、精准、诱惑、毫不掩饰地以产品为核心。每一个章节都是为了展示工具能做什么，而网站本身就是概念证明。

**核心特征：**
- 纯黑（`#000000`）虚空画布——绝对深色，非暖色或灰色调
- GT Walsheim 展示字体，极度负字间距（110px 时 -5.5px）
- Framer 蓝（`#0099ff`）作为唯一强调色——冷峻、电气、精确
- 胶囊形按钮（40px–100px 圆角）——交互元素无尖角
- 产品截图作为英雄艺术品——工具本身即是营销
- 深色表面上使用 `rgba(255, 255, 255, 0.1)` 的磨砂玻璃按钮变体
- Inter 大量使用 OpenType 特性，实现精致的微排版

## 2. 色彩体系与角色分工

### 主色
- **纯黑** (`#000000`)：主背景，定义 Framer 深色优先身份的虚空画布
- **纯白** (`#ffffff`)：深色表面上的主文字颜色，强调背景上的按钮文字
- **Framer 蓝** (`#0099ff`)：主强调色——链接、边框、光环阴影、交互高亮

### 次色与强调色
- **静默银** (`#a6a6a6`)：次要文字、低调标签、深色表面上的暗化描述
- **近黑** (`#090909`)：浮起深色表面，微妙层深分隔的阴影光环颜色

### 表面与背景
- **虚空黑** (`#000000`)：页面背景、主画布
- **磨砂白** (`rgba(255, 255, 255, 0.1)`)：半透明按钮背景，深色上的玻璃效果表面
- **微妙白** (`rgba(255, 255, 255, 0.5)`)：悬停状态下稍不透明的磨砂元素

### 中性色与文字
- **纯白** (`#ffffff`)：标题文字，高强调正文
- **静默银** (`#a6a6a6`)：正文、描述、次要信息
- **幽灵白** (`rgba(255, 255, 255, 0.6)`)：第三级文字、深色表面上的占位文字

### 语义色与强调色
- **Framer 蓝** (`#0099ff`)：链接、交互边框、焦点环
- **蓝色光晕** (`rgba(0, 153, 255, 0.15)`)：焦点环阴影，交互元素周围的微妙蓝色光晕
- **默认链接蓝** (`#0000ee`)：标准浏览器链接颜色（内容区域中稀少使用）

### 渐变系统
- 无明显渐变使用——Framer 依赖纯平黑色表面，偶尔使用极低不透明度的 Framer 蓝色调光晕增加深度
- 产品截图背后极低不透明度的 Framer 蓝微妙辐射光晕效果

## 3. 排版规则

### 字体家族
- **展示字体**：`GT Walsheim Framer Medium` / `GT Walsheim Medium`——定制几何无衬线字体，字重 500。备用字体：`GT Walsheim Framer Medium Placeholder`，系统无衬线
- **正文/UI**：`Inter Variable` / `Inter`——可变无衬线字体，大量 OpenType 特性。备用字体：`Inter Placeholder`、`-apple-system`、`system-ui`
- **强调字体**：`Mona Sans`——GitHub 的开源字体，用于特定元素，超轻字重（100）
- **等宽字体**：`Azeret Mono`——代码和技术标签的配套等宽字体
- **圆形字体**：`Open Runde`——用于微型标签的小型圆形配套字体

### 排版原则
- **压缩即个性**：GT Walsheim 极度负字间距（110px 时 -5.5px）是定义性的排版姿态——标题感觉如弹簧蓄力、紧迫，几乎令人窒息
- **OpenType 最大化主义**：Inter 同时使用 6+ 个 OpenType 特性（`cv01`、`cv05`、`cv09`、`cv11`、`ss03`、`ss07`），即使在正文尺寸下也创造出微妙的定制感
- **展示字重克制**：所有 GT Walsheim 使用字重 500（中等）——从不粗体，从不常规。这创造出自信而非咄咄逼人的展示语调
- **超紧行高**：展示文字行高 0.85 意味着字母在垂直方向几乎重叠——刻意的密度，适合在臂长距离阅读

## 4. 组件样式

### 按钮
- **磨砂胶囊**：`rgba(255, 255, 255, 0.1)` 背景，黑色文字（`#000000`），胶囊形（40px 圆角）。生活在深色表面的玻璃效果按钮——半透明、环境感、微妙
- **实色白色胶囊**：`rgb(255, 255, 255)` 背景，黑色文字（`#000000`），完全胶囊形（100px 圆角），内边距 `10px 15px`。主 CTA——在深色背景上干净、高对比度、不可错过
- **幽灵**：无可见背景，白色文字，仅靠文字样式维持。悬停时显示微妙磨砂背景
- **过渡**：基于缩放的动画（带 0.85 缩放因子的矩阵变换），透明度过渡用于显示效果

### 卡片与容器
- **深色表面卡片**：黑色或近黑（`#090909`）背景，`rgba(0, 153, 255, 0.15) 0px 0px 0px 1px` 蓝色光环阴影边框，圆角（10px–15px）
- **浮起卡片**：多层阴影——`rgba(255, 255, 255, 0.1) 0px 0.5px 0px 0.5px`（微妙顶部高亮）+ `rgba(0, 0, 0, 0.25) 0px 10px 30px`（深环境阴影）
- **产品截图**：深色容器内全宽或带内边距，软件 UI 预览 8px–12px 圆角
- **悬停**：Framer 蓝光环阴影微妙增强光晕，或磨砂表面的亮度变化

### 输入框与表单
- 营销站点上表单存在感极弱
- 输入框遵循深色主题：深色背景，微妙边框，白色文字
- 焦点状态：Framer 蓝（`#0099ff`）光环边框，`1px solid #0099ff`
- 占位文字颜色 `rgba(255, 255, 255, 0.4)`

### 导航
- **深色浮动导航栏**：黑色背景带磨砂玻璃效果，白色文字链接
- **导航链接**：Inter 15px，字重 400，白色文字，微妙悬停不透明度变化
- **CTA 按钮**：胶囊形，白色或磨砂，位于导航右端
- **移动端**：折叠为汉堡菜单，保持深色主题
- **粘性行为**：导航在滚动时保持固定在顶部

### 图片处理
- **产品截图作为英雄艺术品**：全宽嵌入 UI 截图，圆角（8px–12px）
- **深色背景上的深色构图**：截图放置于黑色背景上，微妙阴影实现深度分离
- **16:9 及自定义宽高比**：产品演示填满容器
- **无装饰性图像**：所有图像均为功能性——展示工具、产出成果或工作流程

### 信任与社会证明
- 深色表面上的客户标志和评价，以静默灰呈现
- 最小装饰——产品截图本身即是信任信号

## 5. 布局原则

### 间距系统
- **基础单位**：8px
- **比例**：1px、2px、3px、4px、5px、6px、8px、10px、12px、15px、20px、30px、35px
- **章节内边距**：大型垂直间距（章节间 80px–120px）
- **卡片内边距**：15px–30px 内部填充
- **组件间距**：相关元素之间 8px–20px

### 网格与容器
- **最大宽度**：约 1200px 容器，居中
- **列式布局**：全宽主图，2 列功能章节，单列产品展示
- **非对称布局**：功能章节常将文字（40%）与截图（60%）配对

### 留白哲学
- **通过黑暗呼吸**：章节间宽裕的垂直间距——黑色背景意味着留白以虚空形式呈现，在内容块之间创造戏剧性停顿
- **内部密集，之间疏朗**：单个组件构成紧密（紧行高、压缩文字），但漂浮在充裕的周围空间中
- **以产品为先的密度**：截图区域允许密集且信息丰富，与稀疏的营销文字形成对比

### 圆角比例
- **1px**：微型元素，近乎方形的精确边缘
- **5px–7px**：小型 UI 元素，图片缩略图——轻微柔化
- **8px**：标准组件圆角——代码块、按钮、交互元素
- **10px–12px**：卡片、产品截图——舒适圆润
- **15px–20px**：大型容器、功能卡片——慷慨圆润
- **30px–40px**：导航胶囊、分页——显著圆润
- **100px**：完全胶囊形——主 CTA，标签元素

## 6. 层深与高度

| 级别 | 处理方式 | 用途 |
|-------|-----------|-----|
| 第 0 级（平面） | 无阴影，纯黑表面 | 页面背景、空白区域 |
| 第 1 级（光环） | `rgba(0, 153, 255, 0.15) 0px 0px 0px 1px` | 卡片边框、交互元素轮廓——Framer 蓝光晕环 |
| 第 2 级（限制） | `rgb(9, 9, 9) 0px 0px 0px 2px` | 近黑光环，深色表面上的微妙限制 |
| 第 3 级（浮动） | `rgba(255, 255, 255, 0.1) 0px 0.5px 0px 0.5px, rgba(0, 0, 0, 0.25) 0px 10px 30px` | 浮起卡片、浮动元素——微妙白色顶边高亮 + 深环境阴影 |

### 阴影哲学
Framer 的高度系统与传统浅色主题设计相反。不同于浅色背景上的深色阴影，Framer 使用：
- **蓝色调光环阴影**，极低不透明度（0.15）用于限制——这一标志性举措为每个带边框的元素带来微妙的品牌烙印
- **白色边缘高亮**（0.5px），位于浮起元素的顶边——模拟光线照射在顶部表面
- **深环境阴影**，用于真正的浮动元素——`rgba(0, 0, 0, 0.25)` 配大扩散（30px）

### 装饰性层深
- **蓝色光晕：** 关键交互区域后方微妙的 Framer 蓝（`#0099ff`）辐射渐变
- **无背景模糊/玻璃质感**：尽管有磨砂按钮效果，但无重度玻璃模糊使用——半透明通过简单的 rgba 不透明度实现

## 7. 应做与禁止

### 应做
- 使用纯黑（`#000000`）作为主背景——不是深灰，不是炭灰
- 在 GT Walsheim 展示文字上应用极度负字间距（-3px 至 -5.5px）
- 保持所有按钮为胶囊形（40px+ 圆角）——绝不使用方角或轻微圆角按钮
- 仅将 Framer 蓝（`#0099ff`）用于交互强调——链接、边框、焦点状态
- 在深色背景的磨砂玻璃表面使用 `rgba(255, 255, 255, 0.1)`
- 将 GT Walsheim 保持在字重 500——中等字重即是品牌
- 在 Inter 文字上使用大量 OpenType 特性（cv01, cv05, cv09, cv11, ss03, ss07）
- 让产品截图成为视觉中心——工具自我营销
- 为卡片限制使用蓝色光环阴影（`rgba(0, 153, 255, 0.15) 0px 0px 0px 1px`）

### 禁止
- 使用暖深色背景（不用 `#1a1a1a`、`#2d2d2d` 或棕色调黑色）
- 在 GT Walsheim 展示文字上使用粗体（700+）——仅使用中等 500
- 在 Framer 蓝之外引入额外强调色——这是单一强调色系统
- 在非交互元素上使用大圆角（卡片使用 10px–15px，只有按钮使用 40px+）
- 添加装饰性图像、插图或图标——产品即插图
- 在标题上使用正字间距——一切都是压缩的、负字间距
- 创建重投影——深度通过微妙光环和极小环境阴影传达
- 在内容章节后方放置浅色/白色背景——虚空是神圣的
- 使用衬线或展示字重字体——系统仅限几何无衬线体

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 移动端 | <809px | 单列，堆叠功能章节，减小主图文字（62px→40px），汉堡导航 |
| 平板 | 809px–1199px | 开始 2 列功能，导航链接部分可见，截图缩小 |
| 桌面端 | >1199px | 完整布局，包含所有链接 + CTA 的扩展导航，110px 展示主图，并排功能 |

### 触摸目标
- 胶囊按钮：最低 40px 高度，10px 垂直内边距——超过 WCAG 最低 44px 要求
- 导航链接：15px 文字配宽裕内边距，确保触摸无障碍
- 移动端 CTA 按钮：移动端全宽胶囊，便于拇指触及

### 折叠策略
- **导航**：完整水平导航 → 移动端断点处折叠为汉堡菜单
- **主图文字**：110px 展示 → 85px → 62px → 约 40px，跨断点保持极度负字间距比例
- **功能章节**：并排（文字 + 截图）→ 移动端垂直堆叠
- **产品截图**：在容器内响应式缩放，保持宽高比
- **章节间距**：按比例减小——桌面端 120px → 移动端 60px

### 图片行为
- 产品截图在容器边界内响应式缩放
- 无艺术指导变化——跨断点使用相同裁剪
- 深色背景确保截图在任何尺寸下都保持视觉冲击
- 截图在用户滚动进入视口时懒加载

### 迭代指南
精修使用此设计系统生成的现有界面时：
1. 一次专注于一个组件——深色画布让每个元素都珍贵
2. 始终验证 GT Walsheim 标题的字间距——极度负字间距不可妥协
3. 确认 Framer 蓝仅出现在交互元素上——绝不作为非链接的装饰背景或文字颜色
4. 确保所有按钮都是胶囊形——任何方角立即破坏 Framer 美学
5. 通过检查磨砂玻璃表面是否恰好为 `rgba(255, 255, 255, 0.1)` 来测试——太不透明看起来像 bug，太透明则消失不见
