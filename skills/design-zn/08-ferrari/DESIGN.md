# Ferrari 设计系统灵感参考

## 1. 视觉主题与氛围

Ferrari 的网站是一本数字杂志——一本精心策划的杂志，以艺术机构的庄严与意大利手工车身制造的精准，呈现着马术品牌。页面在绝对黑暗的宽幅上展开，只有标志性的跃马纹章孤独地漂浮在自身的氛围中。其后，内容在笔墨般深沉的电影感章节与清爽的白色编辑面板之间戏剧性地交替展开。这种明暗对比的节奏——黑暗让位于光明，机械让位于人的故事——更像是翻阅一本 Ferrari 年鉴，而非滚动一个商业网站。每个章节都是一个精心策划的小品：从阴影中浮现的概念车，摆出雕塑般静止姿态的两位 F1 车手，以宝石色调列队游行的量产车型。

色彩语言对一个以速度和激情著称的品牌来说修道院般克制。Ferrari 红（`#DA291C`）以近乎外科手术般的吝啬出现——保留给"订阅"CTA 和需要立即吸引注意力的强调时刻。界面的绝大部分生活在黑色、白色和精心校准的灰阶中（从 `#303030` 深色表面经 `#8F8F8F` 中间调至 `#D2D2D2` 浅色边框）。两种黄色——赛车黄（`#FFF200`）和更深的摩德纳黄（`#F6E500`）——作为特殊场景的传承强调色存在于 token 系统中，向 Ferrari 的赛车渊源致敬。这种克制意味着当红色出现时，它承载着整个品牌的分量。

排版依赖 FerrariSans——一种专有无衬线字体家族，中等字重标题（500–700）和紧凑比例。展示文字在章节标题上以 24–26px 运行，而 UI 框架在 12–16px、常规到粗体的字重范围内生活。次级"Body-Font"自定义字体处理说明文字和工具文字，以大写渲染并带宽字间距（1px），创造出标签式的编辑品质。这一双字体系统——FerrariSans 用于叙事权威，Body-Font 用于结构注释——赋予网站印刷杂志般的层级。没有任何文字装饰是多余的。标题字间距紧凑，标签字间距刻意扩展，创造出在紧迫与沉静之间交替的视觉节奏。

**核心特征：**
- 明暗对比布局，在深黑章节和干净白色编辑面板之间交替
- Ferrari 红（`#DA291C`）极度克制使用——强调而非氛围
- 跃马纹章作为黑色虚空场上的孤立英雄元素
- FerrariSans 专有字体，紧凑比例和中等字重
- 新闻摄影图像：概念渲染、车手肖像、车型列队——每个章节都是一个故事
- Body-Font 大写标签，宽字间距（1px）用于编辑注释
- 接近零的圆角（默认 2px），反映精密工程美学
- 双框架架构（PrimeReact + Element Plus），驱动 32+ 个交互组件
- 轮播驱动的主图，带编辑幻灯片和箭头/圆点导航

## 2. 色彩体系与角色分工

### 主色
- **Ferrari 红** (`#DA291C`)：标志性的 Rosso Corsa——主要强调色和 CTA 颜色。用于"订阅"按钮、关键操作触发器和需要最高视觉权威的品牌时刻。系统中最重要的颜色（--f-color-accent-100）
- **纯白** (`#FFFFFF`)：编辑内容面板的主表面、深色背景上的导航文字和按钮填充。在深色电影章节之间提供喘息空间的画布（--f-color-ui-0）

### 次色与强调色
- **深红** (`#B01E0A`)：Ferrari 红的较深变体，用于悬停/按下状态和高对比度场景——在不引入新色相的情况下为品牌色增添维度感（--f-color-accent-90）
- **最深红** (`#9D2211`)：最饱和的深红——用于活跃状态和需要比深红更重分量的额外强调（--f-color-accent-80）
- **赛车黄** (`#FFF200`)：来自 Ferrari 赛车涂装的传承强调——保留给特殊高亮和赛车相关场景（--f-color-yellow-hypersail）
- **摩德纳黄** (`#F6E500`)：比赛车黄稍暖、更金黄——用于次级传承强调和类别标记（--f-color-yellow）

### 表面与背景
- **绝对黑** (`#000000`)：主图章节、电影感背景和主导深色表面——令图像和跃马纹章漂浮的虚空
- **深色表面** (`#303030`)：页脚区域、通讯章节和分层深色面板的次级深色表面——从纯黑略微提亮以实现深度区分（--f-color-ui-90）
- **浅灰表面** (`#D2D2D2`)：白色面板上的分隔符和边框处理的微妙替代表面（--f-color-ui-20）
- **深色遮罩** (`hsla(0, 0%, 7%, 0.8)`)：弹窗遮罩和图片说明背景的半透明近黑（--f-color-overlay-darker）

### 中性色与文字
- **近黑** (`#181818`)：浅色表面上的主正文颜色——从绝对黑略微软化，提高可读性（链接默认颜色）
- **深灰** (`#666666`)：次要文字和低调 UI 标签——用于文字需要从主层级退后的地方（--f-color-black-60）
- **中灰** (`#8F8F8F`)：元数据、时间戳和支持性内容的第三级文字（--f-color-black-50）
- **银灰** (`#969696`)：占位文字和禁用状态指示器（--f-color-black-55）

### 语义色与强调色
- **警告红** (`#F13A2C`)：无障碍警告状态——比 Ferrari 红更亮、更偏橙，以区分语义警告和品牌表达（--f-color-accessible-warning）
- **成功绿** (`#03904A`)：确认和积极状态指示器（--f-color-accessible-success）
- **信息蓝** (`#4C98B9`)：信息性提示、工具提示和中性状态消息（--f-color-accessible-info）
- **链接悬停蓝** (`#3860BE`)：文字链接的交互悬停状态——一种庄重的海军蓝，传达交互性而不与 Ferrari 红竞争

### 渐变系统
- token 系统中无明显渐变
- 深度通过摄影和黑白表面之间的二元对比实现
- 深色遮罩颜色（`hsla(0, 0%, 7%, 0.8)`）通过在图像上的透明叠加创造深度
- 偶尔的摄影渐变（工作室拍摄中的光线衰落）在图像内容中提供大气深度

## 3. 排版规则

### 字体家族
- **FerrariSans**：标题、导航、按钮和编辑内容的主字体。一种专有无衬线字体，默认字重为中等（500），紧凑 x 字高，精确的字间距控制。备用字体：Arial、Helvetica、sans-serif
- **Body-Font**：说明文字、标签和工具文字的次级字体。常以大写渲染，带扩展字间距（1px），呈现编辑标签美学。用于类别标签和小型注释文字
- **Arial / Helvetica**：系统备用字体，用于 Cookie 同意弹窗、表单元素和第三方组件框架

### 排版原则
- **专有身份**：FerrariSans 是 Ferrari 独有的——不可替代，否则失去品牌识别。字体的紧凑比例和默认中等字重（500）传达工程精准
- **双声调系统**：FerrariSans 处理叙事声音（标题、内容、按钮），Body-Font 处理结构注释（标签、标记、微说明文字）——这反映了编辑文字与技术标签的印刷杂志惯例
- **大写作为强调工具**：Body-Font 说明文字使用 `text-transform: uppercase` 配扩展字间距（1px），创造视觉上独特的标签层，读起来像"信息覆层"而非主要内容
- **紧凑行高**：标题使用紧凑行高（1.00–1.30），创造密集有力的文字块，而正文展开至 1.50 以舒适阅读——压缩标题与宽松正文之间的对比创造视觉张力
- **字重范围 400–700**：系统中活跃着四种字重（400、500、600、700）——比特斯拉范围明显更宽，但仍受控制。500 是默认"声音"，700 用于强调，400 用于正文，600 用于导航

## 4. 组件样式

### 按钮
Ferrari 的按钮是极简白色矩形，接近零圆角——CTA 哲学是"建筑，非装饰"。

**主 CTA（白色）**——默认操作按钮：
- 默认：背景 `#FFFFFF`，文字 `#000000`，字体尺寸 16px（FerrariSans），字间距 1.28px，内边距 12px 10px，圆角 2px，边框 1px solid `#000000`
- 悬停：背景 `#1EAEDB`（蓝绿），文字 `#FFFFFF`，不透明度 0.9
- 焦点：背景 `#1EAEDB`，文字 `#FFFFFF`，边框 1px solid `#FFFFFF`，轮廓 2px solid `#000000`，不透明度 0.9
- 用于："配置"操作，浅色背景上的次级 CTA

**订阅 CTA（红色）**——高强调操作按钮：
- 默认：背景 `#DA291C`（Ferrari 红），文字 `#FFFFFF`，圆角 2px，内边距 12px 10px
- 用于：邮件订阅，深色背景上的主要转化操作
- 唯一使用 Ferrari 红的按钮——保留给最高视觉优先级

**幽灵按钮（白色边框）**——用于深色背景：
- 默认：背景透明，文字 `#FFFFFF`，边框 1px solid `#FFFFFF`，圆角 2px，内边距 12px 10px
- 悬停：背景 `#1EAEDB`（蓝绿），文字 `#FFFFFF`，不透明度 0.9
- 焦点：与主 CTA 焦点状态相同
- 用于：覆叠在深色图像和电影感章节上的操作

**文字链接**——行内导航：
- 默认：文字 `#181818`（浅色表面上）或 `#FFFFFF`（深色表面上），无边框，无背景
- 悬停：颜色切换至 `#3860BE`（链接悬停蓝），装饰移除下划线
- 深色表面上的白色变体默认带下划线装饰
- 根据上下文使用 FerrariSans 或 Body-Font（大写标签链接使用 Body-Font）

### 卡片与容器

**编辑卡片**（内容章节）：
- 背景：白色
- 边框：无
- 阴影：无
- 布局：图片在上，标题 + 说明在下
- 图片处理：卡片内全宽，图片无圆角
- 文字：FerrariSans 标题（16–24px）+ Body-Font 说明（12–13px 大写）

**深色电影卡片**（主图/功能章节）：
- 背景：`#000000`（绝对黑）
- 带文字覆层的全出血图像
- 无边框，无阴影——黑暗即是容器
- 文字：白色，以精心的负空间定位

**车辆阵容**（车型轮播）：
- 车辆缩略图的水平可滚动行
- 每辆车在中性/白色背景上
- 导航：箭头按钮 + 圆点指示器
- 背景切换以展示所选车型的颜色场景

### 输入框与表单

**邮件订阅输入框**（页脚章节）：
- 背景：深色表面上的透明
- 文字：白色
- 边框：1px solid `#CCCCCC`
- 占位文字：`#969696`（银灰）
- 焦点：边框颜色过渡（标准浏览器焦点环）
- 标签：Body-Font 大写，12px，1px 字间距

**Cookie 同意**（弹窗）：
- 背景：白色
- 圆角：8px（对话框）
- 阴影：`rgb(153, 153, 153) 1px 1px 1px 0px`
- 按钮：超大（45px Arial），白色背景配黑色边框
- 使用标准 PrimeReact/Element Plus 弹窗框架

### 导航
- **桌面端**：跃马纹章居中于页面顶部，下方为主导航——非传统水平导航栏，而是黑色背景上的全宽头部块
- **标志**：居中跃马纹章（44×42px）在绝对黑上——单一最显著的 UI 元素
- **链接**：FerrariSans，13px，字重 600，深色背景上的白色文字
- **移动端**：汉堡折叠至垂直导航抽屉
- **页脚**：`#303030`（深色表面）上的多列布局，链接类别使用 Body-Font 大写
- **无粘性导航行为**——页面随头部自然滚动至视口外

### 图片处理
- **主图**：黑色背景上的全宽编辑摄影——大气工作室灯光下的概念车，电影构图的编辑人像
- **宽高比**：混合——主图章节横向（16:9），人像/车手图像接近正方形，车辆阵容宽全景
- **全出血与带内边距**：主图是从边到边全出血的；编辑内容图像在白色容器内带内边距
- **懒加载**：折叠下方章节使用渐进式加载（PrimeReact 框架处理）
- **图片质量**：工作室灯光的高分辨率摄影——无用户生成或生活方式图像。每张图像都经过艺术指导

### 轮播组件
- 多幻灯片的编辑轮播
- 幻灯片位置的圆点指示器
- 幻灯片边缘的箭头导航（左/右）
- 自动前进，支持手动覆盖
- 内容：混合编辑——活动回顾、车型发布、赛车亮点

## 5. 布局原则

### 间距系统
- **基础单位**：8px（检测到的系统基础）
- **比例**：1px、2px、4px、5px、6px、9px、10px、11.2px、12px、13px、15px、16px、19px、20px、25px
- **按钮内边距**：垂直 12px，水平 10px——紧凑精准
- **章节内边距**：主要内容块之间宽裕的垂直间距（估计 40–80px）
- **卡片间距**：网格项目之间 16–20px
- **页脚内边距**：深色页脚块内章节水平 25px

### 网格与容器
- **最大宽度**：1920px（最大断点），内容在较窄宽度受限
- **主图**：黑色全出血，内容居中
- **编辑章节**：2 列布局，图片 + 文字，交替侧面
- **车辆阵容**：水平滚动/轮播，桌面宽度可见 5–6 个车型
- **页脚**：链接类别 4 列网格

### 留白哲学
Ferrari 将留白视为画廊墙壁。每个章节——无论是黑色虚空上的概念车渲染还是中性灰上的两位 F1 车手——都被赋予自己的"房间"呼吸空间。交替的黑色/白色章节创造节奏：深色 = 沉浸时刻，白色 = 编辑内容，深色 = 沉浸时刻。这种节奏使滚动感觉像翻阅奢侈品出版物的页面。编辑卡片之间的留白适中（非特斯拉式极端），因为 Ferrari 在讲述故事，不是在展示单一物品。

### 圆角比例
| 值 | 场景 |
|-------|---------|
| 1px | 小型行内元素（span）的微妙软化 |
| 2px | 按钮、输入框和交互元素的默认——几乎无感，剃刀精准 |
| 8px | 弹窗对话框和覆层容器——"最软"的结构圆角 |
| 50% | 圆形元素：轮播圆点、头像缩略图、滑块把手 |

## 6. 层深与高度

| 级别 | 处理方式 | 用途 |
|-------|-----------|-----|
| 第 0 级（平面） | 无阴影，无边框 | 所有内容章节和卡片的默认状态 |
| 第 1 级（微妙） | `rgb(153, 153, 153) 1px 1px 1px 0px` | 罕见——Cookie 同意对话框和下拉菜单 |
| 第 2 级（遮罩） | `hsla(0, 0%, 7%, 0.8)` 背景 | 弹窗遮罩和图片说明背景 |
| 第 3 级（边框） | `1px solid #CCCCCC` | 输入框、表单容器——通过轮廓而非阴影实现深度 |

### 阴影哲学
Ferrari 对高度的处理几乎和特斯拉一样平面，但有不同的理由。特斯拉为了极简主义而避免阴影，Ferrari 避免阴影是因为编辑摄影提供了所需的所有视觉深度。单一阴影 token（`rgb(153, 153, 153) 1px 1px 1px 0px`）极为微妙——仅用于同意对话框等功能性场景的 1 像素低语。网站通过三种策略传达层级：
1. **表面颜色对比**：黑色章节与白色章节创造无误的分层
2. **遮罩透明度**：`--f-color-overlay-darker` 在 80% 不透明度下无需阴影即可创造深度
3. **摄影深度**：带反光、地面阴影和大气霾的工作室灯光汽车图像提供所有视觉维度感

### 装饰性层深
- 界面元素上无 UI 渐变、无光晕、无模糊效果
- 黑色背景上的跃马标志通过纯对比创造"漂浮于虚空"效果——无需光晕或阴影
- 深色到浅色章节过渡是硬切，不是渐变混合——强化编辑翻页隐喻

## 7. 应做与禁止

### 应做
- 克制使用 Ferrari 红（`#DA291C`）——仅用于主 CTA 和品牌关键时刻。其力量来自克制
- 在黑色电影章节和白色编辑章节之间交替，创造标志性的明暗对比节奏
- 将 FerrariSans 字重 500 用作默认标题声音——它是发动机音符的排版等价物
- 将 Body-Font 以大写和 1px 字间距应用于所有标签、类别标签和结构注释
- 所有交互元素保持 2px 圆角——剃刀精准，非圆润友善
- 让摄影承载情感重量——每张图像都应是经过艺术指导的工作室品质
- 将跃马纹章作为黑色背景上的独立英雄元素——绝不用邻近内容挤压它
- 保持 12px/10px 按钮内边距比例——紧凑、有目的、无多余
- 正文文字使用 `#181818`（近黑）而非纯 `#000000`——微妙的暖意提高可读性
- 将黄色强调（`#FFF200`、`#F6E500`）严格保留给赛车运动和赛车传承场景

### 禁止
- 将 Ferrari 红散布于界面作为装饰——它是 CTA 信号，不是主题颜色
- 使用圆形胶囊按钮或大圆角——2px 精准不可妥协
- 为卡片或内容容器添加 box-shadow——深度来自表面颜色对比和摄影
- 在同一文字块中混用 FerrariSans 和 Body-Font——它们服务于不同的层级功能
- 使用彩色背景（蓝色、绿色等）作为章节——调色板仅限黑/白/灰，配以红色和黄色强调
- 对 FerrariSans 标题应用文字变换——大写保留给 Body-Font 标签
- 展示低质量或用户生成图像——每张照片必须符合编辑标准
- 将链接悬停蓝（`#3860BE`）用于交互悬停状态以外的任何用途——它不是品牌色
- 创建有多个竞争焦点的繁忙布局——每个章节应有一个清晰的故事
- 用品牌颜色覆盖语义色彩系统（警告、成功、信息）——`#F13A2C` 警告故意不同于 `#DA291C` 品牌红

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 小型移动端 | ≤375px | 单列，最小内边距（12px），堆叠导航，主图文字缩至约 18px，全宽 CTA |
| 移动端 | 376–600px | 单列，稍大内边距（16px），汉堡导航，正文 13px |
| 小型平板 | 601–768px | 开始 2 列编辑网格，主图全宽，页脚切换至 2 列 |
| 平板 | 769–960px | 完整 2 列布局，轮播显示 3 辆车，内边距增至 20px |
| 桌面端 | 961–1280px | 完整导航，带较大图像的 2 列编辑，车辆阵容显示 5 个车型 |
| 大型桌面端 | 1281–1920px | 最大内容宽度，宽裕留白，主图摄影呈现完整电影规模 |

### 触摸目标
- 主 CTA 按钮：最小 44px 高度，12px 垂直内边距（满足 WCAG AAA 44×44px 目标）
- 导航链接：13px 文字，1.50 行高，项目间适当间距
- 轮播箭头：视口边缘 44px+ 触摸目标
- 页脚链接：充足垂直间距（16–20px）以确保触摸精度

### 折叠策略
- **导航**：完整水平导航在移动端折叠至居中跃马标志 + 汉堡菜单
- **编辑章节**：2 列图片+文字布局折叠至单列，图片堆叠在文字上方
- **车辆阵容**：水平轮播保持滚动行为，但将可见车型从 5 个减至 2–3 个
- **页脚**：4 列链接网格在平板折叠至 2 列，移动端单列手风琴
- **主图轮播**：所有断点全宽，圆点指示器和箭头按比例缩放
- **间距压缩**：章节内边距从 40–80px（桌面端）减至 20–40px（移动端），保持比例呼吸感

### 图片行为
- 主图：所有断点全出血，使用 `object-fit: cover` 保持电影构图
- 编辑图像：在其容器内响应式，保持宽高比
- 车辆阵容：缩略图尺寸缩放，但保持一致的车身与框架比例
- 艺术指导：移动端裁剪可能在车辆主体上收紧，减少环境背景
- 懒加载：PrimeReact 处理折叠下方内容的渐进式图像加载

### 迭代指南
精修使用此设计系统生成的现有界面时：
1. 一次专注于一个组件——Ferrari 的编辑节奏意味着每个章节都是一个独立的小品

6. 每个屏幕保留 Ferrari 红仅出现在一个元素上——如果红色出现在多处，它就失去了权威性
