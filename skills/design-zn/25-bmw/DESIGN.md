# BMW 设计系统灵感

## 1. 视觉主题与氛围

BMW 的网站是视觉化的汽车工程——一套传达精准、性能和德国工业自信的设计系统。页面在深色英雄区（展示全出血汽车摄影）和干净白色内容区域之间交替，创造出令人联想到豪华汽车展厅的电影节奏，车辆在黑暗中聚光灯下展示。BMW CI2020 设计语言（企业形象焕新）定义了每个元素。

排版建立在 BMWTypeNextLatin 上——一种两种变体的专有字体：BMWTypeNextLatin Light（字重 300）用于大型大写展示标题，BMWTypeNextLatin Regular 用于正文和 UI 文字。60px 大写标题字重 300 是定义性的排版手势——轻字重文字低语权威，而非喊叫。回退字体栈包括 Helvetica 和日文字体（Hiragino、Meiryo），反映 BMW 的全球影响力。

BMW 的独特之处在于其 CSS 变量驱动的主题系统。上下文感知变量（`--site-context-highlight-color: #1c69d4`、`--site-context-focus-color: #0653b6`、`--site-context-metainfo-color: #757575`）表明这是一个为多品牌、多上下文部署构建的设计系统，颜色可以全局切换。蓝色高亮色（`#1c69d4`）是 BMW 的标志性蓝色——谨慎用于交互元素和焦点状态，从不用于装饰。未检测到圆角——BMW 的设计是有棱角的、锐利角的，且不妥协地几何化。

**核心特征：**
- BMWTypeNextLatin Light（字重 300）大写展示——低语权威
- BMW 蓝（`#1c69d4`）作为单一强调色——仅用于交互元素
- 未检测到圆角——有棱角、锐利角的工业几何形状
- 深色英雄摄影 + 白色内容区域——展厅照明节奏
- CSS 变量驱动主题：`--site-context-*` 标记用于品牌灵活性
- 导航强调用字重 900——与 300 展示极端对比
- 全程紧凑行高（1.15–1.30）——压缩、高效、德国工程
- 全出血汽车摄影作为主要视觉内容

## 2. 色彩搭配与角色定义

### 主品牌色
- **纯白**（`#ffffff`）：`--site-context-theme-color`，主要表面，卡片背景
- **BMW 蓝**（`#1c69d4`）：`--site-context-highlight-color`，主要交互强调
- **BMW 聚焦蓝**（`#0653b6`）：`--site-context-focus-color`，键盘焦点和激活状态

### 中性刻度
- **近黑**（`#262626`）：浅色表面上的主要文字，深色链接文字
- **元数据灰**（`#757575`）：`--site-context-metainfo-color`，次要文字，元数据
- **银色**（`#bbbbbb`）：三级文字，静音链接，页脚元素

### 交互状态
- 所有链接悬停为白色（`#ffffff`）——表明主要是深色表面导航
- 文字链接悬停时使用无下划线——干净交互

### 阴影
- 极简阴影系统——深度通过摄影和深色/浅色区域对比实现

## 3. 排版规则

### 字体家族
- **展示浅色**：`BMWTypeNextLatin Light`，回退：`Helvetica, Arial, Hiragino Kaku Gothic ProN, Hiragino Sans, Meiryo`
- **正文/UI**：`BMWTypeNextLatin`，相同回退字体栈

### 层级

| 角色 | 字体 | 尺寸 | 字重 | 行高 | 备注 |
|------|------|------|--------|-------------|-------|
| 展示英雄 | BMWTypeNextLatin Light | 60px (3.75rem) | 300 | 1.30（紧凑） | `text-transform: uppercase` |
| 章节标题 | BMWTypeNextLatin | 32px (2.00rem) | 400 | 1.30（紧凑） | 主要章节标题 |
| 导航强调 | BMWTypeNextLatin | 18px (1.13rem) | 900 | 1.30（紧凑） | 导航粗体条目 |
| 正文 | BMWTypeNextLatin | 16px (1.00rem) | 400 | 1.15（紧凑） | 标准正文文字 |
| 按钮粗体 | BMWTypeNextLatin | 16px (1.00rem) | 700 | 1.20–2.88 | CTA 按钮 |
| 按钮 | BMWTypeNextLatin | 16px (1.00rem) | 400 | 1.15（紧凑） | 标准按钮 |

### 原则
- **浅色展示，重字导航**：英雄标题字重 300 创造低语式优雅；导航字重 900 创造鲜明权威。这种极端字重对比（300 vs 900）是标志性的排版张力。
- **通用大写展示**：60px 英雄始终大写——创造纪念碑式、建筑感质量。
- **全程紧凑**：整个系统的行高从 1.15 到 1.30。没有任何呼吸空间——每行都是压缩的、高效的、德国工程式的。
- **单一字体家族**：BMWTypeNextLatin 处理从 60px 展示到 16px 正文的一切——通过一种字体的统一性。

## 4. 组件样式

### 按钮
- 文字：16px BMWTypeNextLatin，主要字重 700，次要字重 400
- 行高：1.15–2.88（大差异表明内边距驱动的尺寸）
- 边框：深色表面上的白色底部边框（`1px solid #ffffff`）
- 无圆角——锐利矩形按钮

### 卡片与容器
- 无圆角——所有容器都是锐利角矩形
- 浅色区域白色背景
- 英雄/特性区域深色背景
- 大多数元素无可见边框

### 导航
- 主要导航链接 BMWTypeNextLatin 18px 字重 900
- 深色页眉上的白色文字
- BMW 标志 54x54px
- 悬停：保持白色，无文字装饰
- 页眉中的"首页"文字链接

### 图像处理
- 全出血汽车摄影
- 深色电影感照明
- 边缘到边缘英雄图像
- 汽车摄影作为主要视觉内容

## 5. 布局原则

### 间距系统
- 基本单位：8px
- 刻度：1px、5px、8px、10px、12px、15px、16px、20px、24px、30px、32px、40px、45px、56px、60px

### 网格与容器
- 全宽英雄摄影
- 居中内容区域
- 页脚：多列链接网格

### 空白哲学
- **展厅节奏**：深色英雄区域搭配宽裕内边距创造走过展厅的感觉，每辆车都在自己的空间中聚光灯照明。
- **压缩内容**：正文区域使用紧凑行高和密集间距——信息丰富，不浪费。

### 圆角刻度
- **未检测到任何圆角。** BMW 专门使用锐利角——每个元素都是精确的矩形。这是分析过的最有棱角的设计系统。

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 摄影（第 0 层） | 全出血深色图像 | 英雄背景 |
| 平面（第 1 层） | 白色表面，无阴影 | 内容区域 |
| 聚焦（无障碍） | BMW 聚焦蓝（`#0653b6`） | 焦点状态 |

**阴影哲学**：BMW 几乎不使用阴影。深度完全通过深色摄影区域与白色内容区域之间的对比创造——汽车照明承担层次工作。

## 7. 注意事项

### 应该做
- 所有展示标题使用 BMWTypeNextLatin Light（300）大写
- 保持所有角落锐利（0px 圆角）——有棱角的几何形状是不可妥协的
- 将 BMW 蓝（`#1c69d4`）仅用于交互元素——绝不用于装饰
- 导航强调应用字重 900——极端字重对比是刻意的
- 英雄区域使用全出血汽车摄影
- 全程保持紧凑行高（1.15–1.30）
- 使用 `--site-context-*` CSS 变量进行主题化

### 不应该做
- 不要圆角——零圆角是 BMW 的身份
- 不要将 BMW 蓝用于背景或大面积表面——它仅是强调色
- 不要使用中等字重（500–600）——系统使用 300、400、700、900 极端值
- 不要添加装饰元素——摄影和排版承载一切
- 不要使用宽松行高——BMW 文字始终是压缩的
- 不要减弱深色英雄区域——与白色的对比就是设计本身

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 小手机 | <375px | 支持的最小尺寸 |
| 手机 | 375–480px | 单列 |
| 大手机 | 480–640px | 小幅调整 |
| 小平板 | 640–768px | 两列开始 |
| 平板 | 768–920px | 标准平板 |
| 小桌面 | 920–1024px | 桌面布局开始 |
| 桌面 | 1024–1280px | 标准桌面 |
| 大桌面 | 1280–1440px | 扩展 |
| 超宽 | 1440–1600px | 最大布局 |

### 折叠策略
- 英雄：60px → 缩小，保持大写
- 导航：水平 → 汉堡
- 摄影：各尺寸保持全出血
- 内容区域：垂直堆叠
- 页脚：多列 → 堆叠
