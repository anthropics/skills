# Linear 设计系统灵感

## 1. 视觉主题与氛围

Linear 的网站是深色模式优先产品设计的典范——一块近黑色的画布（`#08090a`），内容从黑暗中涌现如繁星。整体印象是极致精密工程：每个元素都存在于精心校准的亮度层次中，从几乎不可见的边框（`rgba(255,255,255,0.05)`）到柔和发光的文字（`#f7f8f8`）。这不是应用于浅色设计的深色主题——它是以黑暗为原生媒介，通过白色透明度的微妙渐变而非色彩变化来管理信息密度。

排版系统完全建立在 Inter Variable 上，全局启用 OpenType 特性 `"cv01"` 和 `"ss03"`，赋予字体更简洁、更几何化的特性。Inter 使用了极宽的字重范围——从 300（浅色正文）经过 510（中等，Linear 的标志性字重）到 590（次粗强调）。510 字重尤为独特：它介于常规和中等之间，创造出一种不过分显眼的微妙强调。在展示尺寸（72px、64px、48px），Inter 使用激进的负字间距（-1.584px 至 -1.056px），创造出压缩的、权威的标题，感觉是被设计出来的而非经过设计的。Berkeley Mono 作为等宽字体伴侣，用于代码和技术标签，备用字体为 ui-monospace、SF Mono 和 Menlo。

色彩系统几乎完全是消色差的——深色背景配白色/灰色文字——只有一个品牌强调色点缀其中：Linear 的标志性靛蓝紫（`#5e6ad2` 用于背景，`#7170ff` 用于交互强调）。这个强调色使用克制而有意，仅出现在行动按钮、激活状态和品牌元素上。边框系统使用超薄、半透明白色边框（`rgba(255,255,255,0.05)` 至 `rgba(255,255,255,0.08)`），在不造成视觉噪音的情况下创造结构，如同在月光下绘制的线框。

**核心特征：**
- 深色模式原生：`#08090a` 营销背景、`#0f1011` 面板背景、`#191a1b` 抬高表面
- Inter Variable 全局启用 `"cv01", "ss03"` ——几何替代字形，更简洁的美学
- 标志性字重 510（介于常规和中等之间），用于大多数 UI 文字
- 展示尺寸的激进负字间距（72px 时 -1.584px，48px 时 -1.056px）
- 品牌靛蓝紫：`#5e6ad2`（背景）/ `#7170ff`（强调）/ `#828fff`（悬停）——系统中唯一的彩色
- 全页面半透明白色边框：`rgba(255,255,255,0.05)` 至 `rgba(255,255,255,0.08)`
- 按钮背景接近零透明度：`rgba(255,255,255,0.02)` 至 `rgba(255,255,255,0.05)`
- 深色表面的多层阴影及内嵌变体
- Radix UI 原语作为组件基础（检测到 6 个原语）
- 成功绿色（`#27a644`、`#10b981`）仅用于状态指示器

## 2. 色彩系统与角色

### 背景表面
- **营销黑** (`#010102` / `#08090a`)：最深的背景——英雄区域和营销页面的画布。近纯黑色，带有几乎察觉不到的蓝冷底调。
- **面板深色** (`#0f1011`)：侧边栏和面板背景。比营销黑高一个层次。
- **3 级表面** (`#191a1b`)：抬高表面区域、卡片背景、下拉菜单。
- **次级表面** (`#28282c`)：最浅的深色表面——用于悬停状态和略微抬高的组件。

### 文字与内容
- **主文字** (`#f7f8f8`)：带有些许暖意的近白色。默认文字颜色——非纯白，避免深色背景上的视觉疲劳。
- **次级文字** (`#d0d6e0`)：冷银灰色，用于正文、描述和次要内容。
- **三级文字** (`#8a8f98`)：柔和的灰色，用于占位符、元数据和弱化内容。
- **四级文字** (`#62666d`)：最低调的文字——时间戳、禁用状态、细微标签。

### 品牌与强调
- **品牌靛蓝** (`#5e6ad2`)：主品牌色——用于行动按钮背景、品牌标记和关键交互表面。
- **强调紫** (`#7170ff`)：更亮的变体，用于交互元素——链接、激活状态、选中项。
- **强调悬停** (`#828fff`)：更浅、更饱和的变体，用于强调元素的悬停状态。
- **安全薰衣草** (`#7a7fad`)：柔和靛蓝，专门用于安全相关的 UI 元素。

### 状态色彩
- **绿色** (`#27a644`)：主要成功/激活状态。用于"进行中"指示器。
- **翠绿** (`#10b981`)：次级成功——徽章、完成状态。

### 边框与分隔线
- **主边框** (`#23252a`)：实色深边框，用于显著分隔。
- **次边框** (`#34343a`)：略浅的实色边框。
- **三级边框** (`#3e3e44`)：最浅的实色边框变体。
- **细微边框** (`rgba(255,255,255,0.05)`)：超细腻半透明边框——默认值。
- **标准边框** (`rgba(255,255,255,0.08)`)：卡片、输入框、代码块的标准半透明边框。
- **线条着色** (`#141516`)：几乎不可见的线条，用于最细腻的分隔。
- **三级线条** (`#18191a`)：略微可见的分隔线。

### 浅色模式中性色（用于浅色主题场景）
- **浅色背景** (`#f7f8f8`)：浅色模式下的页面背景。
- **浅色表面** (`#f3f4f5` / `#f5f6f7`)：细微的表面着色。
- **浅色边框** (`#d0d6e0`)：浅色场景中的可见边框。
- **浅色边框替代** (`#e6e6e6`)：另一种更浅的边框。
- **纯白** (`#ffffff`)：卡片表面、高光。

### 遮罩
- **主遮罩** (`rgba(0,0,0,0.85)`)：模态/对话框背景——极度深色，用于焦点隔离。

## 3. 字体规则

### 字体系列
- **主字体**：`Inter Variable`，备用字体：`SF Pro Display, -apple-system, system-ui, Segoe UI, Roboto, Oxygen, Ubuntu, Cantarell, Open Sans, Helvetica Neue`
- **等宽字体**：`Berkeley Mono`，备用字体：`ui-monospace, SF Mono, Menlo`

### 原则
- **510 是标志性字重**：Linear 使用 Inter Variable 的 510 字重（介于常规 400 和中等 500 之间）作为默认强调字重。这创造了一种微妙的加粗感，而没有传统中等或半粗的厚重感。
- **随尺寸压缩**：展示尺寸使用逐渐紧凑的字间距——72px 时 -1.584px，64px 时 -1.408px，48px 时 -1.056px，32px 时 -0.704px。低于 24px 时，间距向 normal 放松。
- **OpenType 作为身份**：`"cv01", "ss03"` 不是装饰性的——它们将 Inter 转变为 Linear 的独特字体，赋予其更几何、更有目的的特性。
- **三级字重系统**：400（阅读）、510（强调/UI）、590（强力强调）。300 字重仅在有意弱化的场景中出现。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 尺度：1px、4px、7px、8px、11px、12px、16px、19px、20px、22px、24px、28px、32px、35px
- 7px 和 11px 值表明用于视觉对齐的微调
- 主要节奏：8px、16px、24px、32px（标准 8px 网格）

### 网格与容器
- 最大内容宽度：约 1200px
- 英雄区：居中单列，带宽松垂直内边距
- 功能区域：功能卡片的 2–3 列网格
- 全宽深色区域，带内部最大宽度约束
- 更新日志：单列时间线布局

### 空白哲学
- **黑暗即空间**：在 Linear 的深色画布上，空白不是白色——而是缺席。近黑色背景就是空白，内容从中涌现。
- **压缩标题，宽阔周边**：72px 带 -1.584px 字间距的展示文字密集而压缩，但处于广阔的深色内边距中。排版密度与空间宽裕度之间的对比创造了张力。
- **区域隔离**：每个功能区域都被宽松的垂直内边距（80px+）隔开，无可见分隔线——深色背景提供自然分隔。

### 圆角尺度
- 微型（2px）：行内徽章、工具栏按钮、细微标签
- 标准（4px）：小型容器、列表项
- 舒适（6px）：按钮、输入框、功能元素
- 卡片（8px）：卡片、下拉菜单、弹出框
- 面板（12px）：面板、特色卡片、区域容器
- 大型（22px）：大型面板元素
- 完整胶囊（9999px）：标签、筛选胶囊、状态标签
- 圆形（50%）：图标按钮、头像、状态点

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 平面（0 级） | 无阴影，`#010102` 背景 | 页面背景，最深画布 |
| 细腻（1 级） | `rgba(0,0,0,0.03) 0px 1.2px 0px` | 工具栏按钮、微型层次 |
| 表面（2 级） | `rgba(255,255,255,0.05)` 背景 + `1px solid rgba(255,255,255,0.08)` 边框 | 卡片、输入框、容器 |
| 内嵌（2b 级） | `rgba(0,0,0,0.2) 0px 0px 12px 0px inset` | 凹陷面板、内阴影 |
| 环形（3 级） | `rgba(0,0,0,0.2) 0px 0px 0px 1px` | 边框即阴影技术 |
| 抬高（4 级） | `rgba(0,0,0,0.4) 0px 2px 4px` | 浮动元素、下拉菜单 |
| 对话框（5 级） | 多层叠加：`rgba(0,0,0,0) 0px 8px 2px, rgba(0,0,0,0.01) 0px 5px 2px, rgba(0,0,0,0.04) 0px 3px 2px, rgba(0,0,0,0.07) 0px 1px 1px, rgba(0,0,0,0.08) 0px 0px 1px` | 弹出框、命令面板、模态框 |
| 焦点 | `rgba(0,0,0,0.1) 0px 4px 12px` + 附加层 | 交互元素的键盘焦点 |

**阴影哲学**：在深色表面上，传统阴影（深色叠深色）几乎不可见。Linear 通过使用半透明白色边框作为主要深度指示器来解决这一问题。层次不是通过阴影深度传达，而是通过背景亮度步进——每个层级略微增加表面背景的白色不透明度（`0.02` → `0.04` → `0.05`），创造微妙的堆叠效果。内嵌阴影技术（`rgba(0,0,0,0.2) 0px 0px 12px 0px inset`）为凹陷面板创造独特的"下沉"效果，增加了传统深色主题所缺乏的立体深度。

## 7. 规范与禁忌

### 应该做
- 所有文字使用 Inter Variable 配 `"cv01", "ss03"` ——这些特性是 Linear 字体身份的基础
- 使用字重 510 作为默认强调字重——这是 Linear 标志性的中间字重
- 在展示尺寸应用激进的负字间距（72px 时 -1.584px，48px 时 -1.056px）
- 在近黑色背景上构建：营销用 `#08090a`，面板用 `#0f1011`，抬高表面用 `#191a1b`
- 使用半透明白色边框（`rgba(255,255,255,0.05)` 至 `rgba(255,255,255,0.08)`），而非实色深色边框
- 保持按钮背景接近透明：`rgba(255,255,255,0.02)` 至 `rgba(255,255,255,0.05)`
- 仅将品牌靛蓝（`#5e6ad2` / `#7170ff`）保留给主要行动按钮和交互强调
- 主文字使用 `#f7f8f8` ——非纯 `#ffffff`，后者过于刺眼
- 应用亮度堆叠模型：更深 = 更暗的背景，抬高 = 略浅的背景

### 不应该做
- 不要使用纯白（`#ffffff`）作为主文字——`#f7f8f8` 可防止视觉疲劳
- 不要对按钮使用实色背景——透明度是系统的规则（rgba 白色在 0.02–0.05 之间）
- 不要将品牌靛蓝用于装饰——它专为交互/行动按钮保留
- 不要对展示文字使用正字间距——大尺寸的 Inter 始终使用负值
- 不要在深色背景上使用可见/不透明边框——边框应该是轻薄的半透明白色
- 不要跳过 OpenType 特性（`"cv01", "ss03"`）——没有它们，这只是通用 Inter，而非 Linear 的 Inter
- 不要使用字重 700（粗体）——Linear 的最大字重是 590，510 是主力
- 不要在 UI 外壳中引入暖色——调色板是冷灰色加蓝紫强调色
- 不要使用投影来表现深色表面的层次——应使用背景亮度步进

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|------|----------|
| 小移动端 | <600px | 单列，紧凑内边距 |
| 移动端 | 600–640px | 标准移动端布局 |
| 平板 | 640–768px | 开始双列网格 |
| 小桌面 | 768–1024px | 完整卡片网格，扩展内边距 |
| 桌面 | 1024–1280px | 标准桌面，完整导航 |
| 大桌面 | >1280px | 完整布局，宽松边距 |

### 触控目标
- 按钮使用舒适内边距，最小 6px 圆角
- 导航链接 13–14px，带足够间距
- 胶囊标签有 10px 水平内边距，满足触控可访问性
- 图标按钮 50% 圆角确保圆形、易点击目标
- 搜索触发器位置显著，带宽松点击区域

### 折叠策略
- 英雄区：72px → 48px → 32px 展示文字，字间距等比调整
- 导航：水平链接 + 行动按钮 → 768px 时汉堡菜单
- 功能卡片：3 列 → 2 列 → 堆叠单列
- 产品截图：保持宽高比，可能缩减内边距
- 更新日志：时间线在所有尺寸保持单列
- 页脚：多列 → 堆叠单列
- 区域间距：80px+ → 移动端 48px

### 图像行为
- 仪表板截图在所有尺寸保持边框处理
- 移动端英雄视觉简化（减少浮动 UI 元素）
- 产品截图使用响应式尺寸，保持一致圆角
- 深色背景确保截图在任何视口自然融合

### 迭代指南
1. 始终在所有 Inter 文字上设置 font-feature-settings `"cv01", "ss03"` ——这对 Linear 的外观是不可妥协的
2. 字间距随字号缩放：72px 时 -1.584px，48px 时 -1.056px，32px 时 -0.704px，16px 以下 normal
3. 三种字重：400（阅读）、510（强调/导航）、590（宣告）

6. 边框始终是半透明白色，在深色背景上永远不使用实色深色
7. Berkeley Mono 用于代码或技术内容，Inter Variable 用于其他一切
