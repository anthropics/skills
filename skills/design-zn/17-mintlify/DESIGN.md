# Mintlify 设计系统灵感

## 1. 视觉主题与氛围

Mintlify 的网站是一份将文档作为产品设计的研究——白色、通透、信息丰富的界面，以清晰度为最高美学价值。页面以明亮的白色（`#ffffff`）背景开篇，近黑色（`#0d0d0d`）文字，以及标志性的绿色品牌点缀（`#18E299`）——这种色彩传递出清新与智慧，而不主导整个色彩方案。整体氛围平静、自信，为易读性而精心设计——一套设计系统以每一个像素低声细语地说"我们在乎你的开发者体验"。

Inter 字体系列承担整个排版负担。在展示尺寸（40–64px），它使用紧凑的负字间距（-0.8px 至 -1.28px）和半粗字重（600），创造出专注而紧凑的标题，感觉像精心编写的文档标题。正文文字 16–18px 配 150% 行高，提供慷慨的阅读舒适度。Geist Mono 专门用于代码和技术标签——全大写、增大字距、小号——营销页面内终端的声音。

区别于其他文档平台的是 Mintlify 的氛围感渐变主视觉区。主视觉内容后方柔和的云状绿到白渐变，创造出一种空灵智慧的感觉——文档漂浮于噪音之上。主视觉区下方，页面进入有纪律的节奏：白色区段之间以 5% 透明度边框微妙分隔。卡片使用慷慨内边距（24px+）、大圆角（16px–24px）和如耳语般纤细的边框，创造出开放而非封闭的容器感。

**核心特征：**
- Inter 在展示尺寸配紧凑负字间距（-0.8px 至 -1.28px）——紧凑而可读
- Geist Mono 用于代码标签：全大写，12px，增大字距，终端的声音
- 品牌绿（`#18E299`）谨慎使用——行动召唤按钮，悬停状态，焦点环和点缀触感
- 带云状绿白渐变的氛围感主视觉区
- 超圆角：容器 16px，特色卡片 24px，按钮和胶囊形全圆（9999px）
- 微妙的 5% 透明度边框（`rgba(0,0,0,0.05)`）创造隐约的分隔感
- 8px 基础间距系统，慷慨区段内边距（48px–96px）
- 干净白色画布——无灰色背景，无彩色区段，仅通过边框和留白营造深度

## 2. 色彩方案与角色

### 主色
- **近黑色** (`#0d0d0d`)：主要文字，标题，深色界面。非纯黑——微妙的柔和度提升阅读舒适感。
- **纯白** (`#ffffff`)：页面背景，卡片界面，输入框背景。
- **品牌绿** (`#18E299`)：标志性点缀——行动召唤按钮，悬停链接，焦点环，品牌标识。

### 次要点缀色
- **品牌绿浅色** (`#d4fae8`)：用于徽章、悬停状态、微妙背景的绿色界面。
- **品牌绿深色** (`#0fa76e`)：浅绿色徽章上的文字，品牌元素悬停状态的深绿色。
- **温暖琥珀色** (`#c37d0d`)：警告状态，注意徽章——`--twoslash-warn-bg`。
- **柔和蓝** (`#3772cf`)：标签背景，信息注释——`--twoslash-tag-bg`。
- **错误红** (`#d45656`)：错误状态，破坏性操作——`--twoslash-error-bg`。

### 中性色系
- **灰色 900** (`#0d0d0d`)：主要标题文字，导航链接。
- **灰色 700** (`#333333`)：次要文字，描述，正文。
- **灰色 500** (`#666666`)：第三级文字，弱化标签。
- **灰色 400** (`#888888`)：占位文字，禁用状态，代码注释。
- **灰色 200** (`#e5e5e5`)：边框，分割线，卡片轮廓。
- **灰色 100** (`#f5f5f5`)：微妙界面背景，悬停状态。
- **灰色 50** (`#fafafa`)：近白界面色调。

### 交互色
- **默认链接** (`#0d0d0d`)：链接与文字颜色一致，依赖下划线/上下文。
- **链接悬停** (`#18E299`)：悬停时的品牌绿——`var(--color-brand)`。
- **焦点环** (`#18E299`)：输入框和交互元素的品牌绿焦点轮廓。

### 界面与叠加层
- **卡片背景** (`#ffffff`)：白色背景上的白色卡片，通过边框分隔。
- **微妙边框** (`rgba(0,0,0,0.05)`)：5% 黑色透明度边框——主要分隔机制。
- **中等边框** (`rgba(0,0,0,0.08)`)：交互元素稍强的边框。
- **输入框焦点边框** (`var(--color-brand)`)：焦点输入框的绿色环。

### 阴影与深度
- **卡片阴影** (`rgba(0,0,0,0.03) 0px 2px 4px`)：微妙抬升的近无阴影。
- **按钮阴影** (`rgba(0,0,0,0.06) 0px 1px 2px`)：按钮深度的微阴影。
- **无厚重阴影**：Mintlify 依赖边框而非阴影来营造深度。

- **主字体**：`Inter`，备用字体：`Inter Fallback, system-ui, -apple-system, sans-serif`

### 排版原则
- **展示尺寸紧凑字距**：Inter 在 40–64px 使用 -0.8px 至 -1.28px 字间距。这种压缩创造出刻意而节省空间的标题——文档标题，而非广告牌文案。
- **正文尺寸宽松阅读**：16–18px 正文使用正常字距，配 150% 行高，创造慷慨的阅读通道。文档需要舒适感。
- **双字体系统**：Inter 用于所有人类可读内容，Geist Mono 专门用于技术/代码上下文。边界严格——不混用。
- **全大写作为层级信号**：区段标签和技术标签使用全大写 + 正字距（0.6px–0.65px）作为内容类型之间清晰的视觉分隔符。
- **三种字重**：400（正文/阅读），500（UI/导航/强调），600（标题/题目）。系统中没有粗体（700）。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 比例：2px、4px、5px、6px、7px、8px、10px、12px、16px、24px、32px、48px、64px
- 区段内边距：48px–96px 垂直
- 卡片内边距：24px–32px
- 组件间距：8px–16px

### 栅格与容器
- 内容最大宽度：约 1200px
- 主视觉区：居中单列，顶部内边距慷慨（96px+）
- 功能区段：卡片使用 2–3 列 CSS Grid
- 全宽区段，内容受容器限制
- 一致水平内边距：24px（移动端）至 32px（桌面端）

### 留白哲学
- **文档级呼吸空间**：每个元素都有慷慨的周边留白。Mintlify 销售文档，所以营销页面本身展示了阅读舒适度。
- **区段即章节**：每个功能区段是一个自成一体的单元，配 48px–96px 垂直内边距，创造清晰的"章节分隔"。
- **内容密度低**：与将页面塞满的开发者工具不同，Mintlify 每个区段只用 1–2 个关键信息，配支撑性图像。

### 圆角比例
- 小（4px）：内联代码，小型标签，提示框
- 中（8px）：导航按钮，透明按钮，小容器
- 标准（16px）：卡片，内容容器，图像包装
- 大（24px）：特色卡片，主视觉容器，区段面板
- 全胶囊（9999px）：按钮，输入框，徽章，胶囊——标志性形状

## 6. 深度与层级

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 平面（层级 0） | 无阴影，无边框 | 页面背景，文字块 |
| 微妙边框（层级 1） | `1px solid rgba(0,0,0,0.05)` | 标准卡片边框，分割线 |
| 中等边框（层级 1b） | `1px solid rgba(0,0,0,0.08)` | 交互元素，输入框边框 |
| 环境阴影（层级 2） | `rgba(0,0,0,0.03) 0px 2px 4px` | 带微妙抬升的卡片 |
| 按钮阴影（层级 2b） | `rgba(0,0,0,0.06) 0px 1px 2px` | 按钮微深度 |
| 焦点环（无障碍） | `1px solid #18E299` 轮廓 | 焦点输入框，激活交互元素 |

**阴影哲学**：Mintlify 几乎不使用阴影。深度系统几乎完全由边框驱动——超微妙的 5% 透明度边框创造分隔，不带视觉重量。阴影出现时，它们是氛围感的耳语（0.03 透明度，2px 模糊，4px 扩散），增添最低限度的漂浮感。这种克制使页面感觉平坦而纸质——恰如其分，因为这是一家产品关乎清晰度和可读性的文档公司。

### 装饰性深度
- 主视觉渐变：主视觉内容后方的氛围感绿白云状渐变
- 无背景颜色交替——自始至终白色上的白色
- 深度来自边框透明度变化（5% → 8%）和留白

## 7. 深色模式

### 颜色反转
- **背景**：`#0d0d0d`（近黑色）
- **主要文字**：`#ededed`（近白色）
- **次要文字**：`#a0a0a0`（弱化灰色）
- **品牌绿**：`#18E299`（不变——绿色在两种背景上均有效）
- **边框**：`rgba(255,255,255,0.08)`（8% 透明度白色）
- **卡片背景**：`#141414`（比页面略亮）
- **阴影**：`rgba(0,0,0,0.4) 0px 2px 4px`（更强的阴影以提供对比度）

### 关键调整
- 按钮反转：白色背景深色文字变为深色背景浅色文字
- 徽章背景转变为更深的色调，配更亮的文字
- 焦点环保持品牌绿
- 主视觉渐变转变为深色调的绿色氛围渐变

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 移动端 | <768px | 单列，堆叠布局，汉堡导航 |
| 平板 | 768–1024px | 双列栅格开始，扩展内边距 |
| 桌面端 | >1024px | 完整布局，三列栅格，最大内容宽度 |

### 触控目标
- 全胶囊形按钮配舒适的 8px+ 垂直内边距
- 导航链接间距足够的 16px+ 间隔
- 移动菜单提供全宽点击目标

### 折叠策略
- 主视觉区：64px → 40px 标题，按比例保持紧凑字距
- 导航：水平链接 + 行动召唤按钮 → 768px 时汉堡菜单
- 功能卡片：三列 → 双列 → 单列堆叠
- 区段间距：96px → 移动端 48px
- 页脚：多列 → 单列堆叠
- 信任栏：栅格 → 水平滚动或堆叠

### 图像行为
- 产品截图保持响应式容器中的宽高比
- 主视觉渐变在移动端简化
- 全宽区段保持从边缘到边缘的处理

### 迭代指南
1. 始终对按钮和输入框使用全胶囊圆角（9999px）——这是 Mintlify 的标志性形状
2. 边框保持 5% 透明度（`rgba(0,0,0,0.05)`）——更强的边框会破坏通透感
3. 字间距随字体尺寸缩放：64px 时 -1.28px，40px 时 -0.8px，24px 时 -0.24px，16px 时正常
4. 仅三种字重：400（阅读），500（交互），600（公告）
5. 品牌绿（`#18E299`）谨慎使用——仅用于行动召唤按钮和悬停状态，绝不用于装饰填充
6. Geist Mono 全大写用于技术标签，Inter 用于其他所有内容
7. 区段内边距慷慨：桌面端 64px–96px，移动端 48px
8. 无灰色背景区段——自始至终白色，通过边框和留白实现分隔
