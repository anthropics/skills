# Resend 设计系统灵感

## 1. 视觉主题与氛围

Resend 的网站是一个黑暗的、电影感的画布，将电子邮件基础设施视为奢侈品。整个页面笼罩在纯黑色（`#000000`）中，文字在近白色（`#f0f0f0`）中发光，创造出一种剧院般的体验，内容在空旷舞台上演出，没有任何东西争夺注意力。这不是典型的开发者工具黑暗——而是摄影画廊的受控黑暗，每个元素都被刻意照明，没有任何东西参与竞争。

排版系统是主角。三种精心选择的字体创造出一种既有编辑感又有技术感的层次：Domaine Display（Klim Type Foundry 的衬线字体）以 96px 的巨大尺寸呈现英雄标题，行高极浅（1.00），负字间距（-0.96px），创造感觉像杂志封面的展示文字。ABC Favorit（由 Dinamo 制作）以更激进的字间距（56px 时 -2.8px）处理章节标题，给中层文字带来压缩、精工的质感。Inter 接管正文和 UI，提供让展示字体大放异彩的干净可读性。Commit Mono 为代码块画龙点睛。

Resend 的独特之处在于其冰蓝色调边框系统。不使用中性灰色边框，Resend 使用 `rgba(214, 235, 253, 0.19)`——一种 19% 透明度的冰冷、略带蓝色调的线条，给每个容器和分割线在黑色背景上带来冷冽、晶莹的质感。结合药丸形按钮（9999px 圆角）、多色强调系统（橙色、绿色、蓝色、黄色、红色——每种都有自己的 CSS 变量刻度）以及 OpenType 字体样式集（`"ss01"`、`"ss03"`、`"ss04"`、`"ss11"`），结果是一个感觉高端、精准、安静自信的设计系统。

**核心特征：**
- 纯黑背景，近白色（`#f0f0f0`）文字——戏剧性、画廊般的黑暗
- 三字体层次：Domaine Display（衬线英雄），ABC Favorit（几何章节），Inter（正文/UI）
- 冰蓝色调边框：`rgba(214, 235, 253, 0.19)`——每条边框都有冷冽、晶莹的光泽
- 多色强调系统：橙色、绿色、蓝色、黄色、红色——每种都有编号 CSS 变量刻度
- 药丸形按钮和标签（9999px 圆角），透明背景
- OpenType 字体样式集（`"ss01"`、`"ss03"`、`"ss04"`、`"ss11"`）应用于展示字体
- Commit Mono 用于代码——等宽字体作为设计元素，而非事后补充
- 使用蓝色调环的低语级阴影：`rgba(176, 199, 217, 0.145) 0px 0px 0px 1px`

## 2. 色彩搭配与角色定义

### 主色
- **虚空黑**（`#000000`）：页面背景，定义画布颜色（通过 `--color-black-12` 透明度 95%）
- **近白**（`#f0f0f0`）：主要文字，按钮文字，高对比度元素
- **纯白**（`#ffffff`）：`--color-white`，最高强调文字，链接高亮

### 强调刻度——橙色
- **橙色 4**（`#ff5900`）：`--color-orange-4`，22% 透明度——细微暖光
- **橙色 10**（`#ff801f`）：`--color-orange-10`，主要橙色强调——温暖、充沛
- **橙色 11**（`#ffa057`）：`--color-orange-11`，次要用途的较浅橙色

### 强调刻度——绿色
- **绿色 3**（`#22ff99`）：`--color-green-3`，12% 透明度——微弱翠绿洗涤
- **绿色 4**（`#11ff99`）：`--color-green-4`，18% 透明度——成功指示发光

### 强调刻度——蓝色
- **蓝色 4**（`#0075ff`）：`--color-blue-4`，34% 透明度——中等蓝色强调
- **蓝色 5**（`#0081fd`）：`--color-blue-5`，42% 透明度——更强蓝色
- **蓝色 10**（`#3b9eff`）：`--color-blue-10`，亮蓝色——链接，交互元素

### 强调刻度——其他
- **黄色 9**（`#ffc53d`）：`--color-yellow-9`，暖金色，用于警告或高亮
- **红色 5**（`#ff2047`）：`--color-red-5`，34% 透明度——错误状态，破坏性操作

### 中性刻度
- **银色**（`#a1a4a5`）：次要文字，静音链接，描述
- **深灰**（`#464a4d`）：三级文字，降调内容
- **中灰**（`#5c5c5c`）：悬停状态，细微强调
- **中等灰**（`#494949`）：四级文字
- **浅灰**（`#f8f8f8`）：浅色模式表面（如适用）
- **边框灰**（`#eaeaea`）：浅色环境边框
- **边缘灰**（`#ececec`）：浅色表面细微边框
- **薄雾灰**（`#dedfdf`）：浅色分割线
- **柔灰**（`#e5e6e6`）：替代浅色边框

### 表面与覆盖层
- **霜原色**（`#fcfdff`）：主要颜色标记（轻微蓝色调，94% 透明度）
- **白色悬停**（`rgba(255, 255, 255, 0.28)`）：深色上的按钮悬停状态
- **白色 60%**（`oklab(0.999994 ... / 0.577)`）：静音文字的半透明白色
- **白色 64%**（`oklab(0.999994 ... / 0.642)`）：略亮的半透明白色

### 边框与阴影
- **霜边框**（`rgba(214, 235, 253, 0.19)`）：标志性——19% 透明度的冰蓝色调边框
- **霜边框替代**（`rgba(217, 237, 254, 0.145)`）：列表条目略浅变体
- **环阴影**（`rgba(176, 199, 217, 0.145) 0px 0px 0px 1px`）：蓝色调阴影作为边框
- **焦点环**（`rgb(0, 0, 0) 0px 0px 0px 8px`）：重黑色焦点环
- **细微阴影**（`rgba(0, 0, 0, 0.1) 0px 1px 3px, rgba(0, 0, 0, 0.1) 0px 1px 2px -1px`）：最小卡片层次

## 3. 排版规则

### 字体家族
- **展示衬线**：`domaine`（Klim Type Foundry 的 Domaine Display）——英雄标题
- **展示无衬线**：`aBCFavorit`（Dinamo 的 ABC Favorit），回退：`ui-sans-serif, system-ui`——章节标题
- **正文/UI**：`inter`，回退：`ui-sans-serif, system-ui`——正文、按钮、导航
- **等宽**：`commitMono`，回退：`ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas`
- **次要**：`Helvetica`——特定 UI 环境的回退
- **系统**：`-apple-system, system-ui, Segoe UI, Roboto`——嵌入内容

### 原则
- **三字体编辑层次**：Domaine Display（衬线，英雄），ABC Favorit（几何无衬线，章节），Inter（可读正文）。每种字体有严格的角色——它们从不跨越边界。
- **展示字体上的激进负字间距**：Domaine 为 -0.96px，ABC Favorit 为 -2.8px。展示文字感觉压缩、紧迫、经过设计——就像杂志刊头。
- **导航上的正字间距**：ABC Favorit 导航链接使用 +0.35px 字间距——系统中唯一的正字间距。这创造了与压缩标题形成对比的通透、间隔导航文字。
- **OpenType 作为身份**：`"ss01"`、`"ss03"`、`"ss04"`、`"ss11"` 字体样式集在所有 ABC Favorit 和 Domaine 文字上启用，激活给 Resend 排版带来独特个性的替换字形。
- **Commit Mono 作为设计元素**：等宽字体不藏在代码块中——它突出用于代码示例和技术内容，作为一流的视觉元素处理。

## 4. 组件样式

### 按钮

**主要透明药丸**
- 背景：透明
- 文字：`#f0f0f0`
- 内边距：5px 12px
- 圆角：9999px（完整药丸）
- 边框：`1px solid rgba(214, 235, 253, 0.19)`（霜边框）
- 悬停：背景 `rgba(255, 255, 255, 0.28)`（白色玻璃）
- 用途：深色背景上的主要 CTA

**白色实心药丸**
- 背景：`#ffffff`
- 文字：`#000000`
- 内边距：5px 12px
- 圆角：9999px
- 用途：高对比度 CTA（"开始使用"）

**幽灵按钮**
- 背景：透明
- 文字：`#f0f0f0`
- 圆角：4px
- 无边框
- 悬停：微妙背景色调
- 用途：次要操作，标签条目

### 卡片与容器
- 背景：透明或非常微妙的深色色调
- 边框：`1px solid rgba(214, 235, 253, 0.19)`（霜边框）
- 圆角：16px（标准卡片），24px（大型区域/面板）
- 阴影：`rgba(176, 199, 217, 0.145) 0px 0px 0px 1px`（环阴影）
- 卡片内容为深色产品截图和代码演示
- 无传统 box-shadow 层次

### 输入框与表单
- 文字：深色上的 `#f0f0f0`，浅色上的 `#000000`
- 圆角：4px
- 聚焦：基于阴影的环
- 极简样式——继承深色主题

### 导航
- 带霜边框底部的固定深色页眉：`1px solid rgba(214, 235, 253, 0.19)`
- "Resend" 文字标记左对齐
- ABC Favorit 14px 字重 500，+0.35px 字间距，用于导航链接
- 右对齐药丸形 CTA
- 移动端：汉堡折叠

### 图像处理
- 产品截图和代码演示主导内容区域
- 深色主题截图在深色背景上——无缝集成
- 图像圆角：12px–16px
- 带细微渐变叠加的全宽区域

### 独特组件

**标签导航**
- 带细微选择指示器的水平标签
- 标签条目：8px 圆角
- 带细微背景区分的激活状态

**代码预览面板**
- 使用 Commit Mono 的深色代码块
- 霜边框（`rgba(214, 235, 253, 0.19)`）
- 带多色强调标记的语法高亮（橙色、蓝色、绿色、黄色）

**多色强调徽章**
- 每个产品特性有自己的 CSS 变量刻度强调色
- 徽章使用低透明度（12–42%）的强调色作为背景，全透明度作为文字

## 5. 布局原则

### 间距系统
- 基本单位：8px
- 刻度：1px、2px、4px、5px、6px、7px、8px、10px、12px、16px、20px、24px、30px、32px、40px

### 网格与容器
- 宽裕最大宽度的居中内容
- 带受限内部内容的全宽黑色区域
- 单列英雄，向下扩展为特性网格
- 全宽或受限展示的代码预览面板

### 空白哲学
- **电影感黑色空间**：黑色背景就是空白。章节间宽裕的垂直间距（80px–120px+）创造在黑暗中滚动的体验，每个区域像一个场景浮现。
- **内容紧凑，周围广阔**：文字块和卡片内部紧凑，但漂浮在广阔的黑暗空间中——创造孤立的内容"岛屿"。
- **排版引导节奏**：巨大的展示字体（96px）创造自己的垂直节奏——每个标题是一个视觉事件，锚定周围的空间。

### 圆角刻度
- 锐利（4px）：按钮（幽灵），输入框，小交互元素
- 细微（6px）：菜单面板，导航条目
- 标准（8px）：标签，内容块
- 舒适（10px）：强调元素
- 卡片（12px）：剪贴板按钮，中等容器
- 大（16px）：特性卡片，图像，主要按钮
- 区域（24px）：大型面板，区域容器
- 药丸（9999px）：主要 CTA，标签，徽章

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 平面（第 0 层） | 无阴影，透明背景 | 默认——深色虚空上的大多数元素 |
| 环（第 1 层） | `rgba(176, 199, 217, 0.145) 0px 0px 0px 1px` | 卡片、容器的阴影作为边框 |
| 霜边框（第 1b 层） | `1px solid rgba(214, 235, 253, 0.19)` | 显式边框——按钮、分割线、标签 |
| 细微（第 2 层） | `rgba(0, 0, 0, 0.1) 0px 1px 3px, rgba(0, 0, 0, 0.1) 0px 1px 2px -1px` | 浅色卡片层次 |
| 焦点（第 3 层） | `rgb(0, 0, 0) 0px 0px 0px 8px` | 重黑色焦点环——无障碍 |

**阴影哲学**：Resend 几乎完全不使用阴影。在纯黑背景上，传统阴影是不可见的——你无法在虚空中投下阴影。相反，Resend 通过其标志性霜边框（`rgba(214, 235, 253, 0.19)`）创造深度——在黑暗中捕捉光的细薄、冰蓝色调线条。这创造了"玻璃面板漂浮在空间中"的美感，边框是主要的深度机制。

### 装饰性深度
- 英雄内容后方的细微暖渐变发光（橙色/琥珀色调）
- 产品截图通过其自身内部 UI 创造视觉深度
- 无渐变背景——深度来自边框光泽和内容对比

## 7. 注意事项

### 应该做
- 使用纯黑色（`#000000`）作为页面背景——虚空就是画布
- 对所有结构性线条应用霜边框（`rgba(214, 235, 253, 0.19)`）——它们是蓝色调的标志
- 仅将 Domaine Display 用于英雄标题（96px），ABC Favorit 用于章节标题，Inter 用于其他一切
- 在 Domaine 和 ABC Favorit 文字上启用 OpenType `"ss01"`、`"ss04"`、`"ss11"`
- 对主要 CTA 和标签应用药丸圆角（9999px）
- 使用多色强调刻度（橙色/绿色/蓝色/黄色/红色）搭配透明度变体，用于特定上下文高亮
- 将阴影保持在环级别（`0px 0px 0px 1px`）——在黑色上，传统阴影不起作用
- 在 ABC Favorit 导航链接上使用 +0.35px 字间距——唯一的正字间距

### 不应该做
- 不要将背景调亮至 `#000000` 以上——纯黑虚空是不可妥协的
- 不要使用中性灰色边框——所有边框必须有霜蓝色调
- 不要将 Domaine Display 应用于正文——它是仅限展示的衬线字体
- 不要在同一组件中混合强调色——每个特性获得一种强调色
- 不要在深色背景上使用 box-shadow 作为层次——使用霜边框代替
- 不要跳过 OpenType 字体样式集——它们定义排版个性
- 不要在导航链接上使用负字间距——ABC Favorit 导航使用正 +0.35px
- 不要让深色上的按钮不透明——带霜边框的透明度是模式

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 小手机 | <480px | 单列，紧凑内边距，76.8px 英雄 |
| 手机 | 480–600px | 标准手机，堆叠布局 |
| 桌面 | >600px | 完整布局，96px 英雄，扩展区域 |

*注意：Resend 使用极简断点系统——仅检测到 480px 和 600px。设计是桌面优先，干净的移动端折叠。*

### 触摸目标
- 药丸按钮：充足内边距（最小 5px 12px）
- 标签条目：8px 圆角，舒适的点击区域
- 导航链接间距，带 0.35px 字间距，用于视觉分隔

### 折叠策略
- 英雄：Domaine 96px → 移动端 76.8px
- 导航：水平 → 汉堡
- 特性区域：并排 → 堆叠
- 代码面板：保持宽度，需要时水平滚动
- 间距按比例压缩

### 图像行为
- 产品截图保持宽高比
- 深色截图在所有尺寸与深色背景无缝融合
- 跨断点保持圆角（12px–16px）

### 迭代指南
1. 从纯黑开始——所有内容漂浮在虚空中
2. 霜边框（`rgba(214, 235, 253, 0.19)`）是通用结构元素——不是灰色，不是中性
3. 三种字体，三种角色：Domaine（英雄），ABC Favorit（章节），Inter（正文）——绝不跨越
4. OpenType 字体样式集对展示字体是强制性的——它们定义个性
5. 低透明度（12–42%）的多色强调用于背景，全透明度用于文字
6. CTA 和徽章用药丸形（9999px），容器用标准圆角（4px–16px）
7. 无阴影——对虚空使用霜边框作为深度
