# Vercel 设计系统灵感参考

## 1. 视觉主题与氛围

Vercel 的网站是"让开发基础设施隐于无形"的视觉宣言——一套近乎哲学意味的极度克制设计系统。页面以压倒性的白色（`#ffffff`）配合近黑色（`#171717`）文字，营造出画廊般的空旷感，每一个元素都在为自身的存在争取每一个像素。这不是装饰性的极简主义，而是作为工程原则的极简主义。Geist 设计系统对待界面的方式，就如同编译器对待代码——一切多余的 token 都被剥离，直至只剩结构。

定制的 Geist 字体家族是整个系统的核心亮点。Geist Sans 在展示尺寸下使用极度紧缩的字间距（-2.4px 至 -2.88px），令标题呈现出被压缩的、紧迫的、工程化的质感——如同针对生产环境压缩优化过的代码。在正文尺寸下，字间距有所放松，但几何精度依然保持。Geist Mono 作为单等宽字体伴侣，完成代码、终端输出和技术标签的排版任务。两种字体均全局启用 OpenType `"liga"`（连字），为排版细节增添了一层精致感，值得细细品味。

Vercel 有别于其他单色设计系统的地方，在于其"阴影即边框"的设计哲学。Vercel 不使用传统的 CSS 边框，而是采用 `box-shadow: 0px 0px 0px 1px rgba(0,0,0,0.08)`——零偏移、零模糊、1px 扩散的阴影，在不影响盒模型的前提下形成类似边框的视觉线条。这一技术让边框存在于阴影层中，从而实现更流畅的过渡、圆角不剪裁，以及比传统边框更轻盈的视觉重量。整个层深系统建立在多层、多值阴影叠加的基础上，每一层各司其职：一层形成"边框"，一层提供柔和的高度感，一层营造环境深度。

**核心特征：**
- Geist Sans 在展示尺寸下使用极度负字间距（展示级 -2.4px 至 -2.88px）——文字即被压缩的基础设施
- Geist Mono 用于代码和技术标签，全局启用 OpenType `"liga"`
- 阴影即边框技术：`box-shadow 0px 0px 0px 1px` 替代全站传统边框
- 多层阴影叠加实现细腻层深（单一声明中包含边框 + 高度 + 环境阴影）
- 近纯白画布配以 `#171717` 文字——非纯黑，产生微妙对比柔化效果
- 工作流专属强调色：发布红（`#ff5b4f`）、预览粉（`#de1d8d`）、开发蓝（`#0a72ef`）
- 焦点环系统使用 `hsla(212, 100%, 48%, 1)`——饱和蓝色，满足无障碍要求
- 胶囊状徽章（9999px）配以着色背景，用于状态指示

## 2. 色彩体系与角色分工

### 主色
- **Vercel 黑** (`#171717`)：主文字、标题、深色背景表面。非纯黑——略带暖意，避免过于生硬。
- **纯白** (`#ffffff`)：页面背景、卡片表面、深色背景上的按钮文字。
- **真黑** (`#000000`)：次要用途，`--geist-console-text-color-default`，特定终端/代码场景使用。

### 工作流强调色
- **发布红** (`#ff5b4f`)：`--ship-text`，"发布至生产"工作流步骤——温暖、紧迫的珊瑚红。
- **预览粉** (`#de1d8d`)：`--preview-text`，预览部署工作流——鲜艳的洋红粉。
- **开发蓝** (`#0a72ef`)：`--develop-text`，开发工作流——明亮、专注的蓝色。

### 终端 / 代码色
- **终端蓝** (`#0070f3`)：`--geist-console-text-color-blue`，语法高亮蓝。
- **终端紫** (`#7928ca`)：`--geist-console-text-color-purple`，语法高亮紫。
- **终端粉** (`#eb367f`)：`--geist-console-text-color-pink`，语法高亮粉。

### 交互色
- **链接蓝** (`#0072f5`)：主链接颜色，带下划线装饰。
- **焦点蓝** (`hsla(212, 100%, 48%, 1)`)：`--ds-focus-color`，交互元素的焦点环。
- **光环蓝** (`rgba(147, 197, 253, 0.5)`)：`--tw-ring-color`，Tailwind 光环工具类。

### 中性色阶
- **灰 900** (`#171717`)：主文字、标题、导航文字。
- **灰 600** (`#4d4d4d`)：次要文字、描述文案。
- **灰 500** (`#666666`)：第三级文字、弱化链接。
- **灰 400** (`#808080`)：占位文字、禁用状态。
- **灰 100** (`#ebebeb`)：边框、卡片轮廓、分隔线。
- **灰 50** (`#fafafa`)：微妙表面色调、内阴影高亮。

### 表面与遮罩
- **遮罩背景** (`hsla(0, 0%, 98%, 1)`)：`--ds-overlay-backdrop-color`，弹窗/对话框背景遮罩。
- **选中文字** (`hsla(0, 0%, 95%, 1)`)：`--geist-selection-text-color`，文字选中高亮。
- **徽章蓝底** (`#ebf5ff`)：胶囊徽章背景，着色蓝色表面。
- **徽章蓝字** (`#0068d6`)：胶囊徽章文字，较深蓝色确保可读性。

### 阴影与层深
- **边框阴影** (`rgba(0, 0, 0, 0.08) 0px 0px 0px 1px`)：标志性用法——替代传统边框。
- **微妙高度** (`rgba(0, 0, 0, 0.04) 0px 2px 2px`)：卡片的极轻微浮起感。
- **卡片叠层** (`rgba(0,0,0,0.08) 0px 0px 0px 1px, rgba(0,0,0,0.04) 0px 2px 2px, rgba(0,0,0,0.04) 0px 8px 8px -8px, #fafafa 0px 0px 0px 1px`)：完整多层卡片阴影。
- **光环边框** (`rgb(235, 235, 235) 0px 0px 0px 1px`)：标签页和图片的浅灰光环边框。

## 3. 排版规则

### 字体家族
- **主字体**：`Geist`，备用字体：`Arial, Apple Color Emoji, Segoe UI Emoji, Segoe UI Symbol`
- **等宽字体**：`Geist Mono`，备用字体：`ui-monospace, SFMono-Regular, Roboto Mono, Menlo, Monaco, Liberation Mono, DejaVu Sans Mono, Courier New`
- **OpenType 特性**：所有 Geist 文字全局启用 `"liga"`；特定说明文字启用 `"tnum"` 以对齐数字。

### 排版原则
- **压缩即身份**：Geist Sans 在展示尺寸下使用 -2.4px 至 -2.88px 字间距——这是所有主流设计系统中最激进的负字间距。这令文字呈现出_压缩_感，如同针对生产环境优化的代码。字间距随字号递减逐步放松：32px 时为 -1.28px，24px 时为 -0.96px，16px 时为 -0.32px，14px 时恢复正常。
- **连字无处不在**：每一个 Geist 文字元素均启用 OpenType `"liga"`。连字不是装饰——它们是结构性的，创造出更紧密、更高效的字形组合。
- **三种字重，严格分工**：400（正文/阅读）、500（UI/交互）、600（标题/强调）。除微型徽章外不使用粗体（700）。这一窄字重范围通过尺寸和字间距而非字重来建立层级。
- **等宽字体即身份**：Geist Mono 大写配合 `"tnum"` 或 `"liga"` 作为"开发者终端"的声音——紧凑的技术标签，将营销站点与产品本身相连。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 比例：1px、2px、3px、4px、5px、6px、8px、10px、12px、14px、16px、32px、36px、40px
- 值得注意的跳跃：从 16px 直接跳到 32px——主要比例中无 20px 或 24px

### 网格与容器
- 内容最大宽度：约 1200px
- 主区：居中单列，顶部宽松内边距
- 功能章节：2–3 列网格卡片
- 全宽分隔线使用 `border-bottom: 1px solid #171717`
- 代码/仪表盘截图全宽或带边框的限制宽度

### 留白哲学
- **画廊空旷感**：章节间巨大的垂直内边距（80px–120px+）。留白即是设计——它传达出 Vercel 无需证明自身、无需掩盖任何东西的从容。
- **压缩文字，扩展空间**：标题上激进的负字间距与周围充裕的留白形成反差。文字密集，其周围空间广阔。
- **章节节奏**：白色章节与白色章节交替——章节之间无颜色变化。分隔仅通过边框（阴影边框）和间距来实现。

### 圆角比例
- 微小（2px）：内联代码片段、小型 span
- 微妙（4px）：小型容器
- 标准（6px）：按钮、链接、功能性元素
- 舒适（8px）：卡片、列表项
- 图片（12px）：特色卡片、图片容器（顶部圆角）
- 大（64px）：标签导航胶囊
- 超大（100px）：大型导航链接
- 完全胶囊（9999px）：徽章、状态胶囊、标签
- 圆形（50%）：菜单切换按钮、头像容器

## 6. 层深与高度

| 级别 | 处理方式 | 用途 |
|-------|-----------|-----|
| 平面（第 0 级） | 无阴影 | 页面背景、文字块 |
| 光环（第 1 级） | `rgba(0,0,0,0.08) 0px 0px 0px 1px` | 大多数元素的阴影即边框 |
| 浅色光环（第 1b 级） | `rgb(235,235,235) 0px 0px 0px 1px` | 标签页、图片的浅色光环 |
| 微妙卡片（第 2 级） | 光环 + `rgba(0,0,0,0.04) 0px 2px 2px` | 带轻微浮起感的标准卡片 |
| 完整卡片（第 3 级） | 光环 + 微妙 + `rgba(0,0,0,0.04) 0px 8px 8px -8px` + 内部 `#fafafa` 光环 | 特色卡片、高亮面板 |
| 焦点（无障碍） | `2px solid hsla(212, 100%, 48%, 1)` 轮廓 | 所有交互元素的键盘焦点 |

**阴影哲学**：Vercel 拥有现代网页设计中可以说是最精密的阴影系统。它并非 Material Design 传统意义上用阴影表达高度，而是使用多值阴影叠加——每一层都有特定的架构目的：一层创造"边框"（0px 扩散，1px），一层增加环境柔化感（2px 模糊），一层处理远距深度（8px 模糊配负扩散），内部光环（`#fafafa`）则创造令卡片"从内部发光"的微妙高亮。这种分层方式使卡片感觉是被构建出来的，而非漂浮在上面。

### 装饰性层深
- 主图渐变：主图内容背后柔和的多彩渐变晕染（几乎不可见，营造氛围）
- 章节边框：主要章节间使用 `1px solid #171717`（完整深色线条）
- 无背景色变化——深度完全来自阴影分层和边框对比

## 7. 应做与禁止

### 应做
- 在展示尺寸下使用 Geist Sans 配激进负字间距（48px 时 -2.4px 至 -2.88px）
- 使用阴影即边框（`0px 0px 0px 1px rgba(0,0,0,0.08)`）替代传统 CSS 边框
- 在所有 Geist 文字上启用 `"liga"`——连字是结构性的，不是可选的
- 使用三字重系统：400（正文）、500（UI）、600（标题）
- 仅在工作流上下文中使用工作流强调色（红/粉/蓝）
- 为卡片使用多层阴影叠加（边框 + 高度 + 环境 + 内部高亮）
- 保持色彩无色彩倾向——从 `#171717` 到 `#ffffff` 的灰阶即为系统
- 主文字使用 `#171717` 而非 `#000000`——那微妙的暖意至关重要

### 禁止
- 不要在 Geist Sans 上使用正字间距——始终为负值或零
- 不要在正文中使用字重 700（粗体）——600 是最大值，且仅用于标题
- 不要在卡片上使用传统 CSS `border`——使用阴影边框技术
- 不要在 UI 框架中引入暖色（橙色、黄色、绿色）
- 不要将工作流强调色（发布红、预览粉、开发蓝）用于装饰目的
- 不要使用深重阴影（不透明度 > 0.1）——阴影系统是低语级别的
- 不要增加正文字间距——Geist 是为紧密排列而设计的
- 不要在主要操作按钮上使用胶囊圆角（9999px）——胶囊形仅用于徽章/标签
- 不要在卡片阴影中省略内部 `#fafafa` 光环——它是令整个系统生效的发光效果

### 迭代指南
1. 始终使用阴影即边框而非 CSS 边框——`0px 0px 0px 1px rgba(0,0,0,0.08)` 是基础
2. 字间距随字号变化：48px 时 -2.4px，32px 时 -1.28px，24px 时 -0.96px，14px 时恢复正常
