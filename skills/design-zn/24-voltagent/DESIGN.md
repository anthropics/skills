# VoltAgent 设计系统灵感

## 1. 视觉主题与氛围

VoltAgent 的界面是 AI 时代的深空指挥终端——一个面向开发者的黑暗世界，建立在近纯黑色表面（`#050507`）上，唯一的中断是翠绿色能量的电脉冲。整个体验唤起深夜盯着高性能 IDE 的感觉：黑暗、专注，充满目的地生动着。这不是一个友好的 SaaS 落地页——这是一个工程平台，通过代码片段、架构图和原始技术自信来宣告自己。

绿色强调色（`#00d992`）以外科手术般的精准度使用——它从标题、边框和交互元素中发光，就像承载信号的电路板。在碳黑画布上，这种绿色读作"通电"——一个 AI 智能体工程平台的刻意视觉隐喻。支撑调色板完全由温暖中性灰色（`#3d3a39`、`#8b949e`、`#b8b3b0`）构建，柔化黑暗而不引入颜色噪音，创造出纯蓝灰色所缺乏的驾驶舱般温暖感。

排版依赖系统字体栈的标题——实现最快的渲染速度和原生感的权威——而 Inter 以几何精度承载正文和 UI 文字。代码块使用 SFMono-Regular，与开发者在终端中看到的字体相同，在每次滚动时强化工具的可信度。

**核心特征：**
- 碳黑画布（`#050507`），温暖灰色边框收纳（`#3d3a39`）——不冷漠，不无菌
- 单一强调色身份：翠绿信号绿（`#00d992`）作为唯一色彩能量源
- 双排版系统：system-ui 用于权威标题，Inter 用于精确 UI/正文文字，SFMono 用于代码可信度
- 超紧凑标题行高（1.0–1.11），创造密集、压缩的力量块
- 温暖中性调色板（`#3d3a39`、`#8b949e`、`#b8b3b0`），防止深色主题感觉临床化
- 开发者终端美感，代码片段就是英雄内容
- 绿色发光效果（`drop-shadow`、边框强调），使 UI 元素感觉充满电

## 2. 色彩搭配与角色定义

### 主色
- **翠绿信号绿**（`#00d992`）：核心品牌能量——用于强调边框、发光效果和最高信号交互时刻。这是整个界面的"通电"指示器。
- **VoltAgent 薄荷绿**（`#2fd6a1`）：按钮文字的品牌绿变体——比纯信号绿略温暖且更易读，专门用于深色表面上的 CTA 文字。
- **Tailwind 翠绿**（`#10b981`）：生态系统标准绿色，以低透明度（30%）用于微妙背景色调和默认链接。将 VoltAgent 的定制调色板与 Tailwind 实用类桥接。

### 次色与强调色
- **柔和紫**（`#818cf8`）：一种冷靛蓝紫色，谨慎用于次要分类、代码语法高亮和视觉多样性，不与绿色竞争。
- **钴蓝主**（`#306cce`）：Docusaurus 主要深色——用于文档环境中的链接和交互焦点状态。
- **深钴蓝**（`#2554a0`）：最深的主要色调，保留用于文档 UI 的按下/激活状态。
- **环蓝**（`#3b82f6`）：Tailwind 50% 透明度的环形颜色——仅在键盘焦点时可见，用于无障碍合规。

### 表面与背景
- **深渊黑**（`#050507`）：落地页画布——近纯黑，带最淡的温暖底调，比大多数"深色主题"更暗，与绿色强调形成最大对比。
- **碳表面**（`#101010`）：主要卡片和按钮背景——比深渊黑亮一个色调，创造几乎无感的层次。用于所有容纳表面。
- **温暖木炭边框**（`#3d3a39`）：标志性的收纳颜色——不是冷灰色，而是温暖的、几乎带棕色调的深色，防止边框在黑色画布上显得刺眼。

### 中性色与文字
- **雪白**（`#f2f2f2`）：深色表面上的主要文字颜色——不是纯白（`#ffffff`），而是柔和、护眼的米白色。网站上使用最多的颜色（1008 次）。
- **纯白**（`#ffffff`）：保留用于最高强调时刻——幽灵按钮文字和最大对比度标题。以低透明度（5%）用于细微覆盖效果。
- **温暖羊皮纸**（`#b8b3b0`）：次要正文文字——带轻微粉红底调的温暖浅灰，在深色画布上读起来像"纸张"。
- **钢石板**（`#8b949e`）：三级文字、元数据、时间戳和降调内容。提供清晰层次的冷蓝灰色，低于温暖羊皮纸。
- **雾灰**（`#bdbdbd`）：页脚链接和支撑导航文字——悬停时变亮为纯白。
- **薄雾灰**（`#dcdcdc`）：比雾灰略亮，用于悬停时过渡为明亮绿色的次要链接文字。
- **近白**（`#eeeeee`）：最高对比度次要文字，雪白以下一步。

### 语义色与强调色
- **成功翠绿**（`#008b00`）：文档环境中成功状态和积极确认的深绿。
- **成功浅色**（`#80d280`）：成功背景和细微积极指标的柔和粉绿。
- **警告琥珀**（`#ffba00`）：警告警告和注意状态的明亮琥珀色。
- **警告浅色**（`#ffdd80`）：用于警告背景填充的柔化琥珀。
- **危险珊瑚**（`#fb565b`）：错误状态和破坏性操作警告的鲜艳红色。
- **危险玫瑰**（`#fd9c9f`）：错误背景的柔化珊瑚粉。
- **信息青**（`#4cb3d4`）：信息性标注和提示说明的冷青蓝。
- **虚线边框石板**（`#4f5d75`，40% 透明度）：专门用于工作流图表装饰虚线边框的静音蓝灰。

### 渐变系统
- **绿色信号发光**：`drop-shadow(0 0 2px #00d992)` 动画至 `drop-shadow(0 0 8px #00d992)`——在 VoltAgent 闪电标志和交互元素上创造脉冲"电荷"效果。发光像心跳一样扩张收缩。
- **温暖大气模糊**：`rgba(92, 88, 85, 0.2) 0px 0px 15px`——一种温暖色调的漫射阴影，在提升的卡片周围创造柔和的大气发光，在边缘可见而无锐利边界。
- **深度戏剧层次**：`rgba(0, 0, 0, 0.7) 0px 20px 60px` 搭配 `rgba(148, 163, 184, 0.1) 0px 0px 0px 1px inset`——重度戏剧性向下阴影，搭配微弱内嵌石板环，用于最突出的浮动元素。

## 3. 排版规则

### 字体家族
- **主要（标题）**：`system-ui`，回退：`-apple-system, Segoe UI, Roboto, Ubuntu, Cantarell, Noto Sans, Helvetica, Arial, Apple Color Emoji, Segoe UI Emoji, Segoe UI Symbol`
- **次要（正文/UI）**：`Inter`，回退继承自 system-ui 栈。OpenType 特性：`"calt", "rlig"`（上下文替换和必要连字）
- **等宽（代码）**：`SFMono-Regular`，回退：`Menlo, Monaco, Consolas, Liberation Mono, Courier New, monospace`

### 层级

| 展示/英雄 | system-ui | 60px (3.75rem) | 400 | 1.00（紧凑） | -0.65px | 最大冲击力，压缩块 |
| 章节标题 | system-ui | 36px (2.25rem) | 400 | 1.11（紧凑） | -0.9px | 系统中最紧凑的字间距 |
| 子标题 | system-ui | 24px (1.50rem) | 700 | 1.33 | -0.6px | 此尺寸粗体强调 |
| 子标题浅 | system-ui / Inter | 24px (1.50rem) | 300–400 | 1.33 | -0.6px | 浅字重变体，更柔和层次 |
| 上标线 | system-ui | 20px (1.25rem) | 600 | 1.40 | 0.5px | 大写变换，正字间距 |
| 特性标题 | Inter | 20px (1.25rem) | 500–600 | 1.40 | normal | 卡片标题，特性名称 |

### 原则
- **系统原生权威**：展示标题使用 system-ui 而非定制网络字体——这意味着最大文字即时渲染（无 FOIT/FOUT），并继承操作系统的原生个性。在 macOS 上是 SF Pro，在 Windows 上是 Segoe UI。设计接受这种可变性为特性，而非缺陷。
- **紧凑压缩创造密度**：英雄行高极度压缩（1.0），负字间距（-0.65px 至 -0.9px），创造感觉像密集技术规格而非轻盈营销文案的文字块。
- **字重渐变，而非字重对比**：系统使用温和的 300→400→500→600→700 字重进程。粗体（700）保留用于子标题和代码按钮强调。大多数正文文字在 400–500，创造微妙而非戏剧性的层次。
- **大写是赢得的，且是宽间距的**：大写出现时，总是搭配宽松字间距（0.45px–2.52px），将密集词语转化为间隔开的上标线标签。此处理从不应用于标题。
- **默认 OpenType**：system-ui 和 Inter 都启用了 `"calt"` 和 `"rlig"` 特性，确保全程上下文字符调整和连字渲染。

## 4. 组件样式

### 按钮

**幽灵/轮廓（标准）**
- 背景：透明
- 文字：纯白（`#ffffff`）
- 内边距：舒适（12px 16px）
- 边框：细实线温暖木炭（`1px solid #3d3a39`）
- 圆角：舒适圆角（6px）
- 悬停：背景变暗为 `rgba(0, 0, 0, 0.2)`，透明度降至 0.4
- 轮廓：细微绿色色调（`rgba(33, 196, 93, 0.5)`）
- 默认交互元素——低调但明显可点击

**绿色主 CTA**
- 背景：碳表面（`#101010`）
- 文字：VoltAgent 薄荷绿（`#2fd6a1`）
- 内边距：舒适（12px 16px）
- 边框：无可见边框（基于轮廓的焦点指示器）
- 轮廓：VoltAgent 薄荷绿（`rgb(47, 214, 161)`）
- 悬停：与幽灵按钮相同的变暗行为
- "通电"按钮——深色表面上的绿色文字读作活跃的终端命令

**三级/强调容器按钮**
- 背景：碳表面（`#101010`）
- 文字：雪白（`#f2f2f2`）
- 内边距：宽松（20px 四边）
- 边框：厚实线温暖木炭（`3px solid #3d3a39`）
- 圆角：舒适圆角（8px）
- 用于较大交互表面的卡片式按钮处理（代码复制块，特性 CTA）

### 卡片与容器
- 背景：碳表面（`#101010`）——比页面画布亮一个色调
- 边框：标准收纳用 `1px solid #3d3a39`（温暖木炭）；高亮/激活卡片用 `2px solid #00d992`
- 圆角：内容卡片舒适圆角（8px）；较小内联容器略圆（4–6px）
- 阴影第 1 级：温暖大气模糊（`rgba(92, 88, 85, 0.2) 0px 0px 15px`）用于标准层次
- 阴影第 2 级：深度戏剧（`rgba(0, 0, 0, 0.7) 0px 20px 60px` + `rgba(148, 163, 184, 0.1) 0px 0px 0px 1px inset`）用于英雄/特性展示卡片
- 悬停行为：可能边框颜色向绿色强调变化或微妙透明度增加
- 虚线变体：工作流/图表容器用 `1px dashed rgba(79, 93, 117, 0.4)`——与实线边框内容卡片视觉区分

### 输入框与表单
- 未提取明确输入框标记数据——网站以落地页为中心，表单 UI 极少
- npm 安装命令（`npm create voltagent-app@latest`）作为代码块呈现而非输入框
- 推断样式：碳表面背景，温暖木炭边框，VoltAgent 薄荷绿焦点环，雪白文字

### 导航
- 深渊黑画布上的固定顶部导航栏
- 标志：VoltAgent 闪电图标，带动画绿色发光（`drop-shadow` 循环 2px–8px）
- 导航结构：标志 → 产品下拉 → 用例下拉 → 资源下拉 → GitHub 星数徽章 → 文档 CTA
- 链接文字：雪白（`#f2f2f2`），14–16px Inter，字重 500
- 悬停：链接过渡为绿色变体（`#00c182` 或 `#00ffaa`）
- GitHub 徽章：直接集成在导航中的社会证明元素
- 移动端：折叠为汉堡菜单，单列垂直布局

### 图像处理
- 深色主题产品截图和架构图主导
- 代码块作为主要视觉内容处理——SFMono-Regular 语法高亮
- 智能体工作流可视化作为带绿色连接线的交互节点图
- 装饰性点阵图案背景出现在英雄区域后方
- 在卡片容器内全出血，遵循 8px 圆角

### 独特组件

**npm 安装命令块**
- 突出的代码片段（`npm create voltagent-app@latest`），样式为可复制命令
- 碳表面上的 SFMono-Regular，带复制到剪贴板按钮
- 作为主要 CTA 使用——"先安装，后阅读"的开发者心理

**公司标志滚动条**
- 开发者/公司标志的水平滚动条
- 无限动画（`scrollLeft`/`scrollRight`，25–80s 持续时间）
- 悬停时暂停，并为有运动减少偏好的用户暂停
- 展示生态系统采用而不会混乱布局

**特性章节卡片**
- 结合代码示例和描述文字的大卡片
- 左侧：带语法高亮的代码片段；右侧：特性描述
- 高亮/激活特性上的绿色强调边框（`2px solid #00d992`）
- 内部内边距：宽松（估计 24–32px）

**智能体流程图**
- 展示智能体协调的交互节点图可视化
- 连接线使用 VoltAgent 绿色变体
- 节点样式为温暖木炭边框系统内的小卡片

**社区/GitHub 区域**
- 大型 GitHub 图标作为视觉锚点
- 星数和贡献者指标突出显示
- 温暖社会证明：页脚的 Discord、X、Reddit、LinkedIn、YouTube 链接

## 5. 布局原则

### 间距系统
- 基本单位：8px
- 刻度：2px、4px、5px、6px、6.4px、8px、12px、16px、20px、24px、28px、32px、40px、48px、64px
- 按钮内边距：12px 16px（标准），20px（容器按钮）
- 卡片内部内边距：约 24–32px
- 章节垂直间距：宽松（主要章节间估计 64–96px）
- 组件间距：兄弟卡片/元素间 16–24px

### 网格与容器
- 最大容器宽度：约 1280–1440px，居中
- 英雄区：单列居中，最大呼吸空间
- 特性章节：交替不对称布局（代码左/文字右，然后反转）
- 标志滚动条：全宽水平滚动，突破容器限制
- 卡片网格：特性展示 2–3 列
- 集成网格：合作伙伴/集成图标的响应式多列

### 空白哲学
- **章节间电影感呼吸空间**：巨大的垂直间距创造"逐章节滚动"体验——每个章节感觉像一个新场景。
- **组件内密集**：卡片和代码块内部紧凑，行高紧，内边距受控。信息集中，不分散。
- **边框定义分隔**：VoltAgent 不仅依赖空白，还使用温暖木炭边框系统（`#3d3a39`）来划分内容区域。边框就是空白信号。
- **英雄优先层次**：页面顶部占据最多空间——"AI 智能体工程平台"标题和 npm 命令在第一个内容章节出现前获得最大垂直空间。

### 圆角刻度
- 近方形（4px）：小内联元素、SVG 容器、代码 span——最锐利处理，传达技术精度
- 略微圆角（6px）：按钮、链接、剪贴板操作——交互元素的主力圆角
- 代码专用（6.4px）：代码块、`pre` 元素、剪贴板复制目标——与标准 6px 的刻意微区分
- 舒适圆角（8px）：内容卡片、特性容器、强调按钮——标准收纳圆角
- 药丸形（9999px）：标签、徽章、状态指示器、药丸形导航元素——小分类标签的最圆处理

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 平面（第 0 层） | 无阴影，无边框 | 页面背景（`#050507`），内联文字 |
| 收纳（第 1 层） | `1px solid #3d3a39`，无阴影 | 标准卡片，导航栏，代码块 |
| 强调（第 2 层） | `3px solid #3d3a39`，无阴影 | 大型交互按钮，强调容器 |
| 强调色（第 3 层） | `2px solid #00d992`，无阴影 | 激活/高亮特性卡片，选中状态 |
| 大气发光（第 4 层） | `rgba(92, 88, 85, 0.2) 0px 0px 15px` | 提升卡片，悬停状态，柔和大气提升 |
| 戏剧浮动（第 5 层） | `rgba(0, 0, 0, 0.7) 0px 20px 60px` + `rgba(148, 163, 184, 0.1) 1px inset` | 英雄特性展示，模态框，最大层次内容 |

**阴影哲学**：VoltAgent 主要通过**边框粗细和颜色**传达深度，而非阴影。标准 `1px solid #3d3a39` 边框就是层次——添加 `3px` 边框粗细或切换为绿色（`#00d992`）比添加阴影更能传达重要性。当阴影出现时，它们要么是温暖漫射的（第 4 层），要么是电影戏剧性的（第 5 层）——从不是中等或通用的。

### 装饰性深度
- **绿色信号发光**：VoltAgent 闪电标志以翠绿信号绿中 2px 和 8px 模糊半径之间循环的 `drop-shadow` 动画脉冲。这是最独特的装饰性元素——它使标志感觉"通电了"。
- **温暖木炭收纳线**：`#3d3a39` 边框的温暖色调在冷黑色上创造微妙的视觉温暖，仿佛卡片从内部微微发热。
- **虚线工作流线**：`1px dashed rgba(79, 93, 117, 0.4)` 为架构图创造蓝图般的美感，与实线内容边框视觉区分。

## 7. 注意事项

### 应该做
- 使用深渊黑（`#050507`）作为落地页背景，碳表面（`#101010`）用于所有收纳元素——两层暗色系统至关重要
- 将翠绿信号绿（`#00d992`）专门保留用于高信号时刻：激活边框、发光效果和最重要的交互强调
- 将 VoltAgent 薄荷绿（`#2fd6a1`）用于深色表面上的按钮文字——比纯信号绿更易读
- 保持标题行高压缩（1.0–1.11），负字间距，创造密集、权威的文字块
- 使用温暖灰调色板（`#3d3a39`、`#8b949e`、`#b8b3b0`）用于边框和次要文字——温暖感防止深色主题感觉无菌
- 将代码片段呈现为主要内容——它们是英雄元素，不是辅助插图
- 使用边框粗细（1px → 2px → 3px）和颜色变化（`#3d3a39` → `#00d992`）传达深度和重要性，而非依赖阴影
- system-ui 用于标题，Inter 用于正文文字——原生字体的速度/权威与几何无衬线字体的精度相结合
- 所有代码内容使用 SFMono-Regular——这是开发者可信度信号
- 在所有文字中应用 `"calt"` 和 `"rlig"` OpenType 特性

### 不应该做
- 不要使用明亮或浅色背景作为主要表面——整个身份生活在近黑色上
- 不要引入暖色（橙色、红色、黄色）作为装饰性强调——调色板严格限于绿色 + 暖中性色在黑色上。暖色仅保留用于语义状态（警告、错误）
- 不要在大表面或背景填充上使用翠绿信号绿（`#00d992`）——它是强调色，从不是表面色
- 不要将标题行高增加至 1.33 以上——压缩密度是工程平台身份的核心
- 不要慷慨地使用重阴影——深度来自边框处理，而非 box-shadow。阴影仅保留用于第 4–5 层
- 不要将纯白（`#ffffff`）用作默认正文文字——雪白（`#f2f2f2`）是标准。纯白保留用于最高强调标题和按钮文字
- 不要混入衬线或装饰字体——整个系统是几何无衬线 + 等宽
- 不要在内容卡片上使用大于 8px 的圆角——9999px（药丸）仅用于小标签和徽章
- 不要跳过温暖灰色边框系统——没有 `#3d3a39` 边框的卡片会失去收纳感，在深色画布上模糊漂浮
- 不要激进地动画——动画慢而微妙（滚动条 25–100s 持续时间，温和发光脉冲）。快速运动与"工程精度"氛围相矛盾

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 小手机 | <420px | 最小布局，一切堆叠，英雄文字缩小至约 24px |
| 手机 | 420–767px | 单列，汉堡导航，全宽卡片，英雄文字约 36px |
| 平板 | 768–1024px | 两列网格开始，精简导航，中等英雄文字 |
| 桌面 | 1025–1440px | 完整多列布局，带下拉的展开导航，大英雄（60px） |
| 大桌面 | >1440px | 最大宽度容器居中（估计 1280–1440px），宽裕水平边距 |

*共检测到 23 个断点，范围 360px 至 1992px——表明流式、高度响应的网格系统，而非固定断点捕捉。*

### 触摸目标
- 按钮使用舒适内边距（最小 12px 16px），确保足够的触摸区域
- 导航链接有足够的间距用于拇指导航
- 交互卡片表面足够大，可作为全触摸目标
- 推荐最小触摸目标：44x44px

### 折叠策略
- **导航**：带下拉菜单的完整水平导航在移动端折叠为汉堡菜单
- **特性网格**：3 列 → 2 列 → 单列垂直堆叠
- **英雄文字**：60px → 36px → 24px 渐进缩放，保持压缩比
- **标志滚动条**：调整滚动速度和条目尺寸；保持无限循环
- **代码块**：较小视口上水平滚动而非换行——保持代码可读性
- **章节内边距**：等比例减少，但章节间保持宽裕垂直节奏
- **卡片**：移动端垂直堆叠，全宽处理，保持内部内边距

### 图像行为
- 深色主题截图和图表在容器内等比例缩放
- 智能体流程图在窄视口上简化或水平滚动
- 点阵装饰背景随视口缩放
- 断点间无可见艺术指导变化——相同裁剪，等比例缩放
- 折叠内容懒加载（Docusaurus 默认行为）
