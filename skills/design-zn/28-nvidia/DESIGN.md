# NVIDIA 设计系统灵感

## 1. 视觉主题与氛围

NVIDIA 的网站是一种高对比度、技术前沿的体验，通过设计克制来传达原始计算能力。页面建立在纯黑色（`#000000`）和白色（`#ffffff`）的基础上，以 NVIDIA 的标志性绿色（`#76b900`）为点缀——这种颜色如此特定，以至于它作为品牌指纹发挥作用。这不是自然界的郁郁葱葱的绿色；而是 GPU 渲染光的电气、黄绿色偏移的绿色，一种介于黄绿色和深绿色之间的颜色，立即向技术领域的任何人传达"NVIDIA"。

定制的 NVIDIA-EMEA 字体家族（带 Arial 和 Helvetica 回退）创造了干净、工业化的排版声音。标题 36px 粗体，行高紧凑 1.25，创造密集、权威的文字块。字体缺乏硅谷无衬线字体的几何趣味性——它是欧洲的、务实的、以工程为中心的。正文文字 15-16px，适合阅读但不宽松，保持屏幕空间像 GPU 内存一样被优化的感觉。

将 NVIDIA 的设计与其他深色背景科技网站区分开来的是对绿色强调色的严格使用。`#76b900` 出现在边框（`2px solid #76b900`）、链接下划线（`underline 2px rgb(118, 185, 0)`）和 CTA 中——但从不作为主要内容的背景或大型表面区域。绿色是信号，不是表面。结合深度阴影系统（`rgba(0, 0, 0, 0.3) 0px 0px 5px`）和最小圆角（1-2px），整体效果是以像素渲染的精密工程硬件。

**核心特征：**
- NVIDIA 绿（`#76b900`）作为纯强调色——仅用于边框、下划线和交互高亮
- 黑色（`#000000`）主导背景，深色区域白色（`#ffffff`）文字
- NVIDIA-EMEA 定制字体，Arial/Helvetica 回退——工业化、欧式、干净
- 紧凑行高（标题 1.25）创造密集、权威的文字块
- 最小圆角（1-2px）——锐利、精工角，贯穿始终
- 绿色边框按钮（`2px solid #76b900`）作为主要交互模式
- Font Awesome 6 Pro/Sharp 图标系统，字重 900，用于锐利图标
- 多框架架构（PrimeReact、Fluent UI、Element Plus），支持丰富交互组件

## 2. 色彩搭配与角色定义

### 主品牌色
- **NVIDIA 绿**（`#76b900`）：标志性——边框、链接下划线、CTA 轮廓、激活指示器。从不用作大型表面填充。
- **真黑**（`#000000`）：主要页面背景，浅色表面上的文字，主导色调。
- **纯白**（`#ffffff`）：深色背景上的文字，浅色区域背景，卡片表面。

### 扩展品牌调色板
- **NVIDIA 绿浅**（`#bff230`）：亮莱姆强调色，用于高亮和悬停状态。
- **橙色 400**（`#df6500`）：暖强调色，用于警告、特色徽章或能源相关环境。
- **黄色 300**（`#ef9100`）：次要暖强调色，产品类别高亮。
- **黄色 050**（`#feeeb2`）：用于标注背景的浅暖表面。

### 状态与语义色
- **红色 500**（`#e52020`）：错误状态，破坏性操作，严重警告。
- **红色 800**（`#650b0b`）：严重警告背景的深红。
- **绿色 500**（`#3f8500`）：成功状态，积极指标（比品牌绿更深）。
- **蓝色 700**（`#0046a4`）：信息性强调，链接悬停替代。

### 装饰色
- **紫色 800**（`#4d1368`）：渐变端点深紫，高级/AI 环境。
- **紫色 100**（`#f9d4ff`）：浅紫色表面色调。
- **紫红色 700**（`#8c1c55`）：特殊促销或特色内容的丰富强调色。

### 中性刻度
- **灰色 300**（`#a7a7a7`）：静音文字，禁用标签。
- **灰色 400**（`#898989`）：次要文字，元数据。
- **灰色 500**（`#757575`）：三级文字，占位符，页脚。
- **边框灰**（`#5e5e5e`）：细微边框，分割线。
- **近黑**（`#1a1a1a`）：深色表面，黑色页面上的卡片背景。

### 交互状态
- **链接默认（深色背景）**（`#ffffff`）：深色背景上的白色链接。
- **链接默认（浅色背景）**（`#000000`）：浅色背景上带绿色下划线的黑色链接。
- **链接悬停**（`#3860be`）：所有链接变体上悬停时的蓝色变化。
- **按钮悬停**（`#1eaedb`）：按钮悬停状态的青色高亮。
- **按钮激活**（`#007fff`）：激活/按下按钮状态的亮蓝色。
- **焦点环**（`#000000 solid 2px`）：键盘焦点的黑色轮廓。

### 阴影与深度
- **卡片阴影**（`rgba(0, 0, 0, 0.3) 0px 0px 5px 0px`）：提升卡片的细微环境阴影。

## 3. 排版规则

### 字体家族
- **主要**：`NVIDIA-EMEA`，回退：`Arial, Helvetica, sans-serif`
- **图标字体**：`Font Awesome 6 Pro`（字重 900 用于实心图标，700 用于常规）
- **图标锐利**：`Font Awesome 6 Sharp`（字重 300 用于浅色图标，400 用于常规）

### 原则
- **粗体作为默认声音**：NVIDIA 在标题、按钮、链接和标签上大量使用字重 700。400 字重保留用于正文和描述——其他一切都是粗体，投射自信和权威。
- **紧凑标题，宽松正文**：标题行高始终为 1.25（紧凑），正文文字放宽至 1.50-1.67。这种对比在内容块顶部创造视觉密度，在段落中提供舒适的可读性。
- **大写用于导航**：链接标签使用 `text-transform: uppercase` 配字重 700，创造读起来像硬件规格标签的导航声音。
- **无装饰字间距**：除紧凑按钮（0.144px）外，字间距全程 normal。字体本身携带工业特色，无需调整。

## 5. 布局原则

### 间距系统
- 基本单位：8px
- 刻度：1px、2px、3px、4px、5px、6px、7px、8px、9px、10px、11px、12px、13px、15px
- 主要内边距值：8px、11px、13px、16px、24px、32px
- 章节间距：垂直内边距 48-80px

### 网格与容器
- 最大内容宽度：约 1200px（受限）
- 带受限文字的全宽英雄区域
- 特性章节：产品卡片 2-3 列网格
- 文章/博客内容单列
- 文档的侧边栏布局

### 空白哲学
- **目的性密度**：NVIDIA 比典型 SaaS 网站使用更紧凑的间距，反映技术内容的密度。空白用于分隔概念，而非创造奢侈的空旷感。
- **章节节奏**：深色区域与白色区域交替，使用背景颜色（不仅仅是间距）分隔内容块。
- **卡片密度**：产品卡片紧密排列，间隔 16-20px，创造目录感而非画廊感。

### 圆角刻度
- 微型（1px）：内联 span，微小元素
- 标准（2px）：按钮、卡片、容器、输入框——几乎所有内容的默认值
- 圆形（50%）：头像图像，圆形标签指示器

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 平面（第 0 层） | 无阴影 | 页面背景，内联文字 |
| 细微（第 1 层） | `rgba(0,0,0,0.3) 0px 0px 5px 0px` | 标准卡片，模态框 |
| 边框（第 1b 层） | `1px solid #5e5e5e` | 内容分隔线，区域边框 |
| 绿色强调（第 2 层） | `2px solid #76b900` | 激活元素，CTA，选中条目 |
| 聚焦（无障碍） | `2px solid #000000` 轮廓 | 键盘焦点环 |

**阴影哲学**：NVIDIA 的深度系统极简且功能性。本质上只有一个阴影值——5px 环境模糊，30% 透明度——谨慎用于卡片和模态框。主要深度信号不是阴影而是_颜色对比_：黑色背景紧邻白色区域，黑色表面上的绿色边框。这创造了硬件式的视觉分层，深度来自材料差异，而非模拟光线。

### 装饰性深度
- 英雄内容后方的绿色渐变洗涤
- 区域过渡的深到更深渐变（黑到近黑）
- 无玻璃态或模糊效果——清晰胜于氛围

## 7. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 小手机 | <375px | 紧凑单列，减少内边距 |
| 手机 | 375-425px | 标准手机布局 |
| 大手机 | 425-600px | 较宽手机，部分两列提示 |
| 小平板 | 600-768px | 两列网格开始 |
| 平板 | 768-1024px | 完整卡片网格，展开导航 |
| 桌面 | 1024-1350px | 标准桌面布局 |
| 大桌面 | >1350px | 最大内容宽度，宽裕边距 |

### 触摸目标
- 按钮使用 11px 13px 内边距，舒适的点击目标
- 导航链接 14px 大写，有充足间距
- 绿色边框按钮在深色背景上提供高对比度触摸目标
- 移动端：汉堡菜单折叠，带全屏覆盖层

### 折叠策略
- 英雄：36px 标题等比例缩小
- 导航：完整水平导航约 1024px 时折叠为汉堡菜单
- 产品卡片：3 列到 2 列到单列堆叠
- 页脚：多列网格折叠为单堆叠列
- 章节间距：64-80px 移动端减少至 32-48px
- 图像：保持宽高比，缩放至容器宽度

### 图像行为
- GPU/产品渲染在所有尺寸保持高分辨率
- 英雄图像随视口等比例缩放
- 卡片图像使用一致的宽高比
- 全出血深色区域保持边缘到边缘处理

## 8. 响应式行为（扩展）

### 排版缩放
- 展示 36px 移动端缩小至约 24px
- 章节标题 24px 移动端缩小至约 20px
- 正文文字在所有断点保持 15-16px
- 按钮文字保持 16px，确保一致的点击目标

### 深色/浅色区域策略
- 深色区域（黑色背景，白色文字）与浅色区域（白色背景，黑色文字）交替
- 绿色强调在两种表面类型上保持一致
- 深色上：链接白色，下划线绿色
- 浅色上：链接黑色，下划线绿色
- 这种交替创造自然的滚动节奏和内容分组

### 迭代指南
1. 始终将 `#76b900` 作为强调色，绝不作为背景填充——它是边框、下划线和高亮的信号颜色
2. 按钮默认是透明带绿色边框——填充背景仅在悬停/激活状态出现
3. 字重 700 是所有交互和标题元素的主导声音；400 仅用于正文段落
