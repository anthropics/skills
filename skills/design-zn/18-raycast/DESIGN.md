# Raycast 设计系统灵感

## 1. 视觉主题与氛围

Raycast 的营销网站感觉像一件精密仪器的暗色内部——一块黑曜石雕刻的瑞士腕表表壳。背景不仅仅是暗色，而是一种近乎黑色的蓝色调（`#07080a`），创造出一种身处 macOS 原生应用内部而非网页的感觉。每一个界面元素、每一条边框、每一道阴影都经过精确校准，以唤起高性能桌面工具的感觉：快速、极简、值得信赖。

标志性的手法是借自 macOS 窗口外壳的多层阴影系统：带内嵌高光的多层盒阴影，模拟物理深度，仿佛卡片和按钮是暗色桌面上真实被按压或凸起的玻璃元素。结合 Raycast Red（`#FF6363`）——几乎专门用于主视觉区标志性的对角条纹图案——整套色彩方案创造出一种读起来是"有个性的强大工具"的品牌感。红色不主导；它是点睛之笔。

Inter 被用于所有地方——标题、正文、按钮、说明文字——配以广泛的 OpenType 特性（`calt`、`kern`、`liga`、`ss03`），创造出一致、可读的排版声音。正字间距（0.2px–0.4px 用于正文）对于暗色 UI 来说不同寻常，赋予文字一种通透、可呼吸的质感，与密集的暗色界面形成平衡。GeistMono 出现于代码元素，强化开发者工具的身份。

**核心特征：**
- 近黑色蓝色调背景（`#07080a`）——非纯黑，有微妙蓝色偏移
- 带多层内嵌高光的 macOS 原生阴影系统，模拟物理深度
- Raycast Red（`#FF6363`）作为点睛颜色——主视觉条纹，非遍布全局
- Inter 配正字间距（0.2px）营造通透、可读的暗色模式体验
- Radix UI 组件原语驱动交互层
- 细微的 rgba 白色边框（0.06–0.1 透明度）用于暗色界面上的边界划定
- 带渐变键帽和厚重阴影的键盘快捷键样式

## 2. 色彩方案与角色

### 主色
- **近黑色蓝** (`#07080a`)：主页面背景——带微妙蓝冷底调的基础空间
- **纯白** (`#ffffff`)：主要标题文字，高强调元素
- **Raycast Red** (`#FF6363` / `hsl(0, 100%, 69%)`)：品牌点缀——主视觉条纹，危险状态，关键高亮

### 次要与点缀色
- **Raycast Blue** (`hsl(202, 100%, 67%)` / ~`#55b3ff`)：交互点缀——链接，焦点状态，选中项目
- **Raycast Green** (`hsl(151, 59%, 59%)` / ~`#5fc992`)：成功状态，正向指示器
- **Raycast Yellow** (`hsl(43, 100%, 60%)` / ~`#ffbc33`)：警告点缀，高亮
- **蓝色半透明** (`hsla(202, 100%, 67%, 0.15)`)：交互界面的蓝色调叠加
- **红色半透明** (`hsla(0, 100%, 69%, 0.15)`)：危险/错误界面的红色调叠加

### 界面与背景
- **深色背景** (`#07080a`)：页面画布，最暗的界面
- **界面 100** (`#101111`)：抬升界面，卡片背景
- **键帽起始** (`#121212`)：键盘键渐变起始
- **键帽结束** (`#0d0d0d`)：键盘键渐变结束
- **卡片界面** (`#1b1c1e`)：徽章背景，标签填充，抬升容器
- **按钮前景** (`#18191a`)：浅色背景上按钮文字的暗色界面

### 中性色与文字
- **近白** (`#f9f9f9` / `hsl(240, 11%, 96%)`)：主要正文，高强调内容
- **浅灰** (`#cecece` / `#cdcdce`)：次要正文，描述
- **银色** (`#c0c0c0`)：第三级文字，抑制标签
- **中灰** (`#9c9c9d`)：链接默认颜色，次要导航
- **暗灰** (`#6a6b6c`)：禁用文字，低强调标签
- **深灰** (`#434345`)：弱化边框，非活动导航链接
- **边框** (`hsl(195, 5%, 15%)` / ~`#252829`)：卡片和分割线的标准边框颜色
- **深色边框** (`#2f3031`)：分隔线，表格边框

### 语义与点缀
- **错误红** (`hsl(0, 100%, 69%)`)：错误状态，破坏性操作
- **成功绿** (`hsl(151, 59%, 59%)`)：成功确认，正向状态
- **警告黄** (`hsl(43, 100%, 60%)`)：警告，需注意状态
- **信息蓝** (`hsl(202, 100%, 67%)`)：信息高亮，链接

### 渐变系统
- **键盘键渐变**：从 `#121212`（顶部）到 `#0d0d0d`（底部）的线性渐变——模拟键的物理深度
- **温暖光晕**：`rgba(215, 201, 175, 0.05)` 辐射扩散——特色元素后方微妙的温暖环境光晕

## 3. 排版规则

### 字体系列
- **主字体**：`Inter` — 人文无衬线字体，用于所有地方。备用字体：`Inter Fallback`，系统无衬线字体
- **系统字体**：`SF Pro Text` — Apple 系统字体，用于特定 macOS 原生 UI 元素。备用字体：`SF Pro Icons`、`Inter`、`Inter Fallback`
- **等宽字体**：`GeistMono` — Vercel 的等宽字体，用于代码元素。备用字体：`ui-monospace`、`SFMono-Regular`、`Roboto Mono`、`Menlo`、`Monaco`
- **OpenType 特性**：全局启用 `calt`、`kern`、`liga`、`ss03`；展示文字使用 `ss02`、`ss08`；主视觉标题禁用 `liga`（`"liga" 0`）

### 排版原则
- **暗色界面正字距**：与大多数暗色 UI 使用紧凑或中性字间距不同，Raycast 应用 +0.2px 至 +0.4px——创造通透、可读的感觉，补偿暗色背景
- **500 字重作为基准**：大多数正文使用中等字重（500），而非常规（400）——微妙的额外厚度提升暗色界面上的易读性
- **展示克制**：64px/600 的主视觉文字自信而不过分——Raycast 避免排版奇观，追求功能优雅
- **OpenType 无处不在**：`ss03`（样式集 3）在 Inter 上全局启用，赋予字体略微几何化、工具感的质感

## 4. 组件样式

### 按钮
- **主要胶囊**：透明背景，白色文字，胶囊形（86px 圆角），多层内嵌阴影（`rgba(255, 255, 255, 0.1) 0px 1px 0px 0px inset`）。悬停：透明度 0.6
- **次要按钮**：透明背景，白色文字，6px 圆角，`1px solid rgba(255, 255, 255, 0.1)` 边框，微妙投影阴影（`rgba(0, 0, 0, 0.03) 0px 7px 3px`）。悬停：透明度 0.6
- **幽灵按钮**：无背景或边框，灰色文字（`#6a6b6c`），86px 圆角，相同内嵌阴影。悬停：透明度 0.6，文字变亮至白色
- **行动召唤（下载）**：半透明白色背景（`hsla(0, 0%, 100%, 0.815)`），深色文字（`#18191a`），胶囊形。悬停：完全白色背景（`hsl(0, 0%, 100%)`）
- **过渡**：所有按钮使用透明度过渡而非背景色变化来实现悬停——Raycast 标志性的交互模式

### 卡片与容器
- **标准卡片**：`#101111` 界面，`1px solid rgba(255, 255, 255, 0.06)` 边框，12px–16px 圆角
- **抬升卡片**：外环阴影 `rgb(27, 28, 30) 0px 0px 0px 1px` + 内环 `rgb(7, 8, 10) 0px 0px 0px 1px inset`——创造双环边界效果
- **特色卡片**：16px–20px 圆角，主视觉元素后方微妙温暖光晕（`rgba(215, 201, 175, 0.05) 0px 0px 20px 5px`）
- **悬停**：卡片通过边框透明度增加或微妙阴影增强略微变亮

### 输入框与表单
- 暗色输入字段，`#07080a` 背景，`1px solid rgba(255, 255, 255, 0.08)` 边框，8px 圆角
- 焦点状态：边框变亮，蓝色光晕（`hsla(202, 100%, 67%, 0.15)`）环出现
- 文字：`#f9f9f9` 输入文字颜色，`#6a6b6c` 占位文字
- 标签：`#9c9c9d`，14px，字重 500

### 导航
- **顶部导航**：与页面融合的暗色背景，白色文字链接，16px 字重 500
- **导航链接**：灰色文字（`#9c9c9d`）→ 悬停时白色，悬停时下划线装饰
- **行动召唤按钮**：导航末端的半透明白色胶囊
- **移动端**：折叠为汉堡菜单，保持暗色主题
- **粘性**：顶部固定，带微妙边框分隔符

### 图像处理
- **产品截图**：macOS 窗口外壳风格——圆角（12px），深阴影模拟浮动窗口
- **全出血区段**：暗色截图与暗色背景无缝融合
- **主视觉插图**：Raycast Red 对角条纹图案——抽象、几何、定义品牌
- **应用 UI 嵌入**：展示实际的 Raycast 命令面板和扩展——产品即内容

### 键盘快捷键键帽
- **键帽样式**：渐变背景（`#121212` → `#0d0d0d`），厚重多层阴影（`rgba(0, 0, 0, 0.4) 0px 1.5px 0.5px 2.5px` + 内嵌阴影），营造逼真的物理键外观
- 圆角：单个键 4px–6px

### 徽章与标签
- **中性徽章**：`#1b1c1e` 背景，白色文字，6px 圆角，14px 字体字重 500，`0px 6px` 内边距
- 紧凑、胶囊状的分类处理

## 5. 布局原则

### 间距系统
- **基础单位**：8px
- **比例**：1px、2px、3px、4px、8px、10px、12px、16px、20px、24px、32px、40px
- **区段内边距**：主要区段之间 80px–120px 垂直
- **卡片内边距**：16px–32px 内部间距
- **组件间距**：相关元素之间 8px–16px

### 栅格与容器
- **最大宽度**：约 1200px 容器（断点 1204px），居中
- **列模式**：单列主视觉，2–3 列功能栅格，全宽展示区段
- **应用展示**：产品 UI 呈现在居中窗口框架中

### 留白哲学
- **戏剧性负空间**：区段漂浮于广阔暗色虚空中，在功能之间创造电影感节奏
- **产品密集，营销稀疏**：产品 UI 截图信息密集，但周边营销文案使用极简文字配慷慨间距
- **垂直节奏**：区段内元素之间保持一致的 24px–32px 间距

### 圆角比例
- **2px–3px**：微型元素，代码片段，极小指示器
- **4px–5px**：键盘键，小型交互元素
- **6px**：按钮，徽章，标签——主力圆角
- **8px**：输入框，内联组件
- **9px–11px**：图像，中型容器
- **12px**：标准卡片，产品截图
- **16px**：大型卡片，功能区段
- **20px**：主视觉卡片，突出容器
- **86px+**：胶囊形按钮，导航行动召唤按钮——完整胶囊形

## 6. 深度与层级

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 层级 0（虚空） | 无阴影，`#07080a` 界面 | 页面背景 |
| 层级 1（微妙） | `rgba(0, 0, 0, 0.28) 0px 1.189px 2.377px` | 最小抬升，内联元素 |
| 层级 2（环形） | `rgb(27, 28, 30) 0px 0px 0px 1px` 外环 + `rgb(7, 8, 10) 0px 0px 0px 1px inset` 内环 | 卡片边界，双环技术 |
| 层级 3（按钮） | `rgba(255, 255, 255, 0.05) 0px 1px 0px 0px inset` + `rgba(255, 255, 255, 0.25) 0px 0px 0px 1px` + `rgba(0, 0, 0, 0.2) 0px -1px 0px 0px inset` | macOS 原生按钮按压——顶部白色高光，底部暗色内嵌 |
| 层级 4（键帽） | 5 层阴影叠加，带内嵌按压效果 | 键盘快捷键键帽——物理 3D 外观 |
| 层级 5（浮动） | `rgba(0, 0, 0, 0.5) 0px 0px 0px 2px` + `rgba(255, 255, 255, 0.19) 0px 0px 14px` + 内嵌 | 命令面板，浮动面板——厚重深度配光晕 |

### 阴影哲学
Raycast 的阴影系统是 Web 上最接近 macOS 原生的。多层阴影结合：
- **外环**用于边界划定（取代传统边框）
- **顶部内嵌高光**（`rgba(255, 255, 255, 0.05–0.25)`）模拟来自上方的光源
- **底部内嵌暗色**（`rgba(0, 0, 0, 0.2)`）模拟下方阴影
- 效果是物理感的：元素感觉像玻璃或拉丝金属，而非平面矩形

### 装饰性深度
- **温暖光晕**：`rgba(215, 201, 175, 0.05) 0px 0px 20px 5px`，在特色元素后方——寒冷暗色画布上的微妙温暖光环
- **蓝色信息光晕**：`rgba(0, 153, 255, 0.15)` 用于交互状态强调
- **红色危险光晕**：`rgba(255, 99, 99, 0.15)` 用于错误/破坏性状态强调

## 7. 应做与禁止

### 应做
- 使用 `#07080a`（非纯黑）作为背景——蓝冷底调对 Raycast 感觉至关重要
- 在正文文字上应用正字间距（+0.2px）——这与大多数暗色 UI 刻意不同
- 对交互元素使用带内嵌高光的多层阴影——macOS 原生深度是标志性特征
- 将 Raycast Red（`#FF6363`）作为点睛颜色，而非遍布全局——留给主视觉时刻和错误状态
- 使用 `rgba(255, 255, 255, 0.06)` 边框用于卡片边界——几乎不可见，但结构上必要
- 将 500 字重应用为正文文字基准——中等字重提升暗色模式易读性
- 主要行动召唤按钮使用胶囊形（86px+ 圆角），次要操作使用矩形（6px–8px）
- 在所有 Inter 文字上启用 OpenType 特性 `calt`、`kern`、`liga`、`ss03`
- 使用透明度过渡（悬停：透明度 0.6）实现按钮交互，而非颜色变化

### 禁止
- 不要使用纯黑（`#000000`）作为背景——蓝色调区分 Raycast 与通用暗色主题
- 不要在正文文字上应用负字间距——Raycast 刻意使用正字距以提升可读性
- 不要将 Raycast Blue 用作所有内容的主要点缀——蓝色用于交互/信息，红色才是品牌色
- 不要创建单层平面阴影——多层内嵌系统是 macOS 原生美学的核心
- 不要在 500 可用时对正文使用常规字重（400）——额外字重防止暗色模式文字显得纤细
- 不要混用暖冷边框——坚持使用冷灰（`hsl(195, 5%, 15%)`）边框色板
- 不要应用厚重投影阴影而无内嵌配合——阴影总是成对出现（外部 + 内嵌）
- 不要使用装饰性元素、渐变或彩色背景——暗色虚空是舞台，内容是表演者

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 移动端 | <600px | 单列，堆叠卡片，汉堡导航，主视觉文字缩小至约 40px |
| 小平板 | 600px–768px | 双列栅格开始，导航部分可见 |
| 平板 | 768px–1024px | 2–3 列功能，导航扩展，截图缩放 |
| 桌面端 | 1024px–1200px | 完整布局，所有导航链接可见，64px 主视觉展示 |
| 大桌面端 | >1200px | 最大宽度容器居中，慷慨边距 |

### 触控目标
- 胶囊形按钮：86px 圆角配 20px 内边距——远超 44px 最小值
- 次要按钮：最小 8px 内边距，但边框提供视觉目标扩展
- 导航链接：16px 文字配周边内边距，提供无障碍触控目标

### 折叠策略
- **导航**：完整水平导航 → 移动端汉堡菜单，配滑出菜单
- **主视觉区**：64px 展示 → 48px → 各断点 36px
- **功能栅格**：三列 → 双列 → 单列堆叠
- **产品截图**：在容器内缩放，保持 macOS 窗口外壳比例
- **键盘快捷键展示**：在键盘快捷键不相关的移动端简化或隐藏

### 图像行为
- 产品截图在固定比例容器内响应式缩放
- 主视觉对角条纹图案按比例缩放
- macOS 窗口外壳圆角在所有尺寸保持不变
- 无延迟加载伪影——图像对产品叙事至关重要

### 迭代指南
在优化使用此设计系统生成的现有屏幕时：
1. 检查背景是否为 `#07080a` 而非纯黑——蓝色调至关重要
2. 验证正文文字字间距为正（+0.2px）——负字距会破坏 Raycast 美学
3. 确保阴影同时具有外部和内嵌层——单层阴影看起来平坦且不正确
4. 确认 Inter 启用了 OpenType 特性 `calt`、`kern`、`liga`、`ss03`
5. 测试悬停状态使用透明度过渡（0.6）而非颜色切换——这是核心交互模式
