# Lovable 设计系统灵感参考

## 1. 视觉主题与氛围

Lovable 的网站以克制传递温度。整个页面建立在奶油色、羊皮纸般的背景（`#f7f4ed`）之上，立即与大多数开发工具网站的冷白色惯例区分开来。这不是为了极简主义而极简——而是一种刻意的选择，让人感到平易近人，几乎是模拟感，像一本精心制作的笔记本。这种暖奶油背景上的近黑文字（`#1c1c1c`）创造出对眼睛友好的对比度，同时保持清晰的可读性。

定制的 Camera Plain Variable 字体是系统的秘密武器。与传递"科技公司"信号的几何无衬线字体不同，Camera Plain 带有人文主义的温度——略微圆润的字母终端、有机的曲线和舒适的阅读节奏。在展示尺寸（48px–60px），字重 600 配激进的负字间距（-0.9px 至 -1.5px）将标题压缩成自信的、编辑性的表达。字体将 `ui-sans-serif, system-ui` 作为备用字体，承认定制字体承载着品牌个性。

Lovable 视觉系统的独特之处在于其以不透明度为驱动的深度模型。系统不使用传统灰阶，而是以不同不透明度（0.03、0.04、0.4、0.82–0.83）调制 `#1c1c1c`，创造统一的色调范围。页面上每一种灰色在技术上都是同一色相——只是透明程度不同。这创造出几乎无法通过任意十六进制值实现的视觉一致性。边框系统同理：浅分隔线使用 `1px solid #eceae4`，较强的交互边界使用 `1px solid rgba(28, 28, 28, 0.4)`。

**核心特征：**
- 暖羊皮纸背景（`#f7f4ed`）——非白色，非米色，一种精心挑选的奶油色
- Camera Plain Variable 字体，人文主义温度，展示尺寸带编辑字间距
- 以不透明度为驱动的色彩系统：所有灰色来自不同透明度的 `#1c1c1c`
- 按钮上的内嵌阴影技术：`rgba(255,255,255,0.2) 0px 0.5px 0px 0px inset, rgba(0,0,0,0.2) 0px 0px 0px 0.5px inset`
- 暖中性边框调色板：`#eceae4` 用于微妙边框，`rgba(28,28,28,0.4)` 用于交互元素
- 完全胶囊圆角（`9999px`）广泛用于操作按钮和图标容器
- 焦点状态使用 `rgba(0,0,0,0.1) 0px 4px 12px` 阴影，实现柔和暖调强调
- shadcn/ui + Radix UI 组件基元，配 Tailwind CSS 工具样式

## 2. 色彩体系与角色分工

### 主色
- **奶油色** (`#f7f4ed`)：页面背景、卡片表面、按钮表面。基础——温暖、纸张般、人性化。
- **炭黑** (`#1c1c1c`)：主文字、标题、深色按钮背景。非纯黑——有机暖意。
- **米白** (`#fcfbf8`)：深色背景上的按钮文字、微妙高亮。几乎与纯白无法区分。

### 中性色阶（以不透明度为基础）
- **炭黑 100%** (`#1c1c1c`)：主文字、标题、深色表面。
- **炭黑 83%** (`rgba(28,28,28,0.83)`)：强次要文字。
- **炭黑 82%** (`rgba(28,28,28,0.82)`)：正文。
- **静默灰** (`#5f5f5d`)：次要文字、描述、说明文字。
- **炭黑 40%** (`rgba(28,28,28,0.4)`)：交互边框、按钮轮廓。
- **炭黑 4%** (`rgba(28,28,28,0.04)`)：微妙悬停背景、微色调。
- **炭黑 3%** (`rgba(28,28,28,0.03)`)：几乎不可见的遮罩、背景深度。

### 表面与边框
- **浅奶油** (`#eceae4`)：卡片边框、分隔符、图片轮廓。暖分隔线。
- **奶油表面** (`#f7f4ed`)：卡片背景、章节填充——与页面背景相同，实现无缝融合。

### 交互色
- **环形蓝** (`#3b82f6` 50% 不透明度)：`--tw-ring-color`，Tailwind 焦点环。
- **焦点阴影** (`rgba(0,0,0,0.1) 0px 4px 12px`)：焦点和活跃状态阴影——柔和、暖、漫射。

### 内嵌阴影
- **按钮内嵌** (`rgba(255,255,255,0.2) 0px 0.5px 0px 0px inset, rgba(0,0,0,0.2) 0px 0px 0px 0.5px inset, rgba(0,0,0,0.05) 0px 1px 2px 0px`)：深色按钮上的标志性多层内嵌阴影。

## 3. 排版规则

### 字体家族
- **主字体**：`Camera Plain Variable`，备用字体：`ui-sans-serif, system-ui`
- **字重范围**：400（正文/阅读）、480（特殊展示）、600（标题/强调）
- **特性**：可变字体，连续字重轴——允许精细调节如 480 这样的中间字重。

### 层级体系

| 角色 | 字体 | 尺寸 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|--------|-------------|----------------|-------|
| 展示主标题 | Camera Plain Variable | 60px (3.75rem) | 600 | 1.00–1.10（紧凑） | -1.5px | 最大冲击，编辑性 |
| 展示替代 | Camera Plain Variable | 60px (3.75rem) | 480 | 1.00（紧凑） | normal | 较轻主图变体 |
| 章节标题 | Camera Plain Variable | 48px (3.00rem) | 600 | 1.00（紧凑） | -1.2px | 功能章节标题 |
| 副标题 | Camera Plain Variable | 36px (2.25rem) | 600 | 1.10（紧凑） | -0.9px | 子章节 |
| 卡片标题 | Camera Plain Variable | 20px (1.25rem) | 400 | 1.25（紧凑） | normal | 卡片标题 |
| 大正文 | Camera Plain Variable | 18px (1.13rem) | 400 | 1.38 | normal | 引言 |
| 正文 | Camera Plain Variable | 16px (1.00rem) | 400 | 1.50 | normal | 标准阅读文字 |
| 按钮 | Camera Plain Variable | 16px (1.00rem) | 400 | 1.50 | normal | 按钮标签 |
| 小按钮 | Camera Plain Variable | 14px (0.88rem) | 400 | 1.50 | normal | 紧凑按钮 |
| 链接 | Camera Plain Variable | 16px (1.00rem) | 400 | 1.50 | normal | 下划线装饰 |
| 小链接 | Camera Plain Variable | 14px (0.88rem) | 400 | 1.50 | normal | 页脚链接 |
| 说明文字 | Camera Plain Variable | 14px (0.88rem) | 400 | 1.50 | normal | 元数据、小文字 |

### 排版原则
- **温暖人文声音**：Camera Plain Variable 赋予 Lovable 其平易近人的个性。略微圆润的字母终端和有机曲线，与大多数开发工具使用的锐利几何无衬线字体形成对比。
- **可变字重作为设计工具**：字体支持连续字重值（如 480），超越标准字重档位实现细腻层级。60px 字重 480 创造出一种比半粗体轻但比常规粗的展示样式。
- **规模上的压缩**：标题使用负字间距（-0.9px 至 -1.5px）实现编辑性冲击。正文保持正常字间距以舒适阅读。
- **两种字重，明确角色**：400（正文/UI/链接/按钮）和 600（标题/强调）。窄字重范围通过尺寸和间距而非字重变化建立层级。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 比例：8px、10px、12px、16px、24px、32px、40px、56px、80px、96px、128px、176px、192px、208px
- 比例在顶端慷慨扩展——章节使用 80px–208px 垂直间距，实现编辑性呼吸感

### 网格与容器
- 内容最大宽度：约 1200px（居中）
- 主图：居中单列，巨大垂直内边距（96px+）
- 功能章节：2–3 列网格
- 全宽页脚，多列链接布局
- 展示章节，居中卡片网格

### 留白哲学
- **编辑慷慨**：Lovable 的章节边界间距宽裕（80px–208px）。暖奶油背景让这些广阔区域感觉温馨而非空洞。
- **内容驱动的节奏**：卡片内紧密间距（12–24px）与宽章节间距形成对比，创造在专注内容和视觉休息之间交替的阅读节奏。
- **章节分隔**：页脚使用 `1px solid #eceae4` 边框和 16px 圆角容器。章节通过宽裕间距而非边框线定义。

### 圆角比例
- 微型（4px）：小按钮、交互元素
- 标准（6px）：按钮、输入框、导航菜单
- 舒适（8px）：紧凑卡片、div
- 卡片（12px）：标准卡片、图片容器、模板
- 容器（16px）：大型容器、页脚章节
- 完全胶囊（9999px）：操作胶囊、图标按钮、切换开关

## 6. 层深与高度

| 级别 | 处理方式 | 用途 |
|-------|-----------|-----|
| 平面（第 0 级） | 无阴影，奶油背景 | 页面表面，大多数内容 |
| 带边框（第 1 级） | `1px solid #eceae4` | 卡片、图片、分隔符 |
| 内嵌（第 2 级） | `rgba(255,255,255,0.2) 0px 0.5px 0px inset, rgba(0,0,0,0.2) 0px 0px 0px 0.5px inset, rgba(0,0,0,0.05) 0px 1px 2px` | 深色按钮、主操作 |
| 焦点（第 3 级） | `rgba(0,0,0,0.1) 0px 4px 12px` | 活跃/焦点状态 |
| 环形（无障碍） | `rgba(59,130,246,0.5)` 2px 环形 | 输入框键盘焦点 |

**阴影哲学**：Lovable 的深度系统刻意浅薄。系统不是用戏剧性投影的浮起卡片，而是依靠暖边框（`#eceae4`）对照奶油表面创造柔和的限制感。唯一值得注意的阴影图案是深色按钮上的内嵌阴影——一种微妙的多层技术，白色高亮线位于顶边，深色环和柔和投影处理底部。这创造出触觉的、被压入表面的质感，而非悬浮于上方的质感。暖焦点阴影（`rgba(0,0,0,0.1) 0px 4px 12px`）刻意漫射且大，创造出柔和光晕而非锐利轮廓。

### 装饰性层深
- 主图：主图背后柔和的暖多彩渐变渲染（粉色、橙色、蓝色）——大气感，几乎不可见
- 页脚：带暖色调渐变背景过渡至底部
- 无强硬章节分隔符——间距和背景暖意处理过渡

## 7. 应做与禁止

### 应做
- 使用暖奶油背景（`#f7f4ed`）作为页面基础——它是品牌的标志性温度
- 在展示尺寸使用 Camera Plain Variable 配负字间距（-0.9px 至 -1.5px）
- 从 `#1c1c1c` 以不同不透明度级别推导所有灰色，实现色调统一
- 在深色按钮上使用内嵌阴影技术实现触觉深度
- 使用 `#eceae4` 边框而非阴影进行卡片限制
- 保持字重系统窄：正文/UI 使用 400，标题使用 600
- 完全胶囊圆角（9999px）仅用于操作胶囊和图标按钮
- 活跃状态应用不透明度 0.8，实现响应式触觉反馈

### 禁止
- 不要使用纯白（`#ffffff`）作为页面背景——奶油色是刻意的
- 不要为卡片使用重 box-shadow——边框是限制机制
- 不要引入饱和强调色——调色板刻意是暖中性的
- 不要使用字重 700（粗体）——600 是系统中的最大字重
- 不要在矩形按钮上应用 9999px 圆角——胶囊用于图标/操作切换
- 不要使用锐利焦点轮廓——系统使用基于柔和阴影的焦点指示器
- 不要混用边框样式——`#eceae4` 用于被动，`rgba(28,28,28,0.4)` 用于交互
- 不要增加标题字间距——Camera Plain 设计为在规模上运行紧凑

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 小型移动端 | <600px | 紧凑单列，减小内边距 |
| 移动端 | 600–640px | 标准移动布局 |
| 小型平板 | 640–700px | 开始 2 列网格 |
| 平板 | 700–768px | 卡片网格扩展 |
| 小型桌面端 | 768–1024px | 多列布局 |
| 桌面端 | 1024–1280px | 完整功能布局 |
| 大型桌面端 | 1280–1536px | 最大内容宽度，宽裕边距 |

### 触摸目标
- 按钮：8px 16px 内边距（舒适触摸）
- 导航：项目间适当间距
- 胶囊按钮：9999px 圆角创造大型触摸友好目标
- 菜单切换：带适当尺寸的 6px 圆角按钮

### 折叠策略
- 主图：60px → 48px → 36px 标题缩放，带比例字间距
- 导航：水平链接 → 768px 处汉堡菜单
- 功能卡片：3 列 → 2 列 → 单列堆叠
- 模板画廊：网格 → 垂直堆叠卡片
- 指标栏：水平 → 垂直堆叠
- 页脚：多列 → 堆叠单列
- 章节间距：128px+ → 移动端 64px

### 图片行为
- 模板截图在所有尺寸保持 `1px solid #eceae4` 边框
- 12px 圆角在断点间保持
- 画廊图片响应式，保持一致宽高比
- 主图渐变在移动端软化/简化

### 迭代指南
1. 始终使用奶油色（`#f7f4ed`）作为基础——绝不使用纯白
2. 从 `#1c1c1c` 以不透明度级别推导灰色，而非使用不同十六进制值
3. 使用 `#eceae4` 边框进行限制，而非阴影
4. 字间距随尺寸变化：60px 时 -1.5px，48px 时 -1.2px，36px 时 -0.9px，16px 时恢复正常
5. 两种字重：400（除标题外的一切）和 600（标题）
6. 深色按钮上的内嵌阴影是标志性细节——不要省略
7. Camera Plain Variable 字重 480 仅用于特殊展示时刻
