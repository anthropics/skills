# Design System Inspiration of Stripe

## 1. 视觉主题与氛围

Stripe 的网站是金融科技设计的黄金标准——一个既能让人感觉技术性又奢华、精准又温暖的系统。页面在纯白画布（`#ffffff`）上展开，深海军蓝标题（`#061b31`）和标志性的紫色（`#533afd`）同时发挥品牌锚点和交互强调的双重作用。这不是企业软件那种冷酷、临床感的紫色；而是一种丰富、饱和的蓝紫色，读起来充满自信和高端感。整体印象是一家由世界级字体铸造厂重新设计的金融机构。

定制的 `sohne-var` 可变字体是 Stripe 视觉身份的定义元素。每个文字元素都启用 OpenType `"ss01"` 样式集，该样式集修改字符形状，创造出明显的几何、现代感。在展示尺寸（48px-56px）下，sohne-var 以字重 300 运行——对于标题来说是极轻的字重，创造出一种空灵的、几乎是低语般的权威。这与"粗体英雄标题"惯例截然相反；Stripe 的标题感觉不需要大喊大叫。负字母间距（56px 下 -1.4px，48px 下 -0.96px）将文字压缩成密集的、工程化的块。在较小尺寸下，系统同样使用字重 300 配合按比例缩小的字母间距，以及通过 `"tnum"` 实现的表格数字用于金融数据展示。

真正使 Stripe 与众不同的是其阴影系统。与大多数网站的平面或单层方式不同，Stripe 使用多层、蓝色调阴影：标志性的 `rgba(50,50,93,0.25)` 结合 `rgba(0,0,0,0.1)` 创造出带有凉爽、几乎是大气般深度的阴影——如同元素漂浮在暮色天空中。主要阴影颜色（50,50,93）的蓝灰色底调直接与海军蓝紫色品牌调色板相联系，使即使是层次感也感觉符合品牌。

**核心特征：**
- sohne-var 在所有文字上启用 OpenType `"ss01"`——定义品牌字母形状的定制样式集
- 字重 300 作为标志性标题字重——轻盈、自信、反惯例
- 展示尺寸下的负字母间距（56px 下 -1.4px，向下渐进放宽）
- 使用 `rgba(50,50,93,0.25)` 的蓝色调多层阴影——感觉带有品牌色的层次感
- 深海军蓝（`#061b31`）标题而非黑色——温暖、高端、金融级
- 保守的圆角半径（4px-8px）——没有胶囊形，没有尖锐感
- Ruby（`#ea2261`）和洋红（`#f96bee`）强调色用于渐变和装饰元素
- `SourceCodePro` 作为代码和技术标签的等宽配套字体

## 2. 色彩体系与角色

### 主色
- **Stripe 紫** (`#533afd`)：主要品牌颜色、行动号召背景、链接文字、交互高亮。锚定整个系统的饱和蓝紫色。
- **深海军蓝** (`#061b31`)：`--hds-color-heading-solid`。主要标题颜色。不是黑色，不是灰色——一种非常深的蓝色，为文字增添温暖和深度。
- **纯白** (`#ffffff`)：页面背景、卡片表面、深色背景上的按钮文字。

### 品牌色与深色
- **品牌深色** (`#1c1e54`)：`--hds-color-util-brand-900`。深色区块、页脚背景和沉浸式品牌时刻的深靛蓝色。
- **深海军蓝** (`#0d253d`)：`--hds-color-core-neutral-975`。最深的中性色——几乎是黑色，但带有蓝色底调，实现最大深度而无粗糙感。

### 强调色
- **Ruby** (`#ea2261`)：`--hds-color-accentColorMode-ruby-icon-solid`。图标、警告和强调元素的温暖红粉色。
- **洋红** (`#f96bee`)：`--hds-color-accentColorMode-magenta-icon-gradientMiddle`。渐变和装饰高亮的鲜艳粉紫色。
- **浅洋红** (`#ffd7ef`)：`--hds-color-util-accent-magenta-100`。洋红主题卡片和徽章的色调表面。

### 交互色
- **主紫色** (`#533afd`)：主要链接颜色、激活状态、选中元素。
- **紫色悬停** (`#4434d4`)：主要元素悬停状态的较深紫色。
- **深紫** (`#2e2b8c`)：`--hds-color-button-ui-iconHover`。图标悬停状态的深紫色。
- **浅紫** (`#b9b9f9`)：`--hds-color-action-bg-subduedHover`。低调悬停背景的柔和薰衣草色。
- **中紫** (`#665efd`)：`--hds-color-input-selector-text-range`。范围选择器和输入框高亮颜色。

### 中性色刻度
- **标题** (`#061b31`)：主要标题、导航文字、强标签。
- **标签** (`#273951`)：`--hds-color-input-text-label`。表单标签、次要标题。
- **正文** (`#64748d`)：次要文字、描述、说明文字。
- **成功绿** (`#15be53`)：状态徽章、成功指示器（背景/边框使用 0.2-0.4 alpha）。
- **成功文字** (`#108c3d`)：成功徽章文字颜色。
- **柠檬黄** (`#9b6829`)：`--hds-color-core-lemon-500`。警告和高亮强调色。

### 表面与边框
- **默认边框** (`#e5edf5`)：卡片、分隔线和容器的标准边框颜色。
- **紫色边框** (`#b9b9f9`)：按钮和输入框的激活/选中状态边框。
- **柔和紫色边框** (`#d6d9fc`)：次要元素的微妙紫色调边框。
- **洋红边框** (`#ffd7ef`)：洋红主题元素的粉色调边框。
- **虚线边框** (`#362baa`)：放置区和占位符元素的虚线边框。

### 阴影颜色
- **蓝色阴影** (`rgba(50,50,93,0.25)`)：标志性——蓝色调主要阴影颜色。
- **深蓝阴影** (`rgba(3,3,39,0.25)`)：悬浮元素的更深蓝色阴影。
- **黑色阴影** (`rgba(0,0,0,0.1)`)：深度强化的次要阴影层。
- **环境光阴影** (`rgba(23,23,23,0.08)`)：微妙层次的柔和环境光阴影。
- **轻柔阴影** (`rgba(23,23,23,0.06)`)：轻微抬起的最小环境光阴影。

## 3. 排版规则

### 字体家族
- **主字体**：`sohne-var`，备选字体：`SF Pro Display`
- **等宽字体**：`SourceCodePro`，备选字体：`SFMono-Regular`
- **OpenType 特性**：所有 sohne-var 文字全局启用 `"ss01"`；金融数据和说明文字的表格数字使用 `"tnum"`。

### 排版原则
- **轻字重作为标志**：展示尺寸下字重 300 是 Stripe 最独特的排版选择。其他品牌用 600-700 吸引注意，Stripe 以轻盈作为奢华——文字如此自信，不需要字重来获得权威性。
- **ss01 无处不在**：`"ss01"` 样式集是不可妥协的。它修改特定字形（可能是交替的 `a`、`g`、`l` 字形），在所有 sohne-var 文字中创造出更几何、更现代的感觉。
- **两种 OpenType 模式**：`"ss01"` 用于展示/正文，`"tnum"` 用于金融数据中的表格数字。它们绝不重叠——段落中的数字使用 ss01，数据表格中的数字使用 tnum。
- **渐进式字母间距**：字母间距按尺寸比例收紧：56px 下 -1.4px，48px 下 -0.96px，32px 下 -0.64px，26px 下 -0.26px，16px 及以下 normal。
- **双字重简洁性**：主要使用 300（正文和标题）和 400（UI/按钮）。主字体没有粗体（700）——SourceCodePro 使用 500/700 实现代码对比。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 刻度：1px、2px、4px、6px、8px、10px、11px、12px、14px、16px、18px、20px
- 注意：刻度在小端密集（4-12px 每 2px 一级），反映 Stripe 面向金融数据的精准导向 UI

### 网格与容器
- 最大内容宽度：约 1080px
- 英雄区：居中单列，充裕内边距，轻量标题
- 功能区块：功能卡片的 2-3 列网格
- 全宽深色区块，`#1c1e54` 背景用于品牌沉浸
- 代码/仪表板预览作为带蓝色调阴影的包含卡片

### 留白理念
- **精准间距**：与极简系统的广阔空旷不同，Stripe 使用经过测量的、有目的的留白。每个间隙都是刻意的排版选择。
- **密集数据，宽松框架**：金融数据展示（表格、图表）紧凑排列，但周围的 UI 框架间距充裕。这创造出一种受控的密度感——如同精美框架中组织良好的电子表格。
- **区块节奏**：白色区块与深色品牌区块（`#1c1e54`）交替，创造出戏剧性的明暗节奏，防止单调而不引入随意的颜色。

### 圆角刻度
- 微型（1px）：精细元素，微妙圆角
- 标准（4px）：按钮、输入框、徽章、卡片——主力圆角
- 舒适（5px）：标准卡片容器
- 放松（6px）：导航、大型交互元素
- 大型（8px）：特色卡片、英雄元素
- 复合：`0px 0px 6px 6px` 用于底部圆角容器（标签面板、下拉菜单页脚）

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 平面（0 级） | 无阴影 | 页面背景、内联文字 |
| 环境光（1 级） | `rgba(23,23,23,0.06) 0px 3px 6px` | 微妙卡片抬起、悬停提示 |
| 标准（2 级） | `rgba(23,23,23,0.08) 0px 15px 35px` | 标准卡片、内容面板 |
| 悬浮（3 级） | `rgba(50,50,93,0.25) 0px 30px 45px -30px, rgba(0,0,0,0.1) 0px 18px 36px -18px` | 特色卡片、下拉菜单、弹出框 |
| 深度（4 级） | `rgba(3,3,39,0.25) 0px 14px 21px -14px, rgba(0,0,0,0.1) 0px 8px 17px -8px` | 模态框、浮动面板 |
| 环（无障碍性） | `2px solid #533afd` 轮廓 | 键盘焦点环 |

**阴影理念**：Stripe 的阴影系统建立在色度深度原则之上。大多数设计系统使用中性灰或黑色阴影，而 Stripe 的主要阴影颜色（`rgba(50,50,93,0.25)`）是一种深蓝灰色，呼应品牌的海军蓝调色板。这创造出不仅增添深度、还增添品牌氛围的阴影。多层方式将这种蓝色调阴影与不同偏移量的纯黑次要层（`rgba(0,0,0,0.1)`）配对，创造出视差般的深度，品牌阴影离元素更远，中性阴影更近。负展开值（-30px、-18px）确保阴影不在水平方向超出元素足迹，保持层次感垂直且受控。

### 装饰性深度
- 深色品牌区块（`#1c1e54`）通过背景颜色对比创造沉浸式深度
- Ruby 到洋红过渡的渐变叠层用于英雄装饰
- 阴影颜色 `rgba(0,55,112,0.08)`（`--hds-color-shadow-sm-top`）用于粘性元素的顶边阴影

## 7. 应做与禁止

### 应做
- 所有文字元素使用 sohne-var 搭配 `"ss01"`——样式集就是品牌
- 所有标题和正文使用字重 300——轻盈是标志性选择
- 所有悬浮元素应用蓝色调阴影（`rgba(50,50,93,0.25)`）
- 标题使用 `#061b31`（深海军蓝）而非 `#000000`——温暖感很重要
- 圆角保持在 4px-8px 之间——保守的圆角是有意为之的
- 任何表格/金融数字展示使用 `"tnum"`
- 阴影分层：蓝色调远阴影 + 中性近阴影实现深度视差
- 使用 `#533afd` 紫色作为主要交互/行动号召颜色

### 禁止
- sohne-var 标题不要使用字重 600-700——字重 300 是品牌声音
- 卡片或按钮不要使用大圆角（12px+、胶囊形）——Stripe 是保守的
- 不要使用中性灰阴影——始终用蓝色着色（`rgba(50,50,93,...)`）
- 不要跳过任何 sohne-var 文字的 `"ss01"`——交替字形定义个性
- 标题不要使用纯黑（`#000000`）——始终用 `#061b31` 深海军蓝
- 不要为交互元素使用暖色调强调色（橙色、黄色）——紫色是主色
- 展示尺寸不要应用正字母间距——Stripe 字母间距始终紧凑
- 洋红/Ruby 强调色不要用于按钮或链接——它们仅用于装饰/渐变

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|-------|-------------|
| 移动端 | <640px | 单列，减小标题尺寸，堆叠卡片 |
| 平板 | 640-1024px | 2 列网格，适中内边距 |
| 桌面 | 1024-1280px | 完整布局，3 列功能网格 |
| 大桌面 | >1280px | 居中内容，充裕边距 |

### 触控目标
- 按钮使用舒适内边距（8px-16px 垂直）
- 导航链接 14px，间距充足
- 徽章最小 6px 水平内边距用于点击目标
- 移动端导航切换按钮，6px 圆角

### 折叠策略
- 英雄：56px 展示 → 移动端 32px，保持字重 300
- 导航：横向链接 + 行动号召 → 汉堡菜单切换
- 功能卡片：3 列 → 2 列 → 单列堆叠
- 深色品牌区块：保持全宽处理，减少内部填充
- 金融数据表格：移动端横向滚动
- 区块间距：64px+ → 移动端 40px
- 排版刻度跨断点压缩：英雄尺寸 56px → 48px → 32px

### 图片行为
- 仪表板/产品截图在所有尺寸保持蓝色调阴影
- 英雄渐变装饰在移动端简化
- 代码块保持 `SourceCodePro` 处理，可能横向滚动
- 卡片图片保持一致的 4px-6px 圆角

### 迭代指南
1. 始终在 sohne-var 文字上启用 `font-feature-settings: "ss01"`——这是品牌的排版 DNA
2. 字重 300 是默认值；仅按钮/链接/导航使用 400
3. 阴影公式：`rgba(50,50,93,0.25) 0px Y1 B1 -S1, rgba(0,0,0,0.1) 0px Y2 B2 -S2`，Y1/B1 较大（远阴影），Y2/B2 较小（近阴影）
4. 标题颜色是 `#061b31`（深海军蓝），正文是 `#64748d`（蓝灰），标签是 `#273951`（深蓝灰）
5. 圆角保持在 4px-8px 范围内——绝不使用胶囊形或大圆角
6. 表格、图表或金融展示中的任何数字使用 `"tnum"`
7. 深色区块使用 `#1c1e54`——不是黑色，不是灰色，而是深品牌靛蓝
8. SourceCodePro 代码使用 12px/500 搭配 2.00 行高（非常充裕以提升可读性）
