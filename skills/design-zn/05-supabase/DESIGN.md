# Supabase 设计系统灵感参考

## 1. 视觉主题与氛围

Supabase 的网站是一个以深色模式为原生形态的开发者平台，传递出高级代码编辑器的美学气质——深黑背景（`#0f0f0f`、`#171717`）配以翠绿色强调（`#3ecf8e`、`#00c573`），呼应品牌的开源、PostgreSQL 绿色身份。这套设计系统仿佛诞生于终端窗口，在演化为成熟的营销表面之后，仍未失去其开发者灵魂。

排版建立在"Circular"之上——一种几何无衬线字体，圆润的字母终端软化了技术的锋芒。在 72px 配合 1.00 行高时，主图文字被压缩至绝对最小的垂直空间，创造出密集、有力的表达，毫不浪费。等宽配套字体（Source Code Pro）在大写技术标签中以 1.2px 字间距克制出现，创造出"开发者终端"标记，将营销站点与产品体验相连。

Supabase 的独特之处在于其精密的 HSL 色彩 token 系统。几乎每一种颜色都不使用平面十六进制值，而是使用带 alpha 通道的 HSL（`--colors-crimson4`、`--colors-purple5`、`--colors-slateA12`），实现细腻的分层系统，让颜色通过透明度相互作用。这从极少的颜色成分中创造出深度——`rgba(46, 46, 46)` 的边框、`rgba(41, 41, 41, 0.84)` 的表面、部分不透明的强调色，全部与深色背景融合，形成丰富的、有维度的调色板。

绿色强调（`#3ecf8e`）选择性出现——在 Supabase 标志、链接颜色（`#00c573`）和边框高亮（`rgba(62, 207, 142, 0.3)`）中——始终作为"这就是 Supabase"的信号，而非装饰元素。主 CTA 使用胶囊形按钮（9999px 圆角），次级元素使用标准 6px 圆角，形成清晰的重要性视觉层级。

**核心特征：**
- 深色模式原生：近黑背景（`#0f0f0f`、`#171717`）——绝非纯黑
- 翠绿品牌强调（`#3ecf8e`、`#00c573`），克制使用作为身份标记
- Circular 字体——几何无衬线体，圆润字母终端
- Source Code Pro 用于大写技术标签（1.2px 字间距）
- 基于 HSL 的色彩 token 系统，带 alpha 通道实现半透明分层
- 主 CTA 使用胶囊按钮（9999px），次级使用 6px 圆角
- 中性灰阶从 `#171717` 经 `#898989` 至 `#fafafa`
- 边框系统使用深灰色（`#2e2e2e`、`#363636`、`#393939`）
- 极少阴影——通过边框对比和透明度实现深度
- Radix 颜色基元（crimson、purple、violet、indigo、yellow、tomato、orange、slate）

## 2. 色彩体系与角色分工

### 品牌色
- **Supabase 绿** (`#3ecf8e`)：主品牌色、标志、强调边框
- **绿色链接** (`#00c573`)：交互绿，用于链接和操作
- **绿色边框** (`rgba(62, 207, 142, 0.3)`)：微妙绿色边框强调

### 中性色阶（深色模式）
- **近黑** (`#0f0f0f`)：主按钮背景，最深表面
- **深色** (`#171717`)：页面背景，主画布
- **深色边框** (`#242424`)：水平分隔线，章节分隔
- **卡片边框** (`#2e2e2e`)：卡片边框、标签边框
- **中间边框** (`#363636`)：按钮边框、分隔符
- **浅色边框** (`#393939`)：次级边框
- **炭灰** (`#434343`)：第三级边框、深色强调
- **深灰** (`#4d4d4d`)：重次要文字
- **中灰** (`#898989`)：静默文字、链接颜色
- **浅灰** (`#b4b4b4`)：次要链接文字
- **近白** (`#efefef`)：浅色边框、微妙表面
- **米白** (`#fafafa`)：主文字、按钮文字

### Radix 颜色 Token（基于 HSL）
- **Slate 阶**：`--colors-slate5` 至 `--colors-slateA12`——中性渐进
- **紫色**：`--colors-purple4`、`--colors-purple5`、`--colors-purpleA7`——强调色谱
- **紫罗兰**：`--colors-violet10`（`hsl(251, 63.2%, 63.2%)`）——鲜明强调
- **深红**：`--colors-crimson4`、`--colors-crimsonA9`——暖强调 / 警告
- **靛蓝**：`--colors-indigoA2`——微妙蓝色渲染
- **黄色**：`--colors-yellowA7`——注意/警告
- **番茄红**：`--colors-tomatoA4`——错误强调
- **橙色**：`--colors-orange6`——暖强调

### 表面与遮罩
- **深色玻璃** (`rgba(41, 41, 41, 0.84)`)：半透明深色遮罩
- **Slate Alpha** (`hsla(210, 87.8%, 16.1%, 0.031)`)：极微妙蓝色渲染
- **固定阶 Alpha** (`hsla(200, 90.3%, 93.4%, 0.109)`)：浅色霜冻遮罩

### 阴影
- Supabase 在其深色主题中**几乎不使用阴影**。深度通过边框对比和表面颜色差异而非 box-shadow 来创造。焦点状态使用 `rgba(0, 0, 0, 0.1) 0px 4px 12px`——最小化，功能性。

## 3. 排版规则

### 字体家族
- **主字体**：`Circular`，备用字体：`custom-font, Helvetica Neue, Helvetica, Arial`
- **等宽字体**：`Source Code Pro`，备用字体：`Office Code Pro, Menlo`

### 层级体系

| 角色 | 字体 | 尺寸 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|--------|-------------|----------------|-------|
| 展示主标题 | Circular | 72px (4.50rem) | 400 | 1.00（紧凑） | normal | 最大密度，零浪费 |
| 章节标题 | Circular | 36px (2.25rem) | 400 | 1.25（紧凑） | normal | 功能章节标题 |
| 卡片标题 | Circular | 24px (1.50rem) | 400 | 1.33 | -0.16px | 轻微负字间距 |
| 副标题 | Circular | 18px (1.13rem) | 400 | 1.56 | normal | 次要标题 |
| 正文 | Circular | 16px (1.00rem) | 400 | 1.50 | normal | 标准正文 |
| 导航链接 | Circular | 14px (0.88rem) | 500 | 1.00–1.43 | normal | 导航项目 |
| 按钮 | Circular | 14px (0.88rem) | 500 | 1.14（紧凑） | normal | 按钮标签 |
| 说明文字 | Circular | 14px (0.88rem) | 400–500 | 1.43 | normal | 元数据、标签 |
| 小字 | Circular | 12px (0.75rem) | 400 | 1.33 | normal | 细则、页脚链接 |
| 代码标签 | Source Code Pro | 12px (0.75rem) | 400 | 1.33 | 1.2px | `text-transform: uppercase` |

### 排版原则
- **字重克制**：几乎所有文字使用字重 400（常规/书体）。字重 500 仅出现在导航链接和按钮标签上。系统中未检测到粗体（700）——层级通过尺寸而非字重创建。
- **1.00 主图行高**：主图文字被压缩至绝对零行距。这是定义性的排版姿态——像终端命令一样密集、高效、不浪费垂直空间的文字。
- **卡片负字间距**：卡片标题使用 -0.16px 字间距，微妙的收紧，使之有别于正文而不过于明显。
- **等宽字体作为仪式**：Source Code Pro 大写配合 1.2px 字间距是"开发者终端"的声音——克制地用于连接产品体验的技术标签。
- **几何个性**：Circular 的圆润终端在本可冷峻、技术的界面中创造温度。字体是人性化的元素。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 比例：1px、4px、6px、8px、12px、16px、20px、24px、32px、40px、48px、90px、96px、128px
- 值得注意的大型跳跃：48px → 90px → 96px → 128px，用于主要章节间距

### 网格与容器
- 居中内容，宽裕最大宽度
- 全宽深色章节，限制宽度的内部内容
- 功能网格：基于图标的网格，一致的卡片尺寸
- "受信任" 章节的标志网格
- 页脚：深色背景上的多列

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 移动端 | <600px | 单列，堆叠布局 |
| 桌面端 | >600px | 多列网格，扩展布局 |

*注：Supabase 使用明显极简的断点系统——主要是单个 600px 断点，表明采用移动优先的渐进增强方法。*

### 留白哲学
- **戏剧性章节间距**：主要章节间 90px–128px 创造电影般的节奏——每个章节在深色虚空中都是独立的场景。
- **密集内容块**：章节内，间距紧凑（16px–24px），创造集中的信息群组。
- **边框定义空间**：Supabase 使用深色背景上的细边框而非留白加阴影进行分隔——通过线条而非间距实现分隔。

### 圆角比例
- 标准（6px）：幽灵按钮、小元素
- 舒适（8px）：卡片、容器
- 中等（11px–12px）：中型面板
- 大（16px）：功能卡片、主要容器
- 胶囊（9999px）：主按钮、标签指示器

## 6. 层深与高度

| 级别 | 处理方式 | 用途 |
|-------|-----------|-----|
| 平面（第 0 级） | 无阴影，边框 `#2e2e2e` | 默认状态，大多数表面 |
| 微妙边框（第 1 级） | 边框 `#363636` 或 `#393939` | 交互元素、悬停 |
| 焦点（第 2 级） | `rgba(0, 0, 0, 0.1) 0px 4px 12px` | 仅焦点状态 |
| 绿色强调（第 3 级） | 边框 `rgba(62, 207, 142, 0.3)` | 品牌高亮元素 |

**阴影哲学**：Supabase 刻意避免阴影。在深色模式原生设计中，阴影几乎不可见且毫无用处。深度通过精密的边框层级传达——从 `#242424`（几乎不可见）经 `#2e2e2e`（标准）至 `#393939`（显著）。绿色强调边框（`rgba(62, 207, 142, 0.3)`）以 30% 不透明度是"浮起"状态——品牌颜色本身成为深度信号。

## 7. 应做与禁止

### 应做
- 使用近黑背景（`#0f0f0f`、`#171717`）——深度来自灰色边框层级
- 克制地应用 Supabase 绿（`#3ecf8e`、`#00c573`）——它是身份标记，不是装饰
- 几乎所有内容使用 Circular 字重 400——500 仅用于按钮和导航
- 将主图文字设置为 1.00 行高——零行距是排版标志
- 通过边框颜色差异创造深度（`#242424` → `#2e2e2e` → `#363636`）
- 仅对主 CTA 和标签页使用胶囊形（9999px）
- 使用带 alpha 的 HSL 颜色实现半透明分层效果
- 将 Source Code Pro 大写标签用于开发者上下文标记

### 禁止
- 不要添加 box-shadow——在深色背景上不可见，且破坏边框定义的深度系统
- 不要使用粗体（700）字重——系统仅使用 400 和 500
- 不要将绿色应用于背景或大型表面——它用于边框、链接和小型强调
- 不要将暖色（深红、橙色）作为主要设计元素——它们作为状态的语义 token 存在
- 不要将主图行高增加到 1.00 以上——密度是刻意的
- 不要在按钮上使用大圆角（16px+）——胶囊（9999px）或标准（6px），无中间值
- 不要将主表面背景提亮至 `#171717` 以上——黑暗是结构性的
- 不要忘记半透明边框——`rgba` 边框颜色是分层机制

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 移动端 | <600px | 单列，堆叠功能，精简导航 |
| 桌面端 | >600px | 多列网格，完整导航，扩展章节 |

### 折叠策略
- 主图：72px → 按比例缩小
- 功能网格：多列 → 单列堆叠
- 标志行：水平 → 换行网格
- 导航：完整 → 汉堡菜单
- 章节间距：90–128px → 48–64px
- 按钮：行内 → 全宽堆叠
