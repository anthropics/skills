# HashiCorp 设计系统灵感

## 1. 视觉主题与氛围

HashiCorp 的网站是将企业基础设施具体化的展现——一个必须在保持亲和力的同时传达云基础设施管理复杂性的设计系统。视觉语言分为两种模式：干净的白色浅色模式用于信息性区域，以及戏剧性的深色模式（`#15181e`、`#0d0e12`）用于英雄区域和产品展示，创造一种映射"在光明中构建，在黑暗中部署"开发者工作流的日/夜双重性。

排版以定制品牌字体（HashiCorp Sans，加载为 `__hashicorpSans_96f0ca`）为锚点，字面意义上承载着实质的重量。标题使用 600–700 字重配紧凑行高（1.17–1.19），创造密集、权威的文字块，传递企业自信。82px 字重 600 配 OpenType `"kern"` 启用的英雄标题不是装饰性的——它是基础设施级别的排版。

HashiCorp 的独特之处在于其多产品色彩系统。产品组合中的每个产品都有自己的品牌颜色——Terraform 紫色（`#7b42bc`）、Vault 黄色（`#ffcf25`）、Waypoint 青色（`#14c6cb`）、Vagrant 蓝色（`#1868f2`）——这些颜色通过 CSS 自定义属性系统（`--mds-color-*`）作为强调标记出现在整个页面中。这在设计系统内创造了一个设计系统：父品牌是黑白配蓝色强调，而每个子产品注入了自己的色彩身份。

组件系统使用 `mds`（Markdown 设计系统）前缀，表明一种系统化的、标记驱动的方法，其中颜色、间距和状态都通过 CSS 变量管理。阴影非常微妙——使用 `rgba(97, 104, 117, 0.05)` 的双层微阴影，几乎不可见，但提供了足够的深度将交互表面与背景分离。

**核心特征：**
- 双模式：干净白色区域 + 戏剧性深色（`#15181e`）英雄/产品区域
- HashiCorp Sans 定制字体，字重 600–700 配 `"kern"` 特性
- 通过 `--mds-color-*` CSS 自定义属性的多产品色彩系统
- 产品品牌色：Terraform 紫色、Vault 黄色、Waypoint 青色、Vagrant 蓝色
- 大写字间距说明文字（13px，字重 600，1.3px 字间距）
- 微阴影：双层 0.05 不透明度——通过低语而非呼喊体现深度
- 带语义变量名的标记驱动 `mds` 组件系统
- 紧凑圆角：2px–8px，无胶囊形或圆形
- 次级文字的 system-ui 备用字体栈

## 2. 色彩系统与角色

### 品牌主色
- **黑色** (`#000000`)：主品牌色，浅色表面上的文字，`--mds-color-hcp-brand`
- **深炭灰** (`#15181e`)：深色模式背景、英雄区域
- **近黑** (`#0d0e12`)：最深的深色模式表面，深色背景上的表单输入

### 中性色系
- **浅灰** (`#f1f2f3`)：浅色背景、细微表面
- **中灰** (`#d5d7db`)：边框、深色背景上的按钮文字
- **冷灰** (`#b2b6bd`)：边框强调（0.1–0.4 不透明度）
- **深灰** (`#656a76`)：辅助文字、次级标签，`--mds-form-helper-text-color`
- **炭灰** (`#3b3d45`)：浅色背景上的次级文字、按钮边框
- **近白** (`#efeff1`)：深色表面上的主文字

### 产品品牌色
- **Terraform 紫色** (`#7b42bc`)：`--mds-color-terraform-button-background`
- **Vault 黄色** (`#ffcf25`)：`--mds-color-vault-button-background`
- **Waypoint 青色** (`#14c6cb`)：`--mds-color-waypoint-button-background-focus`
- **Waypoint 青色悬停** (`#12b6bb`)：`--mds-color-waypoint-button-background-hover`
- **Vagrant 蓝色** (`#1868f2`)：`--mds-color-vagrant-brand`
- **紫色强调** (`#911ced`)：`--mds-color-palette-purple-300`

### 语义色
- **操作蓝** (`#1060ff`)：深色背景上的主要操作链接
- **链接蓝** (`#2264d6`)：浅色背景上的主要链接
- **亮蓝** (`#2b89ff`)：激活链接、悬停强调

### 阴影
- **微阴影** (`rgba(97, 104, 117, 0.05) 0px 1px 1px, rgba(97, 104, 117, 0.05) 0px 2px 2px`)：默认卡片/按钮层次
- **焦点轮廓**：`3px solid var(--mds-color-focus-action-external)` ——系统化焦点环

## 3. 字体规则

### 字体系列
- **主品牌**：`__hashicorpSans_96f0ca`（HashiCorp Sans），备用字体：`__hashicorpSans_Fallback_96f0ca`
- **系统 UI**：`system-ui, -apple-system, BlinkMacSystemFont, Segoe UI, Helvetica, Arial`

### 层级

| 角色 | 字体 | 尺寸 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|------|------|--------|------|
| 展示英雄 | HashiCorp Sans | 82px (5.13rem) | 600 | 1.17（紧凑） | normal | 启用 `"kern"` |
| 区域标题 | HashiCorp Sans | 52px (3.25rem) | 600 | 1.19（紧凑） | normal | 启用 `"kern"` |
| 功能标题 | HashiCorp Sans | 42px (2.63rem) | 700 | 1.19（紧凑） | -0.42px | 负字间距 |
| 次级标题 | HashiCorp Sans | 34px (2.13rem) | 600–700 | 1.18（紧凑） | normal | 功能块 |
| 卡片标题 | HashiCorp Sans | 26px (1.63rem) | 700 | 1.19（紧凑） | normal | 卡片和面板标题 |
| 小标题 | HashiCorp Sans | 19px (1.19rem) | 700 | 1.21（紧凑） | normal | 紧凑标题 |
| 强调正文 | HashiCorp Sans | 17px (1.06rem) | 600–700 | 1.18–1.35 | normal | 粗体正文 |
| 大正文 | system-ui | 20px (1.25rem) | 400–600 | 1.50 | normal | 英雄描述 |
| 正文 | system-ui | 16px (1.00rem) | 400–500 | 1.63–1.69（宽松） | normal | 标准正文 |
| 导航链接 | system-ui | 15px (0.94rem) | 500 | 1.60（宽松） | normal | 导航项 |
| 小正文 | system-ui | 14px (0.88rem) | 400–500 | 1.29–1.71 | normal | 次级内容 |
| 说明 | system-ui | 13px (0.81rem) | 400–500 | 1.23–1.69 | normal | 元数据、页脚链接 |
| 大写标签 | HashiCorp Sans | 13px (0.81rem) | 600 | 1.69（宽松） | 1.3px | `text-transform: uppercase` |

### 原则
- **品牌/系统分工**：HashiCorp Sans 用于标题和品牌关键文字；system-ui 用于正文、导航和功能性文字。品牌字体承载重量，system-ui 承载文字。
- **字距调整始终开启**：所有 HashiCorp Sans 文字启用 OpenType `"kern"` ——字距调整不可妥协。
- **紧凑标题**：每个标题使用 1.17–1.21 行高，创造密集、堆叠的文字块，感觉像基础设施——坚固、承重。
- **宽松正文**：正文使用 1.50–1.69 行高（显著宽松），在密集标题下方创造舒适的阅读节奏。
- **大写标签作为导航**：13px 大写配 1.3px 字间距作为系统性分类/区域标记——始终使用 HashiCorp Sans 字重 600。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 尺度：2px、3px、4px、6px、7px、8px、9px、11px、12px、16px、20px、24px、32px、40px、48px

### 网格与容器
- 最大内容宽度：约 1150px（xl 断点）
- 带内容约束的全宽深色英雄区
- 卡片网格：2–3 列布局
- 桌面比例上宽松的水平内边距

### 断点
| 名称 | 宽度 | 主要变化 |
|------|------|----------|
| 小移动端 | <375px | 紧凑单列 |
| 移动端 | 375–480px | 标准移动端 |
| 小平板 | 480–600px | 细微调整 |
| 平板 | 600–768px | 开始双列网格 |
| 小桌面 | 768–992px | 完整导航可见 |
| 桌面 | 992–1120px | 标准布局 |
| 大桌面 | 1120–1440px | 最大宽度内容 |
| 超宽 | >1440px | 居中，宽松边距 |

### 空白哲学
- **企业呼吸空间**：区域之间宽松的垂直间距（48px–80px+）传递稳定性和严肃性。
- **密集标题，宽松正文**：紧凑行高的标题位于宽松正文上方，在每个区域顶部创造视觉"重量"。
- **黑暗即画布**：深色英雄区域使用额外的垂直内边距，让 3D 插图和渐变呼吸。

### 圆角尺度
- 极小（2px）：链接、小型行内元素
- 细腻（3px）：复选框、小型输入框
- 标准（4px）：次级按钮
- 舒适（5px）：主要按钮、徽章、输入框
- 卡片（8px）：卡片、容器、图像

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 平面（0 级） | 无阴影 | 默认表面、文字块 |
| 低语（1 级） | `rgba(97, 104, 117, 0.05) 0px 1px 1px, rgba(97, 104, 117, 0.05) 0px 2px 2px` | 卡片、按钮、交互表面 |
| 焦点（2 级） | `3px solid var(--mds-color-focus-action-external)` 轮廓 | 焦点环——颜色匹配上下文 |

**阴影哲学**：HashiCorp 可以说使用了现代网页设计中最微妙的阴影系统。双层 5% 不透明度的阴影几乎不可见——它们的存在不是为了创造视觉深度，而是为了发出交互性信号。如果你能看到阴影，它就太强了。这种克制传达了稳定的企业价值——没有任何东西是漂浮的，没有任何东西是不确定的。

## 7. 规范与禁忌

### 应该做
- 标题和品牌文字使用 HashiCorp Sans，正文和 UI 文字使用 system-ui
- 所有 HashiCorp Sans 文字启用 `"kern"`
- 产品品牌色仅用于各自产品（Terraform = 紫色，Vault = 黄色，等等）
- 区域标记应用 13px 字重 600 配 1.3px 字间距的大写标签
- 将阴影保持在"低语"级别（0.05 不透明度双层）
- 使用 `--mds-color-*` 标记系统保持一致的颜色应用
- 保持紧凑标题/宽松正文节奏（1.17–1.21 对比 1.50–1.69 行高）
- 使用 `3px solid` 焦点轮廓确保无障碍访问

### 不应该做
- 不要在产品上下文之外使用产品品牌色（Vault 内容上不用 Terraform 紫色）
- 不要将阴影不透明度增加超过 0.1——低语级别是有意为之的
- 不要使用胶囊形按钮（>8px 圆角）——锐利的极小圆角是结构性的
- 不要在标题上跳过 `"kern"` 特性——字体需要它
- 不要将 HashiCorp Sans 用于小号正文——它专为 17px+ 的标题设计
- 不要在同一组件中混用产品颜色——每个产品有一种颜色
- 不要使用纯黑（`#000000`）作为深色背景——使用 `#15181e` 或 `#0d0e12`
- 不要忘记不对称按钮内边距——9px 9px 9px 15px 是有意为之的

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|------|----------|
| 移动端 | <768px | 单列、汉堡导航、堆叠行动按钮 |
| 平板 | 768–992px | 双列网格，导航开始扩展 |
| 桌面 | 992–1150px | 完整布局，超大菜单导航 |
| 大型 | >1150px | 最大宽度居中，宽松边距 |

### 折叠策略
- 英雄区：82px → 52px → 42px 标题尺寸
- 导航：超大菜单 → 汉堡菜单
- 产品卡片：3 列 → 2 列 → 堆叠
- 深色区域保持全宽但压缩内边距
- 按钮：行内 → 移动端全宽堆叠
