# PostHog 设计系统灵感

## 1. 视觉主题与氛围

PostHog 的网站感觉像一个内部创业 wiki 逃进了野外——温暖、不羁，刻意反企业化。背景不是开发者工具惯用的清爽白色或深色虚空；而是温暖的鼠尾草奶油色（`#fdfdf8`），给每个表面一种手工制作、类纸张的质感。颜色倾向于土质橄榄绿和柔和鼠尾草色，而非 SaaS 世界惯用的蓝色和紫色。仿佛有人在一个舒适的花园棚屋里设计了一个开发者分析平台。

个性是主角：手绘刺猬插图、怪诞的手办模型和俏皮的图像取代了 B2B SaaS 常见的库存摄影和抽象渐变。IBM Plex Sans Variable 作为排版基础——一种具有真正技术可信度的字体（IBM 创作，广泛用于开发者背景），在标题上以粗体字重（700、800）部署，正文文字行高宽松。排版表达"我们是认真的工程师"，而周围的一切却在说"但我们不会太把自己当回事儿"。

交互设计承载着同样的精神：悬停状态闪烁 PostHog 橙色（`#F54E00`）文字——一种在静止时不出现但在交互时惊喜登场的隐藏品牌颜色。近黑色按钮（`#1e1f23`）在悬停时使用透明度降低而非颜色变化，激活状态略微缩放。边框系统使用鼠尾草色调的灰色（`#bfc1b7`），与橄榄文字调色板协调。基于 Tailwind CSS，搭配 Radix UI 和 shadcn/ui 基础组件，技术基础是现代且组件驱动的，但视觉输出顽固地独特。

**核心特征：**
- 温暖鼠尾草/橄榄色调色板，代替惯用蓝色——土质且平易近人
- IBM Plex Sans Variable 字体，标题使用粗体字重（700/800），行高宽松 1.50+
- 隐藏品牌橙色（`#F54E00`）仅在悬停交互时出现——令人愉快的惊喜
- 手绘刺猬插图和俏皮图像——刻意反企业化
- 鼠尾草色调边框（`#bfc1b7`）和背景（`#eeefe9`），创造统一的暖绿系统
- 近黑色 CTA（`#1e1f23`），基于透明度的悬停状态
- 内容丰富的编辑式布局——网站读起来像杂志，不像典型的落地页
- Tailwind CSS + Radix UI + shadcn/ui 组件架构

## 2. 色彩搭配与角色定义

### 主色
- **橄榄墨水**（`#4d4f46`）：主要文字颜色——一种独特的橄榄灰，给所有文字带来温暖、土质的色调
- **深橄榄**（`#23251d`）：链接文字和高强调标题——带绿色底调的近黑色
- **PostHog 橙色**（`#F54E00`）：隐藏品牌强调——仅在悬停状态出现，令人惊喜的鲜艳橙色

### 次色与强调色
- **琥珀金**（`#F7A501`）：深色按钮上的次要悬停强调——与橙色搭配的温暖金色
- **金色边框**（`#b17816`）：特殊按钮边框——特色 CTA 的琥珀金
- **焦点蓝**（`#3b82f6`）：焦点环颜色（Tailwind 默认）——系统中唯一的蓝色，专为无障碍保留

### 表面与背景
- **温暖羊皮纸**（`#fdfdf8`）：主要页面背景——带黄绿色底调的温暖近白色
- **鼠尾草奶油**（`#eeefe9`）：输入框背景，次要表面——淡鼠尾草色调
- **浅鼠尾草**（`#e5e7e0`）：按钮背景，三级表面——柔和鼠尾草绿
- **温暖棕褐**（`#d4c9b8`）：特色按钮背景——用于强调的温暖棕褐/卡其色
- **悬停白**（`#f4f4f4`）：通用悬停背景状态

### 中性色与文字
- **橄榄墨水**（`#4d4f46`）：主要正文和 UI 文字
- **静音橄榄**（`#65675e`）：次要文字，浅色背景上的按钮标签
- **鼠尾草占位符**（`#9ea096`）：占位文字，禁用状态——温暖鼠尾草绿
- **鼠尾草边框**（`#bfc1b7`）：主要边框颜色——所有边框的橄榄色调灰
- **浅边框**（`#b6b7af`）：次要边框，工具栏边框——略深的鼠尾草

### 语义色与强调色
- **PostHog 橙色**（`#F54E00`）：悬停文字强调——交互性和品牌个性的信号
- **琥珀金**（`#F7A501`）：深色按钮悬停强调——温暖信号
- **焦点蓝**（`#3b82f6`，50% 透明度）：键盘焦点环——仅无障碍使用的颜色
- **深色文字**（`#111827`）：高对比度链接文字——重要链接的近黑色

### 渐变系统
- 营销网站上没有渐变——PostHog 的视觉语言刻意平面且温暖
- 深度通过表面层叠和边框收纳实现，而非颜色过渡

## 3. 排版规则

### 字体家族
- **展示与正文**：`IBM Plex Sans Variable`——可变字体（100–700+ 字重范围）。回退：`IBM Plex Sans, -apple-system, system-ui, Avenir Next, Avenir, Segoe UI, Helvetica Neue, Helvetica, Ubuntu, Roboto, Noto, Arial`
- **等宽**：`ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, Liberation Mono, Courier New`——系统等宽字体栈
- **代码展示**：`Source Code Pro`——回退：`Menlo, Consolas, Monaco`

### 层级

### 原则
- **粗体标题主导**：标题使用 700–800 字重——PostHog 的排版自信而坚定，不是低语式的
- **宽松正文行高**：正文文字行高 1.50–1.71，创造极舒适的阅读感——网站内容丰富，为长时间阅读优化
- **分数尺寸**：多个尺寸（21.4px、19.3px、13.7px）表明流式/缩放字体系统而非固定档位——可能由非标准基准的 Tailwind rem 比例计算得出
- **大写作为分类信号**：粗体大写标签（18px–20px 字重 700）用于产品分类标题——一种杂志编辑惯例
- **选择性负字间距**：展示文字时字间距收紧（30px 时 -0.75px），正文时放宽至 0px——标题压缩，正文呼吸

## 4. 组件样式

### 按钮
- **深色主要**：`#1e1f23` 背景，白色文字，6px 圆角，`10px 12px` 内边距。悬停：透明度 0.7，琥珀金文字。激活：透明度 0.8，轻微缩放变换。主要 CTA——深色而自信
- **鼠尾草浅色**：`#e5e7e0` 背景，橄榄墨水（`#4d4f46`）文字，4px 圆角，`4px` 内边距。悬停：`#f4f4f4` bg，PostHog 橙色文字。紧凑实用按钮
- **温暖棕褐特色**：`#d4c9b8` 背景，黑色文字，无可见圆角。悬停：同样的橙色文字闪烁。特色/高级操作
- **输入框样式**：`#eeefe9` 背景，鼠尾草占位符（`#9ea096`）文字，4px 圆角，1px `#b6b7af` 边框。看起来像搜索/筛选控件
- **近白幽灵**：`#fdfdf8` 背景，橄榄墨水文字，4px 圆角，透明 1px 边框。存在感极低
- **悬停模式**：所有按钮在悬停时闪烁 PostHog 橙色（`#F54E00`）或琥珀金（`#F7A501`）文字——品牌的标志性交互惊喜

### 卡片与容器
- **带边框卡片**：温暖羊皮纸（`#fdfdf8`）或白色背景，1px `#bfc1b7` 边框，4px–6px 圆角——干净而简约
- **鼠尾草表面卡片**：`#eeefe9` 背景用于次要内容容器
- **阴影卡片**：`0px 25px 50px -12px rgba(0, 0, 0, 0.25)`——提升内容的单一深阴影（模态框、下拉菜单）
- **悬停**：交互卡片上的橙色文字闪烁——与按钮行为一致

### 输入框与表单
- **默认**：`#eeefe9` 背景，`#9ea096` 占位文字，1px `#b6b7af` 边框，4px 圆角，`2px 0px 2px 8px` 内边距
- **聚焦**：50% 透明度的 `#3b82f6` 环（Tailwind 蓝色焦点环）
- **文字颜色**：输入值用 `#374151`——比主要文字颜色深，以提高可读性
- **边框变体**：多种边框模式——部分输入框使用复合边框（仅顶部、左侧、底部）

### 导航
- **顶部导航**：温暖背景，IBM Plex Sans 15px 字重 600
- **下拉菜单**：带产品类别的丰富超级菜单结构
- **链接颜色**：导航链接用深橄榄（`#23251d`），悬停时下划线
- **CTA**：导航中的深色主要按钮（#1e1f23）——"免费开始"
- **移动端**：折叠为汉堡菜单，简化导航

### 图像处理
- **手绘插图**：刺猬吉祥物和怪诞插图——标志性视觉元素
- **产品截图**：嵌入设备框架或干净容器的 UI 截图
- **手办**：刺猬手办的俏皮产品摄影——反企业化
- **信任标志**：企业标志（空客、GOV.UK）展示在静音信任条中
- **宽高比**：混合——插图不规则，截图为 16:9 或宽屏

### AI 聊天小工具
- 浮动的 PostHog AI 助手，带语音气泡——嵌入营销网站的交互式产品演示

## 5. 布局原则

### 间距系统
- **基本单位**：8px
- **刻度**：2px、4px、6px、8px、10px、12px、16px、18px、24px、32px、34px
- **章节内边距**：章节间 32px–48px 垂直间距（内容丰富网站的紧凑选择）
- **卡片内边距**：内部 4px–12px（显著紧凑）
- **组件间距**：相关元素间 4px–8px

### 网格与容器
- **最大宽度**：1536px（最大断点），内容容器可能为 1200px–1280px
- **列模式**：多变——文字内容单列，特性卡片 2-3 列网格，产品演示不对称布局
- **断点**：定义了 13 个——1px、425px、482px、640px、768px、767px、800px、900px、1024px、1076px、1160px、1280px、1536px

### 空白哲学
- **设计上的内容密集**：PostHog 的网站信息丰富——空白是克制的，不是奢侈的
- **编辑节奏**：内容章节像杂志一样流动，变化的布局保持视线移动
- **插图作为呼吸空间**：手绘刺猬艺术自然地打破密集内容章节

### 圆角刻度
- **2px**：小内联元素，标签（`span`）
- **4px**：主要 UI 组件——按钮、输入框、下拉菜单、菜单项（`button`、`div`、`combobox`）
- **6px**：次要容器——较大按钮、列表项、卡片变体（`button`、`div`、`li`）
- **9999px**：药丸形——徽章、状态指示器、圆角标签（`span`、`div`）

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 第 0 层（平面） | 无阴影，温暖羊皮纸背景 | 页面画布，大多数表面 |
| 第 1 层（边框） | `1px solid #bfc1b7`（鼠尾草边框） | 卡片收纳，输入框边框，区域分隔线 |
| 第 2 层（复合边框） | 不同侧的多个 1px 边框 | 输入框分组，工具栏元素 |
| 第 3 层（深阴影） | `0px 25px 50px -12px rgba(0, 0, 0, 0.25)` | 模态框，浮动元素，超级菜单下拉 |

### 阴影哲学
PostHog 的层次系统极其简约——整个系统中只有一个阴影定义。深度通过以下方式传达：
- **边框收纳**：鼠尾草色调边框（`#bfc1b7`）1px 创造温和的温暖分隔
- **表面颜色变化**：从 `#fdfdf8` 到 `#eeefe9` 到 `#e5e7e0` 的移动创造无需阴影的分层深度
- **单一阴影**：唯一定义的阴影（`0 25px 50px -12px`）专为浮动元素保留——模态框、下拉菜单、弹出窗口。这是一个深邃、戏剧性的阴影，在需要时创造清晰的分离

### 装饰性深度
- **插图分层**：手绘刺猬艺术自然地创造视觉深度
- **无渐变或发光**：平面温暖表面系统完全依赖边框和表面颜色区分
- **无玻璃态**：全程完全不透明表面

## 7. 注意事项

### 应该做
- 使用橄榄/鼠尾草颜色系列（#4d4f46、#23251d、#bfc1b7）用于文字和边框——温暖绿色底调对品牌至关重要
- 在悬停状态闪烁 PostHog 橙色（#F54E00）——这是隐藏的品牌签名
- 使用 IBM Plex Sans 粗体字重（700/800）用于标题——字体带有技术可信度
- 保持正文文字宽松的行高（1.50–1.71）——内容丰富的网站要求可读性
- 保持温暖羊皮纸背景（#fdfdf8）——不是纯白，绝不冷
- 为大多数 UI 元素使用 4px 圆角——保持角落细微而实用
- 包含俏皮的手绘插图元素——个性是差异化因素
- 在深色按钮上应用基于透明度的悬停状态（0.7 透明度），而非颜色变化

### 不应该做
- 不要使用蓝色、紫色或典型科技 SaaS 颜色——PostHog 的调色板刻意是橄榄/鼠尾草色
- 不要添加重阴影——系统只为浮动元素使用一个阴影；其他所有内容使用边框
- 不要让设计在传统意义上看起来"精致"或"高端"——PostHog 的魅力在于其不羁、粗犷的能量
- 不要在正文文字上使用紧凑行高——宽松的 1.50+ 间距对内容丰富布局至关重要
- 不要在卡片上应用大圆角（12px+）——PostHog 使用 4px–6px，保持紧凑而实用
- 不要移除橙色悬停闪烁——这是核心交互模式，不是装饰
- 不要用库存摄影替换插图——手绘刺猬艺术就是品牌
- 不要将纯白（#ffffff）用作页面背景——温暖鼠尾草奶油（#fdfdf8）色调是基础

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 小手机 | <425px | 单列，紧凑内边距，堆叠卡片 |
| 手机 | 425px–640px | 小幅布局调整，较大触摸目标 |
| 平板 | 640px–768px | 两列网格开始，导航部分可见 |
| 大平板 | 768px–1024px | 多列布局，展开导航 |
| 桌面 | 1024px–1280px | 完整布局，3 列特性网格，展开超级菜单 |
| 大桌面 | 1280px–1536px | 最大宽度容器，宽裕边距 |
| 超大 | >1536px | 最大宽度居中容器 |

### 触摸目标
- 按钮：4px–6px 圆角，`4px–12px` 内边距——紧凑但可用
- 导航链接：15px 文字，字重 600，充足内边距
- 移动端：汉堡菜单，简化导航
- 输入框：宽松垂直内边距，方便拇指操作的表单

### 折叠策略
- **导航**：带下拉菜单的完整超级菜单 → 移动端汉堡菜单
- **特性网格**：3 列 → 2 列 → 单列堆叠
- **排版**：展示尺寸跨断点缩小（30px → 更小）
- **插图**：在容器内响应式缩放，部分可能在移动端隐藏以节省空间
- **章节间距**：等比例缩减，同时保持可读性

### 图像行为
- 插图在容器内响应式缩放
- 产品截图保持宽高比
- 信任标志在移动端重新排列为多行网格
- AI 聊天小工具可能在小屏幕上重新定位或简化

### 迭代指南
使用此设计系统精炼现有屏幕时：
1. 验证背景是温暖羊皮纸（#fdfdf8）而非纯白——鼠尾草奶油温暖感至关重要
2. 检查所有文字使用橄榄系列（#4d4f46、#23251d），而非纯黑或中性灰
3. 确保悬停状态闪烁 PostHog 橙色（#F54E00）——如果悬停感觉乏味，你错过了这点
4. 确认边框使用鼠尾草色调灰（#bfc1b7）而非中性灰——温暖贯穿每个元素
5. 整体基调应该感觉像一个有趣、朝气蓬勃的创业 wiki——绝不企业化光鲜或冰冷无趣
