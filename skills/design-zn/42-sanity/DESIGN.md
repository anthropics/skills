# Design System Inspiration of Sanity

## 1. 视觉主题与氛围

Sanity 的网站是一个以开发者为导向的内容平台，呈现为一个夜间指挥中心的形态——深邃、精准、结构严密。整个体验坐落在近黑色画布（`#0b0b0b`）之上，感觉不像是"暗黑模式切换"，更像是一个为生活在终端中的人们所构建工具的自然状态。大多数 CMS 营销页面倾向于使用友好的柔和色彩和温柔插图，而 Sanity 却深入挖掘其产品本身的重力感：结构化内容理应有一个结构化的舞台。

标志性的排版声音是 waldenburgNormal——一种独特的、略带几何感的无衬线字体，在展示尺寸下具有紧凑的负字母间距（-0.32px 至 -4.48px），赋予标题一种压缩的、工程化的质感。在 112px 的英雄尺寸下搭配 -4.48px 的字母间距，文字感觉几乎像是机器加工的——如同精密切割的钢铁字形。这与代码和技术标签的 IBM Plex Mono 搭配，创造出双重声音：编辑权威感与开发者可信度相辅相成。

Sanity 的独特之处在于其单色深色调色板与生动、饱和的强调色点缀之间的相互作用。中性色阶从纯黑经由严格控制的灰色坡度（`#0b0b0b` → `#212121` → `#353535` → `#797979` → `#b9b9b9` → `#ededed` → `#ffffff`）延伸，没有任何暖色或冷色偏向——只有纯粹、消色差的精准。在这个严谨的背景下，霓虹绿强调色（display-p3 绿色）和电光蓝（`#0052ef`）如同暗色控制室中的信号灯般产生强烈冲击。橙红色行动号召（`#f36458`）在整体冷系中提供了唯一的暖色触感。

**核心特征：**
- 近黑色画布（`#0b0b0b`）作为默认的自然环境——不是暗黑"模式"，而是主要身份标识
- waldenburgNormal 在展示尺寸下具有极端负字母间距，创造出精密工程化的排版声音
- 纯消色差灰度——没有暖色或冷色底调，纯净中性的纪律
- 生动的强调色点缀：霓虹绿、电光蓝（`#0052ef`）和珊瑚红（`#f36458`）对比深色底面
- 胶囊形主要按钮（99999px 圆角）与次要操作的微圆角矩形（3-6px）形成对比
- IBM Plex Mono 作为编辑展示字面的技术对应物
- 全出血深色区块，内容包含在有节制的最大宽度容器中
- 悬停状态全部切换为电光蓝（`#0052ef`）——所有交互元素一致的"激活"信号

## 2. 色彩体系与角色

### 主品牌色
- **Sanity 黑** (`#0b0b0b`)：主要画布和主导表面颜色。不是纯黑，但足够接近绝对感。整个视觉身份的基础。
- **纯黑** (`#000000`)：用于最高对比度的场景、深色叠层和某些边框强调。
- **Sanity 红** (`#f36458`)：主要行动号召和品牌强调色——一种温暖的珊瑚红，作为主要的行动号召颜色。用于"开始使用"按钮和主要转化点。

### 强调色与交互色
- **电光蓝** (`#0052ef`)：整个系统中通用的悬停/激活状态颜色。按钮、链接和交互元素在悬停时全部切换为此蓝色。也用作 `--color-blue-700` 的焦点环和激活状态。
- **浅蓝** (`#55beff` / `#afe3ff`)：用于强调背景、徽章和暗蓝色表面的次要蓝色变体。
- **霓虹绿** (`color(display-p3 .270588 1 0)`)：用作 `--color-fg-accent-green` 的鲜艳宽色域绿色，用于成功状态和高级功能高亮。sRGB 备选色为 `#19d600`。
- **强调洋红** (`color(display-p3 .960784 0 1)`)：用于特定强调场景的鲜艳宽色域洋红。

### 表面与背景
- **近黑** (`#0b0b0b`)：默认页面背景和主要表面。
- **深灰** (`#212121`)：卡片、次要容器、输入背景及基础画布之上微妙分层的抬高表面颜色。
- **中深** (`#353535`)：三级表面和边框颜色，用于在深色层之间创造深度。
- **纯白** (`#ffffff`)：用于反色区块、深色上的浅色文字和特定按钮表面。
- **浅灰** (`#ededed`)：反色/浅色区块的浅色表面和微妙背景色调。

### 中性色与文字
- **白色** (`#ffffff`)：深色表面上的主要文字颜色，最高可读性。
- **银色** (`#b9b9b9`)：深色表面上的次要文字、正文、静音描述和占位符文字。
- **中灰** (`#797979`)：三级文字、元数据、时间戳和去强调内容。
- **炭灰** (`#212121`)：浅色/反色表面上的文字。
- **近黑文字** (`#0b0b0b`)：白色/浅色按钮表面上的主要文字。

### 语义色
- **错误红** (`#dd0000`)：破坏性操作、验证错误和关键警告——纯粹的高饱和度红色。
- **GPC 绿** (`#37cd84`)：隐私/合规指示绿色。
- **焦点环蓝** (`#0052ef`)：无障碍性的焦点环颜色，与交互蓝色一致。

### 边框系统
- **深色边框** (`#0b0b0b`)：深色容器上的主要边框——几乎不可见，保持最小包含感。
- **微妙边框** (`#212121`)：深色表面上输入框、文本区域和卡片边缘的标准边框。
- **中等边框** (`#353535`)：强调包含和分隔线的更明显边框。
- **浅色边框** (`#ffffff`)：反色/浅色元素或需要对比分离的按钮上的边框。
- **橙色边框** (`color(display-p3 1 0.3333 0)`)：高亮/特色元素的特殊强调边框。

## 3. 排版规则

### 字体家族
- **展示 / 标题**：`waldenburgNormal`，备选字体：`waldenburgNormal Fallback, ui-sans-serif, system-ui`
- **正文 / UI**：`waldenburgNormal`，备选字体：`waldenburgNormal Fallback, ui-sans-serif, system-ui`
- **代码 / 技术**：`IBM Plex Mono`，备选字体：`ibmPlexMono Fallback, ui-monospace`
- **备选 / CJK**：`Helvetica`，备选字体：`Arial, Hiragino Sans GB, STXihei, Microsoft YaHei, WenQuanYi Micro Hei`

*注：waldenburgNormal 是定制字体。外部实现时，使用 Inter 或 Space Grotesk 作为无衬线替代（几何感、略带紧凑感）。IBM Plex Mono 可在 Google Fonts 获取。*

### 排版原则
- **大尺寸下的极端负字母间距**：72px 以上的展示标题使用激进的负字母间距（-2.88px 至 -4.48px），营造出紧凑、工程化的质感，使 Sanity 区别于更宽松的编辑排版。
- **单一字体，多种语域**：waldenburgNormal 同时处理编辑展示和功能 UI 文字。字重范围窄（大多数为 400-425，仅微型标签达 500-600），保持声音一致。
- **OpenType 特性控制**：排版在展示尺寸使用特定特性设置，包括 `"cv01", "cv11", "cv12", "cv13", "ss07"` 和正文的 `"calt" 0`，针对不同上下文微调字符变体。
- **紧凑标题，舒展正文**：标题使用 1.00-1.24 行高（极度紧凑），正文呼吸于 1.50。这种对比创造出清晰的视觉层次。
- **技术标签使用大写**：IBM Plex Mono 说明文字和小标签频繁使用 `text-transform: uppercase` 搭配紧凑行高，为技术元数据创造出"系统读数"美学。

### 网格与容器
- 最大内容宽度：约 1440px（从断点推断）
- 页面间距：桌面端 32px，移动端 16px
- 内容区块使用全出血背景，内容居中且有最大宽度限制
- 多列布局：桌面端 2-3 列，移动端单列
- 卡片网格：CSS Grid，间距一致（16-24px）

### 留白理念
Sanity 在区块之间使用激进的垂直间距（64-120px），在深色画布上创造呼吸空间。区块内部间距更紧（16-32px），形成密集的信息簇，由充裕的空隙分隔。这种节奏赋予页面一种"幻灯片"质感——每个区块都感觉像是独立的聚焦框架。

## 6. 深度与层次

### 阴影系统

| 层级 | 值 | 用途 |
|-------|-------|-------|
| 0 级（平面） | 无 | 大多数元素的默认状态——深色表面仅通过颜色创造深度 |
| 1 级（微妙） | 0px 0px 0px 1px `#212121` | 类边框阴影，无可见边框的最小包含感 |
| 2 级（焦点） | 0 0 0 2px `var(--color-blue-500)` | 输入框和交互元素的焦点环 |
| 3 级（叠层） | 背景虚化 + 半透明深色 | 导航叠层、模态背景 |

### 深度理念
Sanity 的深度系统几乎完全依赖**色度**而非阴影。层次感通过表面颜色变化传达：`#0b0b0b`（地面）→ `#212121`（抬高）→ `#353535`（突出）→ `#ffffff`（反色/最高）。这种方法对于传统投影阴影不可见的深色界面来说是原生的。少数存在的阴影是环形的（0px 0px 0px Npx）或基于虚化的（backdrop-filter），而非偏移阴影，保持了平面、精密工程化的美学。

基于边框的包含感（1px solid `#212121` 或 `#353535`）作为主要空间分隔符，边框深度经过校准，可见但不占主导。该系统避免"浮动卡片"美学——一切都感觉是附着在表面上，而非悬浮其上。

## 7. 应做与禁止

### 应做
- 使用消色差灰度作为基础——保持纯净的中性纪律，无暖/冷色调
- 在所有交互元素上一致地应用电光蓝（`#0052ef`）作为通用悬停/激活状态
- 在 48px 及以上的展示标题上使用极端负字母间距（-2px 至 -4.48px）
- 保持主要行动号召为全胶囊形（99999px 圆角）搭配珊瑚红（`#f36458`）
- 技术标签、标签和系统元数据使用 IBM Plex Mono 大写
- 通过表面颜色（深到浅）而非阴影传达深度
- 在深色画布上保持充裕的垂直区块间距（64-120px）
- 展示排版使用 `"cv01", "cv11", "cv12", "cv13", "ss07"` OpenType 特性

### 禁止
- 不要在中性色刻度中引入暖色或冷色调——Sanity 的灰色是纯消色差的
- 不要使用投影阴影作为层次——深色界面需要色度深度
- 不要应用 13px 到 99998px 之间的圆角半径——系统直接从 12px（大卡片）跳至胶囊形（99999px）
- 不要将珊瑚红行动号召与电光蓝交互色混用于同一元素
- 不要使用重字体（700+）——系统上限为 600，且仅用于 11px 大写标签
- 不要在同一表面放置浅色文字和深色文字而不检查灰色对灰色的对比率
- 不要使用传统偏移投影框阴影——仅用环形阴影（0 0 0 Npx）或基于边框的包含感
- 不要破坏标题的紧凑行高——1.00-1.24 是范围，展示文字绝不要到 1.5+

| 超大桌面 | >= 1640px | 完整布局，最大内容宽度 |
| 桌面 | >= 1440px | 标准桌面布局 |
| 紧凑桌面 | >= 1200px | 略微压缩的桌面 |

| 移动端 | >= 480px | 单列，堆叠布局 |
| 小型移动端 | >= 376px | 最小支持宽度 |

### 折叠策略
- **导航**：横向链接在 768px 以下折叠为汉堡菜单
- **英雄排版**：跨断点从 112px → 72px → 48px → 38px 缩放，保持紧凑的字母间距比例
- **网格布局**：3 列 → 约 960px 时 2 列 → 768px 以下单列
- **卡片网格**：移动端横向滚动而非换行（保留卡片宽高比）
- **区块间距**：移动端垂直填充约减少 40%（120px → 64px → 48px）
- **按钮尺寸**：行动号召胶囊形保持内边距但缩小字体；幽灵按钮保持固定
- **代码块**：横向滚动，保留等宽字体格式

### 移动端特定调整
- 全出血区块延伸至边缘，内部间距 16px
- 触控目标：所有交互元素最小 44px
- 标题字母间距在移动端略微放宽（减少激进的负字母间距）
- 图片容器从固定宽高比切换为全宽自动高度

1. **以深色开始**：背景 `#0b0b0b`，主要文字 `#ffffff`，次要文字 `#b9b9b9`
2. **添加结构**：使用 `#212121` 表面和 `#353535` 边框进行包含——无阴影
3. **应用排版**：Inter（或 Space Grotesk），标题紧凑字母间距，正文行高 1.50
4. **颜色点缀**：行动号召使用 `#f36458`，所有悬停/交互状态使用 `#0052ef`
5. **精细间距**：8px 基础单位，区块内 24-32px，区块间 64-120px
6. **技术细节**：标签和元数据添加 IBM Plex Mono 大写标签
7. **润色**：确保所有交互元素悬停为 `#0052ef`，所有按钮为胶囊形或微妙 5px 圆角，边框为细线（1px）
