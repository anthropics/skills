# OpenCode 设计系统灵感

## 1. 视觉主题与氛围

OpenCode 的网站体现了一种终端原生、等宽字体优先的美学，反映了其作为开源 AI 编程助手的身份认同。整个视觉系统建立在近黑色背景（`#201d1d`）与温暖米白色文字（`#fdfcfc`）的鲜明对比之上。这不是通用的深色主题——而是一种温暖的、略带红棕色的深色，感觉像是一款精致的终端仿真器，而非冷峻的 IDE。深色和浅色中的温暖底调（注意 `#201d1d` 中微妙的红色通道——rgb(32, 29, 29)）创造出一种统一、有历史感的质感。

Berkeley Mono 是唯一的字体，建立起毫不妥协的等宽字体身份。每一个元素——标题、正文、按钮、导航——都共享这一单一字体系列，创造出统一的"万物皆代码"哲学。38px 粗体标题配 1.50 行高宽敞而可读；16px 字重 500 的正文文字提供略重于正常的阅读字重，增强屏幕易读性。等宽字体栅格自然地在整个布局中强制对齐和节奏感。

色彩系统刻意极简。主要调色板仅由三种功能色调组成：温暖近黑色（`#201d1d`）、中等温暖灰（`#9a9898`）和明亮米白色（`#fdfcfc`）。语义颜色借自苹果 HIG 色板——蓝色点缀（`#007aff`），红色危险（`#ff3b30`），绿色成功（`#30d158`），橙色警告（`#ff9f0a`）——赋予界面熟悉可信的信号颜色，而不增加品牌复杂性。边框使用微妙的温暖透明（`rgba(15, 0, 0, 0.12)`），与整个系统的温暖底调呼应。

**核心特征：**
- Berkeley Mono 作为唯一字体——等宽字体无处不在，无无衬线或衬线字体声音
- 温暖近黑色主色（`#201d1d`），带红棕色底调，非纯黑
- 米白色文字（`#fdfcfc`），带温暖色调，非纯白
- 4px 圆角贯穿全局——锐利、实用的角落
- 8px 基础间距系统，延伸至 96px
- 苹果 HIG 启发的语义颜色（蓝、红、绿、橙）
- 使用 `rgba(15, 0, 0, 0.12)` 的温暖透明边框
- 具有慷慨 20px 内边距和 6px 圆角的邮件输入框——系统中最慷慨的组件圆角
- 单一按钮变体：深色背景，浅色文字，紧凑垂直内边距（4px 20px）
- 带下划线链接作为默认链接样式，强化以文字为核心的身份

## 2. 色彩方案与角色

### 主色
- **OpenCode 深色** (`#201d1d`)：主背景，按钮填充，链接文字。带微妙红棕色温暖的近黑色——rgb(32, 29, 29)。
- **OpenCode 浅色** (`#fdfcfc`)：深色界面上的主要文字，按钮文字。避免临床感纯白的略带温暖米白色。
- **中灰** (`#9a9898`)：次要文字，弱化链接。连接深浅两端的中性温暖灰。

### 次要色
- **深色界面** (`#302c2c`)：略亮于主深色，用于抬升界面和微妙差异化。
- **边框灰** (`#646262`)：较强的边框，交互元素上的轮廓环。
- **浅色界面** (`#f1eeee`)：浅色模式界面，微妙背景变化。

### 点缀色
- **点缀蓝** (`#007aff`)：主要点缀，链接，交互高亮。苹果系统蓝。
- **点缀蓝悬停** (`#0056b3`)：悬停状态的深蓝色。
- **点缀蓝激活** (`#004085`)：最深蓝色，用于按压/激活状态。

### 语义色
- **危险红** (`#ff3b30`)：错误状态，破坏性操作。苹果系统红。
- **危险悬停** (`#d70015`)：危险元素悬停的深红色。
- **危险激活** (`#a50011`)：最深红色，用于按压危险状态。
- **成功绿** (`#30d158`)：成功状态，正向反馈。苹果系统绿。
- **警告橙** (`#ff9f0a`)：警告状态，注意信号。苹果系统橙。
- **警告悬停** (`#cc7f08`)：警告元素悬停的深橙色。
- **警告激活** (`#995f06`)：最深橙色，用于按压警告状态。

### 文字比例
- **弱化文字** (`#6e6e73`)：弱化标签，禁用文字，占位内容。
- **次要文字** (`#424245`)：浅色背景上的次要文字，说明文字。

### 边框
- **温暖边框** (`rgba(15, 0, 0, 0.12)`)：主要边框颜色，带红色调的温暖透明黑。
- **选项卡边框** (`#9a9898`)：选项卡下划线边框，2px 实心底部。
- **轮廓边框** (`#646262`)：容器 1px 实心轮廓边框。

## 3. 排版规则

### 字体系列
- **通用字体**：`Berkeley Mono`，备用字体：`IBM Plex Mono, ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, Liberation Mono, Courier New, monospace`

### 层级体系

| 角色 | 尺寸 | 字重 | 行高 | 备注 |
|------|------|--------|-------------|-------|
| 一级标题 | 38px (2.38rem) | 700 | 1.50 | 主视觉标题，页面标题 |
| 二级标题 | 16px (1.00rem) | 700 | 1.50 | 区段标题，粗体强调 |
| 正文 | 16px (1.00rem) | 400 | 1.50 | 标准正文，段落 |
| 中等正文 | 16px (1.00rem) | 500 | 1.50 | 链接，按钮文字，导航项 |
| 紧凑正文 | 16px (1.00rem) | 500 | 1.00（紧凑） | 紧凑标签，选项卡项 |
| 说明文字 | 14px (0.88rem) | 400 | 2.00（宽松） | 脚注，元数据，小型标签 |

### 排版原则
- **一种字体，一种声音**：Berkeley Mono 专门使用。展示、正文和代码之间没有排版变化——所有内容以相同的等宽字体发声。层级仅通过尺寸和字重实现。
- **字重作为层级**：700 用于标题，500 用于交互/中等强调，400 用于正文。三个字重层级创造整个层级体系。
- **慷慨行高**：1.50 作为标准行高，让文字在等宽字体栅格内呼吸。说明文字 2.00 的宽松行高创造清晰的视觉分隔。
- **紧凑用于交互**：交互元素（选项卡，紧凑标签）使用 1.00 行高，营造密集、可点击的目标。

### 图像处理

**邮件收集**

- 舒适打字的慷慨 20px 内边距
- 6px 圆角——系统中最圆润的元素
- 新闻通讯/候补名单模式

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 精细刻度：1px、2px、4px（边框和微调的 8px 以下值）
- 标准刻度：8px、12px、16px、20px、24px
- 扩展刻度：32px、40px、48px、64px、80px、96px
- 系统遵循干净的 4/8px 栅格，持续翻倍

### 栅格与容器
- 内容最大宽度：约 800–900px（窄，阅读优化）
- 单列布局作为主要模式
- 居中内容，慷慨水平外边距
- 主视觉区：全宽暗色终端元素
- 功能区段：单列文字块
- 页脚：多列链接栅格

### 留白哲学
- **等宽节奏**：Berkeley Mono 的固定宽度特性创造自然的垂直栅格。1.50 和 2.00 的行高保持一致节奏。
- **窄而专注**：内容限制在窄列中，创造出将注意力集中在文字上的慷慨侧边距。
- **间距即区段分隔**：无装饰性分割线。区段通过慷慨的垂直间距（48–96px）而非边框或背景变化来分隔。

### 圆角比例
- 微型（4px）：所有元素的默认——按钮，容器，徽章
- 输入框（6px）：表单输入框获得略多的圆润
- 整个系统仅使用两种圆角值，强化实用主义美学

## 6. 深度与层级

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 平面（层级 0） | 无阴影，无边框 | 大多数元素的默认状态 |
| 微妙边框（层级 1） | `1px solid rgba(15, 0, 0, 0.12)` | 区段分割线，输入框边框，水平分隔线 |
| 选项卡边框（层级 2） | `2px solid #9a9898`，仅底部 | 活动选项卡指示器 |
| 轮廓边框（层级 3） | `1px solid #646262` | 容器轮廓，抬升元素 |

**阴影哲学**：OpenCode 的深度系统刻意扁平。提取的令牌中没有盒阴影——零阴影值被检测到。深度完全通过边框处理和背景色变化来传达。这种扁平性与终端美学一致：终端没有阴影，OpenCode 也没有。三个边框层级（温暖透明，选项卡指示器，实心轮廓）无需任何抬升幻觉即可创造足够的视觉层级。

### 装饰性深度
- `#201d1d` 和 `#302c2c` 之间的背景色变化创造微妙的界面差异化
- 12% 透明度的透明边框提供几乎不可见的结构
- 边框颜色中的温暖红色调（`rgba(15, 0, 0, 0.12)`）将边框与整体温暖暗色调色板联系起来
- 无渐变，无模糊，无氛围效果——纯粹扁平的终端美学

## 7. 交互与动效

### 悬停状态
- 链接：颜色从默认切换至点缀蓝（`#007aff`）或下划线样式变化
- 按钮：微妙的背景变亮或边框强调
- 点缀蓝提供三阶段悬停序列：`#007aff` → `#0056b3` → `#004085`（默认 → 悬停 → 激活）
- 危险红：`#ff3b30` → `#d70015` → `#a50011`
- 警告橙：`#ff9f0a` → `#cc7f08` → `#995f06`

### 焦点状态
- 基于边框的焦点：增加边框透明度或实心边框颜色
- 无基于阴影的焦点环——与扁平无阴影美学一致
- 键盘焦点可能使用轮廓或边框颜色切换至点缀蓝

### 过渡动效
- 最小过渡效果——终端启发的界面偏向即时状态变化
- 颜色过渡：100–150ms 用于微妙的状态反馈
- 无缩放、旋转或复杂变换动画

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 移动端 | <640px | 单列，减小内边距，标题缩小 |
| 平板 | 640–1024px | 内容宽度扩展，内边距略增 |
| 桌面端 | >1024px | 完整内容宽度（约 800–900px 居中），最大留白 |

### 触控目标
- 4px 20px 内边距的按钮提供足够的水平触控区域
- 20px 内边距的输入框确保舒适的移动端打字
- 紧凑行高的 16px 选项卡项目可能需要移动端适配

### 折叠策略
- 主视觉标题：小屏幕上 38px → 28px → 24px
- 导航：水平链接 → 移动端汉堡/抽屉
- 功能列表：保持单列，减少水平内边距
- 终端主视觉区：保持全宽，减少内部内边距
- 页脚列：多列 → 单列堆叠
- 区段间距：96px → 64px → 移动端 48px

### 图像行为
- 终端截图保持宽高比和边框处理
- 全宽元素按比例缩放
- 等宽字体因固定宽度特性在所有尺寸保持可读性

### 迭代指南
1. Berkeley Mono 是唯一的字体——绝不引入第二种字体。尺寸和字重创造所有层级。
2. 保持界面扁平：无阴影，无渐变，无模糊效果。仅使用边框和背景色变化。
3. 温暖底调很重要：使用 `#201d1d` 而非 `#000000`，使用 `#fdfcfc` 而非 `#ffffff`。红棕色温暖感微妙但至关重要。
4. 圆角在所有地方为 4px，输入框除外（6px）。绝不使用圆形胶囊或大圆角。
5. 语义颜色遵循苹果 HIG：`#007aff` 蓝，`#ff3b30` 红，`#30d158` 绿，`#ff9f0a` 橙。每种颜色都有悬停和激活的深色变体。
6. 三阶段交互：默认 → 悬停（加深）→ 激活（深度加深），用于所有语义颜色。
7. 边框使用 `rgba(15, 0, 0, 0.12)`——温暖透明的深色，非中性灰。这将边框与温暖调色板联系起来。
8. 间距遵循 8px 栅格：8、16、24、32、40、48、64、80、96px。仅对精细调整使用 4px。
