# Composio 设计系统灵感

## 1. 视觉主题与氛围

Composio 的界面是一个夜间指挥中心——密集的、以开发者为中心的黑暗，被电子青色和深钴蓝信号所点缀。整个体验建立在近乎纯黑色的画布（`#0f0f0f`）上，内容漂浮于几乎看不见的收纳边框内，创造出高科技控制面板的感觉，而非传统营销页面。这是一个对生活在黑暗终端中的开发者低语权威的网站。

视觉语言大量借鉴代码编辑器和终端窗口的美感。JetBrains Mono 与 abcDiatype 的几何精度并肩出现，强化了这是一个*由*开发者*为*开发者构建的工具的信息。装饰元素克制但有冲击力——细微的青蓝渐变发光从卡片和章节中散发出来，像深水中的生物发光体；精选元素上的硬偏移阴影（`4px 4px`）添加了一种原始的、野兽派边缘，防止设计感觉无菌。

Composio 的独特之处在于极简主义与战略性的发光色彩爆发之间的张力。网站从不喧嚣——标题使用紧凑行高（0.87），将文字压缩成密集、权威的块。颜色像稀缺资源一样被配给：主要内容用白色文字，次要用半透明白色（`rgba(255,255,255,0.5-0.6)`），品牌蓝（`#0007cd`）或电子青色（`#00ffff`）专门保留用于交互时刻和强调发光。

**核心特征：**
- 沥青黑画布，带近不可见的白色边框收纳（4-12% 透明度）
- 双字体身份：内容用几何无衬线字体（abcDiatype），技术可信度用等宽字体（JetBrains Mono）
- 超紧凑标题行高（0.87-1.0），创造压缩、有冲击力的文字块
- 生物发光强调策略——青色和蓝色发光，感觉像从内部发射光线
- 精选交互元素上的硬偏移野兽派阴影（`4px 4px`）
- 单色层次，颜色仅在最高信号时刻使用
- 桥接营销与文档的开发者终端美感

## 2. 色彩搭配与角色定义

### 主色
- **Composio 钴蓝**（`#0007cd`）：核心品牌色——深度饱和的蓝色，谨慎用于高优先级交互元素和品牌时刻。以安静的强度锚定身份。

### 次色与强调色
- **电子青色**（`#00ffff`）：引人注目的强调色——以低透明度（`rgba(0,255,255,0.12)`）用于发光按钮背景和卡片高亮。全饱和度时，作为深色画布的充沛对位。
- **信号蓝**（`#0089ff` / `rgb(0,137,255)`）：用于精选按钮边框和交互焦点状态，弥合钴蓝和青色之间的差距。
- **海洋蓝**（`#0096ff` / `rgb(0,150,255)`）：CTA 按钮的强调边框颜色，比信号蓝略温暖。

### 表面与背景
- **虚空黑**（`#0f0f0f`）：主要页面背景——不是纯黑，而是略带温暖，减少在深色显示器上的眼部疲劳。
- **纯黑**（`#000000`）：用于卡片内部和深层嵌套容器，与页面背景形成微妙的深度区分。
- **木炭**（`#2c2c2c` / `rgb(44,44,44)`）：用于次要按钮边框和深色表面分割线。

### 中性色与文字
- **纯白**（`#ffffff`）：深色表面上的主要标题和高强调文字颜色。
- **静音烟**（`#444444`）：降调正文、元数据和三级内容。
- **幽灵白**（`rgba(255,255,255,0.6)`）：次要正文和链接标签——可见但刻意退缩。
- **低语白**（`rgba(255,255,255,0.5)`）：三级按钮文字和占位内容。
- **幻影白**（`rgba(255,255,255,0.2)`）：微妙按钮背景和深度退缩的 UI 装饰。

### 语义色与强调色
- **边框薄雾 12**（`rgba(255,255,255,0.12)`）：最高透明度边框处理——用于突出的卡片边缘和内容分隔。
- **边框薄雾 10**（`rgba(255,255,255,0.10)`）：深色表面上的标准容器边框。
- **边框薄雾 08**（`rgba(255,255,255,0.08)`）：细微区域分隔线和次要卡片边缘。
- **边框薄雾 06**（`rgba(255,255,255,0.06)`）：背景分组的近不可见收纳边框。
- **边框薄雾 04**（`rgba(255,255,255,0.04)`）：最淡的边框——仅用于大气分隔。
- **浅边框**（`#e0e0e0` / `rgb(224,224,224)`）：保留用于浅色表面环境（此网站中罕见）。

### 渐变系统
- **青色发光**：使用 `#00ffff` 极低透明度的径向渐变，在卡片和特性章节后方创造生物发光光晕。
- **蓝到黑渐变**：从 Composio 钴蓝（`#0007cd`）线性渐变到虚空黑（`#0f0f0f`），用于英雄背景和章节过渡。
- **白色雾霭**：从暗到弥散的白色/灰色的页面底部渐变，在页脚附近创造大气"地平线"效果。

## 3. 排版规则

### 字体家族
- **主要**：`abcDiatype`，回退：`abcDiatype Fallback, ui-sans-serif, system-ui, Apple Color Emoji, Segoe UI Emoji, Segoe UI Symbol, Noto Color Emoji`
- **等宽**：`JetBrains Mono`，回退：`JetBrains Mono Fallback, ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, Liberation Mono, Courier New`
- **系统等宽**（回退）：`Menlo`、`monospace` 用于最小内联代码

### 原则
- **压缩创造权威**：标题行高极度紧凑（0.87-1.0），使大文字感觉密集和命令式，而非轻盈和装饰性。
- **双重个性**：abcDiatype 承载营销声音——几何、精确、友好。JetBrains Mono 承载技术声音——可信、功能性、对开发者熟悉。
- **字重克制**：几乎一切都是字重 400（常规）。字重 500（中等）保留用于小标签、徽章和精选卡片标题。字重 700（粗体）仅出现在微小系统等宽环境中。
- **代码负字间距**：JetBrains Mono 使用负字间距（-0.28px 至 -0.98px），用于感觉像真实 IDE 的密集、紧凑代码块。
- **大写是赢得的**：`uppercase` + `letter-spacing` 处理专门保留用于微型上标线标签和技术标签——从不用于标题。

## 4. 组件样式

### 按钮

**主要 CTA（白色填充）**
- 背景：纯白（`#ffffff`）
- 文字：近黑（`oklch(0.145 0 0)`）
- 内边距：舒适（8px 24px）
- 边框：无
- 圆角：略微圆角（基于标记刻度可能为 4px）
- 悬停：可能微妙透明度降低或轻微灰色变化

**青色强调 CTA**
- 背景：12% 透明度的电子青色（`rgba(0,255,255,0.12)`）
- 文字：近黑（`oklch(0.145 0 0)`）
- 内边距：舒适（8px 24px）
- 边框：细实线海洋蓝（`1px solid rgb(0,150,255)`）
- 圆角：略微圆角（4px）
- 在深色背景上创造"从内部发光"效果

**幽灵/轮廓（信号蓝）**
- 背景：透明
- 文字：近黑（`oklch(0.145 0 0)`）
- 内边距：平衡（10px）
- 边框：细实线信号蓝（`1px solid rgb(0,137,255)`）
- 悬停：可能填充或边框颜色变化

**幽灵/轮廓（木炭）**
- 背景：透明
- 文字：近黑（`oklch(0.145 0 0)`）
- 内边距：平衡（10px）
- 边框：细实线木炭（`1px solid rgb(44,44,44)`）
- 用于深色表面上的次要/三级操作

**幻影按钮**
- 背景：幻影白（`rgba(255,255,255,0.2)`）
- 文字：低语白（`rgba(255,255,255,0.5)`）

- 圆角：几乎方形角（内联元素 2px，内容卡片 4px）

- 悬停行为：可能微妙边框透明度增加或微弱发光效果

## 5. 布局原则

### 间距系统
- 基本单位：8px
- 刻度：1px、2px、4px、6px、8px、10px、12px、14px、16px、18px、20px、24px、30px、32px、40px
- 组件内边距：通常 10px（按钮）至 24px（CTA 按钮水平）
- 章节内边距：宽裕垂直间距（主要章节间估计 80-120px）
- 卡片内部内边距：约 24-32px

### 网格与容器
- 最大容器宽度：约 1200px，居中
- 内容章节使用单列或 2-3 列网格用于特性卡片
- 英雄区：居中单列，最大冲击力
- 特性章节：混合文字块与产品截图的不对称布局

### 空白哲学
- **章节间的呼吸空间**：大型垂直间距创造页面滚动中的不同"章节"。
- **组件内密集**：卡片和文字块内部紧凑（紧凑行高，最小内部内边距），创造专注的信息节点。
- **对比驱动分隔**：Composio 不仅依赖空白，还使用边框透明度差异和细微背景变化来划分内容区域。

### 圆角刻度
- 近方形（2px）：内联代码 span、小标签、pre 块——最锐利处理，传达技术精度
- 略微圆角（4px）：内容卡片、图像、标准容器——主力圆角
- 药丸形（37px）：精选按钮和徽章——为关键 CTA 创造更柔和、更平易近人的感觉
- 完全圆形（9999px+）：圆形元素、头像式容器、装饰性圆点

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 平面（第 0 层） | 无阴影，无边框 | 页面背景，内联文字 |
| 收纳（第 1 层） | 边框薄雾 04-08，无阴影 | 背景分组，细微章节 |
| 卡片（第 2 层） | 边框薄雾 10-12，无阴影 | 标准内容卡片，代码块 |
| 野兽派（第 3 层） | 硬偏移阴影（`4px 4px`，15% 黑色） | 精选交互卡片，独特特性高亮 |
| 浮动（第 4 层） | 柔和漫射阴影（`0px 8px 32px`，50% 黑色） | 模态框，覆盖层，深层提升内容 |

**阴影哲学**：Composio 谨慎且刻意对比地使用阴影。硬偏移野兽派阴影是签名——它以原始的、几乎复古计算感打破流畅的黑暗。柔和漫射阴影保留用于真正的浮动元素。大多数深度通过边框透明度渐变传达，而非阴影。

### 装饰性深度
- **青色发光光晕**：特性卡片和图像后方使用电子青色低透明度的径向渐变光晕。创造"屏幕发光"效果，仿佛 UI 元素正在发射光线。
- **蓝黑渐变洗涤**：从 Composio 钴蓝到虚空黑的线性渐变用作章节背景，添加细微的色温变化。
- **白雾地平线**：从暗到弥散的白色/灰色渐变，在页脚前创造大气"黎明"效果。

## 7. 注意事项

### 应该做
- 使用虚空黑（`#0f0f0f`）作为主要页面背景——不要将纯白用于主要表面
- 保持标题行高超紧凑（0.87-1.0），创造压缩、权威的文字块
- 使用白色透明度边框（4-12%）进行收纳——这里边框比阴影更重要
- 将电子青色（`#00ffff`）专保留用于高信号时刻——CTA、发光、交互强调
- 将 abcDiatype 与 JetBrains Mono 搭配，强化开发者工具身份
- 在精选元素上刻意使用硬偏移阴影（`4px 4px`）增添野兽派个性
- 保持按钮文字深色（`oklch(0.145 0 0)`），即使在最深的背景上——按钮携带自己的表面
- 分层使用基于透明度的边框，创造无需阴影的细微深度
- 仅将大写 + 字间距用于微型上标线标签（12px 或更小）

### 不应该做
- 不要使用明亮背景或浅色表面作为主要容器
- 不要到处应用重阴影——深度来自边框透明度，而非 box-shadow
- 不要将 Composio 钴蓝（`#0007cd`）用作文字颜色——深色背景上太暗，浅色背景上太饱和
- 不要将标题行高增加至 1.2 以上——压缩感是身份的核心
- 不要使用粗体（700）字重用于正文或标题文字——400-500 是上限
- 不要混入暖色——调色板严格限于冷色（蓝色、青色、白色、黑色）
- 不要在内容卡片上使用大于 4px 的圆角——近方形角的精度是刻意的
- 不要在大表面上将电子青色放置在全透明度——它是强调色，背景最多使用 12%
- 不要使用装饰性衬线或手写字体——整个身份是几何无衬线 + 等宽
- 不要跳过技术内容的等宽字体——JetBrains Mono 不是装饰性的，它是可信度信号

| 名称 | 宽度 | 主要变化 |

| 手机 | <768px | 单列，汉堡导航，全宽卡片，减少章节内边距，英雄文字缩小至约 28-40px |
| 平板 | 768-1024px | 卡片两列网格，精简导航，略微缩小英雄文字 |
| 桌面 | 1024-1440px | 完整多列布局，带所有链接可见的展开导航，大型英雄排版（64px） |
| 大桌面 | >1440px | 最大宽度容器居中，宽裕水平边距 |

### 触摸目标
- 所有交互元素最小触摸目标：44x44px
- 按钮使用舒适内边距（最小 8px 24px），确保足够触摸区域
- 导航链接间距足够用于拇指导航

### 折叠策略
- **导航**：桌面完整水平导航折叠为移动端汉堡菜单
- **特性网格**：3 列 → 2 列 → 单列堆叠
- **英雄文字**：64px → 40px → 28px 渐进缩放
- **章节内边距**：等比例减少，但保持宽裕垂直节奏
- **卡片**：移动端垂直堆叠，全宽处理
- **代码块**：较小视口上水平滚动而非换行

### 图像行为
- 产品截图在其容器内等比例缩放
- 深色主题图像在所有尺寸保持在深色背景上的对比度
- 渐变发光效果随容器尺寸缩放
- 断点间无可见艺术指导变化——相同裁剪，等比例缩放
