# MongoDB 设计系统灵感

## 1. 视觉主题与氛围

MongoDB 的网站是一种深林与终端融合的体验——一个植根于最深邃的青黑色（`#001e2b`）的设计系统，既唤起数据库的密度，又唤起森林树冠的深度。在这片近黑色的画布上，一种令人震撼的霓虹绿（`#00ed64`）作为品牌强调色脉动——足够明亮，让人感觉充满电气感，又足够有机，让人感觉充满生命力。这不是赛博朋克的冷霓虹；而是黑暗中生长的东西的生物发光绿。

排版系统雄心勃勃：MongoDB Value Serif 用于大型英雄标题（96px），创造编辑性、权威性的存在感——数据库公司规模的衬线字体是一种大胆选择，表明"我们不只是另一家科技公司"。Euclid Circular A 以异常宽泛的字重范围（300–700）承担正文和 UI 文字的重任，而 Source Code Pro 作为具有独特大写处理的代码和标签字体，采用非常宽的字间距（1px–3px）。这个三字体系统创造了一种跨越编辑优雅 → 几何专业 → 工程精确的层级。

MongoDB 的独特之处在于其双模式设计：深色英雄/功能区域世界（`#001e2b` 配霓虹绿强调）和浅色内容世界（白色配青灰色边框 `#b8c4c2`）。这两种模式之间的过渡创造了戏剧性的对比。阴影系统使用青色着色的深色阴影（`rgba(0, 30, 43, 0.12)`），在浅色表面上也保持森林-深色氛围。按钮使用胶囊形状（100px–999px 圆角），配 MongoDB 绿色边框（`#00684a`），整个组件系统引用 LeafyGreen 设计系统。

**核心特征：**
- 深青黑色背景（`#001e2b`）——森林-深色，而非太空-深色
- 霓虹 MongoDB 绿（`#00ed64`）作为唯一品牌强调——电气且有机
- MongoDB Value Serif 用于英雄标题——科技规模下的编辑权威
- Euclid Circular A 正文，字重 300（细体）作为独特的正文字重
- Source Code Pro 配宽大写字间距（1px–3px），用于技术标签
- 青色着色阴影：`rgba(0, 30, 43, 0.12)` ——阴影承载森林颜色
- 双模式：深色青色英雄区域 + 浅色白色内容区域
- 胶囊按钮（100px 圆角），配绿色边框（`#00684a`）
- 链接蓝（`#006cfa`）和悬停过渡至 `#3860be`

## 2. 色彩系统与角色

### 品牌主色
- **森林黑** (`#001e2b`)：主要深色背景——最深的青黑色
- **MongoDB 绿** (`#00ed64`)：主要品牌强调——霓虹绿，用于高光、下划线、渐变
- **深绿** (`#00684a`)：按钮边框、浅色背景上的链接文字——功能使用的柔和绿色

### 交互色
- **操作蓝** (`#006cfa`)：次级强调——链接、交互高光
- **悬停蓝** (`#3860be`)：所有链接悬停状态过渡至此蓝色
- **青色激活** (`#1eaedb`)：按钮悬停背景——明亮青色

### 中性色系
- **深青** (`#1c2d38`)：深色按钮背景、次级深色表面
- **青灰** (`#3d4f58`)：深色表面上的深色边框
- **深石板** (`#21313c`)：深色链接文字变体
- **冷灰** (`#5c6c75`)：深色背景上的弱化文字、次级按钮文字
- **银青** (`#b8c4c2`)：浅色表面上的边框、分隔线
- **浅输入** (`#e8edeb`)：深色表面上的输入框文字
- **纯白** (`#ffffff`)：浅色区域背景、深色背景上的按钮文字
- **黑色** (`#000000`)：浅色表面上的文字、最深的元素

### 阴影
- **森林阴影** (`rgba(0, 30, 43, 0.12) 0px 26px 44px, rgba(0, 0, 0, 0.13) 0px 7px 13px`)：主要卡片层次——青色着色
- **标准阴影** (`rgba(0, 0, 0, 0.15) 0px 3px 20px`)：一般层次
- **细微阴影** (`rgba(0, 0, 0, 0.1) 0px 2px 4px`)：轻量级卡片提升

### 原则
- **衬线作为权威**：英雄规模的 MongoDB Value Serif 在科技领域创造了不寻常的编辑存在感——它传达了 MongoDB 是一个机构，而非初创公司。
- **字重 300 作为正文默认**：Euclid Circular A 使用细体（300）用于正文，创造与密集深色背景形成对比的轻盈阅读体验。
- **宽字间等宽标签**：大写的 Source Code Pro 配 1px–3px 字间距创造出感觉像数据库字段标签的技术标识——系统化、结构化、分类化。
- **四字重范围**：300（细体正文）→ 400（标准）→ 500（UI/导航）→ 700（粗体行动按钮）——比大多数系统更宽的范围，支持精细的层级。

## 4. 组件样式

### 按钮

**主要绿色（深色表面）**
- 背景：`#00684a`（柔和 MongoDB 绿）
- 文字：`#000000`
- 圆角：50%（圆形）或 100px（胶囊）
- 边框：`1px solid #00684a`
- 阴影：`rgba(0,0,0,0.06) 0px 1px 6px`
- 悬停：scale 1.1
- 激活：scale 0.85

**深青色按钮**
- 背景：`#1c2d38`
- 文字：`#5c6c75`
- 圆角：100px（胶囊）
- 边框：`1px solid #3d4f58`
- 悬停：背景 `#1eaedb`，文字白色，translateX(5px)

**轮廓按钮（浅色表面）**
- 背景：透明
- 文字：`#001e2b`
- 边框：`1px solid #b8c4c2`
- 圆角：4px–8px
- 悬停：背景着色

### 卡片与容器
- 浅色模式：白色背景，`1px solid #b8c4c2` 边框
- 深色模式：`#001e2b` 或 `#1c2d38` 背景，`1px solid #3d4f58`
- 圆角：16px（标准）、24px（中等）、48px（大型/英雄）
- 阴影：`rgba(0,30,43,0.12) 0px 26px 44px`（森林着色）
- 图像容器：30px–32px 圆角

### 输入框与表单
- 文本区域：文字 `#e8edeb`，内边距 12px 12px 12px 8px
- 边框：浅色 `1px solid #b8c4c2`，深色 `1px solid #3d4f58`
- 输入框圆角：4px

### 导航
- 森林黑背景上的深色头部
- Euclid Circular A 16px 字重 500 用于导航链接
- MongoDB 标志（叶子图标 + 文字商标）左对齐
- 右对齐绿色行动按钮胶囊
- 带产品类别的超大菜单下拉

### 图像处理
- 深色背景上的仪表板截图
- 截图中绿色强调的 UI 元素
- 图像容器 30px–32px 圆角
- 产品展示的全宽深色区域

### 特色组件

**霓虹绿强调下划线**
- `0px 2px 2px 0px solid #00ed64` ——底部 + 右侧边框创造强调下划线
- 用于功能标题和高亮文字

**Source Code 标签系统**

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 尺度：1px、4px、7px、8px、10px、12px、14px、15px、16px、18px、20px、24px、32px

### 网格与容器
- 内容最大宽度居中
- 带内容约束的深色英雄区域
- 下方浅色内容区域
- 卡片网格：2–3 列
- 全宽深色页脚

### 空白哲学
- **戏剧性模式过渡**：从深色青色区域到白色内容的转变通过对比而非仅仅空间创造了内置的视觉呼吸感。
- **宽松深色区域**：深色英雄和功能区域使用额外的垂直内边距（80px+），让森林-深色背景呼吸。
- **紧凑浅色区域**：白色内容区域更密集，卡片网格更紧，垂直间距更少。

### 圆角尺度
- 极小（1px–2px）：小型 span、徽章
- 细腻（4px）：输入框、小按钮
- 标准（8px）：卡片、链接
- 卡片（16px）：标准卡片、容器
- 切换（20px）：开关元素
- 大型（24px）：大型面板
- 图像（30px–32px）：图像容器
- 英雄（48px）：英雄卡片
- 胶囊（100px–999px）：按钮、导航胶囊
- 完整（9999px）：最大胶囊

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 平面（0 级） | 无阴影 | 默认表面 |
| 细腻（1 级） | `rgba(0,0,0,0.1) 0px 2px 4px` | 轻量级卡片提升 |
| 标准（2 级） | `rgba(0,0,0,0.15) 0px 3px 9px` | 标准卡片 |
| 显著（3 级） | `rgba(0,0,0,0.15) 0px 3px 20px` | 抬高面板 |
| 森林（4 级） | `rgba(0,30,43,0.12) 0px 26px 44px, rgba(0,0,0,0.13) 0px 7px 13px` | 英雄卡片——青色着色 |

**阴影哲学**：MongoDB 的阴影系统是独特的，主要层次阴影使用 `rgba(0, 30, 43, 0.12)` ——一种将森林-深色品牌色带入深度系统的青色着色阴影。这意味着即使在白色表面上，阴影也感觉属于 MongoDB 色彩世界，而非通用中性黑色。

## 7. 规范与禁忌

### 应该做
- 深色区域使用 `#001e2b`（森林黑）——不使用纯黑
- 稀疏使用 MongoDB 绿（`#00ed64`）以实现最大电气冲击
- MongoDB Value Serif 仅用于英雄/展示标题——其他所有用 Euclid Circular A
- 技术标签应用 Source Code Pro 大写配宽字间距（1px–3px）
- 使用青色着色阴影（`rgba(0,30,43,0.12)`）用于主要卡片层次
- 保持深色/浅色区域的双重性——模式之间的戏剧性对比
- 正文使用字重 300——细字重是可读的声音
- 主要操作按钮应用胶囊圆角（100px）

### 不应该做
- 不要使用纯黑（`#000000`）作为深色背景——始终使用青黑色（`#001e2b`）
- 不要将 MongoDB 绿（`#00ed64`）用于背景——它是文字、下划线和小高光的强调色
- 不要使用标准灰色阴影——始终使用青色着色（`rgba(0,30,43,...)`）
- 不要将衬线字体用于正文——MongoDB Value Serif 仅用于英雄
- 不要在 Source Code Pro 标签上使用窄字间距——宽字间距就是身份
- 不要在同一区域内混用深色和浅色区域处理
- 不要使用暖色——调色板严格是冷色（青色、绿色、蓝色）
- 不要忘记绿色强调下划线——它们是标志性的装饰元素

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|------|----------|
| 小移动端 | <425px | 紧凑单列 |
| 移动端 | 425–768px | 标准移动端 |
| 平板 | 768–1024px | 开始双列网格 |
| 桌面 | 1024–1280px | 标准布局 |
| 大桌面 | 1280–1440px | 扩展布局 |
| 超宽 | >1440px | 最大宽度，宽松边距 |

### 触控目标
- 胶囊按钮，带宽松内边距
- 导航链接 16px，间距足够
- 卡片表面作为全区域触控目标

### 折叠策略
- 英雄区：MongoDB Value Serif 96px → 64px → 进一步缩放
- 导航：水平超大菜单 → 汉堡菜单
- 功能卡片：多列 → 堆叠
- 深色/浅色区域在所有尺寸保持其模式
- Source Code Pro 标签保持大写处理

### 图像行为
- 仪表板截图等比缩放
- 深色区域背景保持全宽
- 图像圆角在断点间保持
