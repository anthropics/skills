# Sentry 设计系统灵感

## 1. 视觉主题与氛围

Sentry 的网站是一个以深色模式为优先的开发者工具界面，语言风格源自代码编辑器和终端窗口。整体美感根植于深紫黑色背景（`#1f1633`、`#150f23`），唤起了 Sentry 为之而生的深夜调试时光。在这片墨色画布上，精心挑选的紫色、粉色以及标志性的荧光绿色点缀（`#c2ef4e`），共同构成了一套兼具技术感与活力的视觉体系。

字体搭配经过刻意设计："Dammit Sans" 在主视觉区以超大尺寸（88px，字重 700）出场，作为展示字体，其个性与态度完美契合 Sentry 不羁的品牌声音（"代码会崩，修得更快。"）；Rubik 则作为主力 UI 字体，覆盖所有功能性文字——标题、正文、按钮、说明文字和导航。Monaco 提供了代码片段和技术内容所需的等宽字体层，完成了开发者工具三位一体的字体组合。

Sentry 的独特之处在于，它拥抱"暗色 IDE"的美学却不显冷漠或死板。温暖的紫色调取代了开发者工具常见的冷灰色，大胆的插画元素（3D 角色、色彩丰富的产品截图）点缀于暗色画布之上。按钮系统采用标志性的哑光紫（`#79628c`）配合内嵌阴影，创造出一种触感十足的立体质感——仿佛按钮真的可以被按入界面表面。

**核心特征：**
- 深紫黑色背景（`#1f1633`、`#150f23`）——绝非纯黑
- 温暖紫色调系谱：从深邃（`#362d59`）到中间调（`#79628c`、`#6a5fc1`），再到鲜艳（`#422082`）
- 荧光绿点缀色（`#c2ef4e`）用于高可见度的行动召唤按钮和高亮元素
- 粉色/珊瑚色点缀（`#ffb287`、`#fa7faa`）用于焦点状态和次要高亮
- "Dammit Sans" 展示字体，彰显品牌个性，仅用于主视觉区大标题
- Rubik 作为主要 UI 字体，配合全大写字母间距标签
- Monaco 等宽字体用于代码元素
- 按钮内嵌阴影营造触觉深度
- 毛玻璃效果，使用 `blur(18px) saturate(180%)`

## 2. 色彩方案与角色

### 主品牌色
- **深紫** (`#1f1633`)：主背景色，品牌的定义色
- **更深紫** (`#150f23`)：深色区段、页脚、次要背景
- **边框紫** (`#362d59`)：边框、分割线、微妙的结构线条

### 点缀色
- **Sentry 紫** (`#6a5fc1`)：主要交互色——链接、悬停状态、焦点环
- **哑光紫** (`#79628c`)：按钮背景、次要交互元素
- **深紫罗兰** (`#422082`)：下拉选择框、激活状态、高强调曲面
- **荧光绿** (`#c2ef4e`)：高可见度点缀，特殊链接，徽章高亮
- **珊瑚色** (`#ffb287`)：焦点状态背景，暖色点缀
- **粉色** (`#fa7faa`)：焦点轮廓，装饰性点缀

### 文字颜色
- **纯白** (`#ffffff`)：深色背景上的主要文字
- **浅灰** (`#e5e7eb`)：次要文字，弱化内容
- **代码黄** (`#dcdcaa`)：语法高亮，代码符号

### 界面曲面与叠加层
- **毛玻璃白** (`rgba(255, 255, 255, 0.18)`)：毛玻璃按钮背景
- **毛玻璃深色** (`rgba(54, 22, 107, 0.14)`)：毛玻璃元素上的悬停叠加层
- **输入框白** (`#ffffff`)：表单输入框背景（浅色上下文）
- **输入框边框** (`#cfcfdb`)：表单字段边框

### 阴影
- **环境光晕** (`rgba(22, 15, 36, 0.9) 0px 4px 4px 9px`)：深紫色环境阴影
- **按钮悬停** (`rgba(0, 0, 0, 0.18) 0px 0.5rem 1.5rem`)：悬停时的抬升状态
- **卡片阴影** (`rgba(0, 0, 0, 0.1) 0px 10px 15px -3px`)：标准卡片抬升
- **内嵌按钮** (`rgba(0, 0, 0, 0.1) 0px 1px 3px 0px inset`)：触觉按压效果

## 3. 排版规则

### 字体系列
- **展示字体**：`Dammit Sans` — 品牌个性字体，用于主视觉大标题
- **主 UI 字体**：`Rubik`，备用字体：`-apple-system, system-ui, Segoe UI, Helvetica, Arial`
- **等宽字体**：`Monaco`，备用字体：`Menlo, Ubuntu Mono`

### 层级体系

| 角色 | 字体 | 尺寸 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|--------|-------------|----------------|-------|
| 主视觉展示 | Dammit Sans | 88px (5.50rem) | 700 | 1.20（紧凑） | 正常 | 最大冲击力，品牌声音 |
| 次要展示 | Dammit Sans | 60px (3.75rem) | 500 | 1.10（紧凑） | 正常 | 次要主视觉文字 |
| 区块标题 | Rubik | 30px (1.88rem) | 400 | 1.20（紧凑） | 正常 | 主要区块标题 |
| 副标题 | Rubik | 27px (1.69rem) | 500 | 1.25（紧凑） | 正常 | 功能区块标题 |
| 卡片标题 | Rubik | 24px (1.50rem) | 500 | 1.25（紧凑） | 正常 | 卡片和块状标题 |
| 功能标题 | Rubik | 20px (1.25rem) | 600 | 1.25（紧凑） | 正常 | 强调功能名称 |
| 正文 | Rubik | 16px (1.00rem) | 400 | 1.50 | 正常 | 标准正文文字 |
| 正文强调 | Rubik | 16px (1.00rem) | 500–600 | 1.50 | 正常 | 加粗正文，导航项 |
| 导航标签 | Rubik | 15px (0.94rem) | 500 | 1.40 | 正常 | 导航链接 |
| 全大写标签 | Rubik | 15px (0.94rem) | 500 | 1.25（紧凑） | 正常 | `text-transform: uppercase` |
| 按钮文字 | Rubik | 14px (0.88rem) | 500–700 | 1.14–1.29（紧凑） | 0.2px | `text-transform: uppercase` |
| 说明文字 | Rubik | 14px (0.88rem) | 500–700 | 1.00–1.43 | 0.2px | 通常全大写 |
| 小说明文字 | Rubik | 12px (0.75rem) | 600 | 2.00（宽松） | 正常 | 细微注释 |
| 微型标签 | Rubik | 10px (0.63rem) | 600 | 1.80（宽松） | 0.25px | `text-transform: uppercase` |
| 代码 | Monaco | 16px (1.00rem) | 400–700 | 1.50 | 正常 | 代码块，技术文字 |

### 排版原则
- **双重个性**：Dammit Sans 在展示尺寸带来不羁的品牌气质；Rubik 为所有功能性内容提供简洁的专业感。
- **全大写作为系统**：按钮、说明文字、标签和微型文字均使用 `text-transform: uppercase`，配合微妙的字间距（0.2px–0.25px），在整个系统中形成"技术标签"的统一模式。
- **字重分层**：Rubik 使用 400（正文）、500（强调/导航）、600（标题/加强）、700（按钮/行动召唤）——干净的四级字重体系。
- **标题紧凑，正文宽松**：所有标题使用 1.10–1.25 行高；正文使用 1.50；小说明文字扩展至 2.00，以提高极小尺寸的可读性。

## 4. 组件样式

### 按钮

**主要哑光紫**
- 背景：`#79628c`（rgb(121, 98, 140)）
- 文字：`#ffffff`，全大写，14px，字重 500–700，字间距 0.2px
- 边框：`1px solid #584674`
- 圆角：13px
- 阴影：`rgba(0, 0, 0, 0.1) 0px 1px 3px 0px inset`（触觉内嵌效果）
- 悬停：抬升阴影 `rgba(0, 0, 0, 0.18) 0px 0.5rem 1.5rem`

**毛玻璃白**
- 背景：`rgba(255, 255, 255, 0.18)`（毛玻璃效果）
- 文字：`#ffffff`
- 内边距：8px
- 圆角：12px（左对齐变体：`12px 0px 0px 12px`）
- 阴影：`rgba(0, 0, 0, 0.08) 0px 2px 8px`
- 悬停背景：`rgba(54, 22, 107, 0.14)`
- 用途：深色界面上的次要操作

**白色实心**
- 背景：`#ffffff`
- 文字：`#1f1633`
- 内边距：12px 16px
- 圆角：8px
- 悬停：背景过渡至 `#6a5fc1`，文字变为白色
- 焦点：背景 `#ffb287`（珊瑚色），轮廓 `rgb(106, 95, 193) solid 0.125rem`
- 用途：深色背景上的高可见度行动召唤按钮

**深紫罗兰（下拉/选择框）**
- 背景：`#422082`
- 文字：`#ffffff`
- 内边距：8px 16px
- 圆角：8px

### 输入框

**文本输入框**
- 背景：`#ffffff`
- 文字：`#1f1633`
- 边框：`1px solid #cfcfdb`
- 内边距：8px 12px
- 圆角：6px
- 焦点：边框色保持 `#cfcfdb`，阴影 `rgba(0, 0, 0, 0.15) 0px 2px 10px inset`

### 链接
- **深色背景上的默认链接**：`#ffffff`，带下划线装饰
- **悬停**：颜色过渡至 `#6a5fc1`（Sentry 紫）
- **紫色链接**：默认 `#6a5fc1`，悬停时显示下划线
- **荧光绿点缀链接**：默认 `#c2ef4e`，悬停至 `#6a5fc1`
- **深色上下文链接**：`#362d59`，悬停至 `#ffffff`

### 卡片与容器
- 背景：半透明或深紫色界面
- 圆角：8px–12px
- 阴影：`rgba(0, 0, 0, 0.1) 0px 10px 15px -3px`
- 背景滤镜：`blur(18px) saturate(180%)` 用于毛玻璃效果

### 导航
- 深色透明顶部导航，覆盖在主视觉内容之上
- Rubik 15px 字重 500 用于导航链接
- 白色文字，悬停至 Sentry 紫（`#6a5fc1`）
- 分类标签全大写，字间距 0.2px
- 移动端：汉堡菜单，展开时全宽

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 比例：1px、2px、4px、5px、6px、8px、12px、16px、24px、32px、40px、44px、45px、47px

### 栅格与容器
- 内容最大宽度：1152px（XL 断点）
- 响应式内边距：2rem（移动端）→ 4rem（平板及以上）
- 内容居中于容器内
- 全宽深色区段，内部内容受限于容器

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 移动端 | < 576px | 单列，堆叠布局 |
| 小平板 | 576–640px | 轻微宽度调整 |
| 平板 | 640–768px | 开始双列布局 |
| 小桌面 | 768–992px | 完整导航可见 |
| 桌面 | 992–1152px | 标准布局 |
| 大桌面 | 1152–1440px | 内容最大宽度 |

### 留白哲学
- **深色呼吸空间**：区块之间慷慨的垂直间距（64px–80px+），让深色背景起到视觉休息的作用。
- **内容岛屿**：功能区块是自成一体的模块，漂浮在深紫色海洋中，每个模块有其自身的内部间距节奏。
- **非对称内边距**：按钮使用非对称内边距模式（12px 16px、8px 12px），感觉有机而非死板。

### 圆角比例
- 极小（6px）：表单输入框，小型交互元素
- 标准（8px）：按钮、卡片、容器
- 舒适（10px–12px）：较大容器、毛玻璃面板
- 圆润（13px）：主要哑光按钮
- 胶囊形（18px）：图片容器、徽章

## 6. 深度与层级

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 下沉（层级 -1） | 内嵌阴影 `rgba(0, 0, 0, 0.1) 0px 1px 3px inset` | 主要按钮（触觉按压感） |
| 平面（层级 0） | 无阴影 | 默认界面，深色背景 |
| 界面（层级 1） | `rgba(0, 0, 0, 0.08) 0px 2px 8px` | 毛玻璃按钮，微妙卡片 |
| 抬升（层级 2） | `rgba(0, 0, 0, 0.1) 0px 10px 15px -3px` | 卡片，浮动面板 |
| 突出（层级 3） | `rgba(0, 0, 0, 0.18) 0px 0.5rem 1.5rem` | 悬停状态，模态框 |
| 环境光（层级 4） | `rgba(22, 15, 36, 0.9) 0px 4px 4px 9px` | 主视觉区周围的深紫色环境光晕 |

**阴影哲学**：Sentry 使用内嵌阴影（按钮感觉被按入界面表面）与环境光晕（内容从深色背景中辐射出来）的独特组合。深紫色环境阴影（`rgba(22, 15, 36, 0.9)`）是标志性特征——它创造出一种生物发光的质感，仿佛内容自身发出带有紫色调的光芒。

## 7. 应做与禁止

### 应做
- 使用深紫黑色背景（`#1f1633`、`#150f23`）——绝不使用纯黑（`#000000`）
- 在主要按钮上应用内嵌阴影，营造触觉按压效果
- Dammit Sans 仅用于主视觉/展示标题——其余所有内容使用 Rubik
- 在按钮和标签上应用 `text-transform: uppercase` 配合 `letter-spacing: 0.2px`
- 谨慎使用荧光绿点缀（`#c2ef4e`），以获得最大冲击效果
- 在分层界面上使用毛玻璃效果（`blur(18px) saturate(180%)`）
- 保持温暖紫色的阴影色调——阴影应呈紫色调，而非中性灰
- 使用 Rubik 的四级字重体系：400（正文）、500（导航/强调）、600（标题）、700（行动召唤）

### 禁止
- 不要使用纯黑（`#000000`）作为背景——始终使用温暖的紫黑色
- 不要将 Dammit Sans 应用于正文或 UI 元素——它仅限展示用途
- 不要使用标准灰色（`#666`、`#999`）作为边框——使用带紫色调的灰色（`#362d59`、`#584674`）
- 不要省略按钮的全大写处理——这是系统级的统一模式
- 不要使用直角（0px 圆角）——所有交互元素最小 6px
- 不要在同一组件中混用荧光绿点缀与珊瑚色/粉色点缀
- 不要在主要按钮上使用平面（非内嵌）阴影——触觉质感是标志性特征
- 不要忘记全大写文字上的字间距——最小 0.2px

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 移动端 | <576px | 单列，汉堡导航，堆叠行动召唤按钮 |
| 平板 | 576–768px | 双列功能栅格开始 |
| 小桌面 | 768–992px | 完整导航，并排布局 |
| 桌面 | 992–1152px | 最大宽度容器，完整布局 |
| 大屏 | >1152px | 内容最大宽度保持，慷慨边距 |

### 折叠策略
- 主视觉文字：88px Dammit Sans → 60px → 移动端缩放
- 导航：水平 → 汉堡菜单，滑出式展开
- 功能区块：并排 → 堆叠卡片
- 按钮：内联 → 移动端全宽堆叠
- 容器内边距：4rem → 2rem
