# Expo 设计系统灵感

## 1. 视觉主题与氛围

Expo 的界面是一个明亮、充满自信的开发者平台，其建立前提是：用于构建应用的工具本身应该像所构建的应用一样精致。整体体验建立在一块明亮通透的画布之上——一种带冷色调的灰白色（`#f0f0f3`），为页面注入了微妙的科技感冷静，又没有纯白色的刺眼。这是一个会呼吸的网站：区域之间巨大的垂直间距创造出一种画廊般的节奏，每个功能都拥有自己的"展室"。

设计语言果断地走向单色——最浅背景上纯黑色（`#000000`）的标题，以及一系列冷调蓝灰色（`#60646c`、`#b0b4ba`、`#555860`）处理所有次要信息传达。界面本身几乎不存在彩色；当色彩出现时，它被专门留给产品截图、应用图标和 React 宇宙插图——使实际内容在中性画布上显得格外生动。

Expo 的独特之处在于其胶囊形几何语言。按钮、标签、视频容器乃至图像都使用圆润或完全胶囊形的圆角（24px–9999px），营造出一种有机、亲切的感觉，与典型的锐边开发者工具美学截然相反。结合大型标题上的极端负字间距（64px 时 -1.6px 到 -3px），其结果是一个同时具有高端感和亲切感的设计——就像专为开发者重新想象的 Apple 产品页面。

**核心特征：**
- 明亮的冷调灰白色画布（`#f0f0f3`），画廊般的垂直间距
- 严格单色：纯黑标题、冷调蓝灰正文、无装饰性色彩
- 胶囊形几何无处不在——按钮、标签、容器、图像（24px–9999px 圆角）
- 巨大展示标题（64px），带极端负字间距（-1.6px 到 -3px）
- Inter 作为唯一字体，字重 400–900 覆盖完整表达范围
- 轻若耳语的阴影，仅将元素从表面轻微抬起
- 产品截图作为界面中唯一的色彩来源

## 2. 色彩系统与角色

### 主色
- **Expo 黑** (`#000000`)：绝对锚点——用于主要标题、行动按钮和品牌标识。冷调白色上的纯黑创造最大对比度而不显得攻击性。
- **近黑** (`#1c2024`)：正文内容的主要文字颜色——一种几乎难以察觉的蓝黑色，比纯 `#000` 更柔和，适合长篇阅读。

### 次级与强调
- **链接钴蓝** (`#0d74ce`)：标准链接颜色——一种可信、饱和的蓝色，传递交互性而不与单色层级竞争。
- **法律蓝** (`#476cff`)：用于法律/页脚链接的更亮、更饱和的蓝色——比链接钴蓝稍更吸引注意力。
- **组件天蓝** (`#47c2ff`)：用于组件品牌元素的浅色、友好的青蓝色——系统中最亮的强调色。
- **预览紫** (`#8145b5`)：用于"预览"或测试版功能指示器的丰富紫罗兰色——与标准内容形成清晰的视觉区分。

### 表面与背景
- **云灰** (`#f0f0f3`)：主要页面背景——带有最淡蓝紫色调的冷调灰白色。不温暖，不生硬——恰到好处的科技感。
- **纯白** (`#ffffff`)：卡片表面、按钮背景和抬升的内容容器。在云灰背景上创造清晰的"悬浮"区分。
- **组件深色** (`#1a1a1a`)：深色主题组件和遮罩元素的深色表面。
- **横幅深色** (`#171717`)：最深的表面变体，用于促销横幅和高对比度容器。

### 中性色与文字
- **石板灰** (`#60646c`)：主力次要文字颜色（305 处实例）。一种冷调蓝灰色，权威却不沉重。
- **中石板** (`#555860`)：略深于石板灰，用于强调的次要文字。
- **银色** (`#b0b4ba`)：三级文字、占位符和弱化的元数据。清晰可读但明显后退。
- **铅灰** (`#999999`)：深色环境中的折叠图标和深度弱化的 UI 元素。
- **浅银** (`#cccccc`)：深色环境中的箭头图标和装饰元素。
- **深石板** (`#363a3f`)：深色表面上的边框、开关滑轨和强调容器。
- **炭灰** (`#333333`)：深色模式开关背景和深色次要表面。

### 语义与强调
- **警告琥珀** (`#ab6400`)：用于警告状态的温暖、深沉琥珀色——刻意避免明亮黄色，传递严肃感。
- **危险玫瑰** (`#eb8e90`)：用于禁用的破坏性操作的柔和粉珊瑚色——比典型红色更温和，减少警觉疲劳。
- **边框薰衣草** (`#e0e1e6`)：标准卡片/容器边框——一种冷调薰衣草灰，可见但不沉重。
- **输入框边框** (`#d9d9e0`)：按钮和表单元素边框——比卡片边框略暖/略深，用于交互元素。
- **深色焦点环** (`#2547d0`)：深色主题环境中键盘焦点指示器的深蓝色。

### 渐变系统
- 设计在界面层面**显著无渐变**。视觉丰富度来自产品截图、React 宇宙插图和精心的阴影层叠，而非色彩渐变。这种缺失本身就是设计决策——渐变会破坏临床般的精准感。

## 3. 字体规则

### 字体系列
- **主要**：`Inter`，备用字体：`-apple-system, system-ui`
- **等宽**：`JetBrains Mono`，备用字体：`ui-monospace`
- **系统备用**：`system-ui, Segoe UI, Roboto, Helvetica, Arial, Apple Color Emoji, Segoe UI Emoji`

### 层级

| 角色 | 字体 | 尺寸 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|------|------|--------|------|
| 展示 / 英雄 | Inter | 64px (4rem) | 700–900 | 1.10（紧凑） | -1.6px 到 -3px | 最大冲击力，极端字间距 |
| 区域标题 | Inter | 48px (3rem) | 600 | 1.10（紧凑） | -2px | 功能区域锚点 |
| 副标题 | Inter | 20px (1.25rem) | 600 | 1.20（紧凑） | -0.25px | 卡片标题、功能名称 |
| 大正文 | Inter | 18px (1.13rem) | 400–500 | 1.40 | normal | 引言段落、区域描述 |
| 正文 / 按钮 | Inter | 16px (1rem) | 400–700 | 1.25–1.40 | normal | 标准文字、导航链接、按钮 |
| 说明 / 标签 | Inter | 14px (0.88rem) | 400–600 | 1.00–1.40 | normal | 描述、元数据、徽章文字 |
| 标签 / 小字 | Inter | 12px (0.75rem) | 500 | 1.00–1.60 | normal | 最小无衬线文字、徽章 |
| 代码正文 | JetBrains Mono | 16px (1rem) | 400–600 | 1.40 | normal | 行内代码、终端命令 |
| 代码说明 | JetBrains Mono | 14px (0.88rem) | 400–600 | 1.40 | normal | 代码片段、技术标签 |
| 代码小字 | JetBrains Mono | 12px (0.75rem) | 400 | 1.60 | normal | 大写技术标签 |

### 原则
- **单一字体，完整表达**：Inter 是唯一的无衬线字体，字重从 400（常规）到 900（超黑）全范围使用。这赋予设计统一的声音，同时在轻柔正文和震撼展示标题之间实现了戏剧性对比。
- **大尺寸极端负字间距**：64px 标题使用 -1.6px 到 -3px 字间距，创造出感觉像标志字体的超密集文字块。这种激进的压缩是标志性的排版动作。
- **字重即层级**：展示级 700–900，标题级 600，强调级 500，正文级 400。级差是决定性的——没有模糊的中间字重。
- **一致的 1.40 正文行高**：几乎所有正文和 UI 文字共享 1.40 行高，创造有节奏的垂直一致性。

## 4. 组件样式

### 按钮

**主要（带边框白色）**
- 背景：纯白（`#ffffff`）
- 文字：近黑（`#1c2024`）
- 内边距：0px 12px（紧凑，高度由行高决定）
- 边框：细线 输入框边框（`1px solid #d9d9e0`）
- 圆角：轻微圆角（6px）
- 阴影：悬停时细微组合阴影
- 低调默认款——干净、专业、不张扬

**主要胶囊**
- 与主要按钮相同，但使用胶囊形圆角（9999px）
- 用于英雄行动按钮和高强调操作
- 额外的圆润感传递"从这里开始"

**深色主要**
- 背景：Expo 黑（`#000000`）
- 文字：纯白（`#ffffff`）
- 胶囊形（9999px）或大圆角（32–36px）
- 无边框（黑色即边框）
- 最高强调的行动按钮——专为主要转化操作保留

### 卡片与容器
- 背景：纯白（`#ffffff`）——从云灰页面清晰抬升
- 边框：细线 边框薰衣草（`1px solid #e0e1e6`）用于标准卡片
- 圆角：舒适圆角（8px）用于标准卡片；大圆角（16–24px）用于特色容器
- 阴影一级：耳语（`rgba(0,0,0,0.08) 0px 3px 6px, rgba(0,0,0,0.07) 0px 2px 4px`）——几乎难以察觉的抬升
- 阴影二级：标准（`rgba(0,0,0,0.1) 0px 10px 20px, rgba(0,0,0,0.05) 0px 3px 6px`）——清晰的悬浮层次
- 悬停：可能有细微的阴影加深或背景变化

### 输入框与表单
- 背景：纯白（`#ffffff`）
- 文字：近黑（`#1c2024`）
- 边框：细线 输入框边框（`1px solid #d9d9e0`）
- 内边距：0px 12px（与按钮尺寸对齐）
- 圆角：轻微圆角（6px）
- 焦点：通过 CSS 自定义属性实现的蓝色环形阴影

### 导航
- 透明/模糊背景上的固定顶部导航
- 标志：Expo 文字商标，黑色
- 链接：近黑（`#1c2024`）或石板灰（`#60646c`），14–16px Inter 字重 500
- 行动按钮：右侧黑色胶囊按钮（"注册"）
- GitHub 星标徽章作为社会证明
- 状态指示器（"所有系统运行正常"），带绿色圆点

### 图像处理
- 产品截图和设备模型是视觉主角
- 视频和图像容器使用大圆角（24px）
- 截图展示在真实设备外框中
- 深色 UI 截图在浅色画布上提供对比
- 在圆角容器内全出血

### 特色组件

**React 宇宙标志**
- 动态/插图形式的 React 标志作为视觉中心
- 将 Expo 的身份与 React 生态系统相连
- 否则以摄影为主的页面上唯一的插图元素

**设备预览网格**
- 同时展示多种设备类型（手机、平板、网页）
- 直观展示跨平台能力
- 每台设备使用真实的设备外框

**状态徽章**
- 导航中的"所有系统运行正常"胶囊
- 绿色圆点 + 文字——紧凑的信任信号
- 胶囊形（36px 圆角）

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 尺度：1px、2px、4px、8px、12px、16px、24px、32px、40px、48px、64px、80px、96px、144px
- 按钮内边距：0px 12px（异常紧凑——高度由行高驱动）
- 卡片内部内边距：约 24–32px
- 区域垂直间距：巨大（主要区域间估计 96–144px）
- 组件间距：兄弟元素之间 16–24px

### 网格与容器
- 最大容器宽度：约 1200–1400px，居中
- 英雄区：居中单列，带大量留白
- 功能区域：交替布局（图像左右、全宽展示）
- 卡片网格：2–3 列用于功能亮点
- 全宽区域，内容受约束

### 空白哲学
- **画廊般的节奏**：每个区域都像独立的展品，被大量空白包围。这创造了一种高端、从容的浏览体验。
- **留白即设计**：慷慨的空白本身就是主要设计元素——它传递着自信、品质，以及每个功能都值得单独关注的态度。
- **内容岛屿**：各区域作为孤立的"岛屿"漂浮在白色空间中，通过滚动而非视觉连续性联系在一起。

### 圆角尺度
- 近方形（4px）：小型行内元素、标签
- 轻微圆角（6px）：按钮、表单输入框、组合框——功能性交互圆角
- 舒适圆角（8px）：标准内容卡片、容器
- 大圆角（16px）：功能标签页、内容面板
- 很圆（24px）：按钮、视频/图像容器、标签面板——标志性柔和感
- 高度圆角（32–36px）：英雄行动按钮、状态徽章、导航按钮
- 胶囊形（9999px）：主要操作按钮、标签、头像——最大亲切感

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 平面（0 级） | 无阴影 | 云灰页面背景、行内文字 |
| 表面（1 级） | 白色背景，无阴影 | 云灰上的标准白色卡片 |
| 耳语（2 级） | `rgba(0,0,0,0.08) 0px 3px 6px` + `rgba(0,0,0,0.07) 0px 2px 4px` | 细微卡片抬升、悬停状态 |
| 抬升（3 级） | `rgba(0,0,0,0.1) 0px 10px 20px` + `rgba(0,0,0,0.05) 0px 3px 6px` | 功能展示、产品截图 |
| 模态（4 级） | 深色遮罩（`--dialog-overlay-background-color`）+ 重阴影 | 对话框、遮罩层 |

**阴影哲学**：Expo 将阴影用作轻柔的耳语而非建筑性声明。主要深度机制是**背景颜色对比**——白色卡片漂浮在云灰之上——而非阴影投射。当阴影出现时，它们柔和、扩散、方向向下，创造出纸张悬停在桌面几毫米之上的感觉。

## 7. 规范与禁忌

### 应该做
- 使用云灰（`#f0f0f3`）作为页面背景，纯白（`#ffffff`）用于抬升的卡片——双色调浅色系统至关重要
- 展示标题保持极端负字间距（64px 时 -1.6px 到 -3px），实现标志性的压缩外观
- 主要行动按钮使用胶囊形（9999px）圆角——有机形状是身份的核心
- 将黑色（`#000000`）保留给标题和主要行动按钮——它在浅色画布上承载最大权威
- 次要文字使用石板灰（`#60646c`）——它在可读与后退之间取得精确平衡
- 区域之间保持巨大的垂直间距（96px+）——画廊节奏定义了高端感
- 使用产品截图作为主要视觉内容——界面保持单色，产品带来色彩
- Inter 使用全字重范围（400–900）——字重对比就是层级

### 不应该做
- 不要在界面外框中引入装饰性色彩——单色调色板是有意为之的
- 不要在交互元素上使用锐利边角（圆角 < 6px）——胶囊/圆角几何是标志
- 不要将区域间距缩小至 64px 以下——留白就是设计
- 不要使用厚重的投影阴影——深度来自背景对比和轻柔阴影
- 不要引入额外字体——Inter 处理从展示到说明的一切
- 不要在正文上使用比 -0.25px 更宽的字间距——极端字间距仅用于展示级
- 不要使用超过 2px 的边框——包容是细腻的，通过背景颜色和轻柔边框实现
- 不要在界面中添加渐变——视觉丰富度来自内容，而非装饰
- 不要在语义上下文之外使用饱和色彩——调色板严格是灰度加功能性蓝色

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|------|----------|
| 移动端 | <640px | 单列，汉堡导航，堆叠卡片，英雄文字缩小至约 36px |
| 平板 | 640–1024px | 双列网格，简化导航，中等英雄文字 |
| 桌面 | >1024px | 完整多列布局，展开导航，巨大英雄（64px） |

*仅检测到一个明确断点（640px），表明采用流式、容器查询或基于 min()/clamp() 的响应式系统，而非固定断点吸附。*

### 触控目标
- 按钮使用大圆角（24–36px）创造宽大、适合手指的触控区域
- 导航链接间距充足
- 状态徽章适合触控尺寸（36px 圆角）
- 建议最小值：44×44px

### 折叠策略
- **导航**：带行动按钮的完整水平导航在移动端折叠为汉堡菜单
- **功能区域**：多列 → 堆叠单列
- **英雄文字**：64px → 约 36px 渐进缩放
- **设备预览**：网格 → 堆叠/轮播
- **卡片**：并排 → 垂直堆叠
- **间距**：等比缩减但保持宽阔节奏

### 图像行为
- 产品截图等比缩放
- 设备模型在移动端可能简化或展示更少设备
- 所有尺寸保持圆角
- 折叠以下内容使用懒加载

### 迭代指南
1. 每次专注于一个组件
2. 引用具体的颜色名称和十六进制代码——\"使用石板灰（`#60646c`）\"而非\"让它变灰\"
3. 有意使用圆角值——按钮 6px，卡片 8px，图像 24px，胶囊 9999px
4. 在具体测量值旁边描述\"感觉\"——\"96px 区域间距的巨大留白\"
5. 始终指定 Inter 和确切字重——字重对比就是层级
6. 阴影指定\"耳语阴影\"或参考层级表中的\"标准层次\"
7. 保持界面单色——让产品内容成为色彩
