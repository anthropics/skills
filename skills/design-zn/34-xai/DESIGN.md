# xAI 设计系统灵感

## 1. 视觉主题与氛围

xAI 的网站是深色优先、等宽驱动的粗野主义极简设计的典范——一个设计系统，感觉像是由那些理解克制是最终精致形式的工程师构建的。整体体验锚定于一个近乎黑色的背景（`#1f2228`）配纯白色文字（`#ffffff`），创造出高对比度、终端风格的美学，传递出深刻的技术可信度。没有渐变、没有装饰性插图、没有抢夺注意力的色彩强调。这是一个通过缺席来传达信息的网站。

排版系统分为两种精心选择的字体。`GeistMono`（Vercel 的等宽字体）以非凡的 320px 字重 300 处理展示级标题，并作为按钮字体，使用大写字母配有节奏的字间距（1.4px）。`universalSans` 以简洁、几何无衬线字体的声音处理所有正文和次级标题。将等宽字体作为展示字体的选择是决定性的美学决策——它将 xAI 定位为不是消费产品，而是基础设施，是由生活在终端中的人们构建的东西。

间距系统基于 8px 基础网格，值集中在小端（4px、8px、24px、48px），反映了密集、以信息为中心的布局哲学。圆角极小——网站几乎不对任何东西进行圆角处理，保持锋利的建筑感边缘。没有装饰性阴影，没有渐变，没有分层层次。深度完全通过对比和留白来传达。

**核心特征：**
- 纯深色主题：`#1f2228` 背景配 `#ffffff` 文字——无中间地带
- GeistMono 在极端展示尺寸（320px，字重 300）——等宽作为奢华
- 大写等宽按钮，带 1.4px 字间距——技术感、命令感
- universalSans 用于 16px/1.5 正文和 30px/1.2 标题——清洁对比
- 零装饰元素：无阴影、无渐变、无色彩强调
- 8px 间距网格，稀疏、刻意的尺度
- Heroicons SVG 图标系统——极简、功能性
- Tailwind CSS 配任意值——工具优先的工程方法

## 2. 色彩系统与角色

### 主色
- **纯白** (`#ffffff`)：单一文字颜色、链接颜色，以及所有前景元素。在 xAI 的系统中，白色不是背景——它是声音。
- **深色背景** (`#1f2228`)：画布。一种带有细微蓝色底调的暖近黑色（不是纯黑，不是中性灰）。这个特定色调避免了 `#000000` 的刺眼，同时保持深沉黑暗。

### 交互色
- **白色默认** (`#ffffff`)：默认状态下的链接和交互元素颜色。
- **白色弱化** (`rgba(255, 255, 255, 0.5)`)：链接的悬停状态——刻意的变暗而非变亮，这是不寻常且独特的。
- **白色细腻** (`rgba(255, 255, 255, 0.2)`)：边框、分隔线和细微的表面处理。
- **环形蓝** (`rgb(59, 130, 246) / 0.5`)：Tailwind 默认焦点环颜色（`--tw-ring-color`），用于键盘无障碍焦点状态。

### 表面与边框
- **表面抬高** (`rgba(255, 255, 255, 0.05)`)：细微的卡片背景和悬停表面——几乎不可见的提升。
- **表面悬停** (`rgba(255, 255, 255, 0.08)`)：交互容器略微可见的悬停状态。
- **边框默认** (`rgba(255, 255, 255, 0.1)`)：卡片、分隔线和容器的标准边框。
- **边框强调** (`rgba(255, 255, 255, 0.2)`)：激活状态和按钮轮廓的强调边框。

### 功能色
- **主文字** (`#ffffff`)：所有标题、正文、标签。
- **次文字** (`rgba(255, 255, 255, 0.7)`)：描述、说明、辅助文字。
- **三级文字** (`rgba(255, 255, 255, 0.5)`)：弱化标签、占位符文字、时间戳。
- **四级文字** (`rgba(255, 255, 255, 0.3)`)：禁用文字、极细微注解。

## 3. 字体规则

### 字体系列
- **展示 / 按钮**：`GeistMono`，备用字体：`ui-monospace, SFMono-Regular, Roboto Mono, Menlo, Monaco, Liberation Mono, DejaVu Sans Mono, Courier New`
- **正文 / 标题**：`universalSans`，备用字体：`universalSans Fallback`

### 层级

| 角色 | 字体 | 尺寸 | 字重 | 行高 | 字间距 | 大小写 | 备注 |
|------|------|------|------|------|--------|--------|------|
| 展示英雄 | GeistMono | 320px (20rem) | 300 | 1.50 | normal | 无 | 极端尺度，等宽奢华 |
| 区域标题 | universalSans | 30px (1.88rem) | 400 | 1.20（紧凑） | normal | 无 | 清洁无衬线对比 |
| 正文 | universalSans | 16px (1rem) | 400 | 1.50 | normal | 无 | 标准阅读文字 |
| 按钮 | GeistMono | 14px (0.88rem) | 400 | 1.43 | 1.4px | 大写 | 有节奏等宽，命令感 |
| 标签 / 说明 | universalSans | 14px (0.88rem) | 400 | 1.50 | normal | 无 | 辅助文字 |
| 小字 / 元数据 | universalSans | 12px (0.75rem) | 400 | 1.50 | normal | 无 | 时间戳、脚注 |

### 原则
- **等宽作为展示**：GeistMono 在 320px 不是噱头——它是品牌宣言。极大尺寸的固定宽度字符创造了比例字体无法实现的节奏感、建筑质感。
- **大尺寸轻字重**：320px 标题使用字重 300，防止等宽字体在极端尺寸感觉沉重或粗糙。读起来精确，而非压倒性。
- **大写按钮**：所有按钮文字都是大写 GeistMono 配 1.4px 字间距。这为交互元素创造了明显的技术感、近乎命令行的美学。
- **无衬线用于阅读**：universalSans 在 16px/1.5 为正文内容提供出色的可读性，与等宽展示元素形成清晰对比。
- **两字体清晰度**：系统使用恰好两种字体，角色清晰——等宽用于冲击力和交互，无衬线用于信息和阅读。无重叠，无歧义。

## 4. 组件样式

### 按钮

**主要（深色背景上的白色）**
- 背景：`#ffffff`
- 文字：`#1f2228`
- 内边距：12px 24px
- 圆角：0px（锐利边角）
- 字体：GeistMono 14px 字重 400，大写，字间距 1.4px
- 悬停：`rgba(255, 255, 255, 0.9)` 背景
- 用途：主要行动按钮（"试用 GROK"、"开始使用"）

**幽灵 / 轮廓**
- 背景：透明
- 文字：`#ffffff`
- 内边距：12px 24px
- 圆角：0px
- 边框：`1px solid rgba(255, 255, 255, 0.2)`
- 字体：GeistMono 14px 字重 400，大写，字间距 1.4px
- 悬停：`rgba(255, 255, 255, 0.05)` 背景
- 用途：次级操作（"了解更多"、"查看 API"）

**文字链接**
- 背景：无
- 文字：`#ffffff`
- 字体：universalSans 16px 字重 400
- 悬停：`rgba(255, 255, 255, 0.5)` ——悬停时变暗
- 用途：行内链接、导航项

### 卡片与容器
- 背景：`rgba(255, 255, 255, 0.03)` 或透明
- 边框：`1px solid rgba(255, 255, 255, 0.1)`
- 圆角：0px（锐利）或 4px（细腻）
- 阴影：无——xAI 不使用盒阴影
- 悬停：边框变为 `rgba(255, 255, 255, 0.2)`

### 导航
- 与页面匹配的深色背景（`#1f2228`）
- 品牌文字标志：白色文字，左对齐
- 链接：universalSans 14px 字重 400，`#ffffff` 文字
- 悬停：`rgba(255, 255, 255, 0.5)` 文字颜色
- 行动按钮：白色主要按钮，右对齐
- 移动端：汉堡菜单切换

### 徽章 / 标签
**等宽标签**
- 背景：透明
- 文字：`#ffffff`
- 内边距：4px 8px
- 边框：`1px solid rgba(255, 255, 255, 0.2)`
- 圆角：0px
- 字体：GeistMono 12px 大写，字间距 1px

### 输入框与表单
- 背景：透明或 `rgba(255, 255, 255, 0.05)`
- 边框：`1px solid rgba(255, 255, 255, 0.2)`
- 圆角：0px
- 焦点：带 `rgb(59, 130, 246) / 0.5` 的环
- 文字：`#ffffff`
- 占位符：`rgba(255, 255, 255, 0.3)`
- 标签：`rgba(255, 255, 255, 0.7)`，universalSans 14px

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 尺度：4px、8px、24px、48px
- 尺度是刻意稀疏的——xAI 避免细粒度间距区分，倾向于大跳跃，仅通过留白创造清晰的视觉层级

### 网格与容器
- 最大内容宽度：约 1200px
- 英雄区：全视口高度，居中巨型等宽标题
- 功能区域：简单垂直堆叠，带宽松区域内边距（48px-96px）
- 桌面端功能描述使用双列布局
- 全宽深色区域在整个页面保持单一深色背景

### 空白哲学
- **极度宽松**：xAI 使用大量留白。320px 标题配 48px+ 的周边内边距创造了一种空旷感，这本身就是设计陈述——内容如此重要，需要呼吸空间。
- **垂直节奏优于水平密度**：内容垂直堆叠，区域之间留有大间距，而非水平紧密排布。这创造了一种感觉刻意而电影化的滚动体验。
- **无视觉噪音**：缺少装饰元素、区域间边框和色彩多样性意味着留白是主要结构工具。

### 断点
- 2000px、1536px、1280px、1024px、1000px、768px、640px
- Tailwind 响应式修饰符驱动断点行为

### 圆角尺度
- 锐利（0px）：按钮、卡片、输入框的主要处理——默认值
- 细腻（4px）：次级容器的偶尔柔化
- 近零圆角哲学是品牌粗野主义身份的核心

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 平面（0 级） | 无阴影、无边框 | 页面背景、正文内容 |
| 表面（1 级） | `rgba(255,255,255,0.03)` 背景 | 细微卡片表面 |
| 边框（2 级） | `1px solid rgba(255,255,255,0.1)` 边框 | 卡片、容器、分隔线 |
| 激活（3 级） | `1px solid rgba(255,255,255,0.2)` 边框 | 悬停状态、激活元素 |
| 焦点（无障碍） | 带 `rgb(59,130,246)/0.5` 的环 | 键盘焦点指示器 |

**层次哲学**：xAI 完全拒绝传统的基于阴影的层次系统。网站上任何地方都没有盒阴影。深度通过三种机制传达：（1）基于不透明度的边框在交互时变亮，创造元素"激活"而非"上浮"的感觉；（2）极其细微的背景不透明度变化（`0.03` 至 `0.08`），创造几乎无法察觉的表面差异；（3）320px 展示文字与 16px 正文之间的巨大尺度对比，创造排版深度。这是通过对比和不透明度实现的层次，而非通过模拟光影。

## 7. 规范与禁忌

### 应该做
- 使用 `#1f2228` 作为通用背景——永远不要使用纯黑 `#000000`
- 所有展示标题和按钮文字使用 GeistMono——等宽就是品牌
- 所有按钮标签应用大写 + 1.4px 字间距
- 巨型展示标题（320px）使用字重 300
- 边框保持在 `rgba(255, 255, 255, 0.1)`——勉强可见，并非缺席
- 交互元素悬停时变暗至 `rgba(255, 255, 255, 0.5)` ——与常规做法相反
- 保持锐利边角（0px 圆角）作为默认值——粗野主义的精确
- 所有正文和阅读文字使用 universalSans，16px/1.5

### 不应该做
- 不要使用盒阴影——xAI 的层次没有阴影
- 不要在白色和深色背景之外引入色彩强调——单色调色板是神圣的
- 不要使用大圆角（8px+、胶囊形）——锐利边缘是有意为之的
- 不要使用粗体字重（600-700）用于标题——只有字重 300-400
- 不要在悬停时变亮元素——xAI 变暗至 `0.5` 不透明度
- 不要添加装饰性渐变、插图或色块
- 不要对按钮使用比例字体——GeistMono 大写是强制性的
- 除非绝对必要，不要使用彩色状态指示器——保持一切在白色/深色光谱中

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|------|----------|
| 移动端 | <640px | 单列，英雄标题大幅缩小 |
| 小平板 | 640-768px | 内边距略微增加 |
| 平板 | 768-1024px | 开始双列布局，标题尺寸增大 |
| 桌面 | 1024-1280px | 完整布局，宽松留白 |
| 大型 | 1280-1536px | 更宽的容器，更多呼吸空间 |
| 超大 | 1536-2000px | 最大内容宽度，居中 |
| 超宽 | >2000px | 内容保持居中，极宽边距 |

### 触控目标
- 按钮使用 12px 24px 内边距，触控舒适
- 导航链接间距 24px
- 最小点击目标：44px 高度
- 移动端：全宽按钮，便于拇指操作

### 折叠策略
- 英雄区：320px 等宽标题大幅缩小（移动端约 48px-64px）
- 导航：水平链接折叠为汉堡菜单
- 功能区域：双列到单列堆叠
- 区域内边距：96px → 48px → 24px 跨断点缩减
- 巨型展示文字是第一个调整大小的——必须保持冲击力但不溢出

### 图像行为
- 极少图像——网站依赖排版和留白
- 任何产品截图保持锐利边角
- 全宽媒体随视口等比缩放

## 9. AI 提示指南

### 快速色彩参考
- 背景：深色（`#1f2228`）
- 主文字：白色（`#ffffff`）
- 次文字：白色 70%（`rgba(255, 255, 255, 0.7)`）
- 弱化文字：白色 50%（`rgba(255, 255, 255, 0.5)`）
- 禁用文字：白色 30%（`rgba(255, 255, 255, 0.3)`）
- 默认边框：白色 10%（`rgba(255, 255, 255, 0.1)`）
- 强调边框：白色 20%（`rgba(255, 255, 255, 0.2)`）
- 细腻表面：白色 3%（`rgba(255, 255, 255, 0.03)`）
- 悬停表面：白色 8%（`rgba(255, 255, 255, 0.08)`）
- 焦点环：蓝色（`rgb(59, 130, 246)` 50% 不透明度）
- 主按钮背景：白色（`#ffffff`），文字深色（`#1f2228`）

### 迭代指南
1. 始终从 `#1f2228` 背景开始——永远不要使用纯黑或灰色背景
2. GeistMono 用于展示和按钮，universalSans 用于其他一切——永远不要混淆这些角色
3. 所有按钮必须是 GeistMono 大写配 1.4px 字间距——这是不可妥协的
4. 永远不要使用阴影——深度仅来自边框不透明度和背景不透明度
5. 边框始终是带低不透明度的白色（默认 0.1，强调 0.2）
6. 悬停行为变暗至 0.5 不透明度，而非变亮——与大多数系统相反
7. 默认锐利边角（0px）——仅对特定次级容器使用 4px
8. 正文使用 16px universalSans，行高 1.5，阅读舒适
9. 宽松的区域内边距（48px-96px）——让内容在黑暗中呼吸
10. 单色白色-深色调色板是绝对的——除非功能关键，否则拒绝添加色彩
