# Renault 设计系统灵感

## 1. 视觉主题与氛围

雷诺的网站是一个充满活力的数字展厅，在法国汽车优雅与大胆、前瞻性的能量之间取得平衡——有别于德国或意大利豪华品牌的单色简约主义。页面以一个全屏英雄区开场，将视口笼罩在广阔的极光渐变中——洋红色、紫色和青色的飘带在戏剧性照明的车辆后方流动。这种多彩的表达是网站的标志：虽然界面结构严谨（NouvelR 字体、黑白行动按钮框架、零圆角按钮），但内容充满色彩——英雄轮播上的渐变涂层、饱和的车辆摄影，以及在强调行动按钮上的雷诺黄（`#EFDF00`）点缀。效果是一个感觉充满活力而非安静的展厅。

布局遵循基于卡片的编辑节奏。在英雄轮播下方，内容被组织成一个 PromoCard 网格——每个都是全出血摄影面板，顶部有深色渐变叠加（从 `rgba(0,0,0,0.6)` 淡出至透明），确保白色标题文字在鲜艳图像上保持可读性。这些卡片在浅色和深色模式之间交替：白色编辑面板配黑色文字与黑色 `is-alternative-mode` 区域配白色文字并列，创造棋盘格式的视觉节奏。网格宽松——大型卡片格式主导，给每辆车或活动留出自己的视觉领域。下方的区域转移到完全深色的画布（绝对黑背景）用于 E-Tech 电动和技术展示，建立刻意的情绪转变：电气化存在于黑暗中，传统存在于光明中。

排版在 NouvelR 下统一——一种由 Black[Foundry] 专为雷诺品牌重塑设计的专有几何无衬线字体。字体的标志性"基础 r"具有以 28 度角切割的末端，呼应雷诺菱形标志的角度。提供 6 种字重从细体到超粗体，网站主要使用粗体（700）用于标题，常规体（400）用于正文。展示标题规格大——56px/0.95 行高用于英雄标题，创造出密集、有冲击力的文字块，紧密相连。该字体支持拉丁文、希腊文、西里尔文、希伯来文、阿拉伯文和韩文，反映了雷诺的全球市场覆盖。所有文字渲染感觉精确且工程化，几何比例赋予一种现代感，符合雷诺电动优先的品牌定位。

**核心特征：**
- 在车辆图像后带生动极光渐变背景（洋红色/紫色/青色）的全屏英雄轮播
- 带 28 度"基础 r"切割（与菱形标志几何形状匹配）的 NouvelR 专有字体
- 雷诺黄（`#EFDF00`）作为超级主要强调——谨慎使用，用于最高优先级行动按钮
- 所有按钮零圆角——锐利矩形形式表达精密工程
- 带全出血摄影和深色渐变叠加的基于卡片的编辑网格
- 二元黑白行动按钮系统：主要（黑色背景/白色文字）和幽灵（透明/白色边框）
- PromoCard 深色模式交替，在浅色和深色区域之间创造棋盘节奏
- PrimeReact（21 个组件）+ Element Plus（19 个组件）驱动交互元素
- 链接悬停状态使用雷诺蓝（`#1883FD`）——唯一的彩色交互颜色

## 2. 色彩系统与角色

### 主色
- **雷诺黄** (`#EFDF00`)：品牌的标志性 Pantone——一种鲜艳、饱和的黄色，用于超级主要行动按钮和最高优先级操作按钮。显示为 `.is-cta-super-primary` 类的 `--CtaLink-background-color`。承载着菱形标志的能量
- **绝对黑** (`#000000`)：主要按钮背景、浅色表面上的标题文字，以及整个界面的主导深色表面。整个界面的结构锚点
- **纯白** (`#FFFFFF`)：编辑内容的主要表面、翻转按钮背景、英雄文字颜色，以及主导浅色模式画布（--rt-color-white）

### 次级与强调
- **柔和黄** (`#F8EB4C`)：雷诺黄更浅、更暖的变体——用于黄色行动按钮的悬停/按下状态和次级强调场景
- **雷诺蓝** (`#1883FD`)：所有链接变体的链接悬停颜色——一种明亮、自信的蓝色，传递交互性，但不与黄色品牌强调竞争
- **暖灰** (`#D9D9D6`)：用于禁用状态、非激活 UI 元素和柔和边框的细微暖中性色——带有轻微温度，区别于冷灰色

### 表面与背景
- **纯白** (`#FFFFFF`)：页面背景、浅色编辑区域、导航栏和页脚
- **绝对黑** (`#000000`)：英雄背景、PromoCard 深色模式区域（`is-alternative-mode`）和 E-Tech 展示区域
- **炭灰** (`#222222`)：文字密集的深色区域和页脚子区域的次级深色表面（--rt-color-dark）
- **浅银** (`#F2F2F2`)：用于区域差异化和卡片边框的细微替代浅色表面

### 中性色与文字
- **绝对黑** (`#000000`)：浅色表面上的主标题和正文——雷诺使用真黑色而非近黑色
- **纯白** (`#FFFFFF`)：深色表面上的主文字——英雄标题、深色区域标题和翻转按钮标签
- **暖灰** (`#D9D9D6`)：三级文字、元数据和低调标签
- **边框灰** (`#D1D1D1`)：输入框边框和细微分隔线

### 语义与强调
- **成功绿** (`#8DC572`)：正向状态指示器和确认信息（--rt-color-success）
- **错误玫瑰** (`#BE6464`)：表单验证错误和警告状态（--rt-color-error）
- **警告琥珀** (`#F0AD4E`)：警示提示和需要关注的状态（--rt-color-warning）
- **信息蓝** (`#337AB7`)：信息性提示和中性状态消息（--rt-color-info）

### 渐变系统
- **英雄极光**：应用于英雄幻灯片背景的宽阔多彩渐变（洋红色 → 紫色 → 青色）——网站最具特色的视觉元素。这些是摄影/合成而非 CSS 渐变
- **PromoCard 叠加**：`linear-gradient(rgba(0,0,0,0.6) 0%, rgba(0,0,0,0) 40%)` ——应用于卡片顶部，确保摄影上方的标题文字可读性
- 表面上无平面 CSS 渐变——深度来自摄影处理和黑/白交替

## 3. 字体规则

### 字体系列
- **NouvelR**：唯一字体。由 Black[Foundry] 为雷诺 2021+ 品牌重塑专门设计的几何无衬线字体。具有标志性的"基础 r"，其 28 度末端切割与菱形标志角度匹配。提供 6 种字重（细体至超粗体），支持 6 种书写系统。备用字体：`sans-serif`。在 CSS 中声明为 `"NouvelR, sans-serif"`
- **无次要字体**：与法拉利（FerrariSans + Body-Font）或兰博基尼（LamboType + Open Sans）不同，雷诺对所有文字使用单一字体系列——标题、正文、按钮、说明和导航

### 原则
- **单家族纪律**：NouvelR 处理从 56px 英雄标题到 8.5px 法律说明的一切——字体的几何精确性允许其在这个极端范围内缩放而不失去特性
- **粗体默认标题**：字重 700 主导标题层级。与使用中等（500）标题的品牌不同，雷诺的粗体字重创造了更自信、更充满活力的阅读体验
- **超紧凑展示行高**：英雄和区域标题 0.95 行高——行间距几乎重叠，创造一种压缩、有力的排版纹理，感觉紧迫而现代
- **28 度基础 r**：字体的标志性细节——小写"r"的末端以恰好 28 度切割，以镜像雷诺菱形标志的角度，将品牌身份嵌入每个单词
- **说明文字大小写转换**：部分说明文字使用 `text-transform: capitalize` 进行编辑标注，而微型文字使用 `lowercase` ——用于层级信号的刻意反转

## 4. 组件样式

### 按钮
雷诺的按钮是零圆角的锐利矩形——如同冲压金属车身面板的工业精密感。

**超级主要（黄色）** ——最高强调行动按钮：
- 默认：背景 `#EFDF00`（雷诺黄），文字 `#000000`，圆角 0px，内边距 10px 15px，边框 1px solid `#EFDF00`
- 翻转：背景 `#EFDF00`，文字 `#000000` ——深色背景上相同黄色
- 字号 16px（NouvelR），字重 700，最小高度 46px，最小宽度 46px
- 用于：主要转化操作（配置、立即购买）

**主要（黑色）** ——默认操作按钮：
- 默认：背景 `#000000`，文字 `#FFFFFF`，圆角 0px，内边距 10px 15px，边框 1px solid `#000000`
- 翻转：背景 `#FFFFFF`，文字 `#000000`，边框 1px solid `#FFFFFF` ——深色背景上白色填充
- 字号 16px（NouvelR），字重 700
- 用于："探索"、次级转化操作

**幽灵** ——透明轮廓按钮：
- 深色背景默认：背景透明，文字 `#FFFFFF`，边框 1px solid `#FFFFFF`，圆角 0px，内边距 10px 15px
- 浅色背景默认：背景透明，文字 `#000000`，边框 1px solid `#000000`
- 字号 16px（NouvelR），字重 700
- 用于："抢先了解"、"购买"、次级操作

**文字链接** ——行内导航：
- 浅色默认：文字 `#000000`，无边框，无背景
- 深色默认：文字 `#FFFFFF`
- 悬停：颜色变为 `#1883FD`（雷诺蓝），无文字装饰
- 所有链接变体悬停至相同蓝色——一致的交互反馈

### 卡片与容器

**PromoCard（浅色）** ——编辑内容卡片：
- 背景：白色或透明
- 全出血摄影，顶部深色渐变叠加：`linear-gradient(rgba(0,0,0,0.6) 0%, rgba(0,0,0,0) 40%)`
- 标题：NouvelR 40px/700，白色文字定位于渐变上方
- 圆角：0px——锐利矩形容器
- 无阴影，无可见边框

**PromoCard（深色 / `is-alternative-mode`）** ——电影感卡片：
- 背景：`#000000`（绝对黑）
- 相同渐变叠加处理
- 标题：白色 NouvelR 文字
- 行动按钮：翻转主要（白色背景）或幽灵（白色边框）

**VehicleRangeCard** ——车辆展示：
- 背景：透明
- 车辆图像上方，车型名称和价格/规格下方
- 无阴影，无边框，干净平面处理
- 通过网格间距进行卡片间距控制

### 输入框与表单

**搜索/文本输入框：**
- 背景：`#FFFFFF`
- 文字：`#000000`
- 边框：1px solid `#D1D1D1`（边框灰）
- 圆角：50px（胶囊形——与零圆角按钮系统的不寻常偏差）
- 内边距：6px 35px 6px 15px（搜索图标的额外右内边距）
- 字体：NouvelR，12.8px
- 焦点：标准浏览器焦点环

### 导航
- **桌面**：雷诺菱形标志居中/左对齐，水平导航链接，固定定位
- **背景**：白色，静止时无阴影
- **链接**：NouvelR，13px，字重 700，黑色文字
- **悬停**：颜色变为 `#1883FD`（雷诺蓝）
- **移动端**：汉堡菜单折叠为全屏导航抽屉
- **导航中的行动按钮**：主要转化操作的主要黑色按钮

### 图像处理
- **英雄**：带戏剧性极光渐变背景和艺术指导车辆摄影的全视口轮播——边缘到边缘，无内边距
- **PromoCard**：卡片范围内的全出血摄影，顶部深色渐变叠加确保文字可读性
- **车辆图像**：中性/渐变背景上的透明背景渲染
- **宽高比**：混合——英雄约 16:9 视口，PromoCard 从正方形到宽幅全景不等
- **懒加载**：下折叠区域使用懒加载（框架处理）

### 轮播组件
- 带自动前进幻灯片的全屏英雄轮播
- 每张幻灯片：背景渐变/照片 + 车辆图像 + 标题 + 行动按钮
- 幻灯片位置的点状指示器
- 边缘导航箭头

## 5. 布局原则

### 间距系统
- **基础单位**：8px（检测到的系统基础）
- **尺度**：1px、4px、5px、6px、6.25px、8px、10px、12px、13px、15px、16px、20px、24px、32px、40px
- **按钮内边距**：10px 15px——所有按钮变体一致
- **区域内边距**：主要内容块之间宽松的垂直间距（40–80px）
- **卡片间距**：网格项之间 16–24px
- **最小交互尺寸**：46px（所有按钮的最小宽度和最小高度）

### 网格与容器
- **最大宽度**：1440px（最大定义断点）
- **英雄**：全出血，边缘到边缘，视口高度
- **PromoCard 网格**：2 列和 3 列布局，混合卡片尺寸
- **车辆范围**：可水平滚动的卡片行或网格
- **页脚**：白色背景的多列布局

### 空白哲学
雷诺使用适中的留白——比法拉利更宽松但比特斯拉更少极端。基于卡片的布局意味着内容被组织到定义的容器中，而非漂浮在虚空中。视觉呼吸空间主要来自大型卡片格式和全出血英雄轮播，给每辆车提供自己的电影感时刻。区域之间，间距一致（32–40px），创造有节奏的滚动体验。浅色和深色区域之间的交替也创造了感知的留白——模式切换本身起到视觉分隔的作用。

### 圆角尺度
| 值 | 场景 |
|-----|------|
| 0px | 所有按钮、PromoCard、大多数容器——零圆角默认 |
| 2px | 小型 UI 元素（区域控件） |
| 3px | 内容面板（div、tabpanel） |
| 4px | 标签和标注元素 |
| 46px | 胶囊形元素（搜索输入框、筛选标签） |
| 50px | 搜索/输入框的完整胶囊 |

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 0 级（平面） | 无阴影 | PromoCard、按钮、大多数容器的默认 |
| 1 级（柔和） | `rgba(0,0,0,0.2) 0px 4px 8px` | 卡片悬停状态，细微提升效果 |
| 2 级（中等） | `rgba(0,0,0,0.2) 0px 0px 18px` | 浮动 UI 元素、下拉菜单 |
| 3 级（分层） | `rgba(0,0,0,0) 0px 2px 4px, rgba(50,50,93,0.1) 0px 7px 14px` | 抬高卡片和模态框的复合阴影 |
| 4 级（深度） | `rgba(0,0,0,0.15) 0px 40px 80px` | 大型浮动面板、配置器叠加 |
| 5 级（方向性） | `rgba(0,0,0,0.2) 5px 5px 8px` | 特定组件的偏移方向性阴影 |
| 6 级（环境） | `rgb(199,197,199) 0px 0px 12px 2px` | 高亮元素的环境光晕效果 |

### 阴影哲学
雷诺比法拉利或特斯拉使用更丰富的阴影系统——七种不同的阴影标记，反映更分层、更立体的界面。阴影从微妙的 4px 悬停提升到戏剧性的 80px 深层面板。复合阴影（3 级）的双层方法（紧密的深色阴影加上来自 `rgba(50,50,93,0.1)` 的更宽紫色着色阴影）尤为精致——它创造了一种照片般真实的漂浮效果。暖灰色的环境光晕（6 级）增加了独特的触感，与雷诺更温暖的色彩个性相呼应。

### 装饰性深度
- **英雄极光渐变**：主要装饰性深度元素——鲜艳的色彩渐变在车辆图像后方创造大气深度
- **PromoCard 叠加**：`linear-gradient(rgba(0,0,0,0.6) → transparent)` 通过透明度在卡片内部创造深度
- UI 元素上**无模糊效果**——深度通过阴影和颜色对比传达

## 7. 规范与禁忌

### 应该做
- 雷诺黄（`#EFDF00`）专门用于超级主要行动按钮——它承载着菱形标志身份的全部重量
- 所有按钮保持零圆角——锐利边缘在雷诺系统中是不可妥协的
- 使用 NouvelR 粗体（700）作为默认标题字重——这种自信的字重是品牌充满活力个性的核心
- 在 PromoCard 上应用深色渐变叠加（`rgba(0,0,0,0.6) → transparent`），确保摄影上方的文字可读性
- 英雄展示文字保持超紧凑行高（0.95）——压缩纹理感觉紧迫而现代
- 在黑色和白色区域之间交替，创造标志性的棋盘节奏
- 所有链接悬停状态一致使用 `#1883FD`（雷诺蓝）——一种交互颜色信号
- 所有按钮最小交互尺寸设置为 46×46px——规格中内置的无障碍访问性
- 胶囊形圆角（46–50px）专门用于搜索输入框和筛选元素——永远不用于按钮
- 每张有文字叠加摄影的卡片都使用 PromoCard 渐变叠加

### 不应该做
- 不要将雷诺黄作为区域或表面的背景颜色——它是行动按钮信号，不是氛围颜色
- 不要给按钮添加圆角——零圆角矩形是核心品牌标记
- 不要使用除 NouvelR 以外的任何字体——单一家族纪律是品牌支柱
- 不要在单个区域混用多种彩色强调——调色板是单色加黄色
- 不要将标题字重软化至 400 或 500——NouvelR 粗体是品牌声音，较轻字重感觉不符合品牌
- 不要给 PromoCard 或内容容器添加装饰性边框——分隔来自背景颜色交替
- 不要将语义颜色（成功绿、错误玫瑰）用于装饰目的——它们保留给表单状态
- 不要将 56px 英雄尺寸应用于折叠以下的任何内容——英雄排版尺寸保留给轮播
- 不要创建圆形胶囊按钮——胶囊形状保留给输入框，永远不用于操作元素
- 不要在 UI 表面使用平面 CSS 渐变——唯一的渐变应该是摄影英雄极光和文字可读性叠加

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|------|----------|
| 小移动端 | ≤425px | 单列、全宽卡片、英雄文字缩小至约 32px、堆叠行动按钮、汉堡导航 |
| 移动端 | 426–640px | 单列、略大卡片、英雄文字 32–40px |
| 小平板 | 641–768px | 开始双列 PromoCard 网格，英雄保持全宽 |
| 平板 | 769–896px | 完整双列布局，车辆范围显示 2–3 张卡片 |
| 小桌面 | 897–1024px | 导航完全展开，英雄 56px，双列卡片网格 |
| 桌面 | 1025–1280px | 完整布局，三列卡片网格，宽松留白 |
| 大桌面 | 1281–1440px | 最大内容宽度，居中容器，英雄全电影尺寸 |

### 触控目标
- 所有按钮：最小 46×46px（`minWidth: 46px, minHeight: 46px`）——超过 WCAG AAA 44×44px 要求
- 搜索输入框胶囊：50px 圆角创造大型可触控区域
- 导航链接：NouvelR 13px，项目间距足够
- 轮播导航：视口边缘的大型箭头目标

### 折叠策略
- **导航**：完整水平导航在移动端折叠为雷诺菱形标志 + 汉堡菜单
- **英雄轮播**：所有断点保持全宽，标题从 56px（桌面）缩放至约 32px（移动端）
- **PromoCard 网格**：三列 → 双列 → 视口缩小时单列
- **车辆范围**：所有尺寸保持水平滚动，可见卡片减少
- **行动按钮对**：并排按钮在移动端垂直堆叠
- **页脚**：多列在移动端折叠为单列手风琴

### 图像行为
- 英雄图像：所有断点全出血，使用 `object-fit: cover`
- PromoCard 图像：在卡片容器内响应式，渐变叠加等比缩放
- 车辆图像：透明背景渲染在网格单元内等比缩放
- 艺术指导：移动端可能裁剪至更紧密的车辆视图，减少环境背景

### 迭代指南
优化使用此设计系统生成的现有页面时：
1. 一次专注于一个组件——雷诺系统有清晰的组件边界（PromoCard、VehicleRangeCard、行动按钮变体）
2. 引用具体的颜色名称和十六进制代码——调色板小但每种颜色都有精确的功能
3. 使用自然语言描述，而非 CSS 值——"锐利零圆角矩形"比"border-radius: 0"更好地传达意图
4. 在具体测量值旁边描述所需的"感觉"——"自信的汽车能量"比"font-weight: 700"更好地传达 NouvelR 粗体标题的个性
5. 始终检查区域应该是浅色还是深色——棋盘交替是核心模式
6. 每屏幕保留雷诺黄给一个按钮——如果黄色出现在超过一个行动按钮上，层级就会崩溃
