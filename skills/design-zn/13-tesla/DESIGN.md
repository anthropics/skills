# Tesla 设计系统灵感

## 1. 视觉主题与氛围

Tesla 的网站是极端减法艺术的范本——一个数字展厅，产品就是一切，界面几乎不存在。页面以充满整个视口的主视觉区开篇，用电影感的汽车摄影填满整个屏幕：三辆车排列在抛光混凝土地面上，背景是朦胧的城市天际线，单一的车型名称以半透明白色字体漂浮其上。没有装饰性边框，没有渐变，没有图案，没有阴影。UI 的存在仅仅是为了提供足够的导航结构，然后让开。每一个非产品图像的像素都是留白，这种克制才是整个设计系统最有力的表达。

色彩哲学近乎苦行：单一的蓝色（`#3E6AE1`）用于主要行动召唤按钮，三种深灰色呈现文字层级，其余全部是白色。整体的情感重量完全由摄影承载——宽幅的风景大片、棚拍的车辆侧影、以及覆盖整个视口高度的环境感构图。UI 外壳融入图像之中。导航栏悬浮在主视觉区上方，没有任何可见的背景、边框或阴影——TESLA 文字标志和五个导航标签仅仅存在于空间中，信任着下方内容提供足够的对比度。

排版最近从 Gotham 过渡到 Universal Sans——一种自定义字体系列，分为用于标题的"Display"和用于正文/UI 元素的"Text"，将网站、移动应用和车载软件统一为单一的排版声音。Display 变体以 40px 500 字重渲染主视觉标题，而 Text 变体处理从导航（14px/500）到正文（14px/400）的所有内容。这种字体具有几何精度和略带人文主义的末端，感觉是工程化而非设计出来的——完全契合 Tesla 不需要自我标榜的科技品牌形象。没有文字阴影，没有文字渐变，没有装饰性排版处理。每个字形都通过纯粹的清晰度来赢得其位置。

**核心特征：**
- 全视口主视觉区段（100vh），以电影感汽车摄影为主，UI 叠加层最小化
- 近零 UI 装饰：页面上无处有阴影、渐变、边框、图案
- 单一点缀色——电光蓝（`#3E6AE1`）——专门用于主要行动召唤按钮
- Universal Sans 字体系列（Display + Text），统一网页、应用和车载界面
- 以摄影为先的展示方式，产品图像承载所有情感重量
- 磨砂玻璃导航概念，透明/白色导航漂浮于主视觉内容上方
- 0.33s cubic-bezier 过渡作为所有交互状态变化的通用时序
- 轮播式主视觉，带有圆点指示器和边缘箭头导航，用于多车型展示
- "提问"持久聊天栏，固定在视口底部

## 2. 色彩方案与角色

### 主色
- **电光蓝** (`#3E6AE1`)：主要行动召唤按钮背景——一种自信的中等饱和度蓝色（rgb 62, 106, 225），在整个界面中作为唯一的彩色。专门用于"立即下单"和其他主要操作按钮
- **纯白** (`#FFFFFF`)：所有界面、面板、导航和次要按钮填充的主导背景色——让摄影得以呼吸的画布

### 次要与点缀色
- **促销蓝** (`#3E6AE1`)：蓝色也用于主视觉图像上的促销文字（"0% APR 优惠"），与行动召唤按钮色调相同——在激励信息和操作之间形成视觉关联
- 不存在次要点缀色。Tesla 刻意避免色彩多样性，以保持极致的视觉纪律

### 界面与背景
- **白色画布** (`#FFFFFF`)：页面背景、导航面板、下拉菜单和所有界面容器
- **浅灰** (`#F4F4F4`)：区段差异化的微妙替代界面——与纯白几乎无法察觉的色差（rgb 244, 244, 244）
- **碳深色** (`#171A20`)：主视觉文字叠加和潜在深色模式上下文的深色界面（rgb 23, 26, 32）——带蓝色底调的温暖近黑色
- **磨砂玻璃** (`rgba(255, 255, 255, 0.75)`)：导航在滚动时的 backdrop-filter 效果用半透明白色

### 中性色与文字
- **碳深色** (`#171A20`)：主要标题和导航文字——最深的文字值（rgb 23, 26, 32），用于车型名称、导航标签和浅色背景上的主视觉标题
- **石墨色** (`#393C41`)：正文和次要内容（rgb 57, 60, 65）——默认段落颜色，略比纯灰色温暖
- **铅灰色** (`#5C5E62`)：子链接的第三级文字，次要导航链接如"了解"和"下单"（rgb 92, 94, 98）
- **银雾色** (`#8E8E8E`)：输入字段的占位文字和禁用状态（rgb 142, 142, 142）
- **云灰色** (`#EEEEEE`)：浅色边框和分割线（rgb 238, 238, 238）
- **浅银色** (`#D0D1D2`)：微妙的 UI 边框和划分（rgb 208, 209, 210）

### 语义与点缀
- Tesla 营销网站避免使用语义色编码（无绿/红/黄状态指示器）。表单上下文中的错误、成功和警告状态遵循标准浏览器默认设置
- 蓝色行动召唤按钮（`#3E6AE1`）是唯一的交互颜色信号

### 渐变系统
- 界面中任何地方均不使用渐变
- 深度完全通过摄影、留白和全出血图像与干净白色界面之间的二元对比来实现
- 导航通过透明度（磨砂玻璃效果）而非渐变或阴影来实现分层

## 3. 排版规则

### 字体系列
- **展示字体**：`Universal Sans Display`，-apple-system，Arial，sans-serif — 用于主视觉标题和大型车型名称。一种具有精确工程比例的几何无衬线字体，最近取代 Gotham，以统一 Tesla 的数字生态系统（网站、移动应用、车辆界面）
- **文字/UI**：`Universal Sans Text`，-apple-system，Arial，sans-serif — 用于导航、正文、按钮和所有 UI 文字。针对较小尺寸的易读性进行了优化，比 Display 变体有略宽的比例
- 未检测到 OpenType 特性——排版完全素颜
- 营销网站上未观察到斜体变体

### 层级体系

| 角色 | 尺寸 | 字重 | 行高 | 字间距 | 备注 |
|------|------|--------|-------------|----------------|-------|
| 主视觉标题 | 40px (2.50rem) | 500 | 48px (1.20) | 正常 | Universal Sans Display，深色主视觉图像上的白色文字 |
| 车型名称 | 17px (1.06rem) | 500 | 20px (1.18) | 正常 | Universal Sans Text，导航面板和卡片中的车型名称 |
| 导航项 | 14px (0.88rem) | 500 | 16.8px (1.20) | 正常 | Universal Sans Text，主要导航标签 |
| 正文 | 14px (0.88rem) | 400 | 20px (1.43) | 正常 | Universal Sans Text，段落和描述性内容 |
| 按钮标签 | 14px (0.88rem) | 500 | 16.8px (1.20) | 正常 | Universal Sans Text，行动召唤按钮文字 |
| 子链接 | 14px (0.88rem) | 400 | 20px (1.43) | 正常 | 第三级链接（了解、下单、体验） |
| 促销文字 | 22px (1.38rem) | 400 | 20px (0.91) | 正常 | 主视觉上的白色促销文字（"0% APR 优惠"） |
| 分类标签 | 16px（估算） | 500 | — | 正常 | 分类卡片上的白色文字标签（"运动轿车"） |

### 排版原则
- **"正常"字间距无处不在**：与大多数现代科技品牌对标题使用负字间距不同，Tesla 在所有层级均使用默认字间距。这反映出字体应该自己说话而无需操控的哲学
- **字重克制**：只出现两种字重——500（中等）用于标题/UI，400（常规）用于正文。没有粗体（700），没有细体（300）。系统避免排版戏剧性
- **统一字体尺寸**：大多数 UI 文字集中在 14px，只有主视觉标题（40px）和促销文字（22px）突破这一规律。这种极度统一创造出工程式一致性感
- **Display 与 Text 的分割**：双变体系统（Display 用于主视觉，Text 用于 UI）创造微妙的视觉校正，不带可见的风格差异——它们在不同尺寸下看起来像同一字体
- **无文字转换**：主导航或行动召唤按钮中不出现全大写文字——小写方式强化了 Tesla 低调的自信

## 4. 组件样式

### 按钮
所有按钮使用极小圆角矩形（4px 圆角半径）——创造出锐利、技术感的美学，与车辆的精确性相呼应。

**主要行动召唤按钮** — 主要操作按钮：
- 默认：背景 `#3E6AE1`（电光蓝），文字 `#FFFFFF`，字体 14px，字重 500，内边距 4px，通过 flexbox 居中内容，圆角 4px，最小高度 40px，宽度 200px
- 边框：3px solid transparent（为焦点/激活边框动画预留空间）
- 盒阴影：`rgba(0,0,0,0) 0px 0px 0px 2px inset`（静止时不可见，焦点时动画变为可见）
- 过渡：`border-color 0.33s, background-color 0.33s, color 0.33s, box-shadow 0.25s`
- 悬停：蓝色背景微妙变暗
- 用于："立即下单"行动召唤按钮

**次要行动召唤按钮** — 备选操作按钮：
- 默认：背景 `#FFFFFF`，文字 `#393C41`（石墨色），尺寸和边框样式与主要按钮相同
- 过渡：与主要按钮相同的时序（0.33s）
- 用于：配合主要行动召唤按钮的"查看库存"

**导航按钮** — 顶部导航项：
- 默认：背景透明，文字 `#171A20`（碳深色），字体 14px，字重 500，圆角 4px，内边距 4px 16px，最小高度 32px
- 过渡：`color 0.33s, background-color 0.33s`
- 激活/展开：微妙的背景高亮
- 用于："车辆"、"能源"、"充电"、"探索"、"商店"

**文字链接** — 内容内操作：
- 默认：文字 `#5C5E62`（铅灰色），字体 14px，字重 400，无背景，无边框
- 悬停：带有盒阴影过渡的下划线装饰
- 过渡：`box-shadow 0.33s cubic-bezier(0.5, 0, 0, 0.75), color 0.33s`
- 用于：下拉面板中的"了解"、"下单"、"体验"、"全新"、"二手"链接

### 卡片与容器

**车辆卡片**（导航面板）：
- 背景：透明（继承面板白色）
- 边框：无
- 阴影：无
- 内容：车辆图像（透明 PNG）+ 居中于下方的车型名称 + 两个文字链接
- 布局：下拉面板内的三列栅格
- 卡片本身没有悬停动画——交互通过其下方的文字链接进行

**分类卡片**（首页下部区域）：
- 背景：全出血风景摄影
- 圆角：约 12px（轻微圆润）
- 溢出：hidden（将图像裁剪为圆角）
- 文字：左上角的白色标签（"运动轿车"、"中型 SUV"）
- 尺寸：大型格式，约 2:1 比例
- 无阴影，无边框，无叠加渐变——文字依靠图像暗度提供对比度

### 输入框与表单
- 背景：透明
- 文字颜色：`#171A20`（碳深色）
- 占位文字颜色：`#8E8E8E`（银雾色）
- 边框：极简，继承自浏览器默认
- 字体：Universal Sans Text，14px
- "提问"聊天机器人输入栏位于视口底部，具有干净的白色背景和微妙的边框

### 导航
- **桌面端**：居中水平导航，左侧 TESLA 文字标志（间距大写字母），五个分类按钮居中对齐，右侧三个图标按钮（帮助、地球/语言、账户）
- **背景**：白色（通过类切换 `tds-site-header--white-background` 从深色主视觉上的透明过渡到滚动时的不透明白色）
- **下拉面板**：全宽白色面板，含三列车辆栅格 + 右侧边栏文字链接，无阴影，无边框——无缝衔接于导航下方
- **粘性行为**：`sticky-without-slide` 类——保持在顶部，无滑入动画
- **移动端**：汉堡折叠模式
- **导航与内容之间无可见分隔符**——导航与主视觉融为一体

### 图像处理
- **主视觉**：全视口（100vh）区段，电影感摄影——从边缘到边缘，无内边距，无外边距
- **车辆图像**：透明 PNG 在下拉面板白色背景上呈现，棚拍品质的四分之三角度照片
- **分类卡片**：约 2:1 比例的风景摄影，圆角（12px）
- **轮播**：自动推进，带圆点指示器（3个圆点）和边缘左/右箭头导航
- **延迟加载**：视口以下区段使用延迟加载，在滚动进入视口前呈现为空白白色

### 持久聊天栏
- 固定在视口底部，在所有区段均可见
- 白色背景，带微妙边框
- 包含：聊天图标 + "提问"标签 + 占位文字（"什么是 Dog Mode？"）+ 发送图标 + "立即预约试驾"次要行动召唤按钮
- 预约行动召唤按钮带有蓝绿色图标点缀

## 5. 布局原则

### 间距系统
- **基础单位**：8px
- **常用值**：8px (0.5rem)、16px (1rem)、21.44px (1.34rem)
- **按钮内边距**：4px（最小外边距），通过 flexbox 居中内容，导航项 4px 16px
- **区段内边距**：全视口区段，内容垂直居中
- **卡片间距**：分类卡片之间约 16px

### 栅格与容器
- **最大宽度**：约 1383px（大部分内容使用全视口宽度）
- **主视觉**：全出血，从边缘到边缘，100vh 区段
- **导航面板**：三列车辆卡片栅格，右对齐文字侧边栏（约 70/30 分割）
- **分类卡片**：双栏水平布局（大左卡片 + 小右卡片）

### 留白哲学
Tesla 将留白用作奢侈品信号。区段之间慷慨的垂直间距（每个区段是一个完整的视口高度），意味着你一次只能看到一条"信息"——一辆车、一个车型名称、一对行动召唤按钮。这创造了一种画廊式的浏览体验，每次滚动都是一次刻意的转换，而非连续的信息流。留白不是空洞的——它是将每辆车提升至艺术品地位的画框。

### 圆角比例
| 值 | 场景 |
|-------|---------|
| 0px | 大多数元素——尖角是默认值 |
| 4px | 按钮（主要、次要、导航项）——几乎无法感知的圆润 |
| ~12px | 分类卡片——较大界面上明显但克制的圆润 |
| 50% | 轮播圆点指示器——完美圆形 |

## 6. 深度与层级

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 层级 0（平面） | 无阴影，无边框 | 所有元素的默认状态——静止时的卡片、面板、按钮 |
| 层级 1（磨砂） | `rgba(255,255,255,0.75)` 背景滤镜 | 滚动时的导航栏——磨砂玻璃透明度 |
| 层级 2（叠加） | `rgba(128,128,128,0.65)` | 模态框叠加层和地区/Cookie 弹出框 |
| 层级 3（微妙） | `rgba(0,0,0,0.05)` | 罕见悬停状态下的最小阴影提示 |

### 阴影哲学
Tesla 对层级的处理方式基本上是"无"。网站在其主要界面中完全避免盒阴影。深度通过三种替代策略来传达：
1. **Z 轴分层**：粘性导航通过定位（而非阴影）置于主视觉内容之上
2. **基于透明度的半透明**：磨砂玻璃导航和叠加模态框使用背景颜色透明度（而非阴影）来指示分层
3. **摄影即深度**：全出血图像通过透视、光照和构图创造其自身的视觉深度——使 UI 阴影显得多余

### 装饰性深度
- UI 元素上没有渐变、光晕或氛围效果
- 主视觉图像本身提供所有视觉丰富性——夕阳天空、车身表面反射的光线、棚拍照明产生的地面阴影
- 轮播箭头按钮使用半透明白色背景漂浮于主视觉图像上方，而不影响其观感

## 7. 应做与禁止

### 应做
- 让摄影主宰每个屏幕——产品就是设计
- 将电光蓝（`#3E6AE1`）专门用于主要行动召唤按钮——绝不用于装饰目的
- 主要内容块保持视口高度区段——每屏一条信息
- 只保留 400-500 字重——无粗体，无细体，无极端
- 所有交互元素使用 4px 圆角——精确胜于活泼
- 将留白视为奢侈品信号——不要仅因有空间就填满它
- 所有过渡保持 0.33s——动效的一致性与色彩的一致性同等重要
- 在产品展示中使用透明 PNG 车辆图像于白色背景上
- 将行动召唤按钮水平居中于车型名称下方——垂直节奏是：车型 → 副标题 → 按钮
- 保持 Display/Text 字体分割——Display 仅用于主视觉尺寸文字，Text 用于其他所有内容

### 禁止
- 不要为任何元素添加阴影——通过阴影实现层级与平面画廊美学相矛盾
- 不要使用蓝色行动召唤按钮之外的彩色——调色板刻意是单色加一种颜色
- 不要将渐变、图案或装饰性背景应用于界面——白色和摄影是唯一的背景
- 不要在网页上使用大于 40px 的文字——即使在主视觉尺寸，排版也是刻意克制的
- 不要为卡片或容器添加边框——间距实现分隔，而非线条
- 不要使用全大写文字转换——Tesla 的自信通过小写的平静来表达
- 不要引入圆形胶囊按钮或大圆角——4px 圆角是经过深思熟虑且精确的
- 不要用其他字体覆盖 Universal Sans 系列——跨平台一致性是核心品牌价值
- 不要添加带有缩放/位移变换的悬停动画——Tesla 的交互是仅颜色变化的（背景和边框过渡）
- 不要在视口中堆砌多个行动召唤按钮——每个屏幕最多两个操作按钮

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 移动端 | <768px | 单列布局，汉堡导航取代水平标签，主视觉文字缩小至约 28px，行动召唤按钮垂直堆叠，分类卡片变为全宽 |
| 平板 | 768-1024px | 双列导航面板，主视觉保持全视口高度，行动召唤按钮保持并排，水平内边距减小 |
| 桌面 | 1024-1440px | 完整水平导航，下拉框中的三列车辆栅格，主视觉标题 40px，并排行动召唤按钮 200px/160px 宽度 |
| 大桌面 | >1440px | 内容保持居中，主视觉摄影缩放以填充更宽的视口，导航面板内容的最大宽度容器 |

### 触控目标
- 主要行动召唤按钮：200px × 40px 最小尺寸（远高于 WCAG 44×44px 要求）
- 导航按钮：最小 32px 高度，4px 16px 内边距——足够的触控目标
- 轮播箭头：视口边缘约 44px 方形白色半透明按钮
- 文字链接（"了解"、"下单"）：14px 文字，具有足够行高间距用于触控

### 折叠策略
- **导航**：水平分类按钮（车辆、能源、充电、探索、商店）在移动端折叠为汉堡/抽屉菜单
- **主视觉行动召唤按钮对**：桌面端并排按钮在移动端垂直堆叠
- **分类卡片**：双栏水平布局在移动端折叠为单列全宽
- **车辆栅格**：桌面导航面板中的三列栅格在平板端变为双列，在移动端变为单列
- **间距**：区段垂直内边距保持慷慨（视口高度区段），但水平内边距减小

### 图像行为
- 主视觉图像在各断点完全响应，填充整个视口
- 车辆轮播图像使用 `object-fit: cover` 在各宽度保持电影感构图
- 导航面板中的透明 PNG 车辆图像在其栅格单元内按比例缩放
- 分类卡片图像保持其风景比例，通过带圆角的 `overflow: hidden` 裁剪

### 迭代指南
在优化使用此设计系统生成的现有屏幕时：
1. 一次只专注于一个组件——Tesla 的系统如此极简，每个元素都必须完美像素
2. 引用本文档中的具体颜色名称和 hex 值——整个系统只有 6-7 种颜色
3. 使用自然语言描述，而非 CSS 值——"略微圆润的角"而非"border-radius: 4px"
4. 在具体尺寸之外描述期望的"感觉"——"区段之间的画廊般的静默"比"margin-bottom: 100vh"更能传达留白哲学
5. 始终验证摄影是否在承担情感重量——如果 UI 本身感觉"被设计过"，那就过度了
