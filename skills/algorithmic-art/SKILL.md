## 算法哲学创建

首先，创建一个算法哲学（不是静态图像或模板），将通过以下方式解释：
- 计算过程、涌现行为、数学之美
- 种子随机性、噪声场、有机系统
- 粒子、流动、场、力
- 参数变化和受控混沌

### 关键理解
- 收到什么：用户的一些微妙输入或指令，需要考虑，但作为基础；它不应限制创作自由。
- 创建什么：算法哲学/生成美学运动。
- 接下来发生什么：同一版本接收哲学并用代码表达它——创建 90% 算法生成、10% 基本参数的 p5.js 草图。

考虑这种方法：
- 为生成艺术运动写宣言
- 下一阶段涉及编写使其栩栩如生的算法

哲学必须强调：算法表达。涌现行为。计算之美。种子变化。

### 如何生成算法哲学

**命名运动**（1-2 个词）："有机湍流" / "量子谐波" / "涌现的宁静"

**阐述哲学**（4-6 段——简洁但完整）：

为了捕捉算法本质，表达此哲学如何通过以下方式体现：
- 计算过程和数学关系？
- 噪声函数和随机性模式？
- 粒子行为和场动力学？
- 时间演化和系统状态？
- 参数变化和涌现复杂性？

**关键指南：**
- **避免冗余**：每个算法方面应只提及一次。避免重复关于噪声理论、粒子动力学或数学原理的概念，除非增加新的深度。
- **反复强调工艺**：哲学必须多次强调最终算法应看起来像是经过无数小时开发，经过精心完善，并来自该领域绝对顶尖的人。这种框架是必不可少的——重复诸如"精心制作的算法"、"深厚计算专业知识的产物"、"痛苦的优化"、"大师级实现"之类的短语。
- **留出创作空间**：对算法方向要具体，但要足够简洁，以便下一个 Claude 有空间进行解释性实现选择，具有极高的工艺水平。

哲学必须指导下一个版本以算法方式表达思想，而不是通过静态图像。美在于过程，而不是最终帧。

### 哲学示例

**"有机湍流"**
哲学：受自然法则约束的混乱，从无序中涌现的秩序。
算法表达：由分层 Perlin 噪声驱动的流场。数千个粒子跟随矢量力，它们的轨迹积累成有机密度图。多个噪声倍频程创建湍流区域和平静区域。颜色从速度和密度中涌现——快速粒子燃烧明亮，慢速粒子消退到阴影。算法运行直到平衡——精心调整的平衡，每个参数都由计算美学大师通过无数次迭代完善。

**"量子谐波"**
哲学：表现出波动干扰模式的离散实体。
算法表达：在网格上初始化的粒子，每个携带通过正弦波演化的相位值。当粒子接近时，它们的相位干扰——建设性干扰产生明亮的节点，破坏性产生空洞。简谐运动生成复杂的涌现曼陀罗。这是痛苦频率校准的结果，每个比率都经过精心选择以产生共鸣美。

**"递归耳语"**
哲学：跨尺度的自相似性，有限空间中的无限深度。
算法表达：递归细分的分支结构。每个分支略微随机化但受黄金比例约束。L 系统或递归细分生成感觉既数学又有机的树状形式。微妙的噪声扰动打破完美的对称性。线权重随每个递归级别减小。每个分支角度都是深入数学探索的产物。

**"场动力学"**
哲学：通过其对物质的影响变得可见的不可见力量。
算法表达：从数学函数或噪声构造的矢量场。粒子在边缘出生，沿着场线流动，在达到平衡或边界时死亡。多个场可以吸引、排斥或旋转粒子。可视化仅显示轨迹——不可见力量的幽灵般证据。通过力平衡精心编排的计算舞蹈。

**"随机结晶"**
哲学：结晶成有序结构的随机过程。
算法表达：随机圆填充或 Voronoi 镶嵌。从随机点开始，让它们通过松弛算法演化。细胞相互推挤直到平衡。基于细胞大小、邻居数量或与中心的距离着色。涌现的有机平铺感觉既随机又不可避免。每个种子都产生独特的晶体美——这是大师级生成算法的标志。

*这些是浓缩的示例。实际的算法哲学应该是 4-6 个实质性的段落。*

### 基本原则
- **算法哲学**：创建一个通过代码表达的计算世界观
- **过程优于产品**：始终强调美从算法的执行中涌现——每次运行都是独特的
- **参数表达**：思想通过数学关系、力、行为进行沟通——不是静态构图
- **艺术自由**：下一个 Claude 以算法方式解释哲学——提供创作实现空间
- **纯生成艺术**：这是关于制作活着的算法，而不是具有随机性的静态图像
- **专家工艺**：反复强调最终算法必须感觉精心制作，通过无数次迭代完善，是计算美学领域绝对顶尖的人的深厚专业知识的产物

**算法哲学应该是 4-6 段长。** 用将预期愿景结合在一起的诗意计算哲学来填充它。避免重复相同的点。将此算法哲学作为 .md 文件输出。

### 技术要求

**种子随机性（Art Blocks 模式）**：
```javascript
// 始终使用种子以实现可重复性
let seed = 12345; // 或用户输入的哈希
randomSeed(seed);
noiseSeed(seed);
```

**参数结构 - 遵循哲学**：

要建立从算法哲学中自然涌现的参数，请考虑："这个系统的哪些品质可以调整？"

```javascript
let params = {
  seed: 12345,  // 始终包含种子以实现可重复性
  // 颜色
  // 添加控制您的算法的参数：
  // - 数量（多少？）
  // - 规模（多大？多快？）
  // - 概率（可能性多大？）
  // - 比率（什么比例？）
  // - 角度（什么方向？）
  // - 阈值（行为何时改变？）
};
```

**要设计有效的参数，请专注于系统需要可调谐的属性，而不是以"模式类型"的方式思考。**

**核心算法 - 表达哲学**：

**关键**：算法哲学应该决定要构建什么。

要通过代码表达哲学，请避免思考"我应该使用哪种模式？"，而应思考"如何通过代码表达这个哲学？"

如果哲学是关于**有机涌现**，请考虑使用：
- 随时间积累或增长的元素
- 受自然规则约束的随机过程
- 反馈循环和相互作用

如果哲学是关于**数学之美**，请考虑使用：
- 几何关系和比例
- 三角函数和谐波
- 创建意外模式的精确计算

如果哲学是关于**受控混沌**，请考虑使用：
- 严格边界内的随机变化
- 分叉和相变
- 从无序中涌现的秩序

**算法从哲学中流淌，而不是从选项菜单中。**

为了指导实现，让概念本质告知创意和原始选择。构建表达此特定愿景的东西。

**画布设置**：标准 p5.js 结构：
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 初始化您的系统
}

function draw() {
  // 您的生成算法
  // 可以是静态的（noLoop）或动画的
}
```

### 工艺要求

**关键**：要达到精通水平，请创建看起来像是由大师级生成艺术家通过无数次迭代涌现的算法。仔细调整每个参数。确保每个模式都有目的地涌现。这不是随机噪声——这是通过深厚专业知识完善的受控混沌。

- **平衡**：复杂性而没有视觉噪声，秩序而没有刚性
- **色彩和谐**：深思熟虑的调色板，不是随机的 RGB 值
- **构图**：即使在随机性中，也要保持视觉层次和流动
- **性能**：流畅执行，如果是动画的，则为实时优化
- **可重复性**：相同的种子始终产生相同的输出

### 输出格式

## 交互式制品创建

**提醒：`templates/viewer.html` 应该已经被读取（参见步骤 0）。使用该文件作为起点。**

为了允许探索生成艺术，请创建一个单一的、自包含的 HTML 制品。确保此制品在 claude.ai 或任何浏览器中立即工作——无需设置。将所有内容内联嵌入。

### 关键：固定与可变

`templates/viewer.html` 文件是基础。它包含所需的确切结构和样式。

**固定（始终完全按照显示包含）：**
- 布局结构（页眉、侧边栏、主画布区域）
- Anthropic 品牌标识（UI 颜色、字体、渐变）
- 侧边栏中的种子部分：
  - 种子显示
  - 上一个/下一个按钮
  - 随机按钮
  - 跳转到种子输入 + 转到按钮
- 侧边栏中的操作部分：
  - 重新生成按钮
  - 重置按钮

**可变（为每个作品定制）：**
- 整个 p5.js 算法（setup/draw/classes）
- 参数对象（定义艺术需要什么）
- 侧边栏中的参数部分：
  - 参数控件的数量
  - 参数名称
  - 滑块的最小/最大/步长值
  - 控件类型（滑块、输入等）
- 颜色部分（可选）：
  - 一些艺术需要颜色选择器
  - 一些艺术可能使用固定颜色
  - 一些艺术可能是单色的（不需要颜色控件）
  - 根据艺术的需要决定

**每个作品都应该有独特的参数和算法！** 固定部分提供一致的用户体验——其他所有内容都表达独特的愿景。

### 必需功能

**1. 参数控件**
- 数字参数的滑块（粒子数量、噪声比例、速度等）
- 调色板颜色的颜色选择器
- 参数变化时的实时更新
- 恢复默认值的重置按钮

**2. 种子导航**
- 显示当前种子编号
- "上一个"和"下一个"按钮以循环种子
- "随机"按钮用于随机种子
- 输入字段以跳转到特定种子
- 请求时生成 100 个变体（种子 1-100）

**3. 单个制品结构**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- 来自 CDN 的 p5.js - 始终可用 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 所有样式内联 - 干净、极简 */
    /* 画布在顶部，控件在下方 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 所有参数控件 -->
  </div>
  <script>
    // 所有 p5.js 代码内联在这里
    // 参数对象、类、函数
    // setup() 和 draw()
    // UI 处理程序
    // 所有内容自包含
  </script>
</body>
</html>
```

**关键**：这是一个单一的制品。没有外部文件，没有导入（除了 p5.js CDN）。所有内容内联。

**4. 实现细节 - 构建侧边栏**

侧边栏结构：

**1. 种子（固定）** - 始终完全按照显示包含：
- 种子显示
- 上一个/下一个/随机/跳转按钮

**2. 参数（可变）** - 为艺术创建控件：
```html
<div class="control-group">
    <label>参数名称</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```

添加与参数数量一样多的 control-group div。

**3. 颜色（可选/可变）** - 如果艺术需要可调整的颜色，则包括：
- 如果用户应该控制调色板，则添加颜色选择器
- 如果艺术使用固定颜色，则跳过此部分
- 如果艺术是单色的，则跳过

**4. 操作（固定）** - 始终完全按照显示包含：
- 重新生成按钮
- 重置按钮
- 下载 PNG 按钮

**要求**：
- 种子控件必须工作（上一个/下一个/随机/跳转/显示）
- 所有参数都必须有 UI 控件
- 重新生成、重置、下载按钮必须工作
- 保持 Anthropic 品牌标识（UI 样式，不是艺术颜色）
