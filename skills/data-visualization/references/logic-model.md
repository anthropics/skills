# Logic Model / Theory of Change — Visualization Guide

## Standard Component Sequence

```
Inputs → Activities → Outputs → Short-Term Outcomes → Long-Term Outcomes → Impact
```

Some funders use a condensed version:
```
Resources → Activities → Outputs → Outcomes → Impact
```

Always confirm which version the funder or organization uses before building.

## Visualization Approaches

| Component | Best Visual Treatment |
|---|---|
| Full Logic Model (all components) | Horizontal flowchart with labeled columns and connecting arrows |
| Outcome pathway only | Sankey diagram showing flow of participants through outcome stages |
| Output vs. Outcome progress | Side-by-side bar: planned vs. actual |
| Theory of Change narrative | Annotated flowchart with assumption callout boxes |

## Design Rules

- **Outputs are countable** (# trained, # sessions, # tools delivered) — use bar or number card
- **Outcomes are changes** (% who improved, # who demonstrated skill) — use comparison or trend
- **Impact is long-term and often attribution-challenged** — note this explicitly; use cautious language
- Color-code by component type consistently across the report
- Add an "Assumptions" sidebar listing the key conditions required for the theory to hold

## Funder Expectation Note

United Way, Gates Foundation, and most government funders expect to see:
- Baseline data at the output and outcome levels
- Progress-to-goal indicators (% of target reached)
- A clear distinction between outputs (what you did) and outcomes (what changed)
