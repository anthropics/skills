# Kirkpatrick Framework — Visualization Guide

## Four-Level Structure

| Level | Name | Measured By | Typical Visualization |
|---|---|---|---|
| L1 | Reaction | Satisfaction surveys, NPS, exit tickets | Likert bar chart, gauge |
| L2 | Learning | Pre/post assessments, knowledge checks | Paired bar / slope chart showing gain |
| L3 | Behavior | Observation data, implementation fidelity, walkthroughs | Stacked bar (fidelity tiers), dot plot |
| L4 | Results | Student outcomes, organizational KPIs, long-term indicators | Line trend, benchmark comparison bar |

## Visualization Principles for Kirkpatrick Reports

- **Show the causal chain**: Connect L1→L4 visually using a flow or pathway diagram as the report header
- **Disaggregate at L3 and L4**: Break behavior and results data by subgroup, cohort, or site
- **Anchor L4 to a baseline**: Always show pre-intervention baseline on the same axis
- **Label evidence tier**: Note whether L4 results meet ESSA Tier 1–4 evidence standards

## Sample KPI Card Structure (L1–L4 Dashboard)

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  L1 · 94%   │  │ L2 · +18pts │  │ L3 · 78%    │  │ L4 · +12%   │
│  Satisfied  │  │ Pre→Post    │  │ Fidelity    │  │ Proficiency │
└─────────────┘  └─────────────┘  └─────────────┘  └─────────────┘
```

## Common Pitfalls

- Reporting only L1 (satisfaction) as "evidence of effectiveness" — always push toward L3/L4
- Using % satisfied without showing N (sample size) — always display N
- Missing a comparison group at L4 — recommend adding district average or comparison cohort
