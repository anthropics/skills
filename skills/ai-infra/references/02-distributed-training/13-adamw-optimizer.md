## 从最朴素的 SGD 说起

梯度下降的基本思想——沿负梯度方向迈一步。最朴素的 SGD (Stochastic Gradient Descent) 就是：

```
θ = θ - lr × g
```

其中 `g = ∂loss/∂θ` 是梯度，`lr` 是学习率。

**SGD 的问题**：

```
等高线图 (loss surface):

         ┌──────────────────────────────────┐
         │    ╭──────────────────────╮       │
         │   ╭┤                      ├╮      │
         │  ╭┤│   SGD 的锯齿路径      │├╮     │
         │  │││     ↗↙↗↙↗↙→ ★        │││     │   ★ = 最优解
         │  ╰┤│                      │├╯     │
         │   ╰┤                      ├╯      │
         │    ╰──────────────────────╯       │
         └──────────────────────────────────┘

    - 窄长山谷中，SGD 会来回震荡（纵向梯度大，横向梯度小）
    - 收敛极慢，大量步数浪费在来回弹跳上
```

## 演化路径：SGD → Momentum → Adam → AdamW

```
SGD          →   SGD + Momentum   →    Adam         →   AdamW
(纯梯度)         (加动量，抑制震荡)    (自适应学习率)     (修正权重衰减)
```

### Momentum（动量）—— 第一个改进

想象一个球在山谷里滚动，它不会每步只看当前坡度，而是有"惯性"：

```
v_t = β₁ × v_{t-1} + g_t         ← 当前速度 = 惯性 + 当前梯度
θ = θ - lr × v_t
```

- `β₁` 通常 = 0.9，表示"记住 90% 的历史动量"
- **效果**：纵向震荡因方向反复正负抵消而减弱，横向持续同方向而加速

```
         ┌──────────────────────────────────┐
         │    ╭──────────────────────╮       │
         │   ╭┤                      ├╮      │
         │  ╭┤│  Momentum 的平滑路径  │├╮     │
         │  │││     → → → → → → ★    │││     │   几乎直线到达
         │  ╰┤│                      │├╯     │
         │   ╰┤                      ├╯      │
         │    ╰──────────────────────╯       │
         └──────────────────────────────────┘
```

### Adam（Adaptive Moment Estimation）—— 自适应学习率

Adam 的核心洞察：**不同参数需要不同的学习率**。

- 参数 A 的梯度一直很大 → 说明 loss surface 在这个方向很陡 → 应该用**小步长**
- 参数 B 的梯度一直很小 → 说明 loss surface 在这个方向很平坦 → 应该用**大步长**

Adam 同时维护两个滑动平均：

```
一阶矩 m (均值/动量):    m_t = β₁ × m_{t-1} + (1 - β₁) × g_t
二阶矩 v (方差):         v_t = β₂ × v_{t-1} + (1 - β₂) × g_t²
```

- `m_t`：梯度的**指数加权移动平均**（方向信息，类似 Momentum）
- `v_t`：梯度平方的**指数加权移动平均**（振幅信息，衡量梯度有多"抖"）

#### 偏差修正

因为 `m_0 = 0, v_0 = 0`，训练初期估计值偏小，需要修正：

```
m̂_t = m_t / (1 - β₁ᵗ)        ← t 是当前步数
v̂_t = v_t / (1 - β₂ᵗ)
```

当 `t=1, β₁=0.9` 时：`1 - 0.9¹ = 0.1`，所以 `m̂ = m / 0.1 = 10m`，把偏小的初始值放大回来。

随着 `t` 增大，`β₁ᵗ → 0`，修正系数 `→ 1`，不再产生显著影响。

#### 参数更新

```
θ = θ - lr × m̂_t / (√v̂_t + ε)
```

| 组件 | 含义 |
|------|------|
| `m̂_t` | 平滑后的梯度方向（"该往哪走"） |
| `√v̂_t` | 梯度的历史波动幅度（"这个方向有多陡"） |
| `ε` | 防止除零，通常 = 1e-8 |
| `lr / √v̂_t` | **自适应学习率**：梯度波动大→步长小，波动小→步长大 |

### 一个完整的数值例子

假设某个参数 `w`，连续 3 步的梯度为 `g₁=0.1, g₂=0.3, g₃=0.2`：

```
超参: β₁=0.9, β₂=0.999, lr=0.001, ε=1e-8

Step 1 (g=0.1):
  m₁ = 0.9×0 + 0.1×0.1 = 0.01
  v₁ = 0.999×0 + 0.001×0.01 = 0.00001
  m̂₁ = 0.01 / (1-0.9¹) = 0.01 / 0.1 = 0.1
  v̂₁ = 0.00001 / (1-0.999¹) = 0.00001 / 0.001 = 0.01
  更新: Δw = 0.001 × 0.1 / (√0.01 + 1e-8)
           = 0.001 × 0.1 / 0.1
           = 0.001               ← 注意：几乎和 lr 一样大！

Step 2 (g=0.3):
  m₂ = 0.9×0.01 + 0.1×0.3 = 0.039
  v₂ = 0.999×0.00001 + 0.001×0.09 = 0.0000999 + 0.00009 = 0.0001899
  m̂₂ = 0.039 / (1-0.81) = 0.039 / 0.19 = 0.2053
  v̂₂ = 0.0001899 / (1-0.998) = 0.0001899 / 0.002 = 0.09495
  更新: Δw = 0.001 × 0.2053 / (√0.09495 + 1e-8)
           = 0.001 × 0.2053 / 0.3081
           ≈ 0.000666
```

**关键观察**：无论梯度是 0.1 还是 0.3，更新量 `Δw` 都在 `0.001` 这个量级，Adam 自动"归一化"了步长！这就是**自适应**的含义。

## Adam 的致命问题：权重衰减搞错了

### 什么是权重衰减？

权重衰减是一种**正则化**手段，防止模型参数长得太大（太大容易过拟合）：

```
核心思想：每次更新时，让参数向 0 缩小一点点

L2 正则化: loss_new = loss + (λ/2) × Σ(θ²)     ← 在 loss 上加惩罚项
权重衰减:   θ = θ × (1 - lr × λ) - lr × g        ← 每步直接缩小参数
```

在普通 SGD 中，**L2 正则化和权重衰减是等价的**（可以数学证明）。

### Adam + L2 正则化 ≠ 真正的权重衰减！

原始 Adam 论文中用的是 L2 正则化（把 `λθ` 加到梯度里）：

```
Adam (原始，错误的方式):
  g_t = ∂loss/∂θ + λ × θ           ← 正则项梯度也混入 g
  m_t = β₁ × m_{t-1} + (1-β₁) × g_t
  v_t = β₂ × v_{t-1} + (1-β₂) × g_t²
  θ = θ - lr × m̂_t / (√v̂_t + ε)
```

**问题在哪？** 正则项 `λθ` 也被 `v_t` 的自适应缩放给"归一化"掉了！

```
┌──────────────────────────────────────────────────────────┐
│  Adam + L2 的问题：                                       │
│                                                           │
│  梯度 g = g_task + λθ    ← 正则项混入梯度                 │
│       ↓                                                   │
│  v_t 统计了包含正则项的梯度平方                             │
│       ↓                                                   │
│  自适应学习率 lr/√v̂ 把正则项的效果也一起缩放了             │
│       ↓                                                   │
│  正则化的强度被不均匀地削弱了！                             │
│  梯度大的参数 → 正则化几乎无效                             │
│  梯度小的参数 → 正则化过强                                 │
└──────────────────────────────────────────────────────────┘
```

### AdamW 的修正

Ilya Loshchilov & Frank Hutter 在 2017 年论文 *"Decoupled Weight Decay Regularization"* 中提出了 **AdamW**——把权重衰减**解耦**出来，不通过梯度传递：

```
AdamW (正确的方式):
  g_t = ∂loss/∂θ                        ← 纯梯度，不含正则项！
  m_t = β₁ × m_{t-1} + (1-β₁) × g_t
  v_t = β₂ × v_{t-1} + (1-β₂) × g_t²
  m̂_t = m_t / (1 - β₁ᵗ)
  v̂_t = v_t / (1 - β₂ᵗ)
  θ = θ - lr × (m̂_t / (√v̂_t + ε) + λ × θ)
                ~~~~~~~~~~~~~~~~~~~~~~   ~~~~~~
                Adam 自适应更新部分       权重衰减（直接作用于参数）
```

或者等价地写成两步：

```
θ = θ - lr × m̂_t / (√v̂_t + ε)     ← 第一步：Adam 梯度更新
θ = θ - lr × λ × θ                   ← 第二步：权重衰减（乘法缩小）
  = θ × (1 - lr × λ)                 ← 等价于每步乘以一个小于1的系数
```

**关键区别**：权重衰减不经过 `m_t, v_t` 的自适应处理，**均匀地、等比例地**衰减所有参数。

## AdamW 的完整算法

```
输入: 学习率 lr, 衰减率 λ, β₁=0.9, β₂=0.999, ε=1e-8
初始化: m₀ = 0, v₀ = 0, t = 0

for each training step:
    t = t + 1
    g_t = compute_gradient(loss, θ)       # 纯任务梯度

    # 更新一阶矩（动量）
    m_t = β₁ × m_{t-1} + (1 - β₁) × g_t

    # 更新二阶矩（自适应学习率）
    v_t = β₂ × v_{t-1} + (1 - β₂) × g_t²

    # 偏差修正
    m̂_t = m_t / (1 - β₁ᵗ)
    v̂_t = v_t / (1 - β₂ᵗ)

    # 参数更新 = Adam更新 + 解耦权重衰减
    θ = θ - lr × (m̂_t / (√v̂_t + ε) + λ × θ)
```

## 为什么 AdamW 是 LLM 训练的标配

```
┌─────────────────────────────────────────────────────────────────┐
│                    AdamW 的五大优势                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 自适应学习率                                                 │
│     └── 不同参数自动调整步长，大模型中数十亿参数                  │
│         不可能手工调每个参数的 lr                                 │
│                                                                  │
│  2. 动量平滑                                                     │
│     └── mini-batch 梯度噪声大，动量起到平滑作用                  │
│         避免单个 batch 的异常梯度导致参数剧烈变化                 │
│                                                                  │
│  3. 正确的正则化                                                 │
│     └── 解耦权重衰减让正则化均匀生效                             │
│         大模型尤其需要好的正则化防止过拟合                        │
│                                                                  │
│  4. 超参数稳定                                                   │
│     └── β₁=0.9, β₂=0.999, ε=1e-8 几乎是通用设置               │
│         只需调 lr 和 λ，大幅降低调参成本                         │
│                                                                  │
│  5. 与 LR Schedule 配合好                                        │
│     └── 通常搭配 warmup + cosine decay                          │
│         warmup 阶段让 m 和 v 先"热身"稳定下来                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 显存开销分析

这是 AdamW 在大模型训练中的关键代价：

```python
def calculate_adamw_memory(params):
    # 模型参数 (FP16): 2 bytes
    model_gb = params * 2 / 1e9

    # 梯度 (FP16): 2 bytes
    gradient_gb = params * 2 / 1e9

    # 优化器状态 (AdamW):
    # - FP32 主权重副本: 4 bytes
    # - FP32 momentum: 4 bytes
    # - FP32 variance: 4 bytes
    optimizer_gb = params * 12 / 1e9

    return model_gb, gradient_gb, optimizer_gb
```

**每个参数 AdamW 需要额外存储 12 字节**：

| 状态 | 精度 | 每参数字节 | 7B 模型 |
|------|------|-----------|---------|
| FP32 主权重副本 | FP32 | 4 bytes | 28 GB |
| 一阶矩 m (momentum) | FP32 | 4 bytes | 28 GB |
| 二阶矩 v (variance) | FP32 | 4 bytes | 28 GB |
| **小计** | | **12 bytes** | **84 GB** |

对比之下，模型参数本身 (FP16) 只需 2 bytes/参数 = 14 GB。**优化器状态是参数的 6 倍！**

这正是 ZeRO 优化的切入点——[ZeRO Stage 1](03-data-parallelism.md) 首先把优化器状态切分到多卡上。

## PyTorch 中的实际使用

```python
import torch

model = MyTransformer()

# 典型的 LLM 训练配置
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=3e-4,           # 峰值学习率 (会被 scheduler 调整)
    betas=(0.9, 0.95), # β₁, β₂ (LLM 常用 0.95 而非 0.999)
    eps=1e-8,           # 防除零
    weight_decay=0.1,   # 权重衰减率 λ
)

# 学习率调度器：warmup + cosine decay
scheduler = torch.optim.lr_scheduler.CosineAnnealingWarmRestarts(
    optimizer, T_0=warmup_steps
)

# 训练循环
for step, batch in enumerate(dataloader):
    optimizer.zero_grad()           # 清零旧梯度
    loss = model(batch)             # 前向 + 算 loss
    loss.backward()                 # 反向传播算梯度
    torch.nn.utils.clip_grad_norm_( # 梯度裁剪防爆炸
        model.parameters(), max_norm=1.0
    )
    optimizer.step()                # AdamW 更新参数
    scheduler.step()                # 更新学习率
```

> **注意** `betas=(0.9, 0.95)`：很多 LLM 训练（如 LLaMA）用 `β₂=0.95` 而非默认的 `0.999`，因为训练数据分布变化大，需要二阶矩更快地"遗忘"旧信息，适应新分布。

## 总结对比

| 优化器 | 核心思想 | 优点 | 缺点 |
|--------|---------|------|------|
| SGD | `θ -= lr × g` | 简单、显存少 | 震荡、慢 |
| SGD+Momentum | 加入动量 v | 加速收敛 | 还是全局同一 lr |
| Adam | m(方向) + v(自适应) | 自动调 lr，快 | 权重衰减被扭曲 |
| **AdamW** | **解耦权重衰减** | **正则化正确，LLM 训练标配** | **显存大（12x 参数）** |
