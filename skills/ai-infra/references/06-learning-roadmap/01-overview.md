### 各层次详细内容

#### Level 1：通用基础

| 技能 | 要求 | 学习资源 |
|------|------|----------|
| Python | 熟练编程、常用库 | Python 官方教程 |
| Linux | 文件系统、进程、网络 | Linux 命令行大全 |
| Docker | 容器化、镜像构建 | Docker 官方文档 |
| Git | 版本控制、协作流程 | Pro Git 书籍 |
| 网络基础 | TCP/IP、HTTP | 计算机网络自顶向下 |
| 数据结构与算法 | 基本数据结构、复杂度分析 | LeetCode |

#### Level 2：ML/DL 基础

| 技能 | 要求 | 学习资源 |
|------|------|----------|
| 机器学习基础 | 监督/无监督学习、常用算法 | 吴恩达 ML 课程 |
| 深度学习基础 | 神经网络、反向传播 | fast.ai 课程 |
| PyTorch/TensorFlow | 框架使用、模型训练 | 官方 Tutorial |
| Transformer | Attention 机制、架构理解 | 原论文 + 讲解 |

#### Level 3：AI Infra 核心

| 技能 | 要求 | 学习资源 |
|------|------|----------|
| GPU 架构 | SM、Warp、内存层次 | PMPP 书籍 |
| CUDA | 编程模型、内存管理 | NVIDIA CUDA Guide |
| 分布式训练 | DDP、ZeRO、并行策略 | PyTorch Distributed |
| 推理优化 | 量化、KV Cache、Batching | vLLM 文档 |
| Kubernetes | Pod、调度、资源管理 | K8s 官方文档 |
| MLOps | 实验跟踪、Pipeline、监控 | MLflow 文档 |

#### Level 4：高级主题

| 技能 | 要求 | 学习资源 |
|------|------|----------|
| 大模型训练 | 千亿级模型训练 | Megatron-LM |
| 3D 并行 | DP+TP+PP 组合 | DeepSpeed 文档 |
| LLM 推理优化 | PagedAttention、Flash | vLLM/TGI 源码 |
| 向量数据库 | Embedding 存储检索 | Milvus/Pinecone |
| RAG | 检索增强生成 | LangChain |
| 成本优化 | Spot 实例、资源利用率 | 云厂商文档 |

#### Level 5：专家级

| 技能 | 要求 | 学习资源 |
|------|------|----------|
| Kernel 优化 | 自定义 CUDA Kernel | Triton/CUTLASS |
| 框架源码 | 深入理解框架实现 | PyTorch/vLLM 源码 |
| 系统设计 | 大规模 AI 系统架构 | 业界实践 |
| 前沿研究 | 跟踪最新论文和趋势 | arXiv/顶会论文 |

## 技能层次定义

### 入门 → 熟练 → 精通

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       技能熟练度定义                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   入门 (Beginner)                                                       │
│   ├── 能使用工具/框架完成基本任务                                         │
│   ├── 了解基本概念和术语                                                  │
│   └── 依赖文档和示例                                                      │
│                                                                         │
│   熟练 (Proficient)                                                     │
│   ├── 理解原理，能解释 Why                                               │
│   ├── 能独立排查问题                                                     │
│   └── 能进行性能优化                                                     │
│                                                                         │
│   精通 (Expert)                                                         │
│   ├── 深入源码级理解                                                     │
│   ├── 能设计系统/方案                                                    │
│   └── 能推动技术演进                                                     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 各领域技能定义示例

| 领域 | 入门 | 熟练 | 精通 |
|------|------|------|------|
| **GPU/CUDA** | 能用 PyTorch 写 GPU 代码 | 理解 SM/Warp/内存层次 | 能写高性能 CUDA kernel |
| **分布式训练** | 能用 DDP 训练 | 理解 ZeRO/FSDP 原理 | 能优化通信/设计并行策略 |
| **推理服务** | 能部署 vLLM | 理解 KV Cache/Batching | 能优化延迟/吞吐 |
| **K8s/调度** | 能部署 GPU Pod | 能配置 Quota/调度器 | 能设计多租户方案 |
| **MLOps** | 能用 MLflow | 能建 Pipeline | 能设计 ML 平台 |

## 学习原则

### 高效学习的七大原则

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    高效学习原则                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   1. 实践优先 (Practice First)                                          │
│      ├── 先跑代码，再读理论                                              │
│      ├── 遇到问题再深入学习                                              │
│      └── 70% 时间用于动手                                               │
│                                                                         │
│   2. 项目驱动 (Project-Driven)                                          │
│      ├── 以项目为学习单元                                                │
│      ├── 完整的端到端体验                                                │
│      └── 积累可展示的作品                                                │
│                                                                         │
│   3. 深度优先 (Depth First)                                             │
│      ├── 深入一个领域，而非浅尝多个                                       │
│      ├── 理解原理，而非仅会使用                                          │
│      └── 能解释 Why，而非仅知道 How                                      │
│                                                                         │
│   4. 费曼学习法 (Feynman Technique)                                     │
│      ├── 学完后能教给别人                                                │
│      ├── 写博客/做分享                                                  │
│      └── 简化表达检验理解                                                │
│                                                                         │
│   5. 刻意练习 (Deliberate Practice)                                     │
│      ├── 在舒适区边缘练习                                                │
│      ├── 针对薄弱点强化                                                  │
│      └── 获取反馈并调整                                                  │
│                                                                         │
│   6. 间隔重复 (Spaced Repetition)                                       │
│      ├── 定期复习，防止遗忘                                              │
│      ├── 关联新旧知识                                                    │
│      └── 建立知识网络                                                    │
│                                                                         │
│   7. 社区学习 (Community Learning)                                      │
│      ├── 参与开源项目                                                    │
│      ├── 加入技术社区                                                    │
│      └── 与高手交流                                                      │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 选择你的路径

### 快速路径选择器

回答以下问题，找到适合你的学习路径：

```
问题 1: 你的技术背景是什么？

A. 后端/系统/运维工程师 → 继续问题 2A
B. AI/ML 工程师/数据科学家 → 继续问题 2B
C. 学生/转行者/新人 → 继续问题 2C

问题 2A (后端背景):
你是否了解 K8s 和分布式系统？
├── 是 → 路径 A (后端转型)，从 Phase 1 (ML补课) 开始
└── 否 → 路径 A (后端转型)，但先补 K8s 基础

问题 2B (ML 背景):
你是否能解释 GPU 内存层次？
├── 是 → 路径 B (深入底层)，从 Phase 2 (分布式) 开始
└── 否 → 路径 B (深入底层)，从 Phase 1 (GPU) 开始

问题 2C (新人):
你是否有 Python 编程和 Linux 基础？
├── 是 → 路径 C (从零开始)，从 Month 2 (ML) 开始
└── 否 → 路径 C (从零开始)，从 Month 1 (基础) 开始
```

### 路径对比

| 维度 | 路径 A (后端转型) | 路径 B (ML深入) | 路径 C (从零) |
|------|-------------------|-----------------|---------------|
| 时间 | 4-6 个月 | 3-4 个月 | 6-9 个月 |
| 优势 | 系统基础好 | ML 基础好 | 时间充裕 |
| 重点 | ML + GPU 补课 | 硬件 + 系统 | 全面学习 |
| 难点 | ML 概念理解 | 系统运维知识 | 知识量大 |
| 目标 | AI 平台工程师 | 性能优化专家 | 全栈 AI 工程师 |
