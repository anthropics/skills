## 技能自评表

### 五大核心领域

| 技能领域 | 入门 | 熟练 | 精通 |
|----------|------|------|------|
| **GPU/CUDA** | 能用 PyTorch 写 GPU 代码 | 理解 SM/Warp/内存层次 | 能写高性能 CUDA kernel |
| **分布式训练** | 能用 DDP 训练 | 理解 ZeRO/FSDP 原理 | 能优化通信/设计并行策略 |
| **推理服务** | 能部署 vLLM | 理解 KV Cache/Batching | 能优化延迟/吞吐 |
| **K8s/调度** | 能部署 GPU Pod | 能配置 Quota/调度器 | 能设计多租户方案 |
| **MLOps** | 能用 MLflow | 能建 Pipeline | 能设计 ML 平台 |

### 自评打分指南

```
1分: 听说过，不了解
2分: 了解概念，未实践
3分: 入门 - 能完成基本任务
4分: 熟练 - 理解原理，能独立解决问题
5分: 精通 - 能设计方案，指导他人
```

## 技能发展建议

### 根据当前水平选择发展方向

| 当前水平 | 建议发展方向 |
|----------|--------------|
| 全部入门 | 先深入 1-2 个领域，建立信心 |
| 部分熟练 | 补齐短板，向全面发展 |
| 多数熟练 | 选择 1-2 个方向精通 |
| 部分精通 | 扩展到相关领域，成为专家 |

### 常见发展路径

```
路径 1: 训练方向
GPU/CUDA (入门) → 分布式训练 (熟练) → 分布式训练 (精通) → GPU/CUDA (熟练)

路径 2: 推理方向
推理服务 (入门) → 推理服务 (熟练) → GPU/CUDA (熟练) → 推理服务 (精通)

路径 3: 平台方向
K8s (入门) → K8s (熟练) → MLOps (熟练) → K8s (精通)
```

### 提升技能的方法

| 从入门到熟练 | 从熟练到精通 |
|--------------|--------------|
| 完成官方 Tutorial | 阅读源码 |
| 做入门项目 | 做复杂项目 |
| 理解基本概念 | 理解设计决策 |
| 跟着文档学 | 读论文学原理 |

## 技能评估检查点

### 每季度自评

建议每 3 个月进行一次技能自评：

1. 使用上面的自评表打分
2. 画出能力雷达图
3. 与上次对比，识别进步
4. 制定下一阶段学习目标
