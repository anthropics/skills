# 集群调度与资源管理详解

> GPU 资源昂贵，高效调度直接影响成本和效率。

## 📚 本章概述

本章深入探讨 AI 工作负载的集群调度和资源管理，帮助你构建高效、公平、可扩展的 GPU 集群。

## 🎯 学习目标

完成本章学习后，你将能够：

- [ ] 理解 AI 工作负载的调度挑战
- [ ] 掌握 Kubernetes 中 GPU 资源管理
- [ ] 配置和使用批调度器（Volcano/Yunikorn）
- [ ] 实现多租户资源隔离和配额管理
- [ ] 优化 GPU 集群成本

## 🔧 核心组件速览

| 组件 | 作用 | 关键配置 |
|------|------|----------|
| **Device Plugin** | GPU 资源发现和分配 | nvidia.com/gpu |
| **Volcano** | 批任务调度、Gang Scheduling | minAvailable |
| **Kueue** | 作业排队、配额管理 | ClusterQueue, LocalQueue |
| **ResourceQuota** | 命名空间资源配额 | requests/limits |
| **Priority Class** | 任务优先级和抢占 | value, preemptionPolicy |

## 📝 快速开始

### 安装 NVIDIA Device Plugin

```bash
kubectl apply -f https://raw.githubusercontent.com/NVIDIA/k8s-device-plugin/v0.14.0/nvidia-device-plugin.yml
kubectl get nodes -o json | jq '.items[].status.capacity | select(.["nvidia.com/gpu"])'
```

### 提交 GPU 任务

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: gpu-test
spec:
  template:
    spec:
      containers:
      - name: cuda
        image: nvidia/cuda:12.0-base
        command: ["nvidia-smi"]
        resources:
          limits:
            nvidia.com/gpu: 1
      restartPolicy: Never
```
