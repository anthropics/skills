## 一个学厨师的故事

想象你是一个**零基础的学徒厨师**，目标是学会做一道"西红柿炒蛋"。

你不能看菜谱（没有规则可以编程），只能通过**不断试错**来学习：

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                       │
│  学徒厨师的学习过程:                                                  │
│                                                                       │
│  第1次: 放了一大勺盐 → 师傅尝了一口 → "太咸了！" (评分: 2/10)       │
│         → 下次少放盐                                                  │
│                                                                       │
│  第2次: 少放了盐，但炒糊了 → 师傅: "火太大" (评分: 4/10)             │
│         → 下次火小点                                                  │
│                                                                       │
│  第3次: 盐少了，火小了，但鸡蛋太老 → 师傅: "鸡蛋放晚了" (评分: 6/10)│
│         → 下次先炒蛋再放番茄                                          │
│                                                                       │
│  ...                                                                  │
│                                                                       │
│  第100次: 色香味俱全 → 师傅: "完美！" (评分: 10/10)                  │
│                                                                       │
└──────────────────────────────────────────────────────────────────────┘
```

这个过程和 AI 训练**完全一样**：

| 学厨师 | AI 训练 |
|--------|---------|
| 厨师的手法（放多少盐、火开多大） | **模型参数**（权重 W、偏置 b） |
| 一道菜的食材 | **一个 batch 的训练数据** |
| 炒出一盘菜 | **前向传播**（用当前参数做预测） |
| 师傅打分："差多少分" | **损失函数**（量化预测有多差） |
| 师傅反馈："盐多了、火大了" | **反向传播**（计算每个参数的梯度） |
| 调整手法：少放盐、小点火 | **参数更新**（梯度下降） |
| 不断重复直到满分 | **训练循环**（直到 loss 收敛） |

## 训练循环的五个步骤

```
┌─────────────────────────────────────────────────────────────────┐
│                       单卡训练循环                                 │
│                                                                    │
│  ┌─────────────┐                                                  │
│  │  ① 数据准备  │  拿一批食材（32道菜的原料）                      │
│  └──────┬──────┘                                                  │
│         ↓                                                          │
│  ┌─────────────┐                                                  │
│  │  ② 前向传播  │  按当前手法炒菜（数据流过网络，得到预测）        │
│  └──────┬──────┘                                                  │
│         ↓                                                          │
│  ┌─────────────┐                                                  │
│  │  ③ 计算损失  │  师傅打分（比较预测和真实答案的差距）            │
│  └──────┬──────┘                                                  │
│         ↓                                                          │
│  ┌─────────────┐                                                  │
│  │  ④ 反向传播  │  师傅逐项反馈（计算每个参数对错误的"责任"）     │
│  └──────┬──────┘                                                  │
│         ↓                                                          │
│  ┌─────────────┐                                                  │
│  │  ⑤ 参数更新  │  调整手法（沿着"改进方向"微调每个参数）         │
│  └──────┬──────┘                                                  │
│         │                                                          │
│         └──────────→ 回到 ① 继续下一批                            │
│                                                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 用学厨师类比理解训练循环

### Step ①：数据准备——"今天练习用的食材"

厨师不可能同时炒 10 万道菜来找感觉，每次只拿一小批来练：

```
全校食堂的菜单: [菜1, 菜2, 菜3, ..., 菜100000]
                        ↓ 随机抽出一小批
今天练习批次: [菜42, 菜517, 菜3, ..., 菜891]    ← 32道菜 (Batch Size = 32)
```

| 术语 | 厨师类比 | 训练含义 |
|------|---------|---------|
| **Batch** | 今天要练的这一批菜 | 每次送入网络的一组数据 |
| **Batch Size** | 每批练几道菜 | 每批包含几个样本（32/64/128…） |
| **Epoch** | 把全部菜单练过一遍 | 遍历完整个数据集一次 |
| **Step/Iteration** | 练完一批菜 | 处理完一个 batch |

> **为什么不一次全练？** 10 万道菜（数据）太多，灶台放不下（GPU 显存有限）。每次练 32 道菜是效率和资源的最佳平衡。

### Step ②：前向传播——"按当前手法炒一盘菜"

把食材按照你**当前的手法**（参数），从头到尾做出一盘菜来：

```
食材: 番茄🍅 + 鸡蛋🥚
         ↓
    [切番茄: 切法=参数W₁]     ← 第一层
         ↓
    [打鸡蛋: 打法=参数W₂]     ← 第二层
         ↓
    [下锅翻炒: 火候=参数W₃]   ← 第三层
         ↓
    出锅: 预测结果 ŷ 🍳

前向传播 = 数据依次流过每一层，每层用自己的参数做变换
```

📖 **深入了解**：[03-前向传播详解](03-forward-propagation.md)

### Step ③：计算损失——"师傅打分"

把你炒的菜和"标准答案"（真实标签）做比较：

```
你炒的菜 ŷ: 有点咸，有点糊    (预测值)
标准答案 y:  色香味俱全        (真实标签)

师傅打分:
    loss = 差距有多大？= 6分差距

    loss 大 → 差得远 → 你的"手法"需要大幅调整
    loss 小 → 差不多 → 你快学会了
    loss ≈ 0 → 完美   → 出师了！
```

📖 **深入了解**：[04-损失函数详解](04-loss-function.md)

### Step ④：反向传播——"师傅逐项反馈"

这是最关键的一步。师傅不只说"不好吃"，而是**精确告诉你每个环节差在哪**：

```
师傅的反馈链条（从最终结果往回追溯）：

    "菜不好吃"
        ↓ 为什么？
    "因为太咸了" → 盐的影响: ∂loss/∂盐 = +3.5（盐多了，减少盐能改善）
        ↓ 为什么咸？
    "因为切番茄太碎，水分少" → 刀工的影响: ∂loss/∂刀工 = -1.2
        ↓ 还有呢？
    "火太大，炒糊了" → 火候的影响: ∂loss/∂火候 = +2.8

    反向传播 = 从结果出发，一层层往回追溯每个参数的"责任"
    梯度 = 每个参数对最终结果的影响程度（方向 + 大小）
```

📖 **深入了解**：[05-反向传播详解](05-backpropagation.md)

### Step ⑤：参数更新——"调整手法"

根据师傅的反馈，**微调**每个动作：

```
师傅说: "盐多了（梯度=+3.5），火大了（梯度=+2.8）"

调整幅度 = 学习率(lr) × 梯度

    盐: 当前 10克 → 10 - 0.1 × 3.5 = 9.65克    (少放盐)
    火: 当前 8档 → 8 - 0.1 × 2.8 = 7.72档       (调小火)
    
    学习率(lr) = 0.1 = "每次调整多大幅度"
    太大: 一下减到1克盐——太淡了！来回跳
    太小: 只减0.01克——进步太慢
```

📖 **深入了解**：[06-梯度下降与参数更新详解](06-gradient-descent.md)

## 为什么要"循环"

```
┌───────────────────────────────────────────────────────────────────┐
│                                                                    │
│  为什么一次不够？                                                  │
│                                                                    │
│  1. 一次只看了32道菜（一个batch），不够全面                        │
│     → 换一批菜继续练，见多识广                                    │
│                                                                    │
│  2. 每次只微调一点点（学习率小），不会一步到位                     │
│     → 积少成多，逐步逼近最优                                      │
│                                                                    │
│  3. 参数之间互相影响（调了盐，口感变了，火候也要跟着调）          │
│     → 需要反复迭代找到全局最优平衡点                              │
│                                                                    │
│  典型的 LLM 训练:                                                  │
│  ┌──────────────────────────────────────────────────────────────┐ │
│  │  模型: 7B 参数（70亿个需要调的"手法"）                       │ │
│  │  数据: 1T tokens（上万亿个"菜品"）                           │ │
│  │  Steps: ~100万步训练                                         │ │
│  │  时间: 数百GPU跑数周                                         │ │
│  └──────────────────────────────────────────────────────────────┘ │
│                                                                    │
└───────────────────────────────────────────────────────────────────┘
```

## 一个极简的完整例子

用**只有一个参数**的最简模型，完整走一遍训练循环：

```
任务: 学习 y = 3x（模型要自己发现"乘以3"这个规律）
模型: ŷ = w × x（只有一个参数 w，需要学到 w = 3）
初始: w = 1.0（随机初始化，不知道答案是3）
学习率: lr = 0.01

═══════════════════════════════════════════════════
  Step 1:  训练数据 (x=2, y=6)
═══════════════════════════════════════════════════
  ② 前向传播:  ŷ = 1.0 × 2 = 2.0         (预测: 2.0)
  ③ 计算损失:  loss = (2.0 - 6)² = 16.0   (差很远！)
  ④ 反向传播:  ∂loss/∂w = 2×(2.0-6)×2 = -16.0
               (w 增大 → ŷ 增大 → 更接近6 → loss减小 → 梯度为负)
  ⑤ 参数更新:  w = 1.0 - 0.01×(-16.0) = 1.16
               (w 增大了！朝 3.0 迈了一步)

═══════════════════════════════════════════════════
  Step 2:  训练数据 (x=1, y=3)
═══════════════════════════════════════════════════
  ② ŷ = 1.16 × 1 = 1.16
  ③ loss = (1.16 - 3)² = 3.39
  ④ ∂loss/∂w = 2×(1.16-3)×1 = -3.68
  ⑤ w = 1.16 - 0.01×(-3.68) = 1.197

═══════════════════════════════════════════════════
  Step 3:  训练数据 (x=3, y=9)
═══════════════════════════════════════════════════
  ② ŷ = 1.197 × 3 = 3.59
  ③ loss = (3.59 - 9)² = 29.23
  ④ ∂loss/∂w = 2×(3.59-9)×3 = -32.46
  ⑤ w = 1.197 - 0.01×(-32.46) = 1.522

  ... 继续训练 ...

═══════════════════════════════════════════════════
  训练轨迹:
═══════════════════════════════════════════════════
  Step:   1     2     3    ...   50   ...  100
  w:     1.00  1.16  1.20  ...  2.85  ... 2.998
  loss:  16.0  3.39  29.2  ...  0.2   ... 0.0001

  注意: Step 3 的 loss(29.2) 比 Step 1(16.0) 还大！
  这是正常的 —— 因为每步用的 x 不同 (Step 3 用 x=3，乘出来
  偏差更大)。但 w 一直在朝 3.0 逼近，整体趋势是改善的。

  结论: w 从 1.0 逐步逼近 3.0，模型"学会"了 y = 3x！
```

## 从单卡到分布式

理解了单卡训练循环后，你就能理解为什么需要分布式训练了：

```
单卡训练的瓶颈:

┌──────────────────────────────────────────────────────────┐
│                                                           │
│  以 7B 模型 + AdamW 为例:                                 │
│                                                           │
│  参数 (FP16):     14 GB    ← 70亿个"手法"                │
│  梯度 (FP16):     14 GB    ← 70亿个"反馈"                │
│  优化器 (FP32):   84 GB    ← AdamW: 参数FP32副本+动量+方差  │
│  激活值:          10+ GB    ← 前向传播的中间结果           │
│  ─────────────────────────                                │
│  总计:            122+ GB                                  │
│                                                           │
│  A100 显存:       80 GB    ← 装不下！                     │
│                                                           │
│  → 必须多张卡一起训练（分布式训练）                       │
│                                                           │
└──────────────────────────────────────────────────────────┘
```
