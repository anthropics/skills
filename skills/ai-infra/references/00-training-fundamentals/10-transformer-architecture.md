## 为什么需要 Transformer

### 在 Transformer 之前

```
早期处理文本的方式: RNN (循环神经网络)

    "我" → "今天" → "很" → "开心"
     ↓        ↓       ↓       ↓
    RNN → RNN → RNN → RNN → 输出
     └──传递──┘  └──传递──┘

    问题 1: 串行处理，一个词一个词地读
            → 无法利用 GPU 的并行计算能力
            → 训练很慢!

    问题 2: 长距离遗忘
            读到第 1000 个词时，第 1 个词的信息已经很模糊了
            就像传话游戏，传了 1000 人后原话面目全非

    问题 3: 无法并行
            处理第 5 个词必须等第 4 个处理完
            → GPU 大量算力被浪费
```

### Transformer 的解决方案

```
Transformer 的处理方式:

    "我"   "今天"   "很"   "开心"
     ↓       ↓       ↓       ↓
    ┌─────────────────────────────┐
    │   Self-Attention             │
    │   每个词同时看所有其他词     │
    │   "我"←→"今天"←→"很"←→"开心" │
    └─────────────────────────────┘
     ↓       ↓       ↓       ↓
    输出     输出    输出     输出

    优势 1: 并行处理! 所有词同时计算 → GPU 满载运行
    优势 2: 直接看到! 第 1 个词和第 1000 个词可以直接交互
    优势 3: 灵活性! 自动学习哪些词之间关系重要

    代价: 计算量 ∝ seq² (每个词都要看所有其他词)
          序列长度翻倍 → 注意力计算量翻 4 倍
```

## Transformer 全景图

```
一个 Decoder-Only Transformer (GPT/LLaMA 用的架构):

    输入文本: "今天天气"
         │
         ▼
    ┌─────────────────────────────────────────────────┐
    │  Token Embedding + 位置编码                       │
    │  把文字变成数字向量，并注入位置信息               │
    └───────────────────────┬─────────────────────────┘
                            │
        ┌───────────────────▼───────────────────┐
        │           Transformer Layer × N        │
        │  ┌─────────────────────────────────┐  │
        │  │  Layer Norm                      │  │
        │  │  ↓                               │  │
        │  │  Multi-Head Self-Attention       │  │  ← 词与词之间交流
        │  │  ↓ + 残差连接                    │  │
        │  │  Layer Norm                      │  │
        │  │  ↓                               │  │
        │  │  FFN (前馈网络)                  │  │  ← 每个词独立思考
        │  │  ↓ + 残差连接                    │  │
        │  └─────────────────────────────────┘  │
        │  重复 N 次 (LLaMA-7B: N=32)           │
        └───────────────────┬───────────────────┘
                            │
                            ▼
    ┌─────────────────────────────────────────────────┐
    │  LM Head: 把向量映射到词表概率                    │
    │  → P("很好")=0.35, P("不错")=0.25, ...          │
    └─────────────────────────────────────────────────┘
                            │
                            ▼
                    输出: "很好"
```

## Token Embedding：文字变数字

```
神经网络只能处理数字，所以第一步是把文字变成数字向量。

═══════════════════════════════════════════════════════════════
  Step 1: Tokenization (分词)
═══════════════════════════════════════════════════════════════

    "今天天气很好" → ["今天", "天气", "很", "好"]
    
    每个 token 对应词表中的一个 ID:
    "今天" → 1523
    "天气" → 8901
    "很"   → 234
    "好"   → 567

    词表大小: LLaMA = 32000 个 token
    (包含常见的词、子词、字符)

═══════════════════════════════════════════════════════════════
  Step 2: Embedding (向量化)
═══════════════════════════════════════════════════════════════

    每个 token ID 查表得到一个 4096 维的向量:

    "今天" (1523) → [0.12, -0.34, 0.56, ..., 0.78]  (4096 个数字)
    "天气" (8901) → [0.45, 0.23, -0.67, ..., 0.11]  (4096 个数字)
    "很"   (234)  → [-0.33, 0.89, 0.12, ..., -0.45] (4096 个数字)
    "好"   (567)  → [0.67, -0.12, 0.34, ..., 0.90]  (4096 个数字)

    Embedding 表的大小:
    词表大小 × 隐藏维度 = 32000 × 4096 ≈ 1.3 亿参数

    这些向量是随机初始化的，通过训练学到的!
    训练完成后:
    - 意思相近的词 → 向量靠近 ("好" 和 "棒" 的向量相似)
    - 意思不同的词 → 向量远离 ("好" 和 "坏" 的向量不同方向)

    最终输入形状: [batch, seq, 4096]
    如 4 条句子 × 每条 2048 个 token × 每个 token 4096 维
```

## 位置编码：让模型知道词的顺序

```
问题: Self-Attention 是"集合操作"，不区分词的顺序!
     "猫追狗" 和 "狗追猫" 对它来说一样 → 这不行!

═══════════════════════════════════════════════════════════════
  解决: 给每个位置加上一个"位置信号"
═══════════════════════════════════════════════════════════════

    位置 0 ("今天"): [0.12, -0.34, ...]  +  [1.0, 0.0, 0.84, ...]
                      └──词向量──┘       └──位置编码──┘
                            ↓
                      [1.12, -0.34, 0.84, ...]  (包含了"我是第0个词"的信息)

    位置 1 ("天气"): [0.45, 0.23, ...]  +  [0.54, 0.84, 0.0, ...]
                            ↓
                      [0.99, 1.07, ...]  (包含了"我是第1个词"的信息)

═══════════════════════════════════════════════════════════════
  不同的位置编码方案
═══════════════════════════════════════════════════════════════

  1. 绝对位置编码 (GPT 系列):
     每个位置有一个可学习的向量
     position_embedding = Embedding(max_seq_len, hidden_dim)
     → 简单直接，但超过训练长度就不行了

  2. RoPE 旋转位置编码 (LLaMA/现代主流):
     不直接加到向量上，而是在注意力计算时旋转 Q 和 K
     → 可以外推到更长的序列
     → 这就是为什么 LLaMA 可以从 4K 扩展到 128K

     直觉: 把位置信息"编织"到注意力的查询和匹配过程中
           位置 0 旋转 0°，位置 1 旋转 θ°，位置 2 旋转 2θ°...
           两个词的距离越远，它们的 Q·K 受位置的调制越大
```

## Self-Attention：核心中的核心

### 直觉理解

```
Self-Attention 的核心思想:
    对于每个词，去"查看"所有其他词，
    然后根据相关性加权汇总信息。

类比: 你在写读书笔记的"总结"

    原文: "小明 喜欢 吃 苹果 因为 苹果 很 甜"

    当你在理解第 6 个"苹果"时:
    - "吃" → 高权重 (苹果是被吃的对象)
    - "甜" → 高权重 (苹果的属性)
    - "小明" → 中等权重 (是谁在吃)
    - "因为" → 低权重 (功能词，关系不大)

    Self-Attention 自动学习这种"注意力分配"
```

### Q、K、V 是什么

```
每个词会被投影成三个向量: Query (Q), Key (K), Value (V)

═══════════════════════════════════════════════════════════════
  类比: 图书馆查书
═══════════════════════════════════════════════════════════════

  Q (Query) = 你的"搜索关键词"
      "我想找关于苹果口感的信息"

  K (Key) = 每本书的"标签/索引"
      书1标签: "水果种类"
      书2标签: "水果口感"
      书3标签: "种植技术"

  V (Value) = 每本书的"实际内容"
      书1内容: "苹果是一种水果..."
      书2内容: "苹果吃起来很甜脆..."
      书3内容: "苹果树需要..."

  过程:
  1. Q 和每个 K 做匹配 → 得到相关度分数
     Q·K₁ = 0.3  (有点相关)
     Q·K₂ = 0.9  (非常相关!)
     Q·K₃ = 0.1  (不太相关)

  2. Softmax 归一化 → 变成注意力权重 (加起来=1)
     [0.3, 0.9, 0.1] → [0.2, 0.6, 0.2]

  3. 用权重加权 V → 得到综合结果
     0.2×V₁ + 0.6×V₂ + 0.2×V₃ = "偏重口感信息的综合"

═══════════════════════════════════════════════════════════════
  具体计算 (用 LLaMA-7B 的数字)
═══════════════════════════════════════════════════════════════

  输入: X = [seq_len, 4096]   (每个 token 一个 4096 维向量)

  Step 1: 线性投影得到 Q, K, V
  ────────────────────────────
  Q = X @ W_Q    W_Q: [4096, 4096]   → Q: [seq_len, 4096]
  K = X @ W_K    W_K: [4096, 4096]   → K: [seq_len, 4096]
  V = X @ W_V    W_V: [4096, 4096]   → V: [seq_len, 4096]

  直觉:
  - W_Q 学习"怎么提问" → 把原始向量变成"搜索用的关键词"
  - W_K 学习"怎么被搜到" → 把原始向量变成"方便匹配的标签"
  - W_V 学习"提供什么信息" → 把原始向量变成"要传递的内容"
  
  这三个矩阵是训练出来的! 模型通过训练学会了最优的投影方式。
```

### 注意力分数的计算

```
Step 2: 计算注意力分数
────────────────────────

  Attention(Q, K, V) = Softmax(Q × K^T / √d_k) × V

  逐步拆解:

  ① Q × K^T: 计算每对词之间的"匹配度"
  ─────────────────────────────────────
  Q: [seq_len, d]    K^T: [d, seq_len]
  → 结果: [seq_len, seq_len]   ← 每个词对所有词的分数!

  以 4 个 token 为例:
                    "今天" "天气" "很"  "好"
         "今天"  │  0.8    0.5   0.1   0.2  │
  QK^T = "天气"  │  0.6    0.9   0.3   0.4  │
         "很"    │  0.1    0.3   0.2   0.7  │
         "好"    │  0.2    0.5   0.8   0.3  │

  含义: 第 i 行第 j 列 = 第 i 个词对第 j 个词的关注度
  例: "好"对"很"的关注度=0.8 (因为"很好"常搭配)

  ② 除以 √d_k: 缩放，防止分数太大
  ──────────────────────────────────
  d_k = 每个头的维度 = 4096/32 = 128
  √128 ≈ 11.3

  为什么要缩放？
  如果 d_k=128，Q·K 的结果可能很大 (如 50~100)
  Softmax 对大数值非常敏感 → 几乎变成 one-hot
  → 梯度消失，训练不动!
  除以 √d_k 让分数保持在合理范围

  ③ Causal Mask: 不能偷看后面的词!
  ─────────────────────────────────
  LLM 是自回归的，生成第 3 个词时不能看到第 4 个词
  → 把"未来"的注意力分数设为 -∞ (Softmax 后变成 0)

                   "今天" "天气" "很"  "好"
        "今天"  │  0.8    -∞    -∞    -∞   │
  Mask= "天气"  │  0.6    0.9   -∞    -∞   │
        "很"    │  0.1    0.3   0.2   -∞   │
        "好"    │  0.2    0.5   0.8   0.3  │

  → "今天"只能看到自己
  → "天气"能看到"今天"和自己
  → "好"能看到前面所有词

  ④ Softmax: 归一化为概率分布
  ────────────────────────────
  每行做 Softmax → 每行加起来 = 1

                    "今天" "天气" "很"  "好"
          "今天" │  1.0    0.0   0.0   0.0  │  (只看自己)
  权重 =  "天气" │  0.3    0.7   0.0   0.0  │  (更关注自己)
          "很"   │  0.15   0.25  0.60  0.0  │  (最关注自己)
          "好"   │  0.1    0.2   0.5   0.2  │  (最关注"很")

  ⑤ 乘以 V: 加权汇总
  ─────────────────────
  Output = 权重矩阵 × V    → [seq_len, d]

  "好"的新表示 = 0.1×V("今天") + 0.2×V("天气") + 0.5×V("很") + 0.2×V("好")
  → "好"的表示中融合了大量"很"的信息
  → 模型理解了"很好"这个搭配!
```

### 用一个完整的数值例子走一遍

```
═══════════════════════════════════════════════════════════════
  极简例子: 3 个 token, 维度 d=4 (实际中 d=4096)
═══════════════════════════════════════════════════════════════

  输入 X (已经经过 Embedding):
  "猫"  = [1.0, 0.0, 1.0, 0.0]
  "追"  = [0.0, 1.0, 0.0, 1.0]
  "鼠"  = [1.0, 1.0, 0.0, 0.0]

  假设 W_Q = W_K = W_V = 单位矩阵 (简化, 实际是训练出来的)
  → Q = K = V = X

  Step 1: Q × K^T
  ──────────────────
           "猫"  "追"  "鼠"
   "猫" │  2     0     1   │   (猫·猫=1+0+1+0=2, 猫·追=0, 猫·鼠=1)
   "追" │  0     2     1   │   (追·猫=0, 追·追=2, 追·鼠=1)
   "鼠" │  1     1     2   │   (鼠·猫=1, 鼠·追=1, 鼠·鼠=2)

  Step 2: 除以 √d_k = √4 = 2
  ──────────────────────────────
           "猫"  "追"  "鼠"
   "猫" │  1.0   0.0   0.5  │
   "追" │  0.0   1.0   0.5  │
   "鼠" │  0.5   0.5   1.0  │

  Step 3: Causal Mask (遮住未来)
  ──────────────────────────────
           "猫"  "追"  "鼠"
   "猫" │  1.0   -∞    -∞   │
   "追" │  0.0   1.0   -∞   │
   "鼠" │  0.5   0.5   1.0  │

  Step 4: Softmax (每行)
  ──────────────────────────
   "猫" │ softmax([1.0])       = [1.0,  0.0,  0.0 ] │
   "追" │ softmax([0.0, 1.0])  = [0.27, 0.73, 0.0 ] │
   "鼠" │ softmax([0.5, 0.5, 1.0]) = [0.27, 0.27, 0.45] │

  Step 5: 乘以 V
  ──────────────────
   "猫"的输出 = 1.0×[1,0,1,0] = [1.0, 0.0, 1.0, 0.0]
   (只看到自己, 所以输出不变)

   "追"的输出 = 0.27×[1,0,1,0] + 0.73×[0,1,0,1]
              = [0.27, 0.73, 0.27, 0.73]
   (融合了"猫"和"追"的信息, 偏向自己)

   "鼠"的输出 = 0.21×[1,0,1,0] + 0.21×[0,1,0,1] + 0.58×[1,1,0,0]
              = [0.79, 0.79, 0.21, 0.21]
   (综合了三个词的信息, 偏向自己)

  → 经过 Self-Attention 后，每个词的表示都融合了它能看到的所有词的信息!
```

## Multi-Head Attention：多角度看问题

### 为什么需要多个头

```
一个注意力头只能学一种"关注模式"。

但语言中的关系是多样的:
  "小明昨天在北京吃了一碗很好吃的拉面"

  需要同时捕捉:
  - 语法关系: "小明" ←→ "吃了"  (谁吃的？)
  - 时间关系: "昨天" ←→ "吃了"  (什么时候？)
  - 地点关系: "北京" ←→ "吃了"  (在哪吃的？)
  - 修饰关系: "好吃的" ←→ "拉面" (什么样的？)
  - 量词关系: "一碗" ←→ "拉面"   (多少？)

一个头很难同时学会所有这些关系!
→ 解决: 用多个头, 每个头学一种关系模式
```

### 多头注意力的计算过程

```
═══════════════════════════════════════════════════════════════
  LLaMA-7B: hidden=4096, 32 个头, 每个头 128 维
═══════════════════════════════════════════════════════════════

  Step 1: 投影到 Q, K, V
  ───────────────────────
  Q = X @ W_Q    → [seq, 4096]
  K = X @ W_K    → [seq, 4096]
  V = X @ W_V    → [seq, 4096]

  Step 2: 拆分成多个头
  ─────────────────────
  把 4096 维拆成 32 个头, 每个头 128 维:

  Q: [seq, 4096] → [seq, 32, 128]
                     └─32个头,每个128维─┘

  等价于:
  头 0:  Q₀ = Q[:, 0:128]     (取前 128 维)
  头 1:  Q₁ = Q[:, 128:256]   (取第 128~256 维)
  ...
  头 31: Q₃₁ = Q[:, 3968:4096] (取最后 128 维)
  K, V 同样拆分

  Step 3: 每个头独立做注意力
  ──────────────────────────
  头 0: Attn₀ = Softmax(Q₀ × K₀^T / √128) × V₀  → [seq, 128]
  头 1: Attn₁ = Softmax(Q₁ × K₁^T / √128) × V₁  → [seq, 128]
  ...
  头 31: Attn₃₁ = ...                              → [seq, 128]

  每个头可能学到不同的关注模式:
  头 0: 倾向于关注语法主谓关系
  头 5: 倾向于关注相邻词搭配
  头 12: 倾向于关注长距离指代
  头 20: 倾向于关注标点和结构
  (这些不是人为设定的, 是训练自动学出来的!)

  Step 4: 拼接所有头的结果
  ─────────────────────────
  Concat = [Attn₀ | Attn₁ | ... | Attn₃₁]  → [seq, 4096]
  (32 个头 × 每个 128 维 = 4096 维, 恢复原始维度)

  Step 5: 输出投影
  ─────────────────
  Output = Concat @ W_O    W_O: [4096, 4096]   → [seq, 4096]
  
  这一步把 32 个头的信息融合在一起

═══════════════════════════════════════════════════════════════
  参数量统计 (每层的注意力部分):
  W_Q + W_K + W_V + W_O = 4 × [4096 × 4096] = 4 × 1680万 = 6720万
═══════════════════════════════════════════════════════════════
```

### 为什么多头比单头好

```
┌──────────────────────────────────────────────────────────────┐
│                                                               │
│  单头 (d=4096):                                              │
│  一个 4096×4096 的注意力 → 只能学一种关注模式               │
│                                                               │
│  多头 (32×128):                                              │
│  32 个 128×128 的注意力 → 32 种不同的关注模式               │
│                                                               │
│  参数量完全一样! (4096² = 32 × 128²... 不对, 是切分维度)     │
│  实际上: 多头是把 4096 维"分工"给 32 个头                    │
│  → 不增加参数, 但大幅提升表达能力!                            │
│                                                               │
│  类比:                                                        │
│  单头 = 一个人什么都关注 → 什么都关注不好                    │
│  多头 = 32 个专家各关注一个方面 → 综合起来更全面             │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

## FFN（前馈网络）：独立思考

```
注意力机制之后，每个 token 已经融合了其他 token 的信息。
FFN 让每个 token 独立地做进一步的非线性变换。

═══════════════════════════════════════════════════════════════
  FFN 的结构 (LLaMA-7B):
═══════════════════════════════════════════════════════════════

  标准 FFN (GPT):
      x → Linear(4096→16384) → GeLU → Linear(16384→4096) → output
          └──────升维───────┘          └──────降维───────┘

  LLaMA 的 SwiGLU FFN (更强):
      x → Linear₁(4096→11008)  ─┐
      x → Linear_gate(4096→11008) → SiLU → ×  → Linear₂(11008→4096) → output
                                    └──门控──┘

  为什么先升维再降维？
  ─────────────────────
  类比: 解一道难题

  原始信息 (4096维) → "展开思考" (16384维) → "浓缩结论" (4096维)

  升维: 把信息展开到更高维空间, 在更大的空间里
        更容易找到有用的模式 (就像在更大的白板上画图更容易)
  降维: 把有用的信息压缩回原始大小

  为什么比值是 4×？
  这是 Transformer 论文的经验设计, 后来发现是个好的平衡点:
  - 太小: 表达能力不够
  - 太大: 参数量和计算量爆炸

═══════════════════════════════════════════════════════════════
  参数量 (LLaMA-7B 每层的 FFN):
═══════════════════════════════════════════════════════════════

  SwiGLU: 3 个矩阵
  Linear₁:    4096 × 11008 = 4510万
  Linear_gate: 4096 × 11008 = 4510万
  Linear₂:    11008 × 4096 = 4510万
  每层 FFN 总计: 约 1.35 亿参数

  这比注意力部分 (6720万) 还多!
  → FFN 占了 Transformer 参数量的大头 (~2/3)
```

```
FFN 的直觉作用:

  Self-Attention: "大家互相交流信息" (词与词之间交互)
  FFN: "每个人独立消化、思考" (词自己内部的非线性变换)

  类比:
  Self-Attention = 小组讨论 (听别人说什么)
  FFN = 个人思考 (消化吸收后形成自己的理解)

  研究发现: FFN 更像是一个"知识库"
  它的参数里存储了大量的世界知识 (事实、常识、模式)
  → 这也是为什么模型越大, 知道的"知识"越多
```

## Layer Norm 与残差连接：训练的稳定器

### Layer Norm

```
Layer Norm: 把每个 token 的特征值标准化到均值=0, 方差=1

为什么需要？
─────────────
经过多层变换后, 数值可能变得很大或很小:

  层 1 输出: [0.5, -0.3, 0.8, ...]
  层 5 输出: [50, -30, 80, ...]           ← 数值越来越大!
  层 10 输出: [5000, -3000, 8000, ...]    ← 训练不稳定!

Layer Norm 让数值保持在合理范围:

  输入: [50, -30, 80, 20]
  均值 μ = (50-30+80+20)/4 = 30
  方差 σ² = ... → σ ≈ 40
  标准化: [(50-30)/40, (-30-30)/40, (80-30)/40, (20-30)/40]
        = [0.5, -1.5, 1.25, -0.25]
  → 数值回到合理范围!

  再乘以可学习的 γ, 加上可学习的 β:
  output = γ × [0.5, -1.5, 1.25, -0.25] + β
  → 保持标准化的好处, 同时保留表达能力
```

### 残差连接 (Residual Connection)

```
残差连接: output = x + F(x)   (把输入直接加到输出上)

为什么需要？
─────────────
没有残差连接的深度网络:

  x → 层1 → 层2 → ... → 层32 → output
  信号要穿过 32 层变换, 梯度可能消失!

有残差连接的深度网络:

  x ──────────────────── + → output₁
  │                      ↑
  └→ Attention(x) ──────┘

  output₁ ───────────────── + → output₂
  │                          ↑
  └→ FFN(output₁) ─────────┘

  效果:
  1. 梯度可以"走捷径": 反向传播时梯度不仅走 F(x) 这条路,
     还能直接通过 +x 这条"高速公路"传回去
     → 缓解梯度消失!

  2. 每层只需学"残差" (修正量), 不需要学完整变换
     → 比从零开始学更容易
     → 类比: 修改文章比从头写更容易

  3. 最差情况: 如果某层学不好, F(x)≈0
     → output = x + 0 = x, 至少不会变差!
```

### Pre-Norm vs Post-Norm

```
┌──────────────────────────────────────────────────────────────┐
│                                                               │
│  Post-Norm (原始 Transformer):                               │
│  x → Attention → Add(+x) → LayerNorm → FFN → Add → LN      │
│  · 训练不太稳定, 需要 warmup                                │
│                                                               │
│  Pre-Norm (LLaMA/现代主流):                                  │
│  x → LayerNorm → Attention → Add(+x) → LN → FFN → Add      │
│  · 先 Norm 再计算, 训练更稳定                                │
│  · 几乎所有现代 LLM 都用 Pre-Norm                            │
│                                                               │
│  RMSNorm (LLaMA 实际用的):                                   │
│  · 简化版 LayerNorm, 只做缩放不做平移                        │
│  · 计算更快, 效果一样好                                       │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

## 完整的一层 Transformer

```
把所有组件拼在一起, 这就是 LLaMA-7B 一层的完整流程:

  Input x: [batch, seq, 4096]
     │
     │  ┌───────────────────────────────────────────────────┐
     ├──┤ 残差路径 (直接传过去)                              │
     │  └───────────────────────────────────┬───────────────┘
     ▼                                      │
  RMSNorm                                   │
     │                                      │
     ▼                                      │
  Multi-Head Self-Attention                 │
  (32 头, 每头 128 维)                      │
  Q=x@W_Q, K=x@W_K, V=x@W_V               │
  Attn = Softmax(QK^T/√128 + mask) × V     │
  Output = Attn @ W_O                       │
     │                                      │
     ▼                                      │
  Add (残差连接: + x) ◄─────────────────────┘
     │
     │  ┌───────────────────────────────────────────────────┐
     ├──┤ 残差路径 (直接传过去)                              │
     │  └───────────────────────────────────┬───────────────┘
     ▼                                      │
  RMSNorm                                   │
     │                                      │
     ▼                                      │
  SwiGLU FFN                                │
  gate = SiLU(x @ W_gate)                   │
  up = x @ W_up                             │
  down = (gate × up) @ W_down               │
     │                                      │
     ▼                                      │
  Add (残差连接) ◄──────────────────────────┘
     │
     ▼
  Output: [batch, seq, 4096]


  这一层的参数量:
  ┌──────────────────────────────────────────────────┐
  │ 注意力: W_Q + W_K + W_V + W_O                    │
  │         = 4 × 4096 × 4096 = 6700万              │
  │                                                   │
  │ FFN:    W_gate + W_up + W_down                    │
  │         = 3 × 4096 × 11008 = 1.35亿             │
  │                                                   │
  │ RMSNorm: 2 × 4096 = 8192 (可忽略)               │
  │                                                   │
  │ 每层总计: 约 2.02 亿参数                          │
  │ × 32 层 = 64.6 亿                                │
  │ + Embedding(1.3亿) + LM Head ≈ 70亿 = 7B ✓      │
  └──────────────────────────────────────────────────┘
```

## LLM 如何生成文本

```
理解了 Transformer 的结构, 来看它怎么一个词一个词地生成文本:

═══════════════════════════════════════════════════════════════
  自回归生成 (Auto-Regressive Generation)
═══════════════════════════════════════════════════════════════

  输入: "今天天气"

  Step 1: 整个序列送入 Transformer
  "今天天气" → 32 层 Transformer → 最后一个位置输出向量

  Step 2: LM Head 把向量变成词表概率
  向量 [4096] → Linear [4096, 32000] → Softmax → 概率分布
  P("很好")=0.15, P("不错")=0.12, P("晴朗")=0.10, ...

  Step 3: 采样一个词
  → 选 "很好" (概率最高, 或用 temperature 采样增加随机性)

  Step 4: 把新词拼回去, 重复
  "今天天气很好" → Transformer → P("，")=0.3, P("适合")=0.2, ...
  → 选 "，"
  "今天天气很好，" → Transformer → ...

  直到生成结束符 <EOS> 或达到最大长度

═══════════════════════════════════════════════════════════════
  为什么生成慢？
═══════════════════════════════════════════════════════════════

  生成 100 个 token 需要跑 100 次 Transformer!
  每次都要对所有已有 token 做注意力计算

  优化: KV Cache
  ──────────────
  关键观察: 生成第 N+1 个 token 时, 前 N 个 token 的 K 和 V
           和上次计算的完全一样! → 缓存起来不用重算

  没有 KV Cache:
  第 1 个: 算 1 个 token 的 Q,K,V
  第 2 个: 算 2 个 token 的 Q,K,V  ← 第1个重复算了!
  第 3 个: 算 3 个 token 的 Q,K,V  ← 前2个重复算了!
  → 计算量 ∝ N²

  有 KV Cache:
  第 1 个: 算 1 个 token 的 Q,K,V → 缓存 K₁,V₁
  第 2 个: 只算新 token 的 Q₂,K₂,V₂ → 用 Q₂ 和 [K₁,K₂] 做注意力
  第 3 个: 只算新 token 的 Q₃,K₃,V₃ → 用 Q₃ 和 [K₁,K₂,K₃] 做注意力
  → 计算量 ∝ N (线性!)

  代价: 需要额外显存存 KV Cache
  KV Cache 大小 = 2(K和V) × 层数 × seq_len × hidden × dtype
  LLaMA-7B, seq=2048: 2 × 32 × 2048 × 4096 × 2bytes = 1GB
  → 这就是为什么推理时显存也不少!
```

## 总结

```
┌──────────────────────────────────────────────────────────────┐
│                   Transformer 核心要点                         │
│                                                               │
│  整体结构:                                                    │
│  └── Token Embedding → N 层 Transformer → LM Head            │
│                                                               │
│  一层 Transformer:                                            │
│  ├── RMSNorm → Multi-Head Attention → 残差连接               │
│  └── RMSNorm → FFN (SwiGLU)        → 残差连接               │
│                                                               │
│  Self-Attention (核心):                                       │
│  ├── Q, K, V = 三种线性投影                                  │
│  ├── 注意力分数 = Softmax(QK^T/√d + mask)                    │
│  ├── 输出 = 注意力权重 × V                                   │
│  └── 多头: 32 个头各关注不同方面                              │
│                                                               │
│  FFN (参数大头):                                              │
│  └── 升维 → 非线性 → 降维 (存储世界知识)                     │
│                                                               │
│  稳定训练:                                                    │
│  ├── LayerNorm: 数值稳定                                     │
│  └── 残差连接: 梯度畅通                                      │
│                                                               │
│  生成方式:                                                    │
│  └── 自回归: 一个词一个词生成, 用 KV Cache 加速              │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

| 组件 | 作用 | 生活类比 | 参数占比 |
|------|------|---------|----------|
| Embedding | 文字→向量 | 把文字翻译成数学语言 | ~2% |
| Self-Attention | 词间交互 | 小组讨论，互相参考 | ~33% |
| FFN | 非线性变换 | 个人独立思考消化 | ~65% |
| LayerNorm | 数值稳定 | 中场休息调整状态 | <0.1% |
| 残差连接 | 梯度畅通 | 电梯直达（不用爬楼梯） | 0% (无参数) |
| LM Head | 向量→词表概率 | 把思考结果表达出来 | 共享 |
