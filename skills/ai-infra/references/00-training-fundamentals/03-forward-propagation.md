## 前向传播是什么

### 一句话定义

**前向传播 = 数据从输入层出发，依次流过每一层，最终到达输出层，得到预测结果。**

```
类比: 工厂流水线

    原材料                各道工序               成品
    (输入数据)          (网络的各层)           (预测结果)

    铁矿石 ──→ [冶炼] ──→ [铸造] ──→ [打磨] ──→ [喷漆] ──→ 一辆汽车
                 ↑          ↑          ↑          ↑
              工艺参数    工艺参数    工艺参数    工艺参数
              (W₁, b₁)   (W₂, b₂)   (W₃, b₃)   (W₄, b₄)

    - 每道工序用自己的"参数"（工艺设定）加工半成品
    - 原材料经过所有工序后，变成最终产品
    - 前向传播 = 原材料沿流水线从头走到尾
```

## 一个完整的生活例子：预测房价

### 场景设定

你要训练一个神经网络来预测房价。输入是房子的 3 个特征，输出是预测价格：

```
输入特征:
    x₁ = 面积 (㎡)     例: 100
    x₂ = 楼层          例: 5
    x₃ = 房龄 (年)     例: 10

网络结构: 2层
    输入层: 3个特征
    隐藏层: 4个神经元 (用 ReLU 激活)
    输出层: 1个神经元 (输出预测房价)
```

### 网络长什么样

```
        输入层           隐藏层 (4个神经元)          输出层
        (3个特征)        + ReLU 激活              (1个输出)
        
        ┌─────┐         ┌─────┐
   x₁ = 100 ─┼────┐  ┌─│ h₁  │──┐
        │     │    ├──┤ │ h₂  │──┤           ┌─────┐
   x₂ = 5  ──┼──┐ ├──┤ │ h₃  │──├───────────│ ŷ   │ → 预测房价
        │     │  ├─┤  │ │ h₄  │──┤           └─────┘
   x₃ = 10 ──┼──┘ │  └─│     │──┘
        └─────┘    │    └─────┘
                   │
              全连接: 每个输入连接到每个隐藏神经元
              
    参数:
        W₁: 4×3 矩阵 (4个神经元 × 3个输入) = 12个权重
        b₁: 4×1 向量 (每个神经元1个偏置)     = 4个偏置
        W₂: 1×4 矩阵 (1个输出 × 4个隐藏)   = 4个权重
        b₂: 1×1 标量                         = 1个偏置
        
    总参数: 12 + 4 + 4 + 1 = 21个
```

## 逐层计算详解

### 第一层：线性变换 + 激活

假设网络当前的参数如下（训练初期，随机初始化的）：

```
W₁ = ┌                          ┐     b₁ = ┌      ┐
     │  0.01   0.20  -0.05      │          │  0.5  │
     │  0.15  -0.10   0.08      │          │ -0.3  │
     │ -0.03   0.30   0.02      │          │  0.1  │
     │  0.20   0.05  -0.12      │          │  0.0  │
     └                          ┘          └      ┘
```

**Step 1a: 线性变换（矩阵乘法 + 加偏置）**

```
z₁ = W₁ × x + b₁

计算每个神经元:

神经元1: z₁₁ = 0.01×100 + 0.20×5 + (-0.05)×10 + 0.5
              = 1.0 + 1.0 + (-0.5) + 0.5
              = 2.0

神经元2: z₁₂ = 0.15×100 + (-0.10)×5 + 0.08×10 + (-0.3)
              = 15.0 + (-0.5) + 0.8 + (-0.3)
              = 15.0

神经元3: z₁₃ = (-0.03)×100 + 0.30×5 + 0.02×10 + 0.1
              = (-3.0) + 1.5 + 0.2 + 0.1
              = -1.2

神经元4: z₁₄ = 0.20×100 + 0.05×5 + (-0.12)×10 + 0.0
              = 20.0 + 0.25 + (-1.2) + 0.0
              = 19.05

z₁ = [2.0, 15.0, -1.2, 19.05]
```

**Step 1b: ReLU 激活**

```
h = ReLU(z₁) = [max(0, z) for z in z₁]

    ReLU(2.0)   = 2.0       ✓ 正数，保留
    ReLU(15.0)  = 15.0      ✓ 正数，保留
    ReLU(-1.2)  = 0.0       ✗ 负数，归零！（这个神经元"没激活"）
    ReLU(19.05) = 19.05     ✓ 正数，保留

h = [2.0, 15.0, 0.0, 19.05]

直觉理解:
    神经元1: "面积大，楼层还行" → 激活了，有信号传出
    神经元2: "面积大是主导因素" → 强烈激活
    神经元3: "面积大但取了负权重" → 没激活，这个特征组合不重要
    神经元4: "面积权重最大" → 强烈激活
```

### 第二层：输出预测

```
W₂ = [0.50, 0.30, -0.10, 0.25]     b₂ = 5.0

ŷ = W₂ × h + b₂
  = 0.50×2.0 + 0.30×15.0 + (-0.10)×0.0 + 0.25×19.05 + 5.0
  = 1.0 + 4.5 + 0.0 + 4.7625 + 5.0
  = 15.26 (万元)

预测房价: 15.26万元
```

### 前向传播全流程图

```
输入             第一层线性        ReLU激活         第二层线性       输出
x               z₁ = W₁x+b₁     h = ReLU(z₁)     ŷ = W₂h+b₂

┌─────┐         ┌──────────┐     ┌──────────┐     ┌──────────┐
│ 100 │         │   2.00   │     │   2.00   │     │          │
│   5 │ ──W₁──→│  15.00   │ ──→ │  15.00   │ ──W₂──→ 15.26  │
│  10 │   +b₁  │  -1.20   │     │   0.00 ← 被ReLU截断      │
│     │         │  19.05   │     │  19.05   │   +b₂  │       │
└─────┘         └──────────┘     └──────────┘     └──────────┘

整个过程就是: 矩阵乘法 → 加偏置 → 激活 → 矩阵乘法 → 加偏置 → 输出
```

## 矩阵乘法：前向传播的核心运算

### 为什么是矩阵乘法？

```
类比: 计算期末总成绩

    学生成绩:
    ┌──────┬──────┬──────┐
    │ 数学 │ 英语 │ 物理 │
    ├──────┼──────┼──────┤
    │  90  │  85  │  95  │   ← 小明的成绩
    └──────┴──────┴──────┘

    不同学校的加权方式不同:
    ┌────────┬──────┬──────┬──────┐
    │        │ 数学 │ 英语 │ 物理 │
    ├────────┼──────┼──────┼──────┤
    │ 理科班 │ 0.4  │ 0.2  │ 0.4  │   ← 重视数理
    │ 文科班 │ 0.2  │ 0.5  │ 0.3  │   ← 重视英语
    │ 综合班 │ 0.33 │ 0.33 │ 0.34 │   ← 平均
    └────────┴──────┴──────┴──────┘

    计算各班总分 = 权重矩阵 × 成绩向量:
    
    理科班: 0.4×90 + 0.2×85 + 0.4×95 = 91.0
    文科班: 0.2×90 + 0.5×85 + 0.3×95 = 89.0
    综合班: 0.33×90 + 0.33×85 + 0.34×95 = 90.1

    → 同一组成绩，不同的权重（不同的"看法"）得到不同的总分
    → 这就是矩阵乘法做的事：用不同的权重组合来提取不同的"特征"
```

### 矩阵乘法的 shape 规则

```
矩阵乘法: C = A × B

    A 的 shape: [m, n]    "m行n列"
    B 的 shape: [n, k]    "n行k列"
    C 的 shape: [m, k]    "m行k列"
                 ↑
            A 的列数必须等于 B 的行数！

在神经网络中:
    输入 x:   [batch_size, 输入维度]   = [32, 3]    ← 32个样本，3个特征
    权重 W:   [输出维度, 输入维度]     = [4, 3]     ← 4个神经元，3个输入
    
    输出 = W × xᵀ: [4, 3] × [3, 32] = [4, 32]
    转置后: [32, 4]                    ← 32个样本，每个有4个隐藏值
```

### 为什么 GPU 擅长这个？

```
矩阵乘法的特点: 大量独立的乘加运算

    C[i][j] = Σ A[i][k] × B[k][j]

    矩阵 [4096, 4096] × [4096, 4096]:
    → 需要 4096³ ≈ 687 亿次乘加运算
    → 但每个 C[i][j] 的计算是独立的！
    → 可以并行做 4096² ≈ 1680万个计算

    CPU: 几十个核 → 串行或少量并行 → 慢
    GPU: 上万个核 → 大规模并行 → 快几百倍

    这就是为什么 AI 训练离不开 GPU！
```

## 激活值：必须保存的中间结果

### 为什么要保存？

```
前向传播过程中的中间结果（激活值），反向传播时要用到！

    前向: x → z₁ → h → z₂ → ŷ → loss
    
    保存: [x, z₁, h, z₂, ŷ]    ← 这些都要存！
    
    反向传播时:
        计算 ∂loss/∂W₂ 需要用到 h
        计算 ∂loss/∂W₁ 需要用到 x 和 z₁
        
    如果不保存 → 反向传播时要重新前向算一遍 → 太慢了

类比:
    解一道数学大题，中间有很多步骤。
    如果你边算边擦掉前面的步骤，检查时就得全部重算。
    所以你保留了所有中间过程（草稿纸）。
    
    激活值 = 训练的"草稿纸"
```

### 激活值的显存开销

```
以 7B 模型为例 (batch_size=1, seq_len=2048):

    每层需要保存:
    - Attention 的 Q/K/V:  2048 × 4096 × 3 × 2bytes ≈ 48MB
    - Attention 输出:       2048 × 4096 × 2bytes ≈ 16MB
    - FFN 中间值:           2048 × 11008 × 2bytes ≈ 43MB
    - 各种归一化的输入:     ≈ 16MB
    每层约 ≈ 120MB

    32层总计: 32 × 120MB ≈ 3.8GB

    如果 batch_size=8: 3.8 × 8 ≈ 30GB！

    → 激活值是训练显存的"隐形大户"
    → 这就是为什么有"激活值重算"(Activation Checkpointing) 技术:
      只保存部分层的激活值，其他的反向传播时重新算
      用计算时间换显存空间
```

## Batch 前向传播：一次处理多个样本

### 为什么一次处理一批？

```
类比: 烤面包

    方式1: 一次烤1片 → 烤100片需要100次 → 太慢
    方式2: 一次烤4片 → 烤100片只需25次 → 快4倍
    方式3: 一次烤32片 → 烤100片只需4次 → 最快！
    
    GPU 就像一个大烤箱，一次能同时处理很多数据。
    Batch = 一次塞进烤箱的面包数量。
    
    GPU 计算矩阵乘法:
    处理1个样本:   W × x₁             → 用了 GPU 1% 的算力
    处理32个样本:  W × [x₁,x₂,...x₃₂] → 用了 GPU 30% 的算力
    时间几乎一样！（GPU 的并行能力被利用了）
```

### Batch 前向传播的数学

```
单个样本:
    h = ReLU(W₁ × x + b₁)        x shape: [3]
    ŷ = W₂ × h + b₂              ŷ shape: [1]

Batch (32个样本):
    H = ReLU(W₁ × X + b₁)        X shape: [32, 3]   ← 32个样本一起
    Ŷ = W₂ × H + b₂              Ŷ shape: [32, 1]   ← 32个预测一起出

    数学完全一样，只是 x 变成了矩阵 X（多了一个 batch 维度）
    GPU 可以并行计算所有样本 → 效率大幅提升
```

## LLM 的前向传播

### GPT 类模型的前向传播

```
输入: "今天天气真" → 预测下一个词

┌──────────────────────────────────────────────────────────────────┐
│                                                                   │
│  Step 1: 词嵌入 (Token Embedding)                                │
│    "今天" → [0.12, -0.34, 0.56, ...]    (变成4096维向量)        │
│    "天气" → [0.45, 0.23, -0.11, ...]                             │
│    "真"   → [-0.08, 0.67, 0.33, ...]                             │
│                                                                   │
│  Step 2: 位置编码 (Positional Encoding)                          │
│    告诉模型每个词在句子中的位置                                   │
│    "今天"在位置0，"天气"在位置1，"真"在位置2                   │
│                                                                   │
│  Step 3: 经过 32 层 Transformer                                  │
│    每层做:                                                        │
│    ├── 自注意力: "真"关注"天气"多一些，关注"今天"少一些      │
│    ├── 前馈网络: 进一步变换特征                                  │
│    └── 残差连接 + 归一化: 稳定训练                               │
│                                                                   │
│  Step 4: 输出层 (LM Head)                                        │
│    把最后一个词"真"的隐藏状态映射到词表大小 (32000)              │
│    输出: [P(好)=0.35, P(不)=0.15, P(棒)=0.12, ...]             │
│                                                                   │
│  预测: 下一个词最可能是 "好"                                     │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### LLM 前向传播的计算量

```
以 7B 模型、序列长度 2048 为例:

    每个 token 的前向传播:
    ≈ 2 × 参数量 = 2 × 7B = 14 GFLOP (140亿次浮点运算)
    
    为什么是 2×？
    → 矩阵乘法 [M×K] × [K×N] 需要 2MKN 次运算 (乘+加)
    
    2048 个 token:
    ≈ 2048 × 14 GFLOP ≈ 28.7 TFLOP
    
    A100 (FP16): 312 TFLOPS (每秒万亿次浮点运算)
    → 理论上一次前向传播: 28.7 / 312 ≈ 0.09秒
    
    加上反向传播 (约2倍前向):
    一个训练 step ≈ 0.09 × 3 ≈ 0.27秒
    
    实际上因为内存带宽等限制，通常是理论值的 40-60%
```

## PyTorch 代码实战

### 从零构建并观察前向传播

```python
import torch
import torch.nn as nn

# 定义一个简单的两层网络
class HousePriceNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.layer1 = nn.Linear(3, 4)    # 3个输入特征 → 4个隐藏神经元
        self.relu = nn.ReLU()
        self.layer2 = nn.Linear(4, 1)    # 4个隐藏 → 1个输出
    
    def forward(self, x):
        # 前向传播: 数据依次流过每一层
        z1 = self.layer1(x)       # 线性变换
        h = self.relu(z1)         # 激活
        y = self.layer2(h)        # 输出
        return y

model = HousePriceNet()

# 查看参数
for name, param in model.named_parameters():
    print(f"{name}: shape={param.shape}, 参数量={param.numel()}")

# 输出:
# layer1.weight: shape=torch.Size([4, 3]), 参数量=12
# layer1.bias:   shape=torch.Size([4]),    参数量=4
# layer2.weight: shape=torch.Size([1, 4]), 参数量=4
# layer2.bias:   shape=torch.Size([1]),    参数量=1
# 总参数: 21个

# 一个样本的前向传播
x = torch.tensor([100.0, 5.0, 10.0])    # 100㎡, 5楼, 10年
y_pred = model(x)
print(f"预测房价: {y_pred.item():.2f} 万元")

# 一个 batch 的前向传播
X = torch.randn(32, 3)                   # 32个样本
Y_pred = model(X)                         # 一次得到32个预测
print(f"Batch 预测 shape: {Y_pred.shape}")  # torch.Size([32, 1])
```

### 观察每层的输出

```python
# 手动观察每层的前向传播
x = torch.tensor([100.0, 5.0, 10.0])

# 第一层: 线性变换
z1 = model.layer1(x)
print(f"第一层线性输出 z1 = {z1.data}")
# 例: tensor([2.03, 15.12, -1.18, 19.05])

# ReLU 激活
h = model.relu(z1)
print(f"ReLU 激活后 h = {h.data}")
# 例: tensor([2.03, 15.12, 0.00, 19.05])  ← 负数变成了0！

# 第二层: 输出
y = model.layer2(h)
print(f"最终输出 ŷ = {y.item():.2f}")
```

## 总结

```
┌──────────────────────────────────────────────────────────────────┐
│                       前向传播要点总结                              │
│                                                                   │
│  是什么:                                                          │
│    数据从输入到输出，依次流过每一层                                │
│    每层做: 线性变换 (矩阵乘法 + 偏置) → 激活函数                 │
│                                                                   │
│  核心运算:                                                        │
│    矩阵乘法占 99% 计算量 → 这就是 GPU 的用武之地                 │
│                                                                   │
│  关键输出:                                                        │
│    预测结果 ŷ → 交给损失函数评估好坏                              │
│    激活值 (中间结果) → 保存起来，反向传播要用                     │
│                                                                   │
│  Batch 处理:                                                      │
│    一次处理多个样本，充分利用 GPU 并行能力                        │
│    batch_size 越大 → GPU 利用率越高 → 但显存也越大               │
│                                                                   │
│  与后续步骤的关系:                                                │
│    前向传播(预测) → 损失函数(评分) → 反向传播(算梯度) → 更新参数 │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

| 概念 | 一句话 | 生活类比 |
|------|--------|---------|
| 前向传播 | 数据从输入层流到输出层 | 工厂流水线从原材料到成品 |
| 线性变换 | 矩阵乘法 + 加偏置 | 用不同的权重混合各种因素 |
| 激活函数 | 引入非线性 | "够了就做，不够就不做" |
| 激活值 | 中间结果，反向传播要用 | 解题的草稿纸 |
| Batch | 一次处理多个样本 | 烤箱一次烤多片面包 |
