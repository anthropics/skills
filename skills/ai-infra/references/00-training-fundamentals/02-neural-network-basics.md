## 从人脑到人工神经网络

### 人脑怎么做决策

你决定今天中午吃什么，大脑里发生了什么？

```
输入信号:
    👀 看到麦当劳广告        (视觉信号)
    👃 闻到楼下面馆的香味    (嗅觉信号)
    💰 月底了钱包紧张        (财务信号)
    ⏰ 只有30分钟午休        (时间信号)
    🥗 昨天吃了炸鸡          (记忆信号)

        ↓ 大脑中 860亿个神经元处理 ↓

输出决策:
    → "去吃面吧！便宜、快、昨天吃过油的了"
```

人脑中的一个**生物神经元**：

```
    树突 (接收信号)        细胞体 (处理)           轴突 (发送信号)
    
    信号A ──→ ┐
    信号B ──→ ├──→ [加权求和] ──→ [超过阈值?] ──→ 激发! ──→ 传给下一个神经元
    信号C ──→ ┘         ↑              ↑
                    每个信号的       是否"点火"
                    重要程度不同     的门槛
```

**人工神经网络**就是模仿这个过程：

| 生物神经元 | 人工神经元 |
|-----------|-----------|
| 树突接收信号 | 接收输入数据 x |
| 突触连接强度（有的连接强，有的弱） | **权重 W**（每个输入的重要程度） |
| 激发阈值（刺激够强才点火） | **偏置 b + 激活函数** |
| 轴突发出信号 | 输出值传给下一层 |
| 860亿个神经元 | GPT-4 有约 1.8 万亿参数（传闻） |

## 一个神经元：最小的决策单元

### 用"要不要带伞出门"来理解

```
今天要不要带伞？你的大脑综合考虑以下因素：

输入:
    x₁ = 天气预报说下雨概率 = 0.7   (70%)
    x₂ = 现在天空乌云密布 = 0.9     (很多乌云)
    x₃ = 你带了一个大包 = 0.3       (包不太方便再塞伞)

每个因素的重要程度不同 (权重):
    w₁ = 0.5    (你觉得天气预报还行，中等参考)
    w₂ = 0.8    (你更相信自己的眼睛，权重大)
    w₃ = -0.3   (包太满会让你不想带伞，负面影响)

你的决策偏好 (偏置):
    b = -0.5    (你本来就不太喜欢带伞，有个负面倾向)
```

一个人工神经元的计算过程：

```
Step 1: 加权求和
    z = w₁×x₁ + w₂×x₂ + w₃×x₃ + b
      = 0.5×0.7 + 0.8×0.9 + (-0.3)×0.3 + (-0.5)
      = 0.35 + 0.72 + (-0.09) + (-0.5)
      = 0.48

Step 2: 激活函数 (决定是否"激发")
    输出 = σ(0.48) = 1/(1+e^(-0.48)) = 0.618

    → 0.618 > 0.5，决策：带伞！☂️

    如果 z 是负数，比如 z = -0.3:
    输出 = σ(-0.3) = 0.426
    → 0.426 < 0.5，决策：不带伞
```

### 一个神经元的通用公式

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                   │
│  一个神经元:                                                      │
│                                                                   │
│  输入       权重        加权求和     激活函数     输出             │
│                                                                   │
│  x₁ ──→ ×w₁ ──┐                                                 │
│  x₂ ──→ ×w₂ ──┼──→ z = Σ(wᵢxᵢ) + b ──→ f(z) ──→ output       │
│  x₃ ──→ ×w₃ ──┘                                                 │
│               +b(偏置)                                            │
│                                                                   │
│  公式: output = f(W·x + b)                                       │
│                                                                   │
│  其中:                                                            │
│    W = 权重 (每个输入有多重要)                                    │
│    b = 偏置 (基础倾向)                                            │
│    f = 激活函数 (引入非线性)                                      │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

## 权重和偏置：神经元的"性格"

### 权重 W——"你重视什么"

权重决定了每个输入的重要程度，就像每个人看重的因素不同：

```
选餐厅时，不同人的"权重"不同:

    小明 (吃货):                   小红 (省钱达人):
    w_口味 = 0.9  (最看重)         w_口味 = 0.3  (差不多就行)
    w_价格 = 0.1  (不在乎)         w_价格 = 0.9  (最看重)
    w_距离 = 0.3  (稍微考虑)       w_距离 = 0.5  (也挺重要)

    同样的三家餐厅，两人可能做出完全不同的选择！
    → 权重不同 = 行为不同
    → 训练的目标就是找到最好的权重组合
```

**权重的符号含义**：

```
w > 0 (正权重): 这个输入越大，输出越大 → 正相关
    例: 温度高 → 更想喝冷饮

w < 0 (负权重): 这个输入越大，输出越小 → 负相关
    例: 价格贵 → 越不想买

w ≈ 0 (权重接近零): 这个输入对结果几乎没影响
    例: 餐厅墙壁颜色 → 对口味没影响

|w| 大 (绝对值大): 这个因素影响力大
|w| 小 (绝对值小): 这个因素影响力小
```

### 偏置 b——"你的底线和偏好"

```
还是选餐厅的例子:

    没有偏置: z = w_口味×口味 + w_价格×价格 + w_距离×距离
    → 纯粹靠各因素的加权

    有偏置: z = w_口味×口味 + w_价格×价格 + w_距离×距离 + b
    → b 相当于一个"基础态度"

    b = +2: 你本来就很想出去吃（积极偏向）
             → 即使各因素一般般，也倾向于"去！"
    
    b = -2: 你本来就想叫外卖（消极偏向）
             → 除非各因素特别好，否则倾向于"不去"
    
    b 的作用: 调整"激发"的难易程度（调整决策的门槛）
```

## 激活函数：神经元的"脾气阈值"

### 为什么需要激活函数？

如果没有激活函数，神经元就只做加权求和——这是**线性运算**。

```
生活类比: 如果你的决策完全是"线性"的

    收入涨10% → 消费涨10%
    收入涨20% → 消费涨20%
    收入涨100% → 消费涨100%？？ 
    
    现实显然不是这样！
    收入从 5000→5500: 可能多花 400
    收入从 5000→10000: 可能只多花 2000（存钱了）
    收入从 5000→50000: 消费可能就上了个台阶，但不是10倍
    
    人的行为是非线性的，激活函数就是引入这种非线性
```

**关键问题：没有激活函数，多层 = 一层**

```
假设两层网络，没有激活函数:

    第一层: h = W₁ × x + b₁
    第二层: y = W₂ × h + b₂
    
    代入:
    y = W₂ × (W₁ × x + b₁) + b₂
      = W₂W₁ × x + W₂b₁ + b₂
      = W' × x + b'              ← 等价于一层！
    
    100层没有激活函数 = 1层
    加了几百亿参数，表达能力和一层一样！

    有了激活函数:
    y = W₂ × ReLU(W₁ × x + b₁) + b₂
    → 无法合并！每层都引入了"折叠"，网络能表达复杂的非线性函数
```

### 常见激活函数（用生活类比）

#### ReLU——"要么干、要么不干"

```
ReLU(x) = max(0, x)

    x < 0 → 输出 0 (完全不响应)
    x > 0 → 输出 x (原样传递)

    类比: 你对负面消息的态度
    坏消息: "算了不管了" → 0
    好消息: "多好就多开心" → 原样传递

    图形:
    输出
     │      ╱
     │     ╱
     │    ╱
     │   ╱
    ─┼──╱────── 输入
     │ (这里全是0)
     │
    
    优点: 计算简单（比较大小）、训练快
    缺点: 负数区域永远输出0（"死神经元"问题）
```

#### Sigmoid——"犹豫不决到坚定"

```
Sigmoid(x) = 1 / (1 + e⁻ˣ)

    输出范围: (0, 1)

    类比: 你对一件事的"信心"
    x = -5: σ = 0.007 → "几乎不可能"
    x = 0:  σ = 0.5   → "五五开"
    x = +5: σ = 0.993 → "几乎确定"

    图形:
    1.0 ─────────────────────╱─────
                           ╱
    0.5 ─ ─ ─ ─ ─ ─ ─ ─ ╱─ ─ ─ ─
                       ╱
    0.0 ────────────╱──────────────
                   ↑
              x=0 时输出 0.5

    用途: 输出概率（"下雨的概率是0.8"）
```

#### GELU/SiLU——"现代 Transformer 的选择"

```
GELU ≈ x × Φ(x)    (Φ 是正态分布的累积函数)
SiLU = x × σ(x)     (σ 是 sigmoid)

    类比: "好消息大部分传递，坏消息少量传递"
    → 不像 ReLU 一刀切（负数全部归零）
    → 允许少量负面信号通过，信息更丰富

    图形:
    输出
     │        ╱
     │       ╱
     │      ╱
     │     ╱
    ─┼───╲╱──── 输入
     │  (有个小凹陷，负值不完全是0)
     │

    GPT、LLaMA、BERT 等都使用这类激活函数
```

## 层：把神经元组织起来

### 单个神经元太弱了

一个神经元只能做简单的线性分类（画一条线把两类分开）。要处理复杂问题，需要**大量神经元组织成层**。

```
类比: 公司的组织结构

    一个人只能做简单决策
    一个部门（一层多个神经元）能处理一类问题
    多个部门串起来（多层网络）能处理复杂业务
```

### 三种基本层

```
┌────────────────────────────────────────────────────────────────┐
│                       神经网络的层结构                           │
│                                                                 │
│  输入层              隐藏层(们)              输出层              │
│  ┌─────┐         ┌─────┐  ┌─────┐        ┌─────┐             │
│  │ x₁  │────┐  ┌─│ h₁  │──│ h₁' │─┐  ┌──│  y₁ │             │
│  │ x₂  │──┐ ├──┤ │ h₂  │──│ h₂' │─┤──┤  │  y₂ │             │
│  │ x₃  │─┐├─┤  │ │ h₃  │──│ h₃' │─┤  │  └─────┘             │
│  │ x₄  │─┤├─┘  └─│ h₄  │──│ h₄' │─┘  │                      │
│  └─────┘ ││      └─────┘  └─────┘     │                      │
│          ││                            │                      │
│          └┘    隐藏层1    隐藏层2       │                      │
│                                        │                      │
│  接收原始数据  逐层提取特征           输出最终结果              │
│  (食材)       (切菜→调味→翻炒)     (成品菜)                   │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

| 层类型 | 作用 | 厨房类比 |
|--------|------|---------|
| **输入层** | 接收原始数据 | 食材进厨房 |
| **隐藏层** | 逐层提取和变换特征 | 切菜→调味→翻炒（每步加工） |
| **输出层** | 给出最终结果 | 成品上桌 |

### 每一层做了什么？

```
一个隐藏层的计算（以4个神经元为例）:

输入 x = [x₁, x₂, x₃]

    神经元1: h₁ = f(w₁₁x₁ + w₁₂x₂ + w₁₃x₃ + b₁)
    神经元2: h₂ = f(w₂₁x₁ + w₂₂x₂ + w₂₃x₃ + b₂)
    神经元3: h₃ = f(w₃₁x₁ + w₃₂x₂ + w₃₃x₃ + b₃)
    神经元4: h₄ = f(w₄₁x₁ + w₄₂x₂ + w₄₃x₃ + b₄)

用矩阵表示（这就是为什么 GPU 擅长这个——它天生擅长矩阵乘法）:

    h = f(W × x + b)

    其中 W 是 4×3 的矩阵（4个神经元，每个接收3个输入）
         b 是 4×1 的向量（每个神经元一个偏置）
         f 是激活函数（逐元素应用）
```

## 深度网络：为什么"深"很重要

### 逐层提取特征

```
类比: 看一张猫的照片

人类视觉系统:
    视网膜 → 看到像素点
    初级视觉皮层 → 识别边缘、线条
    中级视觉皮层 → 识别纹理、形状
    高级视觉皮层 → 识别"圆圆的脸"、"尖尖的耳朵"
    决策区域 → "这是一只猫！"

深度神经网络（同样的思路）:
    第1层 → 检测边缘（横线、竖线、斜线）
    第2层 → 组合边缘成纹理（毛发、条纹）
    第3层 → 组合纹理成部件（眼睛、耳朵、鼻子）
    第4层 → 组合部件成物体（猫脸、猫身）
    输出层 → "猫！置信度 95%"
```

```
┌─────────────────────────────────────────────────────────┐
│              深度网络的逐层特征提取                        │
│                                                          │
│  输入: 🖼️ 一张猫的照片 (224×224 像素)                   │
│                                                          │
│  第1层: ╱ ─ ╲ │    检测简单边缘                          │
│         (线条、角)                                       │
│                ↓                                          │
│  第2层: 🔲 🔳 ⬛    组合成纹理                            │
│         (毛发、斑纹)                                     │
│                ↓                                          │
│  第3层: 👁️ 👂 👃    识别部件                              │
│         (眼睛、耳朵)                                     │
│                ↓                                          │
│  第4层: 🐱              识别物体                          │
│         (这是猫！)                                       │
│                ↓                                          │
│  输出: P(猫)=0.95, P(狗)=0.03, P(兔)=0.02              │
│                                                          │
│  越深的层，提取的特征越"高级"、越"抽象"                │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 为什么不用一个超大的单层？

```
类比: 为什么公司要分部门（层），而不是所有人直接向CEO汇报？

    扁平结构 (1层1000个神经元):
    └── 1000个员工直接做事 → 每个人只看到局部，无法协作
    
    深层结构 (10层×100个神经元):
    └── 基层 → 组长 → 经理 → 总监 → VP → CEO
    └── 每层做抽象总结，逐级上报 → 高效、有层次

    深度网络用更少的参数表达更复杂的函数
    （数学上已证明：深度网络比浅层网络指数级高效）
```

## 参数：网络要学的所有东西

### 什么是参数？

```
网络的所有权重 W 和偏置 b，统称为"参数" (Parameters)

    一个神经元: 3个输入 → 3个权重 + 1个偏置 = 4个参数
    一层(100个神经元，接收50个输入): 100×50 + 100 = 5100个参数
    
    GPT-3:   175,000,000,000 个参数 (1750亿)
    LLaMA-7B:  7,000,000,000 个参数 (70亿)
    GPT-4: ~1,800,000,000,000 个参数 (约1.8万亿，传闻)
```

### 参数的直觉

```
类比: 学做菜

    参数 = 你做菜的全部"手法"和"经验"
    
    ┌──────────────────────────────────────────────────┐
    │  切菜力度:     w₁ = 0.7   (中等力度)             │
    │  放盐量:       w₂ = 0.3   (偏少)                 │
    │  火候大小:     w₃ = 0.85  (偏大火)               │
    │  翻炒频率:     w₄ = 0.6   (适中)                 │
    │  ...                                              │
    │  共 70亿个这样的"手法参数"                        │
    │                                                   │
    │  训练 = 通过反复练习，调整这 70亿个参数          │
    │         直到做出来的菜（预测）和标准答案一致      │
    └──────────────────────────────────────────────────┘
```

### 参数量与显存的关系

```
每个参数占多少显存？

    FP32 (全精度): 4 bytes/参数
    FP16 (半精度): 2 bytes/参数
    INT8 (量化):   1 byte/参数

    7B 模型 (FP16):
    7×10⁹ × 2 bytes = 14 GB    ← 仅存储参数就要 14GB

    加上训练所需的梯度和优化器状态:
    参数(FP16): 14 GB
    梯度(FP16): 14 GB  
    优化器(FP32): 84 GB   ← AdamW 需要 12 bytes/参数
    ────────────────────
    总计: 112 GB           ← 单卡 A100 80GB 装不下！
```

## Tensor：数据在网络中的载体

### 什么是 Tensor？

```
Tensor（张量）= 多维数组 = 数据的容器

类比: 不同维度的"表格"

    标量 (0维): 一个数字
        温度 = 25.6℃
    
    向量 (1维): 一行数字
        今天天气 = [温度, 湿度, 风速] = [25.6, 0.7, 3.2]
    
    矩阵 (2维): 一个表格
        一周天气:
        ┌──────┬───────┬──────┬──────┐
        │      │ 温度  │ 湿度 │ 风速 │
        ├──────┼───────┼──────┼──────┤
        │ 周一 │ 25.6  │ 0.7  │ 3.2  │
        │ 周二 │ 28.1  │ 0.6  │ 2.1  │
        │ 周三 │ 22.3  │ 0.8  │ 4.5  │
        └──────┴───────┴──────┴──────┘
    
    3维张量: 一摞表格
        12个月 × 每月30天 × 3个指标 → shape = [12, 30, 3]
    
    4维张量: 一批图片
        32张图 × 3通道(RGB) × 224高 × 224宽 → shape = [32, 3, 224, 224]
```

### 在训练中，一切都是 Tensor

```
┌────────────────────────────────────────────────────────────┐
│  训练中的各种 Tensor:                                       │
│                                                             │
│  输入数据:  shape = [batch_size, features]                  │
│            例: [32, 768] = 32个样本，每个768维特征          │
│                                                             │
│  权重 W:   shape = [输出维度, 输入维度]                     │
│            例: [256, 768] = 把768维映射到256维              │
│                                                             │
│  偏置 b:   shape = [输出维度]                               │
│            例: [256]                                        │
│                                                             │
│  梯度:     shape 与对应参数完全相同                         │
│            W 的梯度: [256, 768]                             │
│            b 的梯度: [256]                                  │
│                                                             │
│  一句话: 参数有多少个，梯度就有多少个，一一对应              │
│                                                             │
└────────────────────────────────────────────────────────────┘
```

---

## Transformer：当前 LLM 的骨架

### 为什么要提 Transformer？

几乎所有现代大语言模型（GPT、LLaMA、Claude 等）都基于 **Transformer 架构**。理解它的基本结构，有助于后续学习分布式训练和推理优化。

### Transformer 的直觉理解

```
类比: 一群人在圆桌讨论，每个人的发言要参考所有人之前说的话

输入: "今天 天气 很 好 适合 出去 玩"
       ↓ 每个词变成一个向量 (Embedding)

┌──────────────────────────────────────────────────────────┐
│  Transformer 的一个层 (Layer):                             │
│                                                           │
│  ① 自注意力 (Self-Attention):                             │
│     "今天" 看看其他所有词，发现和"天气"、"好"关系密切    │
│     "玩" 看看其他所有词，发现和"出去"、"适合"关系密切    │
│     → 每个词综合了与其他词的关系，更新自己的表示           │
│                                                           │
│     类比: 圆桌讨论中，每个人听取所有人的意见，            │
│           对与自己相关的人给更多"注意力"                   │
│                                                           │
│  ② 前馈网络 (FFN):                                        │
│     对每个词的表示做进一步变换                             │
│     类比: 听完讨论后，每个人独立思考、总结                 │
│                                                           │
│  这样的层堆叠 32~96 次 → 就是一个 Transformer              │
│                                                           │
└──────────────────────────────────────────────────────────┘

输出: 预测下一个词的概率 → P("。") = 0.4, P("！") = 0.3, ...
```

### LLM 的规模感

```
┌─────────────────────────────────────────────────────────┐
│  LLaMA-7B 的规模:                                        │
│                                                          │
│  层数:          32 层                                    │
│  隐藏维度:      4096                                     │
│  注意力头数:    32 个                                    │
│  词表大小:      32000                                    │
│  参数总量:      约 70亿                                  │
│                                                          │
│  每一层包含:                                             │
│    - 自注意力的 Q/K/V 权重: 3 × 4096 × 4096 = 5040万   │
│    - 输出投影权重:          4096 × 4096 = 1680万        │
│    - FFN 权重(SwiGLU):      3 × 4096 × 11008 = 1.35亿   │
│    - 其他(归一化等):        约 8000                      │
│    - 每层约 2.0亿参数                                    │
│                                                          │
│  32层 × 2.0亿 + 词嵌入等 ≈ 70亿参数                     │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 总结

### 核心概念回顾

```
┌──────────────────────────────────────────────────────────────┐
│                    神经网络核心概念                             │
│                                                               │
│  神经元 = 最小决策单元                                        │
│  │      加权求和 + 激活函数                                   │
│  │      类比: 综合多个因素做一个判断                          │
│  │                                                            │
│  ├── 权重 W = "你重视什么"                                   │
│  │   正权重=正相关，负权重=负相关，大小=影响力               │
│  │                                                            │
│  ├── 偏置 b = "你的基础倾向"                                 │
│  │   调整激发门槛的高低                                       │
│  │                                                            │
│  ├── 激活函数 = "脾气阈值"                                   │
│  │   引入非线性，让网络能学习复杂规律                        │
│  │   ReLU / GELU / Sigmoid                                    │
│  │                                                            │
│  层 = 一组神经元                                              │
│  │    类比: 公司里的一个部门                                  │
│  │                                                            │
│  深度网络 = 多层堆叠                                          │
│  │    逐层提取越来越抽象的特征                                │
│  │    类比: 公司的层级结构                                    │
│  │                                                            │
│  参数 = 所有 W 和 b 的集合                                    │
│  │    GPT-3: 1750亿个参数                                    │
│  │    训练 = 找到最优的参数组合                               │
│  │                                                            │
│  Tensor = 数据的载体（多维数组）                              │
│       标量→向量→矩阵→高维张量                               │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 概念速查表

| 概念 | 一句话 | 生活类比 |
|------|--------|---------|
| 神经元 | 接收输入，加权求和，激活输出 | 综合多个因素做决定 |
| 权重 W | 每个输入的重要程度 | 你选餐厅时各因素的优先级 |
| 偏置 b | 基础倾向值 | 你本来就想/不想做某事 |
| 激活函数 | 引入非线性 | "够了就做，不够就不做"的门槛 |
| 层 | 一组并行的神经元 | 公司的一个部门 |
| 深度 | 网络有多少层 | 公司有多少级 |
| 参数 | W 和 b 的总称 | 厨师的全部手法 |
| Tensor | 多维数组 | 不同维度的表格 |
| Transformer | 基于注意力的架构 | 圆桌讨论，每人参考所有人 |
