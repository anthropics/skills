## 梯度下降的直觉

### 蒙眼下山

```
你蒙着眼睛站在山上，要走到最低点:

    ┌───────────────────────────────────────────────────┐
    │                                                    │
    │  Step 1: 感受脚下坡度 (计算梯度)                   │
    │          "左边低一些，坡度是30°"                   │
    │                                                    │
    │  Step 2: 决定步长 (学习率 × 梯度)                  │
    │          "迈一小步试试"                            │
    │                                                    │
    │  Step 3: 朝下坡方向走一步 (更新参数)               │
    │          param = param - lr × gradient              │
    │                                                    │
    │  Step 4: 到达新位置，重复                          │
    │                                                    │
    └───────────────────────────────────────────────────┘

    为什么是"减去"梯度？
    
    梯度指向上坡方向（loss增大最快的方向）
    我们要下山 → 沿相反方向走 → 减去梯度
```

### 一个数字来理解

```
当前参数: w = 5.0
梯度: ∂loss/∂w = +3.0   (w增大 → loss增大 → 我们应该减小w)
学习率: lr = 0.1

更新:
w_new = w - lr × gradient
      = 5.0 - 0.1 × 3.0
      = 5.0 - 0.3
      = 4.7

验证直觉:
    梯度是+3.0 (正) → w增大时loss增大
    → 应该减小w
    → w从5.0变成4.7，确实减小了 ✓
```

## 学习率：步子迈多大

### 学习率是训练中最重要的超参数

```
类比: 开车过弯道

    学习率太大 (lr=1.0):
    ┌──────────────────┐
    │       ╱╲         │   猛打方向盘
    │      ╱  ╲        │   → 来回摇摆
    │     ╱    ╲ ╱╲    │   → 甩出去了！
    │    ╱      ╳  ╲   │   (loss 爆炸)
    │   ╱      ╱ ╲  ╲  │
    └──────────────────┘
    
    学习率太小 (lr=0.0001):
    ┌──────────────────┐
    │                  │   轻微转向
    │  → → → → → →    │   → 几乎走直线
    │                  │   → 要绕很久才到
    │                  │   (收敛太慢)
    └──────────────────┘
    
    学习率合适 (lr=0.001):
    ┌──────────────────┐
    │                  │   适度转向
    │  → → ↘          │   → 平滑到达
    │       → → ★     │   → 又快又稳
    │                  │   (收敛良好)
    └──────────────────┘
```

### 不同任务的典型学习率

```
┌──────────────────────────────────────────────────────────┐
│                    典型学习率参考                          │
│                                                           │
│  从头训练 (Pre-training):                                 │
│    LLM 预训练:       lr = 1e-4 ~ 6e-4                    │
│    CV 模型训练:      lr = 1e-3 ~ 1e-1                    │
│                                                           │
│  微调 (Fine-tuning):                                      │
│    全参数微调:       lr = 1e-5 ~ 5e-5                    │
│    LoRA 微调:        lr = 1e-4 ~ 3e-4                    │
│                                                           │
│  为什么微调的 lr 更小？                                   │
│    从头训练: 参数是随机的，离最优解很远 → 大步快走         │
│    微调: 参数已经不错了，只需微调 → 小步慢走               │
│    类比: 粗加工用大锤，精加工用小锉刀                     │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

## SGD 的问题与改进

### 最基础的 SGD

```
SGD (Stochastic Gradient Descent):
    w = w - lr × g

    简单直接，但有几个严重问题:
```

### 问题1：震荡（窄长山谷）

```
类比: 在一条窄长的山沟里走

    ┌──────────────────────────────────────┐
    │    ╭──────────────────────╮          │
    │   ╭┤                      ├╮         │
    │  ╭┤│   SGD 的锯齿路径      │├╮        │
    │  │││     ↗↙↗↙↗↙→ ★        │││        │
    │  ╰┤│                      │├╯        │
    │   ╰┤                      ├╯         │
    │    ╰──────────────────────╯          │
    └──────────────────────────────────────┘
    
    纵向(窄): 梯度大 → 步子大 → 来回弹
    横向(长): 梯度小 → 步子小 → 前进慢
    
    结果: 大量步数浪费在纵向来回弹跳上
```

### 解决: Momentum（动量）

```
类比: 小球在山谷里滚动

    普通 SGD = 每一步只看当前坡度（无记忆）
    Momentum = 小球有惯性，保留之前的运动方向

    公式:
    v = β × v_prev + gradient          ← 速度 = 惯性 + 当前梯度
    w = w - lr × v                      ← 沿速度方向走

    β = 0.9 (保留90%的历史动量)

    效果:
    ┌──────────────────────────────────────┐
    │    ╭──────────────────────╮          │
    │   ╭┤                      ├╮         │
    │  ╭┤│  Momentum 平滑路径    │├╮        │
    │  │││     → → → → → → ★    │││        │
    │  ╰┤│                      │├╯        │
    │   ╰┤                      ├╯         │
    │    ╰──────────────────────╯          │
    └──────────────────────────────────────┘
    
    纵向: 梯度正负交替 → 动量相互抵消 → 震荡减弱
    横向: 梯度持续同方向 → 动量累积 → 加速前进
```

### 问题2：所有参数共享同一个学习率

```
问题:
    参数A: 梯度一直很大(2.0) → 它在 loss surface 的陡坡上 → 需要小步
    参数B: 梯度一直很小(0.01) → 它在平坦区域 → 需要大步

    但 SGD/Momentum 给所有参数同一个 lr → 不合理

解决: Adam (Adaptive Moment Estimation)
    每个参数根据自己的梯度历史，自动调整学习率

    梯度大 → 自动缩小步长 (已经在陡坡了，小心走)
    梯度小 → 自动放大步长 (太平了，大步走)

→ 详见 [AdamW 优化器详解](../02-distributed-training/13-adamw-optimizer.md)
```

### 优化器演化总结

```
SGD            →    Momentum      →    Adam          →    AdamW
w -= lr×g          加入惯性           自适应学习率        解耦权重衰减
                   减少震荡           不同参数不同步长    正则化更正确
                                                         LLM 训练标配

类比:
走路下山        →    滑雪下山      →    智能导航       →    智能导航+限速
(只看脚下)          (有惯性)           (自动调速)          (调速+防超速)
```

## 学习率调度：变速行驶

### 为什么不能用固定学习率？

```
类比: 开车从高速到市区

    高速公路: 120km/h → 快速前进 (训练初期，快速降loss)
    进入市区: 60km/h  → 放慢速度 (训练中期，精细调整)
    停车入库: 5km/h   → 非常慢   (训练后期，微调到最优)

    如果一直 120km/h → 停车入库时会撞墙 (loss 在最优解附近震荡)
    如果一直 5km/h   → 高速上龟速 → 别人超了你10圈 (太慢)
```

### 常见调度策略

#### Warmup（热身）

```
为什么需要 warmup？

    训练刚开始:
    - 参数是随机的，梯度方向很不靠谱
    - 优化器的动量/方差估计还没稳定
    - 如果一上来就大步走 → 可能走偏了就回不来

    Warmup = 前几百/几千步，学习率从0线性增长到目标值

    lr
    ▲
    │         ╱─── 达到目标学习率
    │        ╱
    │       ╱
    │      ╱
    │     ╱
    │    ╱
    │   ╱
    └──────────→ step
       warmup
       阶段

    类比: 跑步前先热身，不要一上来就冲刺
```

#### Cosine Decay（余弦衰减）

```
warmup 之后，学习率按余弦曲线逐渐降低:

    lr
    ▲
    │    ╱──╲
    │   ╱    ╲
    │  ╱      ╲
    │ ╱        ╲───
    │╱              ╲───
    │                    ╲────────
    └──────────────────────────────→ step
    warmup    cosine decay

    公式: lr_t = lr_min + 0.5 × (lr_max - lr_min) × (1 + cos(π × t/T))

    优点:
    - 开始快速下降（快速探索）
    - 中间平缓下降（稳定学习）
    - 最后趋近最小lr（精细调整）
    
    这是 LLM 训练最常用的调度策略
```

#### LLM 典型配置

```
┌──────────────────────────────────────────────────────────┐
│  LLaMA 训练配置:                                          │
│                                                           │
│  峰值学习率:  3e-4                                        │
│  最小学习率:  3e-5 (峰值的1/10)                           │
│  Warmup步数: 2000 步                                      │
│  衰减策略:   Cosine Decay                                 │
│  总训练步数: ~100万步                                     │
│                                                           │
│  lr                                                       │
│  3e-4 ┤     ╱──╲                                         │
│       │    ╱    ╲                                        │
│       │   ╱      ╲──                                     │
│       │  ╱           ╲──                                 │
│  3e-5 ┤╱                 ╲──────────                     │
│       └──────────────────────────────→ step              │
│       0   2k              500k    1M                      │
│       warmup              cosine decay                    │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

## 局部最优与鞍点

### 局部最优——"假谷底"

```
类比: 你在山里蒙眼下山，走到了一个小坑里

    loss
     ▲
     │ ╲     ╱╲
     │  ╲   ╱  ╲
     │   ╲ ╱    ╲
     │    ★       ╲       ╱
     │  局部最优    ╲     ╱
     │              ╲   ╱
     │               ╲ ╱
     │                ★
     │              全局最优
     └──────────────────────→ 参数

    你走到了局部最优（小坑），四周都是上坡
    → 梯度≈0，以为到底了
    → 但真正的最低点还在更远处
```

### 为什么高维空间不太怕局部最优？

```
┌──────────────────────────────────────────────────────────┐
│                                                           │
│  好消息: 在高维空间中，局部最优几乎不存在！               │
│                                                           │
│  原因: 要形成局部最优，需要所有维度都是"碗底"            │
│        GPT-3 有 1750亿个维度                              │
│        所有维度同时是碗底的概率极低                        │
│                                                           │
│  更常见的是"鞍点":                                       │
│                                                           │
│  鞍点 = 有些方向是最低点，有些方向是最高点                │
│  (像马鞍一样，前后低、左右高)                             │
│                                                           │
│        ╲   ╱                                              │
│         ╲ ╱                                               │
│    ──── ★ ────   ← 鞍点: x方向是谷底，y方向是山顶       │
│         ╱╲                                                │
│        ╱  ╲                                               │
│                                                           │
│  Momentum/Adam 通常能帮助逃离鞍点:                        │
│  - 惯性帮助冲过平坦区域                                   │
│  - 自适应学习率帮助找到下降方向                           │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

## 收敛：怎么知道训练够了

### 收敛的标志

```
类比: 你怎么知道减肥成功了？

    ① 体重不再下降了（loss 稳定了）
    ② 维持了一段时间（不是偶然）
    ③ 饮食没变但体重也没变（参数不需要大幅调整了）

收敛判断:
    ┌──────────────────────────────────────────────┐
    │  ✅ Loss 曲线趋于平稳                        │
    │  ✅ 梯度范数很小（坡度接近0，接近谷底了）    │
    │  ✅ 验证集 loss 也稳定（不只是训练集）       │
    │  ❌ 如果验证集 loss 开始上升 → 过拟合了！    │
    └──────────────────────────────────────────────┘
```

### LLM 训练中的"收敛"

```
LLM 预训练通常不等到完全收敛:

    loss
    ▲
    │╲
    │ ╲
    │  ╲───
    │      ╲───
    │          ╲───────────── (还在缓慢下降)
    │
    └──────────────────────→ tokens 消耗量
    0      500B    1T    2T

    LLaMA-7B: 训练了 1T tokens 后停止
    原因: loss 还在下，但下降速度已经很慢
          继续训练的性价比不高（GPU太贵了！）

    Chinchilla Scaling Law:
    最优训练 tokens ≈ 20 × 参数量
    7B → 约 140B tokens (实际通常训练更多)
```

## 过拟合与正则化

### 过拟合——"死记硬背"

```
类比: 两种学生

    学生A (欠拟合/泛化好):
    看了100道练习题 → 理解了解题思路 → 考试遇到新题也能做对

    学生B (过拟合):
    背了100道练习题的答案 → 一模一样的题能做对 → 换个数字就不会了

    ┌─────────────────────────────────────────┐
    │  过拟合的信号:                            │
    │                                          │
    │  loss                                    │
    │  ▲                                       │
    │  │╲                                      │
    │  │ ╲  训练loss (一直在降)                │
    │  │  ╲─────────────────                   │
    │  │                                       │
    │  │    ╲                                  │
    │  │     ╲  验证loss                       │
    │  │      ╲──╱──╱──╱── (开始上升!)        │
    │  │                                       │
    │  └────────────────────→ step             │
    │          ↑                               │
    │     在这里应该停止 (早停法 Early Stopping)│
    │                                          │
    └─────────────────────────────────────────┘
```

### 权重衰减——"别太膨胀"

```
权重衰减 (Weight Decay) 是最常用的正则化手段:

    核心思想: 每次更新参数时，让参数"缩小"一点点

    w = w × (1 - lr × λ) - lr × gradient
              ──────────────
              这部分让 w 每步缩小一点

    λ = weight_decay = 0.01 ~ 0.1

    类比: 
    你在银行存钱（参数值）
    每月银行收取管理费（权重衰减）
    → 钱不会无限增长，保持在合理范围

    为什么有效？
    参数太大 → 模型"记性太好"，容易死记硬背
    权重衰减迫使参数保持小 → 模型更"简洁"，泛化更好

    LLM 典型值: weight_decay = 0.1
```

### Dropout——"随机遮眼训练"

```
训练时随机"关掉"一部分神经元:

    正常:     [h₁=2.0, h₂=3.0, h₃=1.0, h₄=4.0]
    Dropout:  [h₁=2.0, h₂=0.0, h₃=1.0, h₄=0.0]  ← 随机归零
              (dropout_rate = 0.5, 即50%的神经元被关掉)

    类比: 篮球队训练
    正常训练: 5个人配合得很好
    Dropout训练: 随机让2个人坐板凳
    → 每个人都必须独立能力强，不能过度依赖队友
    → 整个队伍更鲁棒

    注意: 推理时不做 Dropout (所有神经元都参与)
```

## 完整的训练配置实战

### LLM 预训练的典型配置

```python
import math
import torch
from torch.optim import AdamW
from torch.optim.lr_scheduler import LambdaLR

model = MyTransformer()

# ─── 优化器配置 ───
optimizer = AdamW(
    model.parameters(),
    lr=3e-4,            # 峰值学习率
    betas=(0.9, 0.95),  # β₁, β₂ (LLM 常用 0.95)
    eps=1e-8,            # 防除零
    weight_decay=0.1,    # 权重衰减
)

# ─── 学习率调度（Warmup + Cosine Decay） ───
warmup_steps = 2000
total_steps = 1000000
min_lr = 3e-5

def lr_lambda(step):
    if step < warmup_steps:
        return step / warmup_steps  # 线性 warmup
    progress = (step - warmup_steps) / (total_steps - warmup_steps)
    return max(min_lr / 3e-4, 0.5 * (1 + math.cos(math.pi * progress)))

scheduler = LambdaLR(optimizer, lr_lambda)

# ─── 训练循环 ───
for step, batch in enumerate(dataloader):
    optimizer.zero_grad()
    
    loss = model(batch)
    loss.backward()
    
    # 梯度裁剪 (防爆炸)
    torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
    
    optimizer.step()
    scheduler.step()
    
    if step % 100 == 0:
        print(f"Step {step}: loss={loss.item():.4f}, lr={optimizer.param_groups[0]['lr']:.6f}")
```

## 总结

```
┌──────────────────────────────────────────────────────────────────┐
│                    梯度下降与参数更新核心要点                       │
│                                                                   │
│  基本公式: w = w - lr × gradient                                 │
│                                                                   │
│  学习率 lr:                                                       │
│  ├── 太大: 震荡、发散                                            │
│  ├── 太小: 收敛太慢                                              │
│  └── 需要用调度器动态调整 (Warmup + Cosine Decay)                │
│                                                                   │
│  优化器演化:                                                      │
│  SGD → Momentum(减震荡) → Adam(自适应lr) → AdamW(正确正则化)    │
│                                                                   │
│  收敛:                                                            │
│  ├── 训练loss稳定下降 → 正常                                    │
│  ├── 验证loss开始上升 → 过拟合，该停了                           │
│  └── LLM 通常不等完全收敛，按 token 预算停                       │
│                                                                   │
│  正则化 (防过拟合):                                               │
│  ├── 权重衰减: 让参数保持小，模型更简洁                          │
│  ├── Dropout: 随机关闭神经元，防止过度依赖                       │
│  └── 早停法: 验证loss不降了就停                                  │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

| 概念 | 一句话 | 生活类比 |
|------|--------|---------|
| 梯度下降 | 沿负梯度方向更新参数 | 蒙眼下山，脚感到哪边低就往哪走 |
| 学习率 | 每步走多远 | 开车速度 |
| Warmup | 开始时慢慢加速 | 跑步前先热身 |
| Cosine Decay | 后期逐渐减速 | 高速到市区要减速 |
| Momentum | 保留历史运动方向 | 滑雪有惯性 |
| 局部最优 | 假谷底，不是真正最低 | 山里的小坑 |
| 鞍点 | 某些方向是谷、某些是峰 | 马鞍 |
| 过拟合 | 背答案而非学解题思路 | 死记硬背的学生 |
| 权重衰减 | 迫使参数保持小 | 银行收管理费 |
| 收敛 | loss 稳定不再下降 | 体重不再变了 |
