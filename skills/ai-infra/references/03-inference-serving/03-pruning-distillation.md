## 为什么需要模型压缩

```
  生活类比：搬家

  你有一个 200 平的房子 (大模型)，里面堆满了东西。
  现在你要搬到 50 平的公寓 (边缘设备/手机)。

  三种"瘦身"方式:

  方式 1 — 剪枝: 扔掉不需要的东西
    "这个跑步机两年没用了? 扔! 那堆旧杂志? 扔!"
    → 减少物品数量，但留下的都是原件

  方式 2 — 蒸馏: 把精华浓缩到小容器里
    "把整个书架的笔记精华抄到一个笔记本上"
    → 换了一个更小的容器，但保留了核心知识

  方式 3 — 量化: 把每件东西真空压缩
    "衣服用真空袋压缩，能塞进更小的箱子"
    → 东西还是那些，只是每件占用空间变小了
```

## 剪枝 (Pruning)

### 核心思想

```
  大模型中有大量"冗余"参数:

  ┌──────────────────────────────────────────────────────────────────┐
  │  发现: 大模型中 80-90% 的权重接近于 0，对输出几乎没有贡献       │
  │                                                                  │
  │  权重分布 (LLaMA-7B 某一层):                                     │
  │                                                                  │
  │  数量                                                            │
  │   ▲      ╱╲                                                      │
  │   │    ╱    ╲                                                    │
  │   │  ╱        ╲         大量权重集中在 0 附近                    │
  │   │╱     │      ╲───────                                        │
  │   └──────┼──────────────→ 权重值                                │
  │         0.0                                                      │
  │                                                                  │
  │  类比: 一个公司 100 个员工                                       │
  │    20 个核心骨干 → 完成了 80% 的业务 (二八定律)                  │
  │    80 个"摸鱼"的 → 贡献很小                                     │
  │                                                                  │
  │  剪枝 = 裁员! 砍掉贡献小的，保留核心骨干                        │
  └──────────────────────────────────────────────────────────────────┘
```

### 非结构化剪枝 vs 结构化剪枝

```
  ┌──────────────────────────────────────────────────────────────────┐
  │  非结构化剪枝 — 逐个权重砍                                       │
  │                                                                  │
  │  原始权重矩阵:          剪枝后 (50% 稀疏):                      │
  │  ┌──────────────┐       ┌──────────────┐                        │
  │  │ 0.3  0.01 0.8│       │ 0.3   0   0.8│                        │
  │  │ 0.02 0.6  0.1│  →    │  0   0.6   0 │  ← 小于阈值的置零     │
  │  │ 0.7  0.03 0.5│       │ 0.7   0   0.5│                        │
  │  └──────────────┘       └──────────────┘                        │
  │                                                                  │
  │  优点: 压缩率高 (可达 90%+)                                      │
  │  缺点: 零散分布的零，GPU 无法高效利用!                           │
  │                                                                  │
  │  类比: 一本书随机涂掉了一些字                                    │
  │    → 能省墨水 (存储)，但翻页速度不变 (计算不加速)                │
  │    → GPU 还是要处理同样形状的矩阵，只是有些是 0                  │
  │    → 需要特殊稀疏硬件支持 (NVIDIA A100 稀疏 Tensor Core          │
  │       支持 2:4 结构化稀疏)                                       │
  │                                                                  │
  ├──────────────────────────────────────────────────────────────────┤
  │  结构化剪枝 — 整行/整列/整个注意力头砍掉                         │
  │                                                                  │
  │  原始权重矩阵 [4,4]:    移除第 2 列后 [4,3]:                    │
  │  ┌────────────────┐     ┌────────────┐                          │
  │  │ 0.3  0.01 0.8 0.4│   │ 0.3  0.8 0.4│                        │
  │  │ 0.02 0.6  0.1 0.9│ → │ 0.02 0.1 0.9│  ← 整列消失           │
  │  │ 0.7  0.03 0.5 0.2│   │ 0.7  0.5 0.2│                        │
  │  │ 0.1  0.8  0.3 0.6│   │ 0.1  0.3 0.6│                        │
  │  └────────────────┘     └────────────┘                          │
  │                                                                  │
  │  优点: 矩阵真的变小了! GPU 可以直接加速                          │
  │  缺点: 压缩率受限 (通常 20-50%)，精度损失可能更大                │
  │                                                                  │
  │  类比: 一本书直接撕掉几章                                        │
  │    → 书真的变薄了 (计算量减少)                                   │
  │    → 但可能丢失了重要章节                                        │
  └──────────────────────────────────────────────────────────────────┘
```

### 在 LLM 中怎么剪？

```
  ┌──────────────────────────────────────────────────────────────────┐
  │  Attention Head 剪枝                                             │
  │                                                                  │
  │  LLaMA-7B: 32 个 Attention Head                                  │
  │                                                                  │
  │  发现: 不是所有 Head 都同等重要!                                  │
  │                                                                  │
  │  Head 重要性评估:                                                 │
  │  Head 0:  ████████████████████  (重要! 负责位置关注)             │
  │  Head 1:  ███████████           (中等)                           │
  │  Head 5:  ██                    (不重要，可以砍)                  │
  │  Head 12: ████████████████      (重要! 负责语义关联)             │
  │  Head 28: █                     (不重要，可以砍)                  │
  │  ...                                                             │
  │                                                                  │
  │  移除 Head 5 和 28 后:                                           │
  │  - 参数减少: 2/32 = 6.25%                                       │
  │  - KV Cache 也减少 6.25%                                         │
  │  - 精度损失: 通常 < 1% (在下游任务上)                            │
  │                                                                  │
  │  如何评估 Head 重要性？                                           │
  │  方法 1: 看注意力熵 — 熵高 = 注意力分散 = 不重要                 │
  │  方法 2: 移除后看 Loss 变化 — 变化小 = 不重要                    │
  │  方法 3: 看梯度大小 — 梯度小 = 对训练贡献小                      │
  │                                                                  │
  ├──────────────────────────────────────────────────────────────────┤
  │  MLP 层宽度剪枝                                                  │
  │                                                                  │
  │  LLaMA-7B MLP: hidden_size 4096 → 中间层 11008 → hidden_size     │
  │                                                                  │
  │  中间层的 11008 个神经元中，并非都有用:                           │
  │  → 可以移除不重要的神经元，将 11008 缩减到比如 8192              │
  │  → 参数减少约 25%                                                │
  │                                                                  │
  ├──────────────────────────────────────────────────────────────────┤
  │  整层移除 (Layer Pruning)                                        │
  │                                                                  │
  │  LLaMA-7B 有 32 层 Transformer Block                             │
  │                                                                  │
  │  研究发现: 中间层 (如 Layer 15-20) 的表示变化较小                 │
  │           说明这些层在做"微调"，不是"大的变换"                    │
  │                                                                  │
  │  Layer 0:  ─────────→ 大变化 (词嵌入到语义表示)                  │
  │  Layer 5:  ─────→     中等变化                                   │
  │  Layer 15: ──→        微小变化 (可考虑移除!)                     │
  │  Layer 16: ──→        微小变化 (可考虑移除!)                     │
  │  Layer 25: ─────→     中等变化                                   │
  │  Layer 31: ─────────→ 大变化 (语义到预测)                        │
  │                                                                  │
  │  移除 Layer 15-16:                                               │
  │  - 参数减少: 2/32 = 6.25%                                       │
  │  - 推理速度提升: ~6%                                             │
  │  - 精度影响: 需要验证，通常可接受                                 │
  └──────────────────────────────────────────────────────────────────┘
```

### 前沿方法: SparseGPT 和 Wanda

```
  ┌──────────────────────────────────────────────────────────────────┐
  │  SparseGPT (2023) — 无需重训练的非结构化剪枝                     │
  │                                                                  │
  │  灵感: 类似 GPTQ (量化) 的"逐列处理 + 误差补偿"                 │
  │                                                                  │
  │  流程:                                                           │
  │  1. 确定要剪的比例 (如 50%)                                      │
  │  2. 逐列处理: 先确定该列是否剪掉                                 │
  │  3. 如果剪掉，把误差补偿到后面的列                               │
  │  → 50% 稀疏度下，LLaMA-7B 困惑度仅增加 ~0.5                     │
  │                                                                  │
  ├──────────────────────────────────────────────────────────────────┤
  │  Wanda (2023) — 更快更简单的剪枝                                  │
  │                                                                  │
  │  核心: 权重大小 × 激活值大小 = 重要性分数                        │
  │                                                                  │
  │  Score(w_ij) = |w_ij| × ‖X_j‖₂                                  │
  │                 ↑          ↑                                      │
  │           权重绝对值   该通道激活值的范数                          │
  │                                                                  │
  │  比 SparseGPT 快得多 (不需要计算 Hessian)                        │
  │  效果接近 SparseGPT                                              │
  │                                                                  │
  │  类比: 裁员标准                                                  │
  │  SparseGPT: 做复杂的绩效评估，还要考虑团队协作影响               │
  │  Wanda: 简单看"个人能力 × 岗位重要性"，快速决定                  │
  └──────────────────────────────────────────────────────────────────┘
```

### 剪枝代码示例

```python
# 结构化剪枝: 移除不重要的 Attention Heads
def prune_attention_heads(model, heads_to_prune):
    """
    heads_to_prune: dict, {layer_idx: [head_idx1, head_idx2, ...]}
    例如: {5: [0, 3], 12: [1]}  → 移除第5层的Head 0/3，第12层的Head 1
    """
    for layer_idx, heads in heads_to_prune.items():
        layer = model.transformer.h[layer_idx]
        layer.attn.prune_heads(heads)

# 使用 Wanda 进行非结构化剪枝 (简化版)
def wanda_prune(weight, activation_norms, sparsity=0.5):
    """
    weight: [out_features, in_features] 权重矩阵
    activation_norms: [in_features] 每个输入通道的激活值范数
    sparsity: 稀疏度 (0.5 = 剪掉 50%)
    """
    # 计算重要性分数: |权重| × 激活范数
    importance = weight.abs() * activation_norms.unsqueeze(0)
    
    # 找到每行中分数最低的 50%
    threshold = torch.kthvalue(
        importance, int(sparsity * importance.shape[1]), dim=1
    ).values.unsqueeze(1)
    
    # 创建掩码: 低于阈值的置零
    mask = importance > threshold
    return weight * mask
```

## 知识蒸馏 (Distillation)

### 核心思想

```
  ┌──────────────────────────────────────────────────────────────────┐
  │  类比: 名师带徒弟                                                │
  │                                                                  │
  │  大模型 (Teacher, 老师傅):                                       │
  │    经验丰富，什么都知道，但"工资高"(算力贵)                      │
  │                                                                  │
  │  小模型 (Student, 学徒):                                         │
  │    刚入行，但学习能力强，"工资低"(算力便宜)                      │
  │                                                                  │
  │  蒸馏过程 = 师傅手把手教徒弟:                                    │
  │                                                                  │
  │    ┌──────────────┐                                              │
  │    │  Teacher (7B) │  "这个问题，答案A有60%概率，                │
  │    │  (不参与训练)  │   B有30%，C有10%"                          │
  │    └──────┬───────┘   ← 这就是 soft labels (软标签)             │
  │           │ 传授                                                 │
  │           │ "暗知识"                                             │
  │           ▼                                                      │
  │    ┌──────────────┐                                              │
  │    │ Student (1B)  │  学习 Teacher 的输出分布                    │
  │    │ (参与训练)    │  而不仅仅是最终答案                          │
  │    └──────────────┘                                              │
  │                                                                  │
  │  关键洞察: 为什么 soft labels 比 hard labels 好？                │
  │                                                                  │
  │    问题: "北京是哪个国家的首都？"                                 │
  │                                                                  │
  │    Hard label: [中国=1, 其他=0]                                  │
  │    → 只告诉你"答案是中国"，没有其他信息                          │
  │                                                                  │
  │    Teacher 的 Soft label:                                        │
  │    [中国=0.92, 日本=0.03, 韩国=0.02, 法国=0.01, ...]             │
  │    → 日本和韩国比法国得分高!                                     │
  │    → 因为它们也是东亚国家，和"中国的首都"有地理关联              │
  │    → 这种"错误答案之间的相对关系"就是 dark knowledge (暗知识)     │
  │                                                                  │
  │  "暗知识"让 Student 学到的远比正确答案本身多得多!                 │
  └──────────────────────────────────────────────────────────────────┘
```

### 温度参数 T 的作用

```
  ┌──────────────────────────────────────────────────────────────────┐
  │  Softmax with Temperature:                                       │
  │                                                                  │
  │    P(i) = exp(z_i / T) / Σ exp(z_j / T)                         │
  │                                                                  │
  │  T = 1 (正常温度):                                               │
  │    logits = [5.0, 2.0, 0.5]                                     │
  │    probs  = [0.92, 0.05, 0.03]  ← 分布很"尖"                   │
  │                                                                  │
  │  T = 3 (升温):                                                   │
  │    logits / T = [1.67, 0.67, 0.17]                               │
  │    probs  = [0.51, 0.30, 0.19]  ← 分布变"平"                   │
  │                                                                  │
  │  T = 10 (高温):                                                  │
  │    logits / T = [0.5, 0.2, 0.05]                                 │
  │    probs  = [0.39, 0.33, 0.28]  ← 几乎均匀                     │
  │                                                                  │
  │                                                                  │
  │  T=1                T=3                T=10                      │
  │  ▐█                 ▐██                ▐███                      │
  │  ▐█                 ▐██                ▐███                      │
  │  ▐█                 ▐██                ▐███                      │
  │  ▐█                 ▐██                ▐███                      │
  │  ▐█                 ▐██                ▐███                      │
  │  ▐█   ▐  ▐          ▐██  ▐█  ▐         ▐███ ▐██ ▐██             │
  │  A    B  C          A    B   C         A    B   C               │
  │                                                                  │
  │  为什么要升温？                                                   │
  │  → T=1 时 Teacher 太"自信"，软标签接近 hard label               │
  │  → 升温后"暗知识"被放大，Student 能学到更多信息                  │
  │  → 常用 T=2~4                                                   │
  │                                                                  │
  │  为什么 Loss 要乘 T²？                                           │
  │  → 升温后 softmax 的梯度会按 1/T² 缩小                          │
  │  → 乘 T² 补偿回来，保持梯度尺度不变                              │
  └──────────────────────────────────────────────────────────────────┘
```

### 蒸馏的 Loss 设计

```
  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │  Total Loss = α × Hard Loss + (1-α) × Soft Loss × T²            │
  │                                                                  │
  │  Hard Loss (硬标签损失):                                          │
  │    = CrossEntropy(Student 输出, 真实标签)                         │
  │    → Student 自己学习"标准答案"                                   │
  │    类比: 学生做练习册，对照答案                                   │
  │                                                                  │
  │  Soft Loss (软标签损失):                                          │
  │    = KL_Divergence(Student 分布, Teacher 分布)                    │
  │    → Student 模仿 Teacher 的"思考方式"                           │
  │    类比: 学生观察老师解题过程，学习推理逻辑                       │
  │                                                                  │
  │  α (平衡系数):                                                    │
  │    α = 1.0 → 纯标准答案学习 (没有蒸馏)                           │
  │    α = 0.0 → 纯模仿老师 (没有标准答案)                           │
  │    α = 0.5 → 各占一半 (常用)                                     │
  │                                                                  │
  │  实际经验:                                                        │
  │    数据量大时: α 可以小一些 (更多依赖 Teacher)                    │
  │    数据量小时: α 应该大一些 (更多依赖标准答案)                    │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘
```

### LLM 蒸馏的特殊之处

```
  ┌──────────────────────────────────────────────────────────────────┐
  │  传统蒸馏 vs LLM 蒸馏                                            │
  │                                                                  │
  │  传统蒸馏 (BERT 时代):                                           │
  │    Teacher 和 Student 结构类似，只是层数/维度不同                 │
  │    直接在 logits 层面对齐                                         │
  │                                                                  │
  │  LLM 蒸馏 (GPT-4 时代):                                          │
  │    Teacher 可能是闭源 API (如 GPT-4)                              │
  │    无法获取 logits，只能拿到文本输出!                              │
  │    → 退化为"用 Teacher 生成高质量数据，然后训练 Student"          │
  │                                                                  │
  │  ── 三种 LLM 蒸馏方式 ──                                         │
  │                                                                  │
  │  1. 白盒蒸馏 (有 Teacher 完整访问权):                             │
  │     可以拿到 logits → 经典 KL 散度蒸馏                           │
  │     例: LLaMA-7B → TinyLLaMA-1.1B                                │
  │                                                                  │
  │  2. 黑盒蒸馏 (只有 Teacher API):                                  │
  │     只能拿到文本输出 → 用 Teacher 生成训练数据                   │
  │     例: 用 GPT-4 生成指令数据训练 Phi 系列                       │
  │                                                                  │
  │  3. 自蒸馏 (Self-Distillation):                                   │
  │     模型自己当 Teacher → 不同层之间互相学习                       │
  │     或: 先用大 batch 训练得到"好的" checkpoint，                  │
  │         再蒸馏到同架构的 Student                                  │
  │                                                                  │
  │  ── 数据蒸馏的典型流程 ──                                        │
  │                                                                  │
  │     GPT-4 (Teacher)                                               │
  │         │                                                         │
  │         │ 生成 10 万条高质量指令-回复对                            │
  │         ▼                                                         │
  │    ┌──────────┐                                                  │
  │    │ 指令数据集 │  "解释量子力学" → GPT-4 的高质量回答            │
  │    │ (10万条)  │  "写一首诗" → GPT-4 的高质量回答                │
  │    └────┬─────┘                                                  │
  │         │                                                         │
  │         │ 用这些数据微调                                          │
  │         ▼                                                         │
  │    Phi-2 (Student, 2.7B)                                          │
  │    → 小模型获得了接近大模型的能力!                                │
  └──────────────────────────────────────────────────────────────────┘
```

### 蒸馏代码示例

```python
import torch
import torch.nn.functional as F

def distillation_loss(student_logits, teacher_logits, labels, alpha=0.5, T=2.0):
    """
    student_logits: Student 模型的原始输出 [batch, vocab_size]
    teacher_logits: Teacher 模型的原始输出 [batch, vocab_size]
    labels: 真实标签 [batch]
    alpha: Hard loss 的权重
    T: 温度参数 (越高分布越平滑)
    """
    # Hard label loss — Student 学习标准答案
    hard_loss = F.cross_entropy(student_logits, labels)
    
    # Soft label loss — Student 模仿 Teacher 的输出分布
    # 除以温度 T 让分布变平滑，暴露更多"暗知识"
    soft_loss = F.kl_div(
        F.log_softmax(student_logits / T, dim=-1),   # Student 的平滑分布
        F.softmax(teacher_logits / T, dim=-1),        # Teacher 的平滑分布
        reduction='batchmean'
    ) * (T ** 2)  # 乘 T² 补偿梯度缩放
    
    return alpha * hard_loss + (1 - alpha) * soft_loss
```

### 常见蒸馏案例

```
  ┌──────────────────────────────────────────────────────────────────┐
  │                       经典蒸馏案例                                │
  ├──────────┬─────────────┬───────────┬────────────────────────────┤
  │ Teacher  │ Student     │ 压缩比    │ 效果                       │
  ├──────────┼─────────────┼───────────┼────────────────────────────┤
  │ BERT     │ DistilBERT  │ 12→6 层   │ 保留 97% 能力，快 60%      │
  │ BERT     │ TinyBERT    │ 12→4 层   │ 保留 96% 能力              │
  │ LLaMA-7B │ TinyLLaMA   │ 7B→1.1B   │ 保留 ~80% 能力             │
  │ GPT-4    │ Phi 系列    │ ?→2.7B    │ 微软数据蒸馏研究           │
  │ GPT-4    │ Orca        │ ?→13B     │ 学习推理过程               │
  └──────────┴─────────────┴───────────┴────────────────────────────┘

    把 12 层 BERT 蒸馏为 6 层
    → 模型小了 40%，推理快了 60%
    → 在 GLUE 基准上保留了 97% 的性能
    → 证明: Teacher 的"暗知识"真的有效!
```

## 剪枝 vs 蒸馏 vs 量化 对比

```
  ┌──────────────────────────────────────────────────────────────────────┐
  │                    三种压缩方法全面对比                               │
  ├──────────┬──────────────┬──────────────┬──────────────────────────┤
  │  维度    │  剪枝         │  蒸馏         │  量化                    │
  ├──────────┼──────────────┼──────────────┼──────────────────────────┤
  │ 类比     │ 裁员          │ 师傅带徒弟    │ 真空压缩                 │
  │ 做什么   │ 删除参数      │ 训练小模型    │ 降低精度                 │
  │ 模型结构 │ 改变 (变稀疏) │ 改变 (更小)   │ 不变                     │
  │ 需要训练 │ 有时需要      │ 必须          │ 通常不需要 (PTQ)         │
  │ 压缩率   │ 2-10x         │ 2-7x          │ 2-4x                    │
  │ 精度损失 │ 小-中          │ 小            │ 小                       │
  │ 推理加速 │ 需要特殊支持  │ 天然加速      │ 天然加速                 │
  │ 实现难度 │ 中            │ 高 (需要Teacher)│ 低                      │
  │ 可组合   │ 剪枝+量化     │ 蒸馏+量化     │ 与其他方法组合           │
  │ 典型流程 │ 几小时        │ 几天到几周    │ 几分钟到几小时           │
  │ 最佳场景 │ 去除冗余Head  │ 模型换代      │ 部署加速                 │
  └──────────┴──────────────┴──────────────┴──────────────────────────┘

  实践中最常见的组合:
    蒸馏出小模型 → 量化到 INT4 → 部署
    例: GPT-4 蒸馏→ Phi-2 (2.7B) → INT4 量化 (1.4GB) → 手机端运行
```
