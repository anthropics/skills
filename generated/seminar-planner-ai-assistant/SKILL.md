# セミナー企画運営 AIアシスタント

オンライン生成AIセミナー企画運営として「業務フローが確立されていないのにも関わらず複数案件同時進行するため負荷が掛かる」を解決するための5機能統合スキル。

## 機能一覧

| # | 機能名 | トリガー | 効果 |
|---|--------|----------|------|
| 1 | セミナータスク生成 | 「セミナーのタスクを洗い出して」 | 案件ごとの全タスクを即座に把握 |
| 2 | プロジェクトチェックリスト | 「チェックリストを作って」 | 抜け漏れ防止・進捗可視化 |
| 3 | SOP文書化支援 | 「業務フローを文書化して」 | 標準フロー確立・属人化解消 |
| 4 | 講師依頼メール作成 | 「講師への依頼メールを書いて」 | 依頼業務の時間短縮 |
| 5 | ガントチャート骨子作成 | 「スケジュールを可視化して」 | 複数案件の全体像把握 |

---

## 機能1: セミナータスク生成

### Purpose
セミナー案件の情報を入力すると、企画から振り返りまでの全タスクを網羅的に生成する。

### Workflow
1. セミナーの基本情報（テーマ、日程、形式、講師有無）を受け取る
2. フェーズ別（企画→講師調整→集客→当日→フォロー）にタスクを生成
3. 優先度・担当・期限目安を付与して出力

### Input
- セミナー名/テーマ
- 開催予定日
- 形式（ライブ/録画/ハイブリッド）
- 講師情報（決定済み/候補段階/未定）
- 集客目標人数

### Output
```markdown
## 【案件名】タスク一覧

### Phase 1: 企画（開催8週間前〜）
- [ ] テーマ・コンセプト確定 【優先:高】
- [ ] 講師候補リストアップ 【優先:高】
- [ ] 開催日時確定 【優先:高】
...

### Phase 2: 講師調整（開催6週間前〜）
- [ ] 講師へ打診メール送付
- [ ] 条件交渉・契約締結
...

### Phase 3: 集客（開催4週間前〜）
...

### Phase 4: 当日準備（開催1週間前〜）
...

### Phase 5: 開催当日
...

### Phase 6: フォローアップ（開催後〜）
...
```

### Example
**入力**:
```
セミナー名: 生成AI活用最前線セミナー
開催予定日: 2025年2月15日
形式: ライブ配信（Zoom）
講師: 候補3名から選定中
集客目標: 100名
```

**出力**: 6フェーズ・約40タスクの詳細リスト

## 機能2: プロジェクトチェックリスト

### Purpose
特定フェーズや特定作業の詳細チェックリストを生成し、抜け漏れを防止する。

### Workflow
1. チェックリストの対象（フェーズ or 作業）を特定
2. 必須項目・推奨項目・オプション項目に分類
3. 確認タイミングと担当を明記

### Input
- 対象フェーズ（例: 当日準備、講師調整）
- または特定作業（例: Zoom設定、告知文作成）

### Output
```markdown
## 【当日準備】チェックリスト

### 1週間前（必須）
- [ ] Zoomウェビナー設定完了
- [ ] 講師へリハーサル日程連絡
- [ ] 参加者リマインドメール準備

### 3日前（必須）
- [ ] スライド最終版受領
- [ ] 録画設定テスト
- [ ] バックアップ配信環境確認

### 当日朝（必須）
- [ ] 配信URL最終確認
- [ ] 講師との接続テスト（開始30分前）
- [ ] 参加者向けリマインド送信

### あると良い（推奨）
- [ ] トラブル時の緊急連絡フロー確認
- [ ] 代替講師の連絡先確保
```

### Example
**入力**: 「Zoom配信のチェックリストを作って」
**出力**: 配信設定に特化した15項目のチェックリスト

## 機能3: SOP文書化支援

### Purpose
繰り返し行う業務を標準作業手順書（SOP）として文書化し、業務フローを確立する。

### Workflow
1. 文書化したい業務の概要を聞き取る
2. 現状のやり方（暗黙知）をヒアリング形式で引き出す
3. 手順・判断基準・注意点を構造化してSOP形式で出力

### Input
- 業務名（例: 講師への初回打診、参加者フォローアップ）
- 現状のやり方（箇条書きでOK）
- よくあるトラブル・例外ケース

### Output
```markdown
# SOP: 講師への初回打診

## 1. 目的
外部講師に登壇を依頼し、興味・可否を確認する。

## 2. 対象
- セミナー講師候補（外部有識者、企業担当者）

## 3. 事前準備
- [ ] 候補者の経歴・過去登壇実績を調査
- [ ] 希望日程を3候補以上用意
- [ ] 謝礼・条件の社内承認を取得

## 4. 手順

### Step 1: 打診メール作成
- テンプレートを使用（機能4参照）
- 必須記載事項: 開催概要、日程候補、謝礼目安

### Step 2: 送信・追跡
- 送信後3営業日で返信なければリマインド
- 電話フォローは5営業日後

### Step 3: 返答対応
| 返答 | 対応 |
|------|------|
| OK | 詳細条件の擦り合わせへ |
| NG | 理由確認、別候補へ |
| 保留 | 期限設定してフォロー |

## 5. 注意事項
- 同時打診は最大3名まで（競合防止）
- 謝礼交渉は事前に上限確認

## 6. 関連文書
- 講師依頼メールテンプレート
- 謝礼基準表
```

### Example
**入力**: 「セミナー後のアンケート回収フローをSOP化して」
**出力**: アンケート設計〜回収〜分析までの標準手順書

## 機能4: 講師依頼メール作成

### Purpose
講師候補への打診・依頼メールを、状況に応じて適切なトーンで作成する。

### Workflow
1. 講師情報と依頼内容を受け取る
2. 関係性・依頼段階に応じたトーンを選択
3. 必要事項を網羅したメール文を生成

### Input
- 講師名・所属
- セミナー概要（テーマ、日程、形式）
- 依頼段階（初回打診/正式依頼/リマインド/お礼）
- 関係性（初めて/過去に接点あり/常連）
- 謝礼・条件（記載する場合）

### Output
```
件名: 【登壇のご相談】生成AI活用セミナーへのご登壇依頼（○○株式会社）

○○先生

突然のご連絡失礼いたします。
○○株式会社の△△と申します。

○○先生の「□□」に関するご知見に感銘を受け、
弊社主催のオンラインセミナーへのご登壇をお願いできないかと
ご連絡させていただきました。

【セミナー概要】
・テーマ: 生成AI活用最前線〜業務効率化の実践事例〜
・日程候補: 2025年2月15日(土) or 2月22日(土) 14:00-15:30
・形式: Zoomウェビナー（ライブ配信）
・想定参加者: 企業の業務改善担当者 約100名
・ご講演時間: 45分程度 + Q&A 15分

【条件】
・謝礼: ○○円（税込）
・事前打ち合わせ: 30分程度（オンライン）

ご多忙のところ恐縮ですが、ご検討いただけますと幸いです。
まずはご興味をお持ちいただけるか、ご返信いただければと存じます。

何卒よろしくお願いいたします。
```

### Example
**入力**: 「過去に1度登壇いただいた山田先生にリピート依頼したい」
**出力**: 過去の御礼を含めた丁寧かつ親しみのあるメール文

## 機能5: ガントチャート骨子作成

### Purpose
複数セミナー案件のスケジュールを可視化し、リソース配分と依存関係を把握する。

### Workflow
1. 進行中の案件リストを受け取る
2. 各案件のマイルストーンを抽出
3. 時系列で並べ、重複・競合を可視化

### Input
- 案件リスト（セミナー名、開催日）
- 現在のフェーズ
- 主要マイルストーン（あれば）

### Output
```markdown
## 複数案件スケジュール（2025年1月〜3月）

### 案件一覧
| # | セミナー名 | 開催日 | 現在フェーズ |
|---|-----------|--------|-------------|
| A | 生成AI活用セミナー | 2/15 | 講師調整中 |
| B | DX推進ワークショップ | 2/28 | 企画中 |
| C | AI倫理セミナー | 3/20 | 企画中 |

### タイムライン

```
1月        2月              3月
|----|----|----|----|----|----|----|----|
[A] ████████████████░░░░  2/15開催
    講師調整 → 集客 → 当日

[B]     ░░░░████████████████░░  2/28開催
        企画 → 講師 → 集客 → 当日

[C]         ░░░░░░░░████████████  3/20開催
            企画 → 講師 → 集客
```

### 重複・注意ポイント
- ⚠️ 2月第2週: A集客とB講師調整が重複 → リソース分散注意
- ⚠️ 2月第4週: A当日準備とB集客が重複 → 役割分担必要
- ✅ 3月はC案件に集中可能

### 推奨アクション
1. 今週中にAの講師確定 → 余裕を持ってBに着手
2. Bの企画を早めに固め、2月前半の負荷を分散
```

### Example
**入力**: 「今動いている3案件のスケジュールを整理して」
**出力**: タイムライン図 + 重複警告 + 推奨アクション

## 使い方ガイド

### インストール方法

**Claude.ai (web):**
1. このZIPファイルをダウンロード
2. Claude.ai にログイン
3. Settings → Capabilities → Custom Skills
4. ZIPをアップロード
5. スキルをオンにする

**Claude Code:**
1. ZIPを展開
2. フォルダを `~/.claude/skills/` に配置

### 推奨利用順序

1. **まず「SOP文書化支援」** から始める
   - 理由: 業務フロー確立がペインの根本原因。まず主要業務をSOP化

2. 次に **「セミナータスク生成」** を活用
   - 理由: 新規案件のタスク漏れを防止

3. **「ガントチャート骨子作成」** で全体把握
   - 理由: 複数案件の重複・競合を可視化

4. 日常業務で **「講師依頼メール作成」** を活用
   - 理由: 高頻度業務の時間短縮

5. 各フェーズで **「チェックリスト」** を生成
   - 理由: 抜け漏れ防止の最後の砦

## 注意事項

- **DO**: 案件情報を具体的に入力するほど精度が上がる
- **DO**: SOPは一度作ったら定期的に見直し・更新する
- **DO**: ガントチャートは週1で更新して全体像を把握
- **DO NOT**: AIの出力をそのまま送信しない（特にメール）
- **DO NOT**: 機密情報（講師の個人連絡先等）を入力しない
