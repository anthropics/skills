# 採用担当者 AIアシスタント

80名規模企業の採用担当者（採用目標3名）として、
「業務フローが出来ていない」「求人票が昭和の書き方」を解決するための5機能統合スキル。

## 機能一覧

| # | 機能名 | トリガー | 効果 |
|---|--------|----------|------|
| 1 | 令和ラブレター求人票生成 | 「求人票を令和風に書いて」 | 候補者の心に響く求人票 |
| 2 | 採用フローSOP生成 | 「採用フローを作って」 | 業務フロー確立・属人化解消 |
| 3 | 候補者管理チェックリスト | 「候補者管理のチェックリスト」 | 選考の抜け漏れ防止 |
| 4 | スカウト文パーソナライズ | 「スカウト文を書いて」 | 返信率向上するスカウト |
| 5 | 求人キャッチコピー生成 | 「キャッチコピーを考えて」 | 目を引く見出し・訴求文 |

---

## 機能1: 令和ラブレター求人票生成

### Purpose
昭和の「条件羅列型」求人票を、令和の「ラブレター型」求人票に完全刷新する。
候補者の心に響く、「この会社で働きたい！」と思わせる求人票を生成。

### Workflow
1. 現在の求人情報（職種、条件、会社情報）を受け取る
2. 「令和のラブレター」フレームワークで再構成
3. 候補者目線の魅力的な求人票を生成

### Input
- 職種名
- 仕事内容（箇条書きでOK）
- 必須条件・歓迎条件
- 給与・福利厚生
- 会社の特徴・カルチャー
- （任意）現在の求人票テキスト

### Output

**令和ラブレター求人票の構成:**

```markdown
## 🎯 あなたへのラブレター

【ここに入れる一言】
「あなたの○○な経験、私たちは本気で必要としています」

---

## 💌 まず、正直に伝えたいこと

私たちは80名の会社です。大企業ではありません。
でも、だからこそ[具体的な魅力]があります。

今回、3名の仲間を探しています。
「3名」という数字に意味があります。[理由を説明]

---

## 🌟 この仕事で得られる「3つの未来」

1. **[未来1]**: 具体的にどう成長できるか
2. **[未来2]**: どんなスキルが身につくか
3. **[未来3]**: キャリアにどうプラスになるか

---

## 📝 お願いしたいこと（仕事内容）

堅苦しい「業務内容」ではなく、
「あなたにお願いしたいこと」として記載

- 最初の1ヶ月: ○○を一緒にやりましょう
- 3ヶ月後: ○○を任せたいと思っています
- 1年後: ○○を目指してほしい

---

## 🤝 こんな方と働きたい

【必須ではなく「こんな方と働きたい」】
- ○○な経験がある方（目安: ○年）
- ○○に興味がある方
- ○○を大切にできる方

【こんな方は特に歓迎】
- ○○の経験がある方
- ○○を一緒に作りたい方

---

## 💰 お約束できること

**給与**: ○○万円〜○○万円
（あなたの経験と希望を聞いて、誠実に決めましょう）

**働き方**:
- リモート: ○○
- フレックス: ○○
- 残業: 正直に○○時間/月くらい

**福利厚生**:
- [候補者が嬉しいものを優先]

---

## 📮 最後に

この求人票を読んで、少しでも「気になる」と思ったら、
ぜひ一度お話しませんか？

カジュアル面談も大歓迎です。
「まだ転職を決めていない」という方も、お気軽にどうぞ。

あなたからのご連絡を、心からお待ちしています。

[担当者名]より
```

### Example

**入力**:
```
職種: 経理担当
会社: 80名のIT企業
条件: 経理経験3年以上
給与: 400-500万
特徴: リモート可、裁量大きい
```

**出力**: 上記フレームワークに沿った、心に響く求人票

## 機能2: 採用フローSOP生成

### Purpose
「誰が・いつ・何を・どうやって」を明確にした採用業務の標準手順書を生成し、
業務フロー未確立の状態を解消する。

### Workflow
1. 現在の採用プロセス（あれば）を聞き取る
2. 80名規模・3名採用に最適なフローを設計
3. 各ステップの担当・期限・判断基準を明確化

### Input
- 採用目標人数（例: 3名）
- 採用職種
- 利用している採用チャネル（媒体、エージェント等）
- 現在の課題や困っていること

### Output

```markdown
# 採用業務SOP（標準作業手順書）

## 1. 採用フロー全体像

```
[求人公開] → [応募受付] → [書類選考] → [1次面接] → [2次面接] → [内定] → [入社]
   ↓           ↓           ↓           ↓           ↓         ↓
  1日目      随時       3営業日     1週間      1週間     3日
```

## 2. 各フェーズの詳細

### Phase 1: 求人公開（担当: 採用担当）

**目的**: 母集団形成の開始

**手順**:
1. [ ] 求人票を作成（機能1使用）
2. [ ] 社内承認を取得
3. [ ] 媒体に掲載（○○、△△）
4. [ ] スカウト送付開始

**完了基準**: 全媒体への掲載完了

**期限**: 採用開始から1営業日以内

---

### Phase 2: 応募受付・書類選考

**目的**: 候補者の一次スクリーニング

**手順**:
1. [ ] 応募通知を確認（毎日AM/PM）
2. [ ] 候補者情報を管理シートに転記
3. [ ] 書類選考を実施（評価シート使用）
4. [ ] 結果連絡（通過/お見送り）

**判断基準**:
| 項目 | 通過 | 要検討 | 見送り |
|------|------|--------|--------|
| 経験年数 | 3年以上 | 1-2年 | 未経験 |
| ... | ... | ... | ... |

**期限**: 応募から3営業日以内に返信

---

### Phase 3: 1次面接
...

### Phase 4: 2次面接
...

### Phase 5: 内定・オファー
...

### Phase 6: 入社準備
...

## 3. 緊急時対応

| 状況 | 対応 |
|------|------|
| 優秀候補者が他社選考中 | 選考スピードアップ |

## 4. 使用ツール・テンプレート

- 候補者管理シート（機能3で生成）
- 面接評価シート
- 各種メールテンプレート
```

## 機能3: 候補者管理チェックリスト

### Purpose
候補者ごとの選考進捗を可視化し、対応漏れを防止する。

### Workflow
1. 選考フェーズ情報を受け取る
2. 各フェーズのチェック項目を生成
3. 候補者ステータス管理表を作成

### Input
- 候補者名（任意）
- 現在の選考フェーズ
- または「全体管理用テンプレートが欲しい」

### Output

```markdown
## 候補者管理チェックリスト

### 📋 候補者ステータス一覧

| 候補者 | フェーズ | 次アクション | 期限 | 担当 |
|--------|----------|--------------|------|------|
| Aさん | 1次面接後 | 2次日程調整 | 1/10 | 自分 |
| Bさん | 書類選考中 | 選考結果連絡 | 1/8 | 自分 |
| Cさん | 内定提示 | 回答待ち | 1/12 | 自分 |

### ✅ フェーズ別チェックリスト

#### 応募受付時
- [ ] 応募情報を管理シートに記録
- [ ] 受付完了メールを送信
- [ ] 書類選考担当をアサイン

#### 書類選考通過時
- [ ] 通過連絡メールを送信
- [ ] 面接候補日を3つ以上提示
- [ ] 面接官のスケジュール確保

#### 面接実施後
- [ ] 評価シートを記入
- [ ] 合否を決定
- [ ] 結果連絡（通過: 24h以内、見送り: 3日以内）

#### 内定提示時
- [ ] オファー内容を社内承認
- [ ] オファー面談を設定
- [ ] 条件通知書を準備

#### 内定承諾後
- [ ] 入社日を確定
- [ ] 入社書類を送付
- [ ] 受け入れ部署へ連絡
```

## 機能4: スカウト文パーソナライズ

### Purpose
候補者のプロフィールに合わせた、返信したくなるスカウト文を生成する。

### Workflow
1. 候補者のプロフィール情報を受け取る
2. パーソナライズポイントを特定
3. 「あなただから送っている」感のあるスカウト文を生成

### Input
- 候補者名
- 候補者の経歴・スキル（LinkedInやWantedlyのプロフィール）
- 募集職種
- 特にアピールしたい自社の特徴

### Output

```markdown
【件名】
[候補者の具体的経験]を拝見してご連絡しました｜80名IT企業の経理

【本文】
○○様

突然のご連絡失礼いたします。
株式会社△△の採用担当、□□と申します。

○○様のプロフィールで、
「[具体的な経験やスキル]」を拝見し、
ぜひ一度お話しさせていただきたくご連絡しました。

特に「[具体的なプロジェクトや実績]」のご経験は、
私たちが今まさに[具体的なチャレンジ]に取り組む中で、
本当に力をお借りしたいと感じたポイントです。

【なぜ○○様なのか】
・[パーソナライズした理由1]
・[パーソナライズした理由2]

【私たちについて少しだけ】
・80名のIT企業です
・[候補者が興味を持ちそうな特徴]
・[働き方や裁量についての魅力]

まずは30分程度のカジュアル面談で、
お互いのことをお話しできればと思います。

ご興味をお持ちいただけましたら、
ぜひご返信いただけると嬉しいです。

どうぞよろしくお願いいたします。
```

### Example

**入力**:
```
候補者: 山田太郎さん
経歴: メガベンチャーで経理5年、IPO準備経験あり
募集: 経理担当（将来のマネージャー候補）
```

**出力**: IPO準備経験に触れた、パーソナライズされたスカウト文

## 機能5: 求人キャッチコピー生成

### Purpose
媒体のタイトルやSNS投稿で目を引く、短くインパクトのあるキャッチコピーを生成。

### Workflow
1. 職種と訴求ポイントを受け取る
2. 複数パターンのキャッチコピーを生成
3. 用途別（媒体タイトル、SNS、スカウト件名）に最適化

### Input
- 職種
- 最も伝えたい魅力（1つ）
- ターゲット層の特徴

### Output

```markdown
## 求人キャッチコピー案

### 📌 職種: 経理担当
### 🎯 訴求ポイント: 裁量が大きい・80名の成長企業

---

### 媒体タイトル用（30文字以内）
1. 「経理の常識、壊しませんか」80名IT企業
2. IPO準備から任せます｜裁量大の経理
3. 80名→300名を一緒に。経理募集
4. 経理×IT。新しいキャリアの形
5. 「経理だけじゃ物足りない」あなたへ

### SNS投稿用（カジュアル）
1. 経理募集中！でも普通の経理じゃないんです...
2. 「え、経理なのにそこまでやるの？」←これ、褒め言葉です
3. 80名IT企業の経理、実は○○もやってます
4. 経理の概念、変えてみませんか？

### スカウト件名用（40文字以内）
1. [○○様のIPO経験を拝見して]80名IT企業より
2. ○○様、経理の裁量でお困りではないですか？
3. 経理×成長企業。○○様にお話ししたいことがあります
4. 【本気で口説かせてください】○○様へ

### 選び方のポイント
- **媒体タイトル**: 1か3がインパクト大
- **SNS**: 2が読みたくなる構成
- **スカウト**: 1がパーソナライズ感あり
```

## 使い方ガイド

### インストール方法

**Claude.ai (web):**
1. このZIPファイルをダウンロード
2. Claude.ai にログイン
3. Settings → Capabilities → Custom Skills
4. ZIPをアップロード
5. スキルをオンにする

**Claude Code:**
1. ZIPを展開
2. フォルダを `~/.claude/skills/` に配置

### 推奨利用順序

1. **まず「採用フローSOP生成」** から始める
   - 理由: 業務フロー確立がペインの根本原因

2. 次に **「令和ラブレター求人票生成」** で求人刷新
   - 理由: 母集団の質向上に直結

3. **「求人キャッチコピー生成」** で目を引く見出しを
   - 理由: クリック率・応募率向上

4. 採用活動開始後は **「スカウト文パーソナライズ」**
   - 理由: スカウト返信率向上

5. 日常的に **「候補者管理チェックリスト」** で進捗管理
   - 理由: 抜け漏れ防止・スピード維持

## 80名規模・3名採用に最適化されたポイント

- **少数精鋭採用**: 1人1人を丁寧に口説く姿勢
- **スピード重視**: 大企業より早く動けることが武器
- **裁量アピール**: 中小企業ならではの魅力を全面に
- **ラブレター型**: 条件勝負しない、想い勝負の訴求

## 注意事項

- **DO**: 求人票はそのまま使わず、社内確認を経て掲載
- **DO**: スカウト文は候補者ごとに微調整
- **DO**: SOPは運用しながら改善
- **DO NOT**: AIの出力をコピペで送信しない
- **DO NOT**: 個人情報を含む候補者データを入力しない（管理は別ツールで）
