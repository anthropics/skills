# 🌙 盘后复盘 · 数据更新

> **执行时间**：每个交易日 **15:00 - 次日8:00**（重点在15:00-20:00）
> **核心原则**：用数据说话，不用情绪复盘。只看「做对了什么」和「信号变化了什么」。

---

## ⚡ 快速指令

```
帮我今日复盘              → 触发完整盘后复盘（含交互式引导）
导入今日操作记录          → 引导用户提交东方财富CSV或手动描述操作
复盘 [日期]               → 对指定日期补做复盘（如：复盘 2026-03-01）
```

## 🚀 执行 SOP（Agent必须按此顺序执行）

当用户触发此 Skill 时，**必须按以下步骤顺序完成**，不跳步，不合并：

### Step 0：交互式预检（运行脚本前先确认）

在执行任何脚本之前，Agent 主动问用户以下问题（**一次性问完，不要逐个追问**）：

```
Agent 提问模板：
「收盘后复盘开始～先确认几件事：
1. 今天有做任何买卖操作吗？（有的话告诉我，或者把东方财富导出的CSV贴过来）
2. 持仓快照是否已更新到今天？（sync_holdings.py 最后运行时间是？）
3. 有没有特别想关注的 ETF 或信号？」
```

若用户说**有操作**：
- 引导用户提供操作数据（见「操作记录导入 SOP」）
- 将操作写入 `output/{date}/operations.json` 后再运行脚本

若用户说**没有操作**：
- 直接进入 Step 1

### Step 1：数据采集

```bash
cd finance-assistant/post-market-review/scripts

# 采集今日全部数据（15:30后运行）
python generate_report.py --force

# 或指定历史日期补做复盘
python generate_report.py --date YYYY-MM-DD
```

各模块单独运行（如需调试）：
```bash
python fetch_market_data.py [--date YYYY-MM-DD]  # 大盘指数/涨跌停
python fetch_fund_flow.py   [--date YYYY-MM-DD]  # 资金流向（含重试机制）
python fetch_etf_shares.py  [--date YYYY-MM-DD]  # ETF份额变化（差异化评分）
```

### Step 2：生成报告

```bash
python generate_report.py [--date YYYY-MM-DD]
```

报告自动保存到两处：
- `scripts/output/{date}/review_{date}.md` — 原始数据目录
- `finance-assistant/reports/{date}-post-market.md` — 归档目录（推荐查阅）

### Step 3：Agent 深度解读

报告生成后，Agent 必须：
1. 读取 `reports/{date}-post-market.md`
2. 提炼「今日三大信号」
3. 点出「最值得警惕的背离」（份额涨但主力净流出、价格涨但机构赎回等）
4. 给出明日重点关注事项（不超过3条）

## 📊 ETF份额评分规则（差异化）

> 幅度越大，评分越高，不再固定+15。

| 份额变化幅度 | 信号 | 评分影响 |
|------------|------|---------|
| ≥ 100% | 🔥 极强申购（翻倍以上） | **+20** |
| 50% ~ 100% | 🔥 极强申购 | **+17** |
| 20% ~ 50% | 🟢 大幅申购 | **+14** |
| 10% ~ 20% | 🟢 大幅申购 | **+11** |
| 5% ~ 10% | 💚 小幅申购 | **+8** |
| 2% ~ 5% | 💚 轻微申购 | **+5** |
| ±2% 以内 | ⚪ 基本持平 | 0 |
| -5% ~ -2% | 🟠 小幅赎回 | **-5** |
| -10% ~ -5% | 🟠 中幅赎回 | **-8** |
| -20% ~ -10% | 🔴 大幅赎回 | **-11** |
| -50% ~ -20% | 🔴 大幅赎回 | **-14** |
| < -50% | 🚨 极强赎回 | **-17** |

**背离信号识别**（自动生成）：
- 份额大幅申购 + 二级市场主力净流出 → 「一级申购/二级出货」，机构借新资金减仓，**短期谨慎**
- 份额大幅赎回 + 价格上涨 → 「借涨出货」，追高风险极大

## 📊 资金流向解读框架

```
全市场主力净流向判断：
  净流入 > 100亿  → 主力积极建仓，偏多
  净流入 0~100亿  → 主力小幅布局，中性偏多
  净流出 0~100亿  → 主力小幅减仓，中性
  净流出 > 100亿  → 主力大幅出货，偏空

主力 vs 散户背离：
  主力流出 + 散户流入 → 「接盘」格局，追高危险
  主力流入 + 散户流出 → 主力低吸，信号正面

行业资金持续性：
  净流入连续3日+  → 主线确立，可加仓相关ETF
  净流出连续3日+  → 主线结束，应减仓相关ETF
```

## ⚙️ 执行原则

### ✅ Do
- **15:30开始复盘**，趁记忆清晰记录今日操作原因
- **数据优先**：用数据更新评分，不用"感觉"覆盖判断
- **记录错误**：每次未执行止损的原因必须写下来
- **评分及时更新**：有重大新信号时立即修正评分
- **关注背离**：份额大增但主力流出 = 一级/二级背离，是最重要的短期信号
- **导入操作**：即使没买卖，也要记录「计划执行了吗」和「为什么没执行」

### ❌ Don't
- 不在盘后"后悔"没追的股票（幸存者偏差）
- 不因今日亏损而明日冲动报复性交易
- 不把ETF份额变化当成100%可靠的申购赎回数据（可能是成交量代理）
- 不把每日小波动当成"信号变化"来频繁更新评分
- 不跳过 Step 0 交互预检直接运行脚本

---
