# 研报 → 知识库 转换

## 概述

将 yanbao 技能产出的 Word 研报（.docx）拆解为 Obsidian 知识库中的结构化笔记，并更新所有 MOC 导航页。

知识库路径：`C:\Users\49285\Desktop\投研\`

## 知识库目录结构

```
投研/
├── 00-总纲/          ← MOC 总纲、待探索问题、模板、仪表盘
├── 01-因子研究/       ← 因子定义、公式、信号规则
├── 02-形态识别/       ← K线形态、双顶、拥挤高点
├── 03-策略回测/       ← 回测设计、结果、扫参
├── 04-市场分析/       ← 专题研究、案例分析
├── 05-核心概念/       ← 原子化知识点
├── 06-原始数据映射/    ← 原始文件路径索引
└── 2026.*/           ← 原始数据目录（不动）
```

## 四步流水线

### Step 1 — 提取研报文本

```bash
python -c "
import zipfile
from xml.etree import ElementTree as ET
with zipfile.ZipFile('{{docx_path}}') as z:
    with z.open('word/document.xml') as f:
        tree = ET.parse(f)
    ns = {'w': 'http://schemas.openxmlformats.org/wordprocessingml/2006/main'}
    text = '\n'.join(''.join(t.text or '' for t in p.findall('.//w:t', ns)) for p in tree.findall('.//w:p', ns))
with open('{{output_txt}}', 'w', encoding='utf-8') as f:
    f.write(text)
print(f'Extracted {len(text)} chars')
"
```

### Step 2 — 识别知识单元并创建笔记

逐段阅读研报内容，按以下分类识别知识单元：

| 研报里有什么 | 创建到 | 用哪个模板 | 命名规则 |
|-------------|--------|-----------|---------|
| 新的因子/信号/指标（含公式、触发条件、历史信号数） | `01-因子研究/` | 因子模板 | 描述性中文名 |
| 回测设计+扫参结果+命中率表格 | `03-策略回测/` | 回测模板 | 描述性中文名 |
| 新的概念/定义/判读规则 | `05-核心概念/` | 概念模板 | 概念的核心术语 |
| 案例研究/历史事件分析 | `04-市场分析/` | 自由格式 | `xxx案例分析` |
| 专题市场分析/风格切换全景 | `04-市场分析/` | 自由格式 | 主题名 |

**模板位置：** `00-总纲/模板/`

**笔记规范：**
- 每个笔记开头有 YAML frontmatter：`tags` + `created` 日期
- 与其他已有笔记做 `[[双向链接]]`
- 引用原始文件路径（`2026.*/xxx.docx`）
- 在笔记中直接写完整内容，不建空占位笔记

### Step 3 — 更新 MOC

每创建一批新笔记后，立即更新以下 MOC 页面：

| MOC 文件 | 更新内容 |
|---------|---------|
| `01-因子研究/因子总览.md` | 新因子加一行到对应表格 |
| `03-策略回测/回测总览.md` | 新回测加一行到研究清单 |
| `06-原始数据映射/原始数据索引.md` | 新 docx 文件登记 + 标注"已完全拆解" |
| `00-总纲/研究问题-待探索.md` | 划掉解决了的问题 `[x]`，添加新发现的问题 `[ ]` |
| `00-总纲/投研知识库总纲.md` | 如有新的"核心发现"或"快速导航"条目则更新 |

### Step 4 — 最终验证

```bash
cd "C:/Users/49285/Desktop/投研"
echo "=== 新增笔记 ===" && find "01-因子研究" "03-策略回测" "04-市场分析" "05-核心概念" -name "*.md" -newer "00-总纲/投研知识库总纲.md" | sort
echo "=== 知识库总数 ===" && find "00-总纲" "01-因子研究" "02-形态识别" "03-策略回测" "04-市场分析" "05-核心概念" "06-原始数据映射" -name "*.md" | wc -l
```

## 笔记内容要求

### 必须包含
- **入口链接**：新笔记被谁引用？更新引用者的笔记
- **出口链接**：新笔记引用了哪些已有概念？用 `[[]]` 链接
- **来源文件**：标明原始 docx 路径
- **与其他笔记的差异化定位**：不要写已有笔记已经覆盖的内容

### 不应包含
- 大段原文复制（提炼核心，原文留在 docx 里）
- 空占位笔记（必须填完内容再创建）
- 孤立的笔记（必须至少有一个入口链接和一个出口链接）

## 交互流程

1. 用户提供 docx 路径
2. 执行 Step 1 提取文本
3. 阅读全文，列出将要创建的笔记清单（向用户确认）
4. 批量创建笔记（Write 工具）
5. 批量更新 MOC（Edit 工具）
6. 执行 Step 4 验证
7. 报告：新增 N 篇笔记，知识库总数 N 篇，更新的 MOC 列表

## 与 yanbao 技能的协作

```
yanbao 技能                    research-to-kb 技能
   ↓                               ↓
生成 Word 研报 (.docx)  ──────→  拆解为知识库笔记
生成图表 (.png)                 注册到原始数据索引
输出回测数据 (.csv)              创建因子/回测/概念笔记
                                 更新所有 MOC
```

两个技能独立运行。yanbao 产出文件 → 用户确认研报质量 → 调用 research-to-kb 入库。

## 常见问题

| 问题 | 处理 |
|------|------|
| 研报中的因子与已有因子高度相似 | 在已有笔记中新增一节"变体"，不创建独立笔记 |
| docx 包含大量图表无法提取文字 | 基于图表标题和上下文描述性写入笔记 |
| 研报主要内容已在知识库中 | 只更新"当前市场状态"部分，不重复拆解 |
| 研报中有多个独立主题 | 每个主题创建独立笔记，共用一个数据索引条目 |
