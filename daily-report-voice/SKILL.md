# 音声入力業務日報作成スキル

音声で話した業務内容を自動的に構造化された日報として作成し、Google Drive連携でワークフローを自動化します。

## 基本ワークフロー

1. **音声入力受付**: 自然言語での業務報告を受信
2. **内容解析・構造化**: AI処理で項目別に振り分け
3. **不足項目ヒアリング**: 必要に応じて追加質問
4. **Excel生成**: 5列形式の日報ファイル作成
5. **Google Drive保存**: 指定フォルダに自動保存
6. **Workspace Flows連携**: 本部スプレッドシート蓄積のトリガー

## 音声入力パターン例

```
「今日はA社のシステム設計を3時間実施しました。
データベース設計で課題が発生しています。
明日はB社との要件定義会議の予定です。」
```

## 処理手順

### 1. 音声内容の解析

音声テキストから以下の項目を抽出:
- **作業内容**: 何をしたか（顧客名、業務内容、工数含む）
- **課題**: 発生した問題や懸念事項
- **次回予定**: 明日以降の予定

### 2. 不足項目の特定とヒアリング

必須項目が不足している場合は追加質問:
- 工数不明: "作業時間はどのくらいでしたか？"
- 顧客不明: "どちらの顧客でしょうか？"  
- 課題なし: "特に課題はありませんでしたか？"
- 予定なし: "明日の予定はありますか？"

### 3. データ構造化

5列形式でデータを整理:
```
日付: 2025-11-23 (自動取得)
報告者: Ken (デフォルト)
作業内容: A社システム設計3h
課題: データベース設計で課題発生  
次回予定: B社要件定義会議
```

### 4. Excel生成と保存

- シンプル日報テンプレートを使用
- 当日の行にデータを挿入
- Google Drive指定フォルダに保存
- ファイル名: `MIRAICHI_日報_YYYY-MM-DD.xlsx`

## MIRAICHI業務特化の考慮事項

### 典型的な業務パターン
- **システム開発**: 要件定義、設計、開発、テスト
- **コンサルティング**: ヒアリング、分析、提案書作成
- **AI/DX支援**: 導入支援、教育、ツール開発
- **セミナー運営**: 資料作成、実施、フォローアップ

### 地域企業との連携
- 金融機関（信用金庫、銀行）
- 協同組合
- 地方自治体
- 製造業

## エラーハンドリング

- **音声認識エラー**: 再入力を促す
- **項目抽出失敗**: 手動入力モードに切り替え
- **ファイル保存エラー**: ローカル保存後、再試行
- **Google Drive接続エラー**: オフライン保存、後で同期

## Google Workspace Flows連携

保存完了後のワークフロー:
1. Google Driveフォルダの変更検知
2. Workspace Flowsトリガー実行
3. 本部スプレッドシートへデータ蓄積
4. 処理完了通知

使用時は assets/template.xlsx のシンプル5列テンプレートを基に処理を行います。
