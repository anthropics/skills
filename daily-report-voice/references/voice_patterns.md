# 音声入力パターン解析リファレンス

## 抽出パターン定義

### 作業内容抽出パターン

#### パターン1: 基本動作表現
```regex
今日は(.+?)(?:しました|やりました|実施|完了|作成)
```
**例:**
- "今日はA社のシステム設計をしました" → "A社のシステム設計"
- "今日はセミナー資料を作成しました" → "セミナー資料を作成"

#### パターン2: 時間帯指定
```regex
午前中?(.+?)(?:の作業|を実施|をやった)
```
**例:**
- "午前中はB社の打ち合わせをやった" → "B社の打ち合わせ"

#### パターン3: 工数込みの表現
```regex
(.+?)(?:に|で|を)(\d+時間?|半日|一日)
```
**例:**
- "データベース設計に3時間" → "データベース設計"
- "要件定義を半日" → "要件定義"

### 課題抽出パターン

#### パターン1: 直接的課題表現
```regex
(?:課題|問題)(?:が|として|は)(.+?)(?:です|ます|。)
```
**例:**
- "課題としてデータ移行があります" → "データ移行"
- "問題が発生しています" → "発生"

#### パターン2: 困難表現
```regex
(.+?)(?:で問題|が課題|に困って)
```
**例:**
- "API連携で問題" → "API連携"
- "スケジュールに困って" → "スケジュール"

### 予定抽出パターン

#### パターン1: 明日の予定
```regex
明日(?:は|の予定は?)(.+?)(?:です|ます|。)
```
**例:**
- "明日はC社との会議です" → "C社との会議"
- "明日の予定はコードレビューです" → "コードレビュー"

#### パターン2: 将来の予定
```regex
来週(.+?)(?:予定|する予定)
```
**例:**
- "来週テストする予定" → "テスト"

## エラーハンドリングパターン

### 抽出失敗時の対応
1. **作業内容不明**: 全体テキストを作業内容として扱う
2. **課題なし**: "特になし"で設定
3. **予定不明**: "未定"で設定

### 品質チェック項目
- 顧客名の一貫性（略称vs正式名称）
- 工数表記の統一（時間単位）
- 技術用語の正確性
